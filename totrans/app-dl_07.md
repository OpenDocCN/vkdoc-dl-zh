# 7. 超参数调整

在本章中，你将了解如何找到最佳超参数以从你的模型中获得最佳结果的问题。首先，我将描述什么是黑盒优化问题，以及这类问题与超参数调整有何关联。你将看到解决这类问题的三种最知名的方法：网格搜索、随机搜索和贝叶斯优化。我将通过示例展示在什么条件下哪种方法有效，并给你一些在对数尺度上改进优化和采样的非常有用的技巧。在本章的结尾，我将展示如何使用这些技术来调整深度模型，使用 Zalando 数据集。

## 黑盒优化

超参数调整问题只是更一般类型问题的一个子类：黑盒优化。一个黑盒函数 *f* (*x*)

![$$ f(x):{\mathbb{R}}^n\to \mathbb{R} $$](img/463356_1_En_7_Chapter_TeX_Equa.png)

是一个解析形式未知的函数。黑盒函数可以评估以获得其定义的所有 *x* 值的值，但无法获得其他信息（例如其梯度）。通常，通过术语“黑盒函数的全局优化”（有时称为黑盒问题），我们是指我们试图找到 *f* (*x*) 的最大值或最小值，有时在一定的约束条件下。以下是一些这类问题的例子：

+   寻找给定机器学习模型的最大化所选优化指标的超参数

+   寻找只能通过数值或我们无法查看的代码评估的函数的最大值或最小值。在某些行业环境中，可能存在非常复杂的遗留代码，并且有一些函数必须基于其结果进行最大化。

+   寻找最佳的石油钻井位置。在这种情况下，你的函数将是你可以找到的石油量，而 *x* 是你的位置。

+   寻找过于复杂以至于无法建模的情况的最佳参数组合，例如，在太空中发射火箭时，如何优化燃料量、火箭各阶段的直径、精确轨迹等。

这是一个非常迷人的问题类别，它产生了智能的解决方案。你将看到其中的三个：网格搜索、随机搜索和贝叶斯优化。如果你对这个主题感兴趣，可以查看黑盒优化竞赛[`goo.gl/LY7huY`](https://goo.gl/LY7huY)。竞赛的规则反映了现实生活中的问题。为这个问题设定了一个场景，你必须通过黑盒接口优化一个函数（找到最大值或最小值）。你可以得到函数在所有 *x* 值下的值，但你无法获取任何其他信息，例如其梯度。

为什么寻找神经网络最佳超参数构成一个黑盒问题？因为我们无法计算关于超参数的信息，例如网络输出的梯度，特别是当使用复杂优化器或自定义函数时，我们需要其他方法，以便找到最佳超参数，以最大化选择的优化指标。请注意，如果我们能够获得梯度，我们可以使用梯度下降等算法来找到最大值或最小值。

### 注意

我们的黑盒函数*f*将是我们的神经网络模型（包括优化器、损失函数形式等），它以优化指标作为输出，以超参数作为输入，而*x*将是包含超参数的数组。

问题可能看起来相当简单。为什么不尝试所有可能性呢？好吧，这可能在你之前章节中看到的例子中是可能的，但如果你正在处理一个问题，并且训练模型需要一周时间，这可能是一个挑战。因为，通常情况下，你将有几个超参数，尝试所有可能性将不可行。让我们通过一个例子来更好地理解这一点。假设我们正在训练一个具有多个层的神经网络模型。我们可能决定考虑以下超参数，以查看哪种组合效果更好：

+   *学习率*：假设我们想尝试*n*·10^(-4)的值，其中*n* = 1, …, 10²。（100 个值）

+   *正则化参数*：0、0.1、0.2、0.3、0.4 和 0.5（6 个值）

+   *优化器选择*：GD、RMSProp 或 Adam（3 个值）

+   *隐藏层数量*：1、2、3、5 和 10（5 个值）

+   *隐藏层中的神经元数量*：100、200 和 300（3 个值）

考虑到你将需要训练你的网络

![$$ 100\times 6\times 3\times 5\times 3=27000 $$](img/463356_1_En_7_Chapter_TeX_Equb.png)

如果你想测试所有可能的组合，那么需要多次尝试。如果你的训练需要 5 分钟，你将需要 13.4 周的计算机时间。如果训练需要数小时或数天，你将没有机会。例如，如果训练需要一天，你将需要 73.9 年才能尝试所有可能性。大多数超参数的选择将来自经验。例如，你总是可以安全地使用 Adam，因为它是可用的更好优化器（在几乎所有情况下）。但你将无法避免尝试调整其他参数，例如隐藏层的数量或学习率。你可以通过经验（就像优化器一样）或一些智能算法来减少你需要尝试的组合数量，正如你将在本章后面看到的。

## 黑盒函数注意事项

黑盒函数通常分为两大类：

+   *低成本函数*：可以评估数千次的函数

+   *高成本函数*：只能评估几次的函数，通常少于 100 次

如果黑盒函数便宜，优化方法的选择并不关键。例如，我们可以通过数值计算相对于*x*的梯度，或者简单地通过在大量点上评估函数来搜索最大值。如果函数昂贵，我们需要更智能的方法。其中之一是贝叶斯优化，我将在本章后面讨论，以给您一个了解这些方法如何工作以及它们有多复杂的概念。

### 备注

尤其是在深度学习领域，神经网络几乎总是昂贵的函数。

对于昂贵的函数，我们必须找到用尽可能少的评估次数解决我们问题的方法。

## 超参数调整问题

在探讨如何找到最佳超参数之前，我想快速回顾一下神经网络，并讨论我们可以在深度模型中调整什么。通常，当谈到超参数时，初学者只会想到数值参数，例如学习率或正则化参数，例如。记住，以下也可以变化，看看您是否能得到更好的结果：

+   *训练轮数*：有时，简单地训练您的网络更长一些会得到更好的结果。

+   *优化器选择*：您可以尝试选择不同的优化器。如果您正在使用普通的梯度下降，您可以尝试使用 Adam，看看是否能得到更好的结果。

+   *调整正则化方法*：如前所述，有几种应用正则化的方法。尝试改变方法可能值得尝试。

+   *激活函数选择*：尽管之前章节中用于隐藏层神经元的激活函数一直是 ReLU，但其他函数可能效果更好。尝试 sigmoid 或 Swish，例如，可能有助于您得到更好的结果。

+   *层数和每层的神经元数量*：尝试不同的配置：尝试具有不同神经元数量的层，例如。

+   *学习率衰减方法*：尝试（如果您没有使用已经执行此操作的优化器）不同的学习率衰减方法。

+   *小批量大小*：调整小批量的大小。当您数据量较少时，可以使用批量梯度下降。当您数据量较多时，小批量更有效率。

+   *权重初始化方法*

让我们将我们可以在模型中调整的参数分为以下三个类别：

+   可以是连续实数或换句话说，可以假设任何值的参数。例如：学习率、正则化参数

+   可以离散但理论上可以假设无限数量的值的参数。例如：隐藏层的数量、每层的神经元数量或训练轮数

+   可以离散且只能假设有限数量的可能性的参数。例如：优化器、激活函数、学习率衰减方法。

对于类别 3，除了尝试所有可能性之外，没有太多可做的事情。通常，这些参数将完全改变模型本身，因此无法模拟它们的影响，这使得测试成为唯一的选择。这也是经验可能最有帮助的类别。众所周知，Adam 优化器几乎是最佳选择，例如，因此你可以在一开始就集中精力在其他地方。对于类别 1 和 2，这要困难一些，我们不得不想出一些聪明的想法来找到最佳值。

## 样本黑盒问题

为了尝试解决一个黑盒问题，让我们创建一个“假”的黑盒问题。问题是这样的：给定公式，找到函数 *f*(*x*) 的最大值

![$$ {\displaystyle \begin{array}{l}g(x)=\cos \frac{x}{4}-\sin \frac{x}{4}-\frac{5}{2}\cos \frac{x}{2}+\frac{1}{2}\sin \frac{x}{2}\\ {}h(x)=-\cos \frac{x}{3}-\sin \frac{x}{3}-\frac{5}{2}\cos \frac{2}{3}x+\frac{1}{2}\sin \frac{2}{3}x\\ {}f(x)=10+g(x)+\frac{1}{2}h(x)\end{array}} $$](img/463356_1_En_7_Chapter_TeX_Equc.png)

假装不知道公式本身。这个公式将允许我们检查我们的结果，但我们将假装它们是未知的。你可能想知道为什么我们使用如此复杂的公式。我想有一个带有几个极大值和极小值的函数，以便给你一个关于方法如何在非平凡示例上工作的概念。*f*(*x*) 可以用以下 Python 代码实现

```py
def f(x):
tmp1 = -np.cos(x/4.0)-np.sin(x/4.0)-2.5*np.cos(2.0*x/4.0)+0.5*np.sin(2.0*x/4.0)
tmp2 = -np.cos(x/3.0)-np.sin(x/3.0)-2.5*np.cos(2.0*x/3.0)+0.5*np.sin(2.0*x/3.0)
return 10.0+tmp1+0.5*tmp2
```

最大值大约在 *x* = 69.18 处，其值为 15.027。我们的挑战是以最有效的方式找到这个最大值，而不了解 *f*(*x*) 的任何信息，除了我们想要点的值。当我们说“有效”时，当然是指尽可能少的评估次数。

## 网格搜索

我们首先考虑的方法是网格搜索，也是最不“智能”的方法。网格搜索只是简单地尝试在规则间隔处尝试函数，并查看对于哪个 *x*，函数 *f*(*x*) 取得最高值。在这个例子中，我们想要找到在两个 *x* 值 *x*[*min*] 和 *x*[*max*] 之间的函数 *f*(*x*) 的最大值。我们将做的是简单地取 *x*[*min*] 和 *x*[*max*] 之间等间距的 *n* 个点，并在这些点上评估函数。我们将定义一个点向量

![x=({x}_{min},{x}_{min}+\frac{\varDelta x}{n},\dots, {x}_{min}+\left(n-1\right)\frac{\varDelta x}{n})](img/463356_1_En_7_Chapter_TeX_Equd.png)

其中我们定义 Δ*x* = *x*[*max*] - *x*[*min*]。然后我们在这些点上评估函数 *f*(*x*)，得到一个包含值的向量 ***f***

![$$ f=\left(f\left({x}_{min}\right),f\left(\ {x}_{min}+\frac{\varDelta x}{n}\right),\dots, f\left({x}_{min}+\left(n-1\right)\frac{\varDelta x}{n}\right)\right) $$](img/463356_1_En_7_Chapter_TeX_Eque.png)

对最大值 ![$$ \left(\tilde{x},\tilde{f}\right) $$](img/463356_1_En_7_Chapter_TeX_IEq1.png) 的估计如下

![$$ \tilde{f}=\underset{0\le i\le n-1}{\max }{f}_i $$](img/463356_1_En_7_Chapter_TeX_Equf.png)

并且，假设最大值在 ![$$ i=\tilde{i} $$](img/463356_1_En_7_Chapter_TeX_IEq2.png) 处找到，我们还将有

![$$ \tilde{x}={x}_{\mathrm{min}}+\frac{\tilde{i}\varDelta x}{n} $$](img/463356_1_En_7_Chapter_TeX_Equg.png)

现在，正如您可能想象的那样，您使用的点越多，您的最大估计值就越准确。问题是，如果对 *f*(*x*) 的评估代价高昂，您将无法像您希望的那样使用那么多点。您需要在点数和精度之间找到一个平衡。让我们通过我之前描述的函数 *f*(*x*) 来探索一个例子。让我们考虑 *x*[*max*] = 80 和 *x*[*min*] = 0，并且让我们取 *n* = 40 个点。我们将有 ![$$ \frac{\varDelta x}{n}=2 $$](img/463356_1_En_7_Chapter_TeX_IEq3.png)。我们可以用以下代码轻松地在 Python 中创建 ***x*** 向量：

```py

```

数组 `gridsearch` 将看起来像这样：


### 注意

网格搜索是一种在黑盒函数便宜时才有效的方法。为了得到好的结果，通常需要大量的采样点。

为了确保你真的得到了最大值，你应该减小步长 Δ*x*，或者增加采样点的数量，直到你找到的最大值不再明显变化。在先前的例子中，如图 7-4 右侧的图所示，当我们的步长 Δ*x* 小于大约 2.0 时，或者换句话说，当采样点的数量大于或大约等于 40 时，我们确信我们已经接近最大值。记住：一开始 40 可能看起来相当少，但如果 *f*(*x*) 评估的是你的深度学习模型的度量标准，并且训练需要 2 小时，例如，你就是在查看 3.3 天的计算机时间。通常，在深度学习领域，2 小时对于训练一个模型来说并不多，所以在开始长时间网格搜索之前，先快速计算一下。此外，请记住，在进行超参数调整时，你是在一个多维空间中移动（你不仅仅是在优化一个参数，而是许多参数），所以所需的评估数量会迅速增加。

让我们快速举一个例子。假设你决定你可以承担 50 次你的黑盒函数的评估。如果你决定你想尝试以下超参数：

+   优化器（RMSProp、Adam 或普通 GD）（3 个值）

+   迭代次数（1000、5000 或 10,000）（3 个值）

您现在已经看到了九个评估结果。那么您能尝试多少个学习率的值呢？只有五个！而且只有五个值，您不太可能接近最优值。这个例子旨在帮助您理解网格搜索仅适用于成本较低的黑盒函数。请记住，时间往往不是唯一的问题。例如，如果您正在使用 Google 云平台来训练您的网络，您将按秒支付您使用的硬件费用。也许您有很多时间可以利用，但成本可能会很快超过您的预算。

## 粗到细优化

仍然有一个优化技巧可以帮助网格或随机搜索。它被称为粗到细优化。假设我们想在 *x*[*min*] 和 *x*[*max*] 之间找到 *f*(*x*) 的最大值。我将解释随机搜索背后的概念，但这也适用于网格搜索。以下步骤为你提供了进行此优化的算法。

1.  在区域 *R*[1] = (*x*[*min*], *x*[*max*]) 中进行随机搜索。让我们用 (*x*[1], *f*[1]) 表示找到的最大值。

1.  现在考虑一个围绕 *x*[1] 的较小区域 *R*[2] = (*x*[1] − *δx*[1], *x*[1] + *δx*[1])，其中 *δx*[1] 我稍后会讨论，并在该区域再次进行随机搜索。假设，当然，真实最大值位于这个区域。我们将用 (*x*[2], *f*[2]) 表示在这里找到的最大值。

1.  在 *x*[2] 附近重复步骤 2，在我们将用 *R*[3] 表示的区域，使用比 *δx*[1] 更小的 *δx*[2]，并在此步骤中找到的最大值用 (*x*[3], *f*[3]) 表示。

1.  现在重复步骤 2，在 *x*[3] 附近，在我们将用 *R*[4] 表示的区域，使用比 *δx*[2] 更小的 *δx*[3]。

1.  按照这种方式，根据需要重复进行，直到区域 *R*[*i* + 1] 中的最大值 (*x*[*i*], *f*[*i*]) 不再改变。

通常情况下，只需要进行一到两次迭代，但理论上，你可以进行大量的迭代。这种方法的问题在于，你无法真正确定你的真实最大值是否位于你的区域 *R*[*i*] 中。但如果这种方法有效，它有一个很大的优点。让我们考虑我们进行标准随机搜索的情况。如果我们想要样本点之间的平均距离为 *x*[*max*] − *x*[*min*] 的 1%，那么如果我们只进行一次随机搜索，就需要大约 100 个点，因此我们必须进行 100 次评估。现在让我们考虑我刚才描述的算法。我们可以从区域 *R*[1] = (*x*[*min*],  *x*[*max*]) 中的 10 个点开始。在这里，我们将用 (*x*[1], *f*[1]) 表示我们找到的最大值。然后让我们取 ![$$ 2\delta x=\frac{x_{max}-{x}_{min}}{10} $$](img/463356_1_En_7_Chapter_TeX_IEq12.png) 并再次在区域 *R*[2] = (*x*[1] − *δx*, *x*[1] + *δx*) 中取 10 个点。在 (*x*[1] − *δx*, *x*[1] + *δx*) 区间内，我们将有平均距离为 *x*[*max*] − *x*[*min*] 的 1%，但我们只采样了函数 20 次，而不是 100 次，所以减少了 5 倍！例如，让我们用 10 个点在 *x*[*min*] = 0 和 *x*[*max*] = 80 之间采样我们之前使用的函数，代码如下

```py
np.random.seed(5)
randomsearch = np.random.random([10])*80
x = 0
m = 0.0
for i, val in np.ndenumerate(f(randomsearch)):
if (val > m):
m = val
x = randomsearch[i]
```

这给我们提供了 *x*[1] = 69.65 和 *f*[1] = 14.89 的最大位置和值，还不错，但还没有像真实值 69.18 和 15.027 那样精确。现在让我们再次在区域 *R*[2] = (*x*[1] − 4, *x*[1] + 4) 附近对找到的最大值进行采样。

```py
randomsearch = x + (np.random.random([10])-0.5)*8
x = 0
m = 0.0
for i, val in np.ndenumerate(maxim(randomsearch)):
if (val > m):
m = val
x = randomsearch[i]
```

这给出了结果 69.189 和 15.027。相当精确的结果，只对函数进行了 20 次评估。如果我们用 *500*（比我们刚才做的多 25 倍）个采样点进行简单的随机搜索，我们得到 *x*[1] = 69.08 和 *f*[1] = 15.022。这个结果展示了这个技巧如何真正有用。但记住风险：如果你的最大值不在你的区域 *R*[*i*] 中，你将永远找不到它，因为你仍在处理随机数。所以，总是选择相对较大的区域 (*x*[*i*] − *δx*[*i*]，*x*[*i*] + *δx*[*i*]) 是一个好主意，以确保它们包含你的最大值。大小取决于，就像深度学习世界中的几乎所有事情一样，你的数据集和问题，这些可能事先无法知道。不幸的是，需要进行测试。在图 7-9 中，你可以看到函数 *f*(*x*) 上的采样点。在左侧的图中，你可以看到前 10 个点，在右侧，有额外的 10 个点的区域 *R*[2]。左侧图中的小矩形标记了 *x* 区域 *R*[2]。

## 贝叶斯优化

在本节中，我们将探讨一种寻找黑盒函数最小值（或最大值）的具体且有效的方法。这是一个相当巧妙的算法，基本上包括以比随机选择或网格选择更智能的方式选择采样点来评估函数。要理解它是如何工作的，你必须首先了解一些新的数学概念，因为这种方法并不简单，需要理解一些更高级的概念。让我们从 Nadaraya-Watson 回归开始。

### Nadaraya-Watson 回归

### 高斯过程预测

现在我们已经到达了有趣的部分：给定向量 ***f***，我们如何估计任意点 *x* 处的 *f*(*x*)？因为我们处理的是随机过程，我们将估计未知函数取给定值 *f*(*x*) 的概率。从数学上讲，我们将预测以下量：

![$$ p\left(f(x)\ |\ f\right) $$](img/463356_1_En_7_Chapter_TeX_Equq.png)

或者换句话说，给定由所有点 *f*(*x*[1])，…，*f*(*x*[*n*])组成的向量 ***f***，得到值 *f*(*x*) 的概率。

假设 *f*(*x*) 是一个平稳高斯过程，其 *m* = 0，预测可以证明由以下公式给出

![$$ p\left(f(x)\ |\ f\right)=\frac{p\left(f(x),f\left({x}_1\right),\dots, f\left({x}_n\right)\right)}{p\left(f\left({x}_1\right),\dots, f\left({x}_n\right)\right)}=\frac{\mathcal{N}\left(f(x),f\left({x}_1\right),\dots, f\left({x}_n\right)\ |\ 0,\kern0.375em \tilde{C}\right)}{\mathcal{N}\left(f\left({x}_1\right),\dots, f\left({x}_n\right)\ |\ 0,\kern0.375em C\right)} $$](img/463356_1_En_7_Chapter_TeX_Equr.png)

```py
f_ = (f1-np.average(f1))
k = K(x-randompoints, 2 , sigm_)
C = np.zeros([randompoints.shape[0], randompoints.shape[0]])
Ctilde = np.zeros([randompoints.shape[0]+1, randompoints.shape[0]+1])
for i1,x1_ in np.ndenumerate(randompoints):
for i2,x2_ in np.ndenumerate(randompoints):
C[i1,i2] = K(x1_-x2_, 2.0, sigm_)
Ctilde[0,0] = K(0, 2.0, sigm_)
Ctilde[0,1:randompoints.shape[0]+1] = k.T
Ctilde[1:,1:] = C
Ctilde[1:randompoints.shape[0]+1,0] = k
mu = np.dot(np.dot(np.transpose(k), np.linalg.inv(C)), f_)
sigma2 = K(0, 2.0,sigm_)- np.dot(np.dot(np.transpose(k), np.linalg.inv(C)), k)
ybayes.append(mu)
sigmabayes_.append(np.abs(sigma2))
ybayes = np.asarray(ybayes_)+np.average(f1)
sigmabayes = np.asarray(sigmabayes_)
```

现在请花些时间来理解它。在`ybayes`列表中，我们将找到在`xsampling`数组中的值上评估的*μ*(*x*)的值。以下是一些有助于你理解代码的提示：

+   我们在一个*x*点的范围内进行循环，其中我们想要评估我们的函数，使用代码`for x in xsampling:`。

+   我们使用代码为向量的每个元素构建 ***k*** 和 ***f*** 向量：`k = K(x-randompoints, 2 , sigm_)` 和 `f1 = f(randompoints)`。对于核，我们为参数 `l` 选择了一个值，如函数 `2` 中定义的那样。我们在向量 ***f*** 中减去平均值，以获得 *m*(*x*) = 0，以便能够应用推导出的公式。

+   我们构建矩阵 *C* 和 ![~C](img/463356_1_En_7_Chapter_TeX_IEq16.png)。

+   我们使用 `mu = np.dot(np.dot(np.transpose(k), np.linalg.inv(C)), f_)` 和标准差来计算 *μ*。

+   最后，我们以相反的顺序重新应用我们为使过程平稳所做的所有转换，简单地将 *f*(*x*) 的平均值再次添加到评估的代理函数 `ybayes = np.asarray(ybayes_)+np.average(f1)` 中。

让我们暂停一下，思考一下我们为什么要做所有这些。目的是找到一个所谓的代理函数 ![~f](img/463356_1_En_7_Chapter_TeX_IEq17.png)，它近似我们的函数 *f* 并具有以下属性

![~f 的最大值 ≈ f 的最大值](img/463356_1_En_7_Chapter_TeX_Equy.png)

这个代理函数必须具有另一个非常重要的属性：它必须易于评估。这样，我们就可以轻松找到 ![~f](img/463356_1_En_7_Chapter_TeX_IEq18.png) 的最大值，并且，使用前面的属性，我们将得到 *f* 的最大值，根据假设，这是昂贵的。

但正如你所看到的，知道我们是否有足够的数据点来找到最大值的正确值可能非常困难。毕竟，根据定义，我们根本不知道我们的黑盒函数看起来像什么。那么，如何解决这个问题呢？

该方法背后的主要思想可以通过以下算法来描述：

1.  我们从一个小的样本点集合开始，这些点随机选择（它应该有多小将取决于你的问题）。

1.  我们使用这个点集来获取一个代理函数，如描述的那样。

1.  我们使用一种特定的方法（我将在稍后讨论）向我们的集合中添加一个额外的点，并重新评估代理函数。

1.  如果我们用代理函数找到的最大值继续变化，我们将继续添加点，就像步骤 3 中那样，直到最大值不再变化，或者我们耗尽时间或预算，无法进行任何进一步的评估。

如果我在步骤 3 中暗示的方法足够聪明，我们将能够相对快速和准确地找到我们的最大值。

### 获取函数

但如何在上一节步骤 3 中提到的附加点中进行选择呢？思路是使用所谓的获取函数。算法以这种方式工作：

1.  我们选择一个函数（我们将在一会儿看到一些可能性），称为获取函数。

1.  然后我们选择，作为额外的点 *x*，获取函数达到最大值的位置。

我们可以使用几种获取函数。在这里，我将只描述我们将使用的一个来了解这种方法是如何工作的，但还有几个你可能想检查的，例如熵搜索、改进概率、期望改进和上置信界。

### 上置信界 (UCB)

在文献中，你可以找到这种获取函数的两个变体。我们可以将函数写成

![$$ {a}_{UCB}(x)=\mathbb{E}\tilde{f}(x)+\eta \sigma (x) $$](img/463356_1_En_7_Chapter_TeX_Equz.png)

其中，我们用 ![$$ \mathbb{E}\tilde{f}(x) $$](img/463356_1_En_7_Chapter_TeX_IEq19.png) 表示代理函数在我们问题中的 *x* 范围内的“期望”值。期望值不过是函数在给定 *x* 范围内的平均值。*σ*(*x*) 是我们在点 *x* 处使用我们的方法计算的代理函数的方差。我们选择的新点是 *a**UCB* 最大的点。*η* > 0 是一个权衡参数。这个获取函数基本上选择方差最大的点。请参阅图 7-15。该方法选择方差较大的点，因此尽可能远离我们已有的点。这样，近似值往往会变得越来越好。

UCB 获取函数的另一种变体如下：

![$$ {\tilde{a}}_{UCB}(x)=\tilde{f}\left(\mathrm{x}\right)+\eta \sigma (x) $$](img/463356_1_En_7_Chapter_TeX_Equaa.png)

这次，获取函数将在选择代理函数最大值周围的点和其方差最大的点之间进行权衡。第二种方法最适合快速找到*f*的最大值的良好近似，而第一种方法则倾向于在整个*x*范围内给出*f*的良好近似。在下一节中，我们将看到这些方法是如何工作的。

### 示例

让我们从本章开头描述的复杂三角函数开始，并考虑*x*的范围[0, 40]。我们的目标是找到它的最大值并近似该函数。为了方便我们的编码，让我们定义两个函数：一个用于评估代理函数，另一个用于评估新点。为了评估代理函数，我们可以使用以下函数：

```py
def get_surrogate(randompoints):
ybayes_ = []
sigmabayes_ = []
for x in xsampling:
f1 = f(randompoints)
sigm_ = np.std(f_)**2
f_ = (f1-np.average(f1))
k = K(x-randompoints, 2.0, sigm_ )
C = np.zeros([randompoints.shape[0], randompoints.shape[0]])
Ctilde = np.zeros([randompoints.shape[0]+1, randompoints.shape[0]+1])
for i1,x1_ in np.ndenumerate(randompoints):
for i2,x2_ in np.ndenumerate(randompoints):
C[i1,i2] = K(x1_-x2_, 2.0, sigm_)
Ctilde[0,0] = K(0, 2.0)
Ctilde[0,1:randompoints.shape[0]+1] = k.T
Ctilde[1:,1:] = C
Ctilde[1:randompoints.shape[0]+1,0] = k
mu = np.dot(np.dot(np.transpose(k), np.linalg.inv(C)), f_)
sigma2 = K(0, 2.0, sigm_)- np.dot(np.dot(np.transpose(k), np.linalg.inv(C)), k)
ybayes_.append(mu)
sigmabayes_.append(np.abs(sigma2))
ybayes = np.asarray(ybayes_)+np.average(f1)
sigmabayes = np.asarray(sigmabayes_)
return ybayes, sigmabayes
```

此函数与我们在前几节示例中已经讨论过的代码相同，但它被封装在一个函数中，该函数返回包含在数组`ybayes`中的代理函数，以及包含在数组`sigmabayes`中的*σ*²。此外，我们还需要一个函数来评估新点，使用获取函数*a**UCB*。我们可以很容易地用以下函数得到它

```py
def get_new_point(ybayes, sigmabayes, eta):
idxmax = np.argmax(np.average(ybayes)+eta*np.sqrt(sigmabayes))
newpoint = xsampling[idxmax]
return newpoint
```

为了使事情更简单，我决定在函数外部定义包含所有我们想要*x*值的数组。让我们从六个随机选择的点开始。为了检查我们的方法表现如何，让我们先定义一些东西。

### 注意

不同的获取函数类型在近似黑盒函数时将给出不同的策略。例如，*a**UCB* 将在方差最高的区域添加点，而 ![~a_{UCB}(x)](img/463356_1_En_7_Chapter_TeX_IEq45.png) 将通过 *η* 调节代理函数的最大值和具有高方差区域之间的平衡来添加点。

对所有不同类型的获取函数的分析将超出本书的范围。要获得足够的经验并了解不同的获取函数如何工作和表现，需要进行大量的研究和阅读已发表的论文。

如果你想在 TensorFlow 模型中使用贝叶斯优化，你不必完全从头开始开发该方法。你可以尝试由 Nicolas Knudde 等人描述的库 GPflowOpt，该库在论文“GPflowOpt: A Bayesian Optimization Library using TensorFlow”中有描述，可在 [`goo.gl/um4LSy`](https://goo.gl/um4LSy) 或 [arXiv.org](http://arxiv.org) 上找到。

## 对数尺度上的采样

有一个最后的微小细节我想讨论一下。有时，你可能会发现自己处于想要尝试一个参数可能值的广泛范围的情况，但你根据经验知道最佳值可能在一个特定的范围内。假设你想找到你模型的学习率最佳值，你决定测试从 10^(-4) 到 1 的值，但你知道，或者至少怀疑，最佳值可能位于 10^(-3) 和 10^(-4) 之间。现在假设你正在使用网格搜索，并采样 1000 个点。你可能认为你有了足够的点，但你将会得到

+   介于 10^(-4) 和 10^(-3) 之间的 0 个点

```py
def build_model(number_neurons):
n_dim = 784
tf.reset_default_graph()
# Number of neurons in the layers
n1 = number_neurons # Number of neurons in the hidden layer
n2 = 10 # Number of neurons in output layer
cost_history = np.empty(shape=[1], dtype = float)
learning_rate = tf.placeholder(tf.float32, shape=())
X = tf.placeholder(tf.float32, [n_dim, None])
Y = tf.placeholder(tf.float32, [10, None])
W1 = tf.Variable(tf.truncated_normal([n1, n_dim], stddev=.1))
b1 = tf.Variable(tf.constant(0.1, shape = [n1,1]) )
W2 = tf.Variable(tf.truncated_normal([n2, n1], stddev=.1))
b2 = tf.Variable(tf.constant(0.1, shape = [n2,1]))
# Let's build our network...
Z1 = tf.nn.relu(tf.matmul(W1, X) + b1) # n1 x n_dim * n_dim x n_obs = n1 x n_obs
Z2 = tf.matmul(W2, Z1) + b2 # n2 x n1 * n1 * n_obs = n2 x n_obs
y_ = tf.nn.softmax(Z2,0) # n2 x n_obs (10 x None)
cost = - tf.reduce_mean(Y * tf.log(y_)+(1-Y) * tf.log(1-y_))
optimizer = tf.train.GradientDescentOptimizer(learning_rate).minimize(cost)
init = tf.global_variables_initializer()
return optimizer, cost, y_, X, Y, learning_rate
```

你应该理解这个函数，因为我们已经在书中多次使用过这段代码。这个函数有一个输入参数：`number_neurons`，它将包含隐藏层中的神经元数量。但有一个小的区别：函数返回我们在训练期间必须引用的张量，例如，当我们想要在训练期间评估`cost`张量时。如果我们不将它们返回给调用者，它们将只在这个函数内部可见，我们将无法训练这个模型。训练模型的函数看起来像这样：

```py
def model(minibatch_size, training_epochs, features, classes, logging_step = 100, learning_r = 0.001, number_neurons = 15):
opt, c, y_, X, Y, learning_rate = build_model(number_neurons)
sess = tf.Session()
sess.run(tf.global_variables_initializer())
cost_history = []
for epoch in range(training_epochs+1):
for i in range(0, features.shape[1], minibatch_size):
X_train_mini = features[:,i:i + minibatch_size]
y_train_mini = classes[:,i:i + minibatch_size]
sess.run(opt, feed_dict = {X: X_train_mini, Y: y_train_mini, learning_rate: learning_r})
cost_ = sess.run(c, feed_dict={ X:features, Y: classes, learning_rate: learning_r})
cost_history = np.append(cost_history, cost_)
if (epoch % logging_step == 0):
print("Reached epoch",epoch,"cost J =", cost_)
correct_predictions = tf.equal(tf.argmax(y_,0), tf.argmax(Y,0))
accuracy = tf.reduce_mean(tf.cast(correct_predictions, "float"))
accuracy_train = accuracy.eval({X: train, Y: labels_, learning_rate: learning_r}, session = sess)
accuracy_dev = accuracy.eval({X: dev, Y: labels_dev_, learning_rate: learning_r}, session = sess)
return accuracy_train, accuracy_dev, sess, cost_history
```

你已经多次看到过与这个非常相似的功能。主要部分应该是清晰的。你将发现一些新的东西。首先，我们在函数内部本身构建模型，

```py

```

## 关于径向基函数的简要说明

在完成本章之前，我想讨论一下关于径向基函数的一个小问题

![径向基函数公式](img/463356_1_En_7_Chapter_TeX_Equab.png)

理解参数 *l* 的作用非常重要。在示例中，我选择了 *l* = 2，但我没有讨论为什么这样做。原因是这样的。选择过小的 *l* 将会使获取函数在已有的点周围形成非常狭窄的峰值，如图 7-27 左边的图所示。大的 *l* 值将对获取函数产生平滑效果，如图 7-27 中间和右边的图所示。

![图 7-27](img/463356_1_En_7_Fig27_HTML.jpg)

图 7-27

改变径向基函数中的参数 *l* 的影响

通常，避免使用过小或过大的 *l* 值是一个好的实践，以便能够使方差在已知点之间以平滑的方式变化，如图 7-27 所示，对于 *l* = 1。一个非常小的 *l* 将会使点之间的方差几乎保持不变，因此，算法几乎总是选择点之间的中点，正如你可以从获取函数中看到的那样。选择一个大的 *l* 将会使方差变小，因此，对于某些获取函数来说，可能难以使用。如图 7-27 中 *l* = 5 所示，获取函数几乎保持不变。常用的典型值大约是 1 或 2。
