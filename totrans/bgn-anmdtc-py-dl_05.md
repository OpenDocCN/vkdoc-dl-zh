## 深度学习简介

深度学习是机器学习的一个子集，它涉及使用神经网络来执行建模任务。神经网络可以由成百上千层相互连接的神经元组成，形成一个“深度”架构，因此得名“深度学习”。

如第四章所述，“传统”机器学习算法是非神经网络架构，如 k-最近邻、支持向量机、随机森林、决策树等。这些模型仍然相当流行，尤其是在 XGBoost（梯度提升树）等形式中，它们在结构化、表格数据上与神经网络保持相当竞争力。其他仍在使用的流行算法包括聚类算法和树。尽管深度学习模型可能受到关注，但传统机器学习算法并未过时，仍然有很好的用途，尤其是在第三章讨论的案例中。

### 什么是深度学习？

深度学习是机器学习的一个子领域，它结合了人工神经元的概念。深度学习与传统机器学习在一般意义上的区别在于，它使用许多相互连接的人工神经元层来构成一个“深度”网络，而不是只有一层或两层的“浅层”网络。层数可以从几个到几百个不等。深度与非线性（本章后面将介绍）的结合使得深度学习模型能够学习执行高度复杂的任务，这些任务直到最近只有人类能够完成，例如高级聊天机器人、生成极其令人印象深刻的艺术作品和自动驾驶。近年来，媒体广泛报道的许多令人印象深刻的突破性技术，如 ChatGPT、深度伪造、语音合成和 AI 艺术，都是由深度学习驱动的。尽管深度学习与机器学习略有不同，本质上更复杂，但机器学习的基本原理仍然适用。

在 Python 中，深度学习主要通过两个流行的框架实现：PyTorch 和 TensorFlow。Keras 也很受欢迎，但自 2.0 版本以来已集成到 TensorFlow 中。从初学者的角度来看，TensorFlow 2.0+ Keras 模块是一个很好的起点。它从你那里抽象掉了深度学习建模工作流程的许多复杂性，让你只需几行代码就能定义模型，并通过一个函数进行训练和推理。这一特性也使得 Keras 成为快速原型设计的优秀工具，允许你进行快速实验，或者在你只需要一个标准的深度学习模型来完成某个任务时，转向一个更容易使用的 API。

如果你需要深入挖掘并定义诸如自定义神经网络层、损失函数和神经网络架构等事物，或者只是需要在低级别上实现工作流程，你可以使用 TensorFlow API 的完整功能，而不仅仅是 Keras。

相比之下，PyTorch 比 Keras 更少抽象化，需要你从头开始定义模型、训练循环和推理循环。你不需要重新实现常用的层，但 PyTorch 不像 Keras 那样“手把手”地引导你。然而，由于 PyTorch 的 Pythonic API 和比 TensorFlow API 更简单的语法，它仍然直观且相对容易使用。由于其灵活性和易用性，PyTorch 也受到研究人员的喜爱。

从历史上看，在 TensorFlow 2.0 集成 Keras 并简化其语法之前，TensorFlow 的语法相当复杂，难以理解。这对初学者来说是一个障碍，因为他们需要克服陡峭的学习曲线才能开始，对研究人员来说也是如此，因为他们必须处理所有的语法，以及一个更难以调试模型的运行环境。当 PyTorch 可用时，它迅速获得了人气。尽管如此，这两个框架至今仍然非常受欢迎，尽管 TensorFlow 在使用历史和社区支持方面占有优势。

这两个框架都适合在工业环境中使用，尽管 TensorFlow 可能由于能够部署到浏览器（通过 TensorFlow.js）和移动设备（TensorFlow Lite）而略占优势。PyTorch 通过其自身的部署能力，如 PyTorch Mobile，正在迅速缩小差距。

在本章中，我们将介绍 PyTorch 和 Keras，并在后续章节的 GitHub 仓库中提供任一框架的代码。然而，由于简洁性，我们将转向 Keras（TensorFlow 2.0+）。

这些神经元之间形成连接，其中树突寻求与末梢轴突的连接。一旦这些连接，称为 *突触*，建立起来，信号就可以根据它从连接的其他神经元接收到的输入信号在单个神经元中传递。图 5-2 展示了两个神经元之间的连接。

![图 5-2](img/483137_2_En_5_Fig2_HTML.jpg)

两个神经元的插图。第一个轴突的末梢分支与第二个神经元的树突相连接。

图 5-2

两个神经元之间的连接，称为突触

突触是两个神经元之间的特殊连接，允许信号从一个神经元传递到另一个神经元。参见图 5-2，当突触前神经元（左侧）放电时，它将信号向外发送到末梢轴突，这个信号被与突触前神经元形成突触的突触后神经元的树突（右侧）拾取。什么决定了神经元是否放电？突触后神经元树突收集的信号输入必须集体通过这个神经元特有的激活阈值，以允许信号传递到其他神经元。

这是对微观层面上真正发生的事情的简化，尽管这足以启发人工神经元背后的原理以及它建立的结构。图 5-3 展示了人工神经元的外观。

![](img/483137_2_En_5_Fig3_HTML.png)

图 5-3

人工神经元结构。输入，从 x[1] 到 x[n] 的向量，与权重向量 w[1] 到 w[n] 元素相乘，然后相加。然后将偏置项 b 偏移，结果通过一个“激活函数”，这是一个基于输入的幅度和符号的数学函数，它根据输入的幅度和符号产生输出信号。一个例子是简单的阶跃函数，如果组合输入超过阈值，则输出 1，否则输出 0。这些现在形成了输出，y[1] 到 y[m]。这个 y 向量现在可以作为另一个神经元的输入

### 激活函数

激活函数是将输入信号映射到某种形式的输出信号，以便后续神经元进行解释的一种方式。如果你不使用激活函数（有时称为“线性激活函数”），那么仿射变换的输出只是神经元的最终输出。

然而，我们想要模拟超出线性建模范围的复杂函数，因此我们想要使用一个将输入映射到某些非线性空间的非线性激活函数。此外，大多数任务和数据都是非线性的，所以如果我们将这些输入映射到线性空间，我们将会严重欠拟合。此类非线性激活函数的例子包括 sigmoid 和双曲正切。

使用非线性激活函数的另一个原因是，如果我们只使用线性激活函数，即使我们有一个深度神经网络，它实际上也只会变成单层。神经网络是一系列线性变换。这个线性变换的序列本身可以用原始输入重写，只进行一次线性变换。

让我们从**sigmoid**激活函数开始，这是一种简单引入非线性方法。如果你想知道如何生成本节中展示的图表，请跟随本节中的笔记本[`https://github.com/apress/beginning-anomaly-detection-python-deep-learning-2e/blob/master/Chapter%205%20Introduction%20to%20Deep%20Learning/chapter5_activation_functions.ipynb`](https://github.com/apress/beginning-anomaly-detection-python-deep-learning-2e/blob/master/Chapter%205%20Introduction%20to%20Deep%20Learning/chapter5_activation_functions.ipynb)。

Sigmoid 函数的公式显示在方程 5-2 中。

![$$ f(x)=\frac{1}{1+{e}^{-x}} $$](img/483137_2_En_5_Chapter_TeX_Equb.png)

这是一个非线性函数，这意味着它不能通过尝试画线来击中所有点来充分建模。图 5-6 显示了 sigmoid 图形的样子。

![](img/483137_2_En_5_Fig6_HTML.jpg)

线图绘制 sigmoid 输出与 x 值的关系。曲线通过以下点：（负 4，0），（负 2，0.1），（0，0.5），（2，0.9），（4，1）。数值为估算值。

图 5-6

Sigmoid 函数的图像。输入值被映射到范围 (0, 1) 内。输入域接受任何和所有的 x 值，但非常大或非常小的 x 值分别趋近于 1 或 0。

如图 5-6 所示，sigmoid 激活函数将其输入映射到范围从 (0, 1) 的曲线输出，函数在其中心附近变化最快。正因为如此，sigmoid 激活函数非常适合用于建模概率分数；例如，“这张图片是狗的概率是多少？”

因此，sigmoid 适合在神经网络的最末端使用，将最后一层的原始输出映射到概率分数。sigmoid 也可以用作中间激活函数，用于网络的不同非输出层。

话虽如此，使用 sigmoid 也有一些问题：

+   **梯度消失**：图像两端的梯度将非常小。神经网络使用梯度来更新其参数，因此微小的梯度会导致微小的更新，从而减慢学习过程，甚至在极端情况下停止。

+   **非零中心**：输出空间本身不是零中心。这对神经网络总体上有几个影响：

    +   **关闭“神经元”**需要负权重且幅度大，以输出接近零。权重通常应该幅度较小，以便模型稳定。否则，神经元输入幅度较大时，可能会过度激活或过度失活神经元，而它本不应该这样做。

    +   **接近零的输入仍然传递信号**，因为如果 x 为零，输出仍然会是 0.5。如果激活函数以零为中心，那么零输入也会产生零输出。结合前一点，如果关闭一个神经元需要权重的大小很小且输入本身接近零，这对模型来说既高效又稳定。

    +   **神经元输出将有自己的分布**，这意味着后续层将不得不在整个训练过程中不断学习前一层的激活输出分布，因为权重在更新。这被称为 **内部协变量偏移**，这无疑是 sigmoid 激活函数的一个潜在问题。

    +   **Zig-zag 梯度更新**。我们通过找到损失函数相对于同一权重的梯度来更新一个权重。在一个具有后续 sigmoid 层的神经网络中，每一层的输入将始终为正，包括梯度。因此，最终梯度步骤的方向完全取决于损失函数相对于输出的梯度。这导致梯度步骤倾向于呈之字形，导致优化效率低下，以及可能收敛性较差。

        由于输出空间以零为中心，这个问题在很大程度上得到了解决，因为激活输出现在可以是正的或负的。由于最终梯度的符号不再仅取决于相对于输出的损失梯度，梯度步可以更加直接，不会像以前那样振荡。

+   **计算成本高**：由于指数运算，sigmoid 函数本身可能有点计算成本高。与简单的激活函数（如我们很快将要介绍的 ReLU）相比，它可能会稍微减慢训练速度。

在实践中，sigmoid 并不是在神经网络层中使用的好激活函数，尽管它作为最终层激活函数来预测概率分数是有用的。你可以有一个单一输出（异常的概率）或者有多个输出类别，每个类别都携带一个概率分数（例如，给定健康指标，疾病 X 的概率，疾病 Y 的概率）。

然而，由于不是以零为中心，这导致了许多问题，因此让我们看看一个类似的以零为中心的函数：双曲正切（**tanh**）函数。其公式由方程 5-3 给出。

![$$ f(x)=\frac{e^x-{e}^{-x}}{e^x+{e}^{-x}} $$](img/483137_2_En_5_Chapter_TeX_Equc.png)

**方程 5-3.** 双曲正切函数的公式

tanh 图像与 sigmoid 图像非常相似，但有一些不同，如图 5-7 所示。

![](img/483137_2_En_5_Fig7_HTML.jpg)

一条线图绘制了 tanh 输出与 x 值的关系。曲线通过以下点：（负 4，0），（负 2，0），（0，0.5），（0，0），（2，1），（4，1）。值是估计的。

图 5-7

双曲正切函数的图像。输入值被映射到输出范围（-1, 1）。当 x 值非常大或非常小时，分别接近 1 或-1。

如图 5-7 所示，tanh 图像以零为中心，并且关于原点对称。由于以零为中心，tanh 相对于 sigmoid 具有以下优势：

+   **没有锯齿形更新**：由于激活输出现在可以是正的或负的，梯度更新不会锯齿形。后续层现在有正负输入，因此层内梯度不再仅是正的。

+   **更好的分布**：以零为中心意味着神经元输出的分布更接近于零的均值，与 sigmoid 不同。与 sigmoid 相比，输出现在在一个更一致的尺度上，尽管内部协变量偏移仍然是潜在问题。均值可能接近零意味着后续层不需要持续学习偏移，因此学习变得更加稳定。

+   **接近零的输入具有接近零的输出**：由于 tanh 的这一特性，关闭神经元变得非常简单，无需使用非常大的权重将输入 x 值转换为非常负的值，以便输出接近零的值，就像 sigmoid 的情况一样。

虽然 tanh 并不完美，但从实际应用的角度来看，将其用作隐藏层激活函数比使用 sigmoid 更可取。然而，tanh 仍然存在梯度消失的问题，因为如果 x 输入的幅度非常大，那么相应的梯度将几乎为零，这可能会阻碍学习。有一些处理这种问题的策略，但在实践中，有更多更适合的激活函数被更频繁地使用。其中一个例子是修正线性单元，或称为**ReLU**。

这就是 ReLU 为什么是一个如此好的激活函数的原因。不仅 ReLU 操作以及求导的计算效率更高，而且它还能作为非线性函数，使网络能够学习复杂函数。因此，ReLU 在实际情况中经常被使用也就不足为奇了。ReLU 有一些变体，如 LeakyReLU，它们不是将负值裁剪为零，而是允许一个非常小的信号通过。无论如何，ReLU 都是训练网络时的一个很好的起点。

与 tanh 相比，使用 ReLU 有以下优点：

+   **没有梯度陷阱**。在图形的任何极端处，梯度不再变得非常小，就像在 tanh 中那样。ReLU 中，梯度在 ReLU 为正的地方是 1，否则为 0。

+   **计算效率高**。没有指数需要计算。它是一个简单的掩码，将任何非正值设置为零。

然而，也有一些可能的缺点：

+   **爆炸输出**. ReLU 在其正上界没有上限。这意味着激活输出可以非常大，并可能导致整个网络不稳定，从而引起发散。这可以通过适当的初始化技术（如 Kaiming He 初始化）在一定程度上得到缓解，以保持学习更加稳定，但仍然存在可能性。

+   **死亡 ReLU**。由于输入小于零的梯度仅仅是零，产生这种输出的神经元不会得到更新。这可能导致许多“死亡”的神经元不再更新，从而导致网络效率低下。He 初始化可以帮助稳定网络以对抗死亡 ReLU，但它并不能完全解决这个问题。

+   **非零中心**。与 sigmoid 类似，值和梯度始终为正，这意味着可能会采取锯齿形的优化步骤。

然而，ReLU 的修改版本，如 LeakyReLU，可以帮助缓解，因为它们允许一些负输入通过（通过 max(0.01x, x)而不是像常规 ReLU 中的 max(0, x)），有助于缓解锯齿形梯度步骤以及死亡 ReLU 问题。

### 神经网络

现在你已经熟悉了神经元和激活函数，是时候深入挖掘前一部分提到的神经元层了。

神经网络中的一个 **层** 是由一些神经元组成的集合，每个神经元使用整个输入计算某个输出值。一个层的输出由该层中所有神经元的输出值组成。神经网络是一系列神经元层，其中一层的输出是下一层的输入。

如前所述，在 ReLU 的上下文中，神经网络的第 1 层是**输入层**，它接收训练数据作为输入。网络的最后一层是**输出层**，它输出用于预测网络被训练执行的任务的值。所有介于输入层和输出层之间的层被称为**隐藏层**。

一个示意图表示了神经网络架构。它有相互连接的节点，从左到右分别代表输入层、隐藏层 1、隐藏层 2 和输出层。

从数学意义上讲，整个网络是一系列链式矩阵乘法操作，并对它们应用激活函数。参考方程 5-5 和 5-6 来了解层计算是如何从数学上计算的。我们使用符号 z[L]表示层 L 的仿射变换输出，a[l]表示使用 z[L]作为输入的激活函数输出。现在 a[L]是层 L+1 的输入，成为 z[L+1]，然后是 a[L+1]。

当数据从最左侧层流向最右侧层时，这被称为**正向传播**。换句话说，正向传播是从某个给定输入计算最终输出的过程。

**反向传播**涉及从最右侧层逐层移动到最左侧层，使用反向传播算法来计算梯度。**反向传播**，简称“误差反向传播”，通过使用链式法则来避免重新计算所有中间项，从而计算每个层的可学习参数的梯度。由于这些过程相互关联，这两个术语有时可以互换使用。然而，区别在于反向传播严格上是逐层反向通过网络的过程，而反向传播（在反向传播中使用）是计算梯度的过程。

一旦计算完所有梯度，就会进行更新可学习参数的优化步骤。我们将在本章的“梯度下降和反向传播”部分中稍后讨论这一点。

现在我们来看数据在正向传播过程中如何通过网络，通过回顾图 5-18 在分类的上下文中的内容，从而产生输出预测。三个输出节点分别对应输出类别 0、1 和 2，分别代表“狗”、“猫”和“鸟”。假设输入数据，尽管在这种情况下只有四个节点宽，是狗、猫或鸟的图像的高度压缩表示。

为了说明，以下图示中假设使用的激活函数是 Sigmoid 函数，激活强度最大的神经元用黑色表示（激活输出接近 1），而激活强度最小的神经元用白色表示（激活输出接近 0）。激活强度更强的神经元会将信号传递到下一层，而激活强度很弱的神经元则不会将信号传递到下一层。

至于权重本身（由连接两个层之间神经元的线条[边]表示），与非常强的权重对应的边用粗线表示，而与较小权重对应的边用细线表示。边上的权重“强度”是以幅度来衡量的。粗线边上的权重可以是强烈负的或强烈正的。

记住，一个神经元与前一层的每个神经元都是全连接的，并且将为这些连接中的每一个分配不同的权重。如果前一个神经元的激活很可能激活后一个神经元，这两个神经元被认为有强烈的连接，这通过连接这两个神经元的边上的较高权重来表示。相反的情况也是真实的，你会在连接两个神经元的边上看到较低的权重，其中前一个神经元的激活很少导致后一个神经元的激活。

假设某个任意神经元很少激活，除非输入数据代表狗。在这种情况下，我们还假设该神经元激活得很强。那么，连接这个神经元到下一层的权重可能因为该神经元激活对输出预测的显著影响而具有较大的幅度。

不幸的是，神经网络如何“理解”输入数据并对其进行解释，在不同架构之间甚至相同架构的不同实例之间都可能有所不同。由于连接数量众多，神经网络的可解释性极差，因此它们通常被视为一个黑盒。当网络做出决策时，我们不知道确切的原因，只知道经过一个包含多达数百万个精心调整的参数的网络的前向传递后，我们得到了一个特定的预测。这与可以告诉我们为什么做出决策的决策树形成对比。此外，重要的是不要将神经网络拟人化，因为它们基本上是大型数学函数，将输入映射到输出。

### 损失函数

损失函数是一种衡量模型预测与模型应该预测的内容之间偏差的方法。你也许还会看到损失函数被称为**代价函数**。在监督学习设置中，这意味着在回归任务中（预测实值输出与目标实值输出之间的损失）或分类任务中（预测整数输出与目标整数输出之间的损失）测量损失。

预测值与目标值之间的差异可以称为“误差”，任何建模任务的目标是尽可能减少这个误差，使得目标值和预测值在数值上尽可能接近。

让我们从回归损失函数开始。

#### 回归

在回归任务中使用的简单损失函数是**均方误差 (MSE)**。回想一下第三章节中提到的，MSE 损失可以用于机器学习中的回归任务。同样的损失也可以用于深度学习任务。作为提醒，MSE 损失方程在方程 5-8 中展示。

![MSE=1/N∑(yi-^yi)²](img/483137_2_En_5_Chapter_TeX_Equk.png)

***方程 5-8.***![^yi](img/483137_2_En_5_Chapter_TeX_IEq1.png) *是对第 *i* 个数据样本 x[*i*] 的预测，y[*i*] 是我们希望模型要达到的相应真实输出。如果模型做出了完美的预测，MSE 将为 0*

还记得从第三章中提到的其他回归损失函数，例如**平均绝对误差（MAE）**（方程 5-9）、**平均绝对百分比误差（MAPE）**（方程 5-10）等，尽管你选择使用哪种损失函数可能取决于任务的性质和对真实输出偏差容忍度的阈值。

![$$ MAE=\frac{1}{N}\sum \limits_{i=1}^N\left|{y}_i-\hat{y_i}\right| $$](img/483137_2_En_5_Chapter_TeX_Equl.png)

***方程 5-9.***平均绝对误差* *找到预测与真实值之间偏差的平均幅度*

![$$ MAPE=\frac{1}{N}\sum \limits_{i=1}^N\left|\frac{y_i-\hat{y_i}}{y_i}\right| $$](img/483137_2_En_5_Chapter_TeX_Equm.png)

***方程 5-10.***平均绝对百分比误差* *以百分比的形式找到预测与真实值之间的偏差，并取平均值*

你应该使用哪种损失/代价函数取决于任务的精确性质。如果你想更严厉地惩罚预测错误，那么由于损失中的平方成分，MSE 损失函数是一个不错的选择。如果你不希望损失对异常值过于敏感，那么 MAE 损失可能是一个好的误差度量，因为它只测量误差的直接幅度。

如果你只想以相对术语查看错误，那么 MAPE 可能是一个不错的选择，因为它以目标值来衡量误差。这可以更适用于时间序列预测任务，例如，其中幅度误差或平方误差差异可能比百分比误差产生更高的损失。例如，如果你预测股票价格将是 10,100，但实际上是 10,000，你的绝对误差将是 100，你的平方误差将是 10,000。然而，你的百分比误差将是 0.01（1%误差）。

#### 分类

在处理分类任务时，你正在处理离散整数标签，所以这并不像寻找目标和预测之间的数值偏差那样简单。此外，模型输出的是输入示例属于某个类别的实值概率。我们希望将这个概率预测纳入损失中，因为它提供了预测强度的指示。尽管 0.71 和 0.99 四舍五入后都是 1，但后者比前者是一个更强的预测。

再次强调，我们可以从第三章中恢复**交叉熵损失/对数损失**函数。对于一个二元分类任务（只有两个可能的标签，要么是 0 要么是 1），我们可以引入一个类似于方程 5-11 的方程。由于这个损失函数只处理两个类别，它也可以被称为**二元交叉熵损失**。

![$$ Log\  Loss=-\frac{1}{N}\sum \limits_{i=1}^N{y}_i\ast \log \left(\hat{y_i}\right)+\left(1-{y}_i\right)\ast \log \left(1-\hat{y_i}\right) $$](img/483137_2_En_5_Chapter_TeX_Equn.png)

***方程 5-11.*** 在这个损失函数中，y*[*i*] 代表与 x*[*i*]* 对应的第 i 个真实标签（一个离散值，要么是 0 要么是 1），而 ![$$ \hat{y_i} $$](img/483137_2_En_5_Chapter_TeX_IEq2.png) 代表模型对输入 x*[*i*]* 输出的概率分数（一个介于 0 和 1 之间的实数值）

然而，我们不仅仅只有两个类别。对于多个类别，我们希望将二元交叉熵损失扩展到一种通用的 **分类交叉熵损失** 函数，该函数能够处理多个类别。

回想一下第二章的内容，**one-hot 编码** 创建了一个除了特定索引处有一个 1 以外其余都是 0 的向量。这个编码中保持 1 的索引值就是整数类别标签。

例如，如果在一个分类任务中有三个类别，那么类别 0 的 one-hot 编码是 [1, 0, 0]，类别 1 的 one-hot 编码是 [0, 1, 0]，类别 2 的 one-hot 编码是 [0, 0, 1]。

在这个背景下，让我们来看一下分类交叉熵的公式。这里将有两个求和：外层求和遍历每个单独的 logits（z）和真实值（y）对。logits 的维度与 one-hot 编码的 y 向量相同。logits 最大值的索引是模型的类别预测。现在参考方程 5-12。

![$$ CE=-\frac{1}{N}\sum \limits_{i=1}^N\sum \limits_{j=1}^C{y}_{i,j}\log \left( softmax{\left({z}_i\right)}_j\right) $$](img/483137_2_En_5_Chapter_TeX_Equo.png)

***方程 5-12.*** **分类交叉熵损失**。在这里，y*[*i*] 是与 x*[*i*]* 对应的第 i 个真实值/目标值，即传递进来的第 i 个输入值。z*[*i*] 指的是神经网络在给定 x*[*i*]* 时产生的模型输出 logits（未应用 `softmax()`）。C 代表类的总数，也是每个 y*[*i*] 向量的长度

方程 5-13 展示了没有 one-hot 编码真实标签的分类交叉熵损失。在这种情况下，真实标签只是一个整数标签的数组。这是方程 5-12 的一个简化版本，不需要对真实值进行额外的 one-hot 编码处理。

![$$ CE=-\frac{1}{N}\sum \limits_{i=1}^N\log \left( softmax{\left({z}_i\right)}_{y_i}\right) $$](img/483137_2_En_5_Chapter_TeX_Equp.png)

***方程 5-13.*** 方程 *5-12* 的一个更简单、等效的版本。所需的一切只是 logits 在 `softmax()` 后 y 标签索引处的分数

这与方程 5-12 等价，因为仔细观察，y 向量的独热编码将除包含 1 的索引外的求和项设置为 0，因此这些项无论如何都不会对损失函数做出贡献。

现在你已经了解了一些我们将用于回归和分类任务的基本损失函数，让我们深入了解神经网络如何使用损失和计算出的梯度来更新权重和偏差。

斜率有一个有趣的性质，它始终指向原始图形上会增加 y 值的方向。例如，在 x=2 处，瞬时斜率（导数的值）是 4。这意味着在 x=2 处，在那个瞬间，y 值随着 x 的 1 个单位增加而增加 4。当我们继续增加 x 时，我们会发现 y 也持续增加。

方程 y = x³ – 5x²及其在 x=2 处的导数，![$$ \frac{dy}{dx}=3{x}²-10x $$](img/483137_2_En_5_Chapter_TeX_IEq5.png)。正如我们所见，这一刻的导数是-8。如果我们朝这个方向（左边）移动，我们会看到 y 值的增加，直到它达到 x=0 处的局部最大值。

这次，我们在 x=2 处的导数是-8。这可以解释为 y 的瞬时变化率随着 x 每增加 1 而减少 8，或者 y 随着 x 每减少 1 而增加 8。记住，斜率是 y 的变化量除以 x 的变化量，所以负号可以应用于分子或分母。

在这种情况下，导数“指向”了 x=0 处的图形的局部最大值。这也恰好是图形的全局最大值。

这个定义在二维设置中效果很好，但机器学习问题往往处理的是可以高达数百维的变量。在三维设置以及更高维的情况下，我们可以扩展导数的概念，称之为**梯度**，它现在是一个向量，而不是一个标量值。对于 y = x² + z²，梯度将是 ![$$ \nabla y=\left\langle \frac{\partial y}{\partial x},\frac{\partial y}{\partial z}\right\rangle $$](img/483137_2_En_5_Chapter_TeX_IEq6.png)，其中*∂*符号表示偏导数。偏导数仅仅是 y 的导数，它是多元的，但只相对于一个变量。现在，正如我们在图 5-25（y = x² + z²的 3D 示例）中看到的那样，同样的原则适用于我们在二维示例中看到的导数。

![](img/483137_2_En_5_Fig25_HTML.jpg)

一个 3-D 图表示 y 函数作为一个抛物面区域。切平面表示为一个对角平面。在 x-z 平面上标记了一个点。

图 5-25

一个三维图，描述了 y 函数以及 x=0.75，z=0.75 处的点以及切平面

由于梯度只是导数在更高维度的扩展，因此可以合理地推断，梯度的反方向应该指向最小值。假设我们已经将 x 和 z 的值初始化为 0.75（如图 5-25 中的红色点所示）。假设我们想要找到导致此图中 y 值最小的 x 和 z 值（x=0，z=0），那么从 x=0.75 和 z=0.75 开始，我们应该沿着梯度的反方向迈出一步。步子不要太大，但要小一些，以免超过最小值。我们可以重复这个过程多次，多次采取小步，直到接近最小值，找到使 y 值有效“最小化”的 x 和 z 的值。

我们可以将这个通用的“优化步骤”表述如下，其中梯度是相对于 x 的：

x = x – 步长 * 梯度

**步长**参数控制每次迭代中更新步的大小。在这种情况下，这个步长也可以称为**学习率**。我们将在“优化器”部分详细介绍学习率。

在讨论了基础理论之后，让我们看看我们是否可以将这些应用到简单的 y=mx+b 直线上，并给定一些数据迭代求解 m 和 b。

首先，让我们设置问题。我们想要使用近似 ![$$ \hat{y} $$](img/483137_2_En_5_Chapter_TeX_IEq7.png) = mx+b（其中 m = 1 和 b = 0 作为初始起始值）来逼近直线 y = 4x + 3。我们将使用的损失是均方误差（MSE）。我们需要找到 MSE 损失函数相对于 m 和 b 的导数。

让我们来看看如何计算这个。首先，我们必须从 MSE 损失函数的定义开始。我们将损失函数推广到 L(y, ![$$ \hat{y} $$](img/483137_2_En_5_Chapter_TeX_IEq8.png))，如方程 5-14 所示。

有一些梯度下降的变体需要了解：

+   **随机梯度下降法**：在计算每个单独数据样本的梯度后执行梯度更新

+   **小批量梯度下降法**：在计算 N 个数据样本的小批量平均梯度后执行梯度更新

+   **批量梯度下降法**：在计算整个数据批次平均梯度后执行梯度更新

对于每个变体都有一些观察结果。

**随机梯度下降法（SGD）可以被视为具有 1 个样本大小的小批量梯度下降，尽管有时将小批量梯度下降称为 SGD。随机梯度下降法的优缺点如下所述。

优点：

+   **计算效率高**：你只需要一个样本大小，因此在资源和计算时间方面，它确实很快，并且不太苛刻。如果你有一个非常大的、计算密集型的模型，有时 SGD 是你在机器上运行它的唯一方式。

+   **逃离局部最小值**：在非常复杂的损失曲线上，可能会有局部最小值（局部山谷），这些最小值不如其他最小值（具有更低损失的其他局部最小值或全局最小值）。有时，梯度可能指向这些次优最小值，学习过程可能会陷入一个不良的解决方案。

    由于 SGD 更新的潜在噪声，我们可能会达到一个参数设置，使其逃离这个山谷，让新的梯度指向更深/更优的最低点。这样，学习过程最终会学习到一组更好的参数，导致比在较早的较浅的最低点停止学习时的损失更低。

缺点：

+   **不稳定**：仅基于一个样本的反馈更新变量可能会引入大量噪声，并导致次优的优化步骤。

    假设你在一瞬间成为意大利的游客，需要找到比萨斜塔。真实的位置在你西北方向某处。你计划在走过一些任意距离（进行梯度下降步骤）后向游客询问。你问了四个人：三个人指向北方（正确的方向），而另一个人指向东南方向。从这些人那里得到的方向代表了指向最小值的单个梯度方向。如果你使用 SGD，你会先向北走三个“步骤”，然后向南东方向走一步。然而，如果你使用小批量或批量梯度下降，你会考虑平均方向，即北方。

+   **收敛性较差**：由于 SGD 优化的潜在噪声和不稳定性质，你可能会发现难以更接近最优解；参数值可能会在损失曲线的最低点附近由于噪声更新而持续振荡。

**小批量梯度下降**在执行梯度更新时，一次考虑 N 个数据样本的小批量（通常简称为具有 N 个批量大小的批量）。这有助于平均噪声，并允许更准确地更新步骤以接近最小值。

优点：

+   **更稳定**：平均噪声有助于采取更准确的更新步骤，并可能避免由于噪声引起的意外振荡。

+   **计算效率高**：由于神经网络的结构，我们可以轻松利用矩阵形式对批量数据进行一次计算，提高吞吐量并利用 GPU 的并行计算架构。如果你有合适的硬件，小批量梯度下降可以高效且快速。

缺点：

+   **批量大小超参数**：小批量大小成为一个需要调整的超参数。太小，可能会使优化更容易受到数据噪声的影响。太大，可能会过度平均梯度，可能导致参数陷入某些局部最小值。

小批量梯度下降通常是在结合梯度下降学习时最优的学习策略，并且在实践中也是最普遍的。它能够导致更优解的更好收敛，因为随机梯度下降（SGD）对噪声过于敏感，容易产生过多的振荡，而批量梯度下降则过度削弱了信号，可能导致无法逃离局部最小值的次优解。

在对梯度下降和训练过程背后的机制有了更好的、更深入的理解之后，让我们继续探讨如何解释神经网络的损失曲线。

### 损失曲线

损失曲线可以告诉我们很多关于模型训练过程以及其对新数据的泛化能力的信息。回想一下，我们可以将数据分为训练集、验证集和测试集。训练集显然是我们训练模型所用的。更具体地说，它是我们通过网络传递的，计算损失并执行梯度下降的。

一个**epoch**代表整个训练数据集通过网络的完整遍历。无论你使用什么批大小（1 代表随机梯度下降，即训练样本的总数，N 代表批量梯度下降，使用小批量梯度下降时批大小介于两者之间），一旦模型看到了训练集中所有样本的数据，你就完成了一个 epoch 的训练。

验证集可以在训练后一个 epoch 后作为盲目的“测试”集来评估你的神经网络。实际上，验证集的使用取决于实现方式，但理想情况下，它将为你提供关于训练过程以及模型泛化能力的良好指示。

测试集是在训练过程完成后用于模型评估的东西。

### 正则化

我们可以使用各种技术来对抗过拟合：

+   **L1/L2 惩罚**：我们可以通过使用 L1 范数（权重绝对值的和）或 L2 范数（权重的平方和）来应用惩罚，激励它们减小幅度以最小化惩罚。

+   **Dropout**：给定保留节点的概率 p，我们随机关闭层内的某些节点，并阻止它们向下一层传递信号。这个过程在每个前向传递中重复，因此没有神经元保证在连续的前向传递中保持活跃。

    这些被激活的神经元也不会对反向传播做出贡献，因此如果它们被删除，它们根本不会参与训练过程。由于后续层不能再依赖于正向传播期间任何单个神经元的存在，它必须学会与尽可能多的其他神经元建立连接。这有助于避免过拟合并促进泛化，因为单个神经元将不再主导与后续层的连接，并且这种连接不能轻易发生。

+   **提前停止**：如前一小节所述，提前停止在最佳泛化性能点（使用模型从未训练过的盲验证数据）停止训练，这样模型就不会在数据上过拟合。在过拟合刚开始发生时简单地停止过拟合是一种正则化形式，以确保你的模型泛化。

+   **更多训练数据**：有时，与测试数据或任何其他新数据相比，你的训练数据无法充分捕捉可能的输入空间。在这些情况下，添加更多、多样化的训练样本可以帮助模型正确学习任务并更好地泛化。

    例如，如果你只在一个模型上训练了清晰的照片，但然后在包含部分遮挡图像的测试数据集上评估该模型，该模型将无法很好地泛化，并且会过拟合到训练数据。在训练数据中添加遮挡图像可以帮助模型正确学习任务并更好地泛化到新数据。

你可以使用这些技术的任何组合来帮助你的模型对新数据有很好的泛化，尽管你不必过度强调正则化，因为这实际上可能导致整体性能更差。

过度正则化的模型（如过于严厉的 L1/L2 惩罚结合过高的 dropout 率）可能无法正确学习训练任务，因此由于训练不佳，在测试集上的表现会更差。

Keras 和 PyTorch 都允许你实现这些技术。参数正则化器是框架本身支持的，当你构建神经网络架构时。我们将在本章后面的监督神经网络示例中介绍这一点。

Dropout 也由这两个框架支持。提前停止在 Keras 中很容易实现，但在 PyTorch 中，你必须手动编码它作为训练循环的一部分。我们将在我们的 PyTorch 示例中实现这一点。

### 优化器

有许多优化器技术超越了简单的梯度下降。我们之前介绍了随机梯度下降，但我们可以对 SGD 进行许多修改来改进优化过程并帮助神经网络收敛到更好的解决方案。

在优化的背景下，**收敛**意味着迭代地到达一个最优的权重集，该权重集在一定范围内最小化损失。我们无法找到全局最优的权重集来找到全局最优的最小值，因为迭代技术永远不会精确地到达最佳解决方案，但它们可以非常接近它。

因此，当模型收敛到解决方案时，它已经在损失曲线上达到某个局部最小值，但不一定是最好的全局最小值。相反的情况是，当模型**发散**或开始增加损失而不是最小化它时。发散的模型是网络或训练设置不稳定的一个迹象，可能表明超参数设置不佳（学习率过高可能导致大的优化步长）。

讨论优化方法会很快变得非常数学化，所以我们只概述每个优化器背后的直觉以及其优缺点。如果您对生成本节可视化效果的代码感兴趣，请参阅笔记本[`github.com/apress/beginning-anomaly-detection-python-deep-learning-2e/blob/master/Chapter%205%20Introduction%20to%20Deep%20Learning/chapter5_optimizers.ipynb`](https://github.com/apress/beginning-anomaly-detection-python-deep-learning-2e/blob/master/Chapter%205%20Introduction%20to%20Deep%20Learning/chapter5_optimizers.ipynb)。

让我们正式回顾一下广义 SGD 算法。在我们深入代码本身之前，让我们回顾一下变量和函数：

+   `compute_gradient()`: 一个抽象函数，它接受损失、数据和模型的可学习参数，并返回相应的梯度。

+   `parameters`: 一些任意神经网络的权重和偏置列表。

+   `gradients`: 变量 `parameters` 中权重和偏置的梯度列表。梯度与参数一一对应，因为顺序被保留。

+   `lr`: 梯度更新中使用的步长。

正如我们在图 5-34 中看到的，SGD 陷入了一个非常浅的局部最小值，在第 52 次迭代时达到了最小损失。我们如何改进并保持优化器向更好的最小值前进？这就是**动量**概念出现的地方。想象一下红色点像是一个滚下山的球。如果球积累了一些使其继续前进的动量，它就能克服那个小山，继续前进，直到它落入一个更深的最低点？

为了实现这一点，我们可以使用**速度**的概念，它是累积梯度的衰减平均值，并添加到梯度更新中。这给优化器提供了一些惯性，有助于克服曲率中的浅谷，并抑制任何振荡行为。

图 5-35

将速度的概念添加到基本 SGD 算法中，并通过 alpha 控制累积梯度更新的衰减

+   `velocities`：一个所有元素均为零的初始向量，就像变量`gradients`一样，通过索引一对一地与每个参数对应。这个变量持有累积梯度更新的衰减平均值，以添加到新的梯度更新中，以增加优化过程中的惯性。

+   `alpha`：当将旧速度加到更新步骤中时应用的一个系数。它控制累积速度的衰减速率。高的`alpha`值会给过去的梯度赋予更多的权重，并增加更多的惯性。`alpha`值接近零时，带有动量的 SGD 表现得更像普通的 SGD，当`alpha`等于零时，正好是普通的 SGD。

优化器在第 190 次迭代时达到了最低损失，并达到了全局最小值。然而，正如我们所看到的，仍然存在相当多的振荡，这很可能是由于保留了大量的惯性需要消散。我们还可以在这里进行另一种修改。如果红色球能够看到它要去哪里，并在需要时减速，那不是很好吗？这样它就可以在达到某些最小值之前开始“刹车”，从而更快地收敛。

这就是**Nesterov 动量**发挥作用的地方。Nesterov 动量的想法是，在计算梯度时，我们不会使用当前迭代的活跃参数，而是使用这些参数的梯度更新的“未来”版本。从某种意义上说，优化器“展望”未来，允许它根据这个“未来”结果计算不同的、最终的梯度更新。这将导致加速/减速以及更准确的梯度更新。现在，优化器不会像以前那样振荡，因为不需要消除的累积速度会少很多——例如，优化器会在触到底部之前提前减速。

虽然这看起来比 SGD 和带有动量的 SGD 有很大的改进，并且是一个相当好的优化器，但它并不完美。一个问题就是学习率保持不变。我们可以手动调整这个值或者使用一个学习率调度器，它会自动随时间改变学习率，但这样学习率就又变成了另一个需要调整的超参数。

有一个动态更新学习率的优化器会很好，不是吗？有一些优化器，如 RMSProp 和 Adadelta，处理一些累积梯度的平方的衰减平均值，以动态调整每个参数的学习率并更好地跟随损失曲线的拓扑结构。它们也非常适合高维问题以及一般稀疏数据，使得 RMSProp 优化器成为训练长短期记忆 (LSTM) 模型（例如自然语言处理 (NLP) 问题中的情感分析或文本生成）的理想选择。

Adam 优化器可以被视为一种“两全其美”类型的优化器，它结合了动量以及像 RMSProp 和 Adadelta 那样的累积平方梯度。

尽管 RMSProp 可以被修改以包含动量（甚至 Nesterov 动量），但 Adam 优化器结合了偏差校正（允许更稳定的训练，尤其是在训练的早期阶段）。Adam 优化器对超参数选择也更加鲁棒，使其成为各种不同训练任务中一个很好的首选优化器。Adam 优化器也被发现通常优于 RMSProp。有了这些优势，Adam 优化器在实践中得到了相当广泛的应用。

然而，没有最好的整体优化器，并且哪个优化器表现最佳可能会根据训练任务而变化。每个优化器都有自己的优缺点。例如，Adam 优化器由于使用了大量变量，因此对内存的需求更大。相比之下，SGD 在内存效率上更高，计算上更轻量，尽管它可能不是表现最佳的优化器。如果你有额外的计算能力，并且性能是一个非常高优先级，那么尝试不同超参数设置的不同优化器可能是有价值的。

在 Keras 中，回调是一组可以在训练过程中调用的函数。图 5-57 展示了 `EarlyStopping()` 回调函数。如果目标指标，如验证损失，在一定的 epoch 数内没有下降或保持在某个阈值内，`EarlyStopping` 将停止训练，并且可以选择恢复表现最佳的 epoch 的权重。这回到了本章前面关于损失曲线和训练过程中的最佳停止点的讨论。

以下列表解释了 `EarlyStopping` 的参数：

+   `patience`: 在没有改进后停止训练的 epoch 数。只要监控指标持续改进，即使改进很小，模型也会继续训练。如果指标在 `patience` 指定的 epoch 数内停止改进，则训练过程将提前停止。

+   `min_delta`: 考虑改进是否显著的阈值。如果监控指标在 epoch 之间的绝对变化小于 `min_delta`，则不认为是改进。如果指标在经过 `patience` 个 epoch 后未能显著改进，则训练将提前停止。

+   `monitor`: 用于早期停止的指标。将其设置为 `'val_loss'` 以监控验证损失可以帮助防止过拟合，因为如果在经过 `patience` 个 epoch 后验证损失停止下降，我们将提前停止。

+   `restore_best_weights`: 如果设置为 `True`，模型的权重将恢复到监控指标最佳值的 epoch 状态。这意味着即使模型在后续 epoch 由于过拟合而性能下降，你仍然会保留最佳模型状态。

Keras 中的其他回调函数包括 `ReduceLROnPlateau`，如果目标指标没有真正改进，它会降低学习率。这可以解决由于步长过大而引起的振荡行为，允许优化器采取更小、更精确的步骤，从而实现更好的收敛。
