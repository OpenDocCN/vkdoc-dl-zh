## 传统机器学习算法

“传统”机器学习算法是非神经网络架构，如 k 近邻、支持向量机、随机森林、决策树等。这些模型仍然相当流行，尤其是在 XGBoost（梯度提升树）等形式中，它们在结构化、表格数据上与神经网络保持相当竞争力。其他仍在使用的传统流行算法包括聚类算法和树。尽管深度学习模型可能受到关注，但传统机器学习算法并未过时，并且仍有很好的用途，尤其是在第三章讨论的案例中。

特别是，我们将介绍隔离森林、单类支持向量机（OC-SVM）以及如何使用它们进行异常检测。

### 隔离森林

**隔离森林**是一种无监督学习算法，它试图发现数据集的潜在结构并隔离异常值。它是一组单个树结构，根据其属性递归地划分数据集。在过程的每次迭代中，都会选择一个随机特征，并根据所选特征的随机选择值进行数据划分。

这个过程会重复进行，直到整个数据集被划分，形成森林中的单个树。异常值通常比正常数据点从根节点到路径的长度要短得多，因为它们更容易被隔离。我们可以通过使用涉及平均路径长度的数据点函数来找到异常分数，这是 scikit-learn 通过其隔离森林的实现提供的。

#### 隔离森林示例

为了更好地理解隔离森林的作用，让我们来看一个假设场景。在一个大湖中，一个不负责任的鱼养殖者繁殖并释放了一种本地鱼类物种的突变版本。以下方面区分了突变物种与本地物种：

+   较长的平均长度

+   较大的平均周长

这为我们提供了三个可以针对每条鱼采取的测量值，以区分入侵、突变物种与本地物种。这类似于在 X 数据集中每个数据点有三个列。

如您所见，平均而言，入侵物种的周长更大，身体更长，尾鳍更长。然而，所有这些测量值都落在分布上，两种物种之间有很大的重叠：一些本地鱼比本地物种的平均值要大，而一些入侵鱼比突变物种的平均值要小。这种重叠使得很难立即判断一条鱼是入侵的还是本地的。

在为当地政府机构工作的生态学家们开始担忧，这种突变物种的侵扰威胁到了湖中本地物种的生存。生态学家们决定对鱼类种群进行一次调查，以调查入侵程度的范围，并雇佣了一群渔民来协助他们。受到统计方法的启发，生态学家们要求渔民在湖中捕获 1,000 条鱼。生态学家们还培训了渔民，根据鱼围、总长度和尾鳍长度的测量，将鱼分类为本地物种（正常）或入侵物种（异常）。基于他们的发现结果，生态学家可以大致估计湖中入侵物种与本地物种的整体分布。

接下来是评估过程，渔民们同意以下流程，并将对总共 1,000 条鱼中的随机几百条进行重复操作：

1.  每位渔民选择一组随机特征子集，并使用每个选定特征的最小值和最大值之间的随机值对数据进行分区。有些人可能会强调长度和周长，而其他人则会关注长度和尾鳍长度等。

1.  每位渔民利用他们的直觉来分离那些立即看起来异常的鱼。有些鱼难以分类，需要更仔细的审查。

1.  一旦每位渔民评估了分配给他们的所有鱼，他们会平均他们的标准，并为每一条 1,000 条鱼生成分数。每个分数代表该鱼在使用步骤 1 中的过程时，与其他鱼分离的难易程度。例如，在一堆较小的鱼中，一条非常大的鱼仅凭体型就能轻松地被分到一边：基于这些分数，可以设置一个阈值，并根据阈值将鱼评估为正常或异常鱼。

在这个例子中，每位渔民代表 Isolation Forest 中的一个隔离树。给定数据的一个随机子集，每棵树尝试通过随机选择的特征子集随机分区。这类似于每位渔民在总共 1,000 条鱼的小子集中查看每条鱼的一个或两个方面，并尝试将它们彼此分离。例如，一条非常大的鱼显然会被放在远离其他鱼的堆里，而其余的鱼可能需要更仔细地测量和按大小排序。在样本中，鱼越难分离，树（以及路径长度）就越深。

当树木形成后，它们聚集在一起形成一个森林（集成）。当渔民聚集起来为每条鱼分配异常分数时，每个渔民根据自己的标准提交自己的分数，这些标准是基于他们随机批次学到的。然后，这些异常分数被平均在一起，以呈现每条鱼的最终异常分数。在隔离森林中，每棵树提交一个异常分数，该分数是路径长度的函数。

样本越容易与其他数据分离，路径长度就越短。个人分数被平均在一起，以呈现一个异常分数，这实际上是平均路径长度。如果所有找到的树都容易隔离数据点，其平均路径长度和异常分数将很低，因此它很可能是异常值。

因此，对于渔民来说，他们各自创建自己的分离标准，作为一个团队提交他们捕获的每条鱼的最终异常分数。这就是隔离森林的工作方式。

然而，这并不是一个完美的系统；正如之前讨论的，由于平均大小突变鱼和平均大小本地鱼的交叉重叠，一些入侵鱼可能会被错误地识别为本地鱼，而一些本地鱼可能会被错误地识别为入侵鱼。这些情况分别代表假阴性和假阳性。

我们只想保留与标签至少有 0.2 幅度相关性的列。换句话说，如果它是至少弱线性相关的，我们只想包括那些变量。以下列表提供了一个关于幅度的通用规则：

+   **0.0 相关性**：没有线性相关性

+   **0.2 到 0.4**：弱线性相关性

+   **0.4 到 0.6**：中等线性相关性

+   **0.6 到 0.8**：强线性相关性

+   **0.8 到 1.0**：非常强的线性相关性

+   **1.0 相关性**：完美的线性相关性，如 y = mx 的关系，其中 m 是一个正系数，所有点都位于一条直线上

相关性也可以是负的。正相关意味着当 y 增加时，x 也增加。负相关意味着当 y 增加时，x 减少。-1.0 相关性的一个例子是 y = mx 的关系，其中 m 是一个负系数。所有点都位于一条直线上。

我们选择 0.2 作为阈值，因为实际上在该点以下几乎没有线性相关性。最终，这又是一个需要调整和实验的超参数，以查看它如何影响最终性能。

##### 超参数调整

Isolation Forest 依赖于超参数设置，这些设置必须根据您的特定数据集进行调整。它不像深度神经网络那样主要依靠自身学习；使用这些传统机器学习算法的大部分工作都是确保超参数调整恰到好处。

使用 Isolation Forest 时，以下是需要调整的超参数：

+   `max_samples`：用于训练森林中每棵树的样本数量

+   `max_features`：每个数据点用于训练森林中每棵树的特征数量

+   `n_estimators`：森林中应包含多少棵树

此外，还有一个功能，**contamination**，它告诉 Isolation Forest 有多少比例的数据被异常值污染。但对于这一点，我们可以从我们的训练集中计算异常值的比例，并直接传递它。

至于超参数调整的方法，我们将使用保留验证结合网格搜索。保留验证，在数据量足够的情况下，有助于获得关于超参数调整方向的大致估计。保留验证在大幅提升预测能力方面做得很好。然而，如果您想要实验上完全可靠且稳健，如果您能承担计算成本或者验证数据量不足，K 折交叉验证是最佳选择。

在实际使用之前，Isolation Forest 模型确实有一些优缺点，您应该考虑。

**优点：**

+   **高维数据**：Isolation Forest 可以高效地采样和处理高维数据。典型的“维度诅咒”（高维数据需要更多的数据样本以便模型学习任务）在这里并不那么强烈，并且只让 Isolation Forest 更容易地隔离样本。

+   **可扩展性**：由于高效的实现，Isolation Forest 可以扩展到更大的数据规模。

+   **异常评分**：Isolation Forest 明确地为每个样本提供异常评分，让您能够解释为什么一个数据点可能是异常数据点而不是正常数据点，以及数据点可能是一个多强的异常。

**缺点：**

+   **对超参数敏感**：如您之前所见，模型对您设置的哪种超参数非常敏感。但话虽如此，一旦调整，模型就能达到标准。

+   **密集点**：当点彼此非常接近时，可能需要更多的分割（因此路径长度更高）来分离数据。这种更高的路径长度可能会错误地暗示一个难以分离的异常实际上是一个正常点，而它并不是。

+   **不平衡数据集**：参数 `contamination` 必须设置以帮助 Isolation Forest 理解期望多少异常。

+   **可解释性**：尽管提供了异常评分，但由于森林中有多少树、它们必须进行多少分割等因素，很难理解一个点是如何被决定为异常点或正常点的。

话虽如此，如果您有大量数据、高维数据，并且愿意坐下来进行超参数调整过程并仔细引导参数选择，Isolation Forest 是一个出色的算法。

### 一类支持向量机

**单类支持向量机（OC-SVM）**是一种修改过的支持向量机模型，非常适合新颖性检测（**半监督异常检测**的一个例子）。其思路是模型在正常数据上训练，并在新数据呈现时用于检测异常。虽然 OC-SVM 可能最适合半监督异常检测，因为只在一个类别上训练意味着在考虑整个数据集时它仍然是“部分标记”的，但它也可以用于无监督异常检测，其中你传递所有数据，让它学习一个最优边界，并通过边界之外的内容来预测异常。

我们将在与 Isolation Forest 示例中使用的相同 KDDCUP 1999 数据集上执行半监督异常检测。

我们刚才所做的是使用**核函数**将数据转换到另一个维度，在这个维度中，数据类别之间有明显的区分。这种数据映射称为**核技巧**。有不同类型的核，包括我们在早期示例中看到的**线性核**。其他类型的核包括**多项式核**，它使用多项式函数将数据映射到某个 n 维，以及**指数核**，它根据指数函数映射数据。

最后，边缘是每个类别的最近数据点到超平面的距离。如前所述，一个**理想边缘**或**最大边缘**涉及支持向量与超平面之间的最大分离。一个**不良边缘**或**次优边缘**会使超平面过于接近一个类别，或者每个点或支持向量到超平面的距离没有尽可能远。

OC-SVM 上的预测任务示例，包括决策边界

OC-SVM 的任务是找到一个超平面，尽可能多地包含其一侧尽可能多的数据点。设置了一个最大训练误差作为超参数，因此超平面必须尽可能多地包含一侧的数据点，同时也要遵守允许的最大训练误差，并故意将一些点放在超平面的另一侧。

关键是找到一个超参数之间的平衡，使得 OC-SVM 尽可能包含尽可能多的数据点，同时将决策边界设置在异常值明显位于其外的位置。正如你很快就会发现的，这是一个非常微妙的平衡。

至于核函数，OC-SVM 通常使用所谓的**径向基函数（RBF**），这是一种特定的函数映射，允许 SVM 形成非线性决策边界。

这很正常，也是可以预料的。我们还没有使用超参数调整。特别是，需要调整的最重要超参数如下：

+   `gamma`：此参数用于径向基函数（RBF）核函数。它决定了点对点之间的影响半径，较小的 gamma 对应较大的半径。我们希望影响半径足够大，以便不过度拟合，同时足够泛化，以便可靠地在新数据中检测到异常。这需要对 gamma 参数进行仔细调整。如果 gamma 太大，那么支持向量机（SVM）将尝试更紧密地拟合训练数据，这可能导致过度拟合。这使得`gamma`成为一个重要的调整参数。

+   `nu`：这是允许的训练错误的分数上限和支撑向量的分数下限。它是一个比 `gamma` 更敏感的超参数，可以相当大地影响模型的输出行为，因此根据你的数据集调整这个值非常重要。即使你的训练数据大小发生变化，你也可能需要重新调整这个值。一个非常小的 `nu` 值对允许的训练错误数量提出了更严格的要求，而较大的 `nu` 值在训练模型时允许更多的训练错误通过。

您现在应该很好地理解了如何使用 scikit-learn 实现自己的 OC-SVM 算法，使用保留验证和网格搜索策略进行调整，并使用发现的超参数设置来创建一个强大的异常检测器。

与隔离森林模型类似，OC-SVM 模型在使用前确实有一些优缺点需要您考虑。

**优点：**

+   **高维数据**：由于使用了核技巧，OC-SVM 能够处理高维数据，因为数据无论如何都会被映射到更高维。但话虽如此，它并不是对“维度诅咒”免疫的，在更高维中，您需要更多的数据来充分填充空间，以便模型能够正确地捕捉到一个好的决策边界。

+   **无监督学习**：OC-SVM 可以以无监督的方式进行训练。然而，如果您确实有部分标签，您可以使用半监督方式，就像我们在本节中所做的那样。

+   **非线性决策边界**：多亏了像 RBF 这样的核函数，OC-SVM 可以建模非线性决策边界，因为数据首先被映射到一个非线性、高维空间，然后再进行线性分离。在原始空间中，得到的决策边界就变成了非线性。

**缺点：**

+   **对超参数敏感**：正如我们所经历的，OC-SVM 非常依赖于超参数，对这些设置的敏感度很高。当您增加训练样本的数量时，可能需要重新调整超参数设置。然而，一旦它开始运行，它就非常好。

+   **噪声数据**：当数据有噪声时，导致正常（但噪声）数据点和异常之间有重叠，决策边界就难以建模。在这些情况下，模型将被迫隔离噪声的正常实例，以针对异常。

+   **扩展性差**：随着数据量的增加，OC-SVM 的扩展性相当差。有一些方法可以绕过这个问题，例如通过使用核映射器使用 RBF 内核分别转换数据，然后使用基于线性、随机梯度下降求解器的 SVM（在 sklearn 中可用）来更快地得到一个近似解。

话虽如此，OC-SVM 仍然是一个非常强大的算法，是你工具箱中的一大法宝。OC-SVM 的版本，如 SGD OC-SVM，可以提供近似类似解决方案的替代方案，如果你在扩展性方面遇到困难，但最终你仍然有一个上限，即你可以训练多少数据。因此，只要你能正确调整你的模型，并遵循数据以及任何可能帮助调整超参数的领域知识，OC-SVM 可以根据数据的具体性质实现出色的结果。
