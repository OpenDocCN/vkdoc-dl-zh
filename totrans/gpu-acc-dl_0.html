1.1.3 Deep Learning Architectures

1.1.3.1 Deep Neural Network

A deep neural network is an extension of a shallow neural network, distinguished by the presence of multiple hidden layers. Each neuron in a hidden layer is connected to several others, and every connection is assigned a weight. These weights determine the extent to which one neuron’s activation influences another, playing a crucial role in learning patterns from data.

The term deep in deep learning originates from the presence of multiple hidden layers, which enhance the network’s ability to capture complex relationships and hierarchical features in data. The effectiveness of a deep neural network largely depends on the appropriate selection of the number of hidden layers, which varies based on the complexity of the problem and the size of the dataset. Increasing the depth can improve performance, but it also requires careful tuning to avoid overfitting.

1.2 Convolutional Neural Networks (CNNs)

Image data is inherently structured as a two-dimensional grid of pixels, with each pixel representing one or more numerical values depending on whether the image is monochromatic or colored. Traditional approaches often disregard this spatial structure by flattening images into one-dimensional vectors before feeding them into fully connected multilayer perceptrons (MLPs). However, this method overlooks crucial spatial relationships between pixels, limiting the model’s ability to learn meaningful patterns efficiently.

To address this limitation, convolutional neural networks (CNNs) were introduced by LeCun in 1995. CNNs are specifically designed to exploit the spatial locality of image data, significantly improving performance in computer vision tasks. Their success was notably demonstrated on the ImageNet dataset where CNNs outperformed traditional architectures. The effectiveness of CNNs arises from their ability to learn hierarchical feature representations while maintaining computational efficiency, benefiting from both biological inspiration and mathematical principles such as group theory.

Beyond image processing, CNNs have found applications in diverse fields, including audio processing, natural language processing, and time series analysis. Additionally, adaptations of CNNs have been employed in graph-structured data and recommender systems.

1.2.1 Evolving from Dense Layers to Convolutional Networks

Traditional models, such as multilayer perceptrons (MLPs), are well-suited for tabular data, where information is structured in rows (instances) and columns (features). In such cases, we may expect complex interactions between features, but we do not assume any inherent structure in their relationships. However, when working with high-dimensional perceptual data, such as images, fully connected networks become impractical due to their enormous parameter requirements.

Consider an image classification task, such as distinguishing between cats and dogs. If we use one-megapixel images, each input would have one million features. Even after reducing this to a hidden layer of 1000 neurons, a fully connected network would still require an overwhelming number of parameters, making training computationally prohibitive. While reducing the image resolution to 100,000 pixels may help, a fully connected architecture would still demand billions of parameters, requiring an unrealistically large dataset for effective training.

Despite these challenges, both humans and modern machine learning systems can accurately differentiate between cats and dogs. This is because images contain inherent spatial structures that can be leveraged instead of treating each pixel as an independent feature. Convolutional neural networks (CNNs) address this challenge by efficiently capturing spatial hierarchies and local patterns in images. By leveraging known properties of natural image structures, CNNs significantly reduce computational complexity, improve generalization, and enhance learning efficiency, making them a powerful alternative to fully connected architectures for image-based tasks.

1.2.2 Introducing Spatial Structure in MLPs

In the context of neural networks, a multilayer perceptron (MLP) can be employed to process two-dimensional image data, where both the input and hidden representations are structured as matrices. Computationally, these representations are treated as two-dimensional tensors, preserving their spatial relationships. This framework underscores the importance of maintaining spatial integrity across network layers.

Let xi,j and hi,j denote the pixel values at position (i,j) in the input image and the hidden representation, respectively. In traditional MLPs, each hidden unit is connected to all input pixels through a weight matrix. However, to better capture spatial dependencies, we transition to a fourth-order weight tensor W. By introducing a bias term b, the transformation is formally expressed as
hi,j=∑k,lWi−k,j−lxk,l+b.
(1.1)

This modification from weight matrices to fourth-order tensors is primarily a reindexing of coefficients, where the weights are now represented as Wi−k,j−l. The indices k and l span both positive and negative offsets across the entire image. As a result, the value of each hidden unit is obtained by summing over neighboring pixels in x, centered around (i,j), and weighted by W.

However, this parameterization presents a substantial challenge. For instance, mapping a 1000×1000 image (1 megapixel) to a hidden representation of the same size would necessitate an enormous number of parameters, far surpassing modern computational capabilities. This computational burden motivates the need for more efficient architectures, such as convolutional neural networks (CNNs), which leverage spatial structure while significantly reducing the number of learnable parameters.

1.2.3 Translation Invariance

To incorporate translation invariance, we apply the principle that shifting the input x should result in an equivalent shift in the hidden representation h. This condition is satisfied only if the weight tensor W and the bias term b are independent of the absolute location (i,j). Thus, we redefine them as Wk,l and a constant bias b, leading to a simplified expression for hi,j:
hi,j=∑k,lWk,lxi−k,j−l+b.
(1.2)

This formulation represents a convolution operation. Here, each pixel in x, located within the neighborhood of (i,j), is weighted by the corresponding coefficients in W to compute hi,j. Unlike the fully connected case, where the number of parameters depends on the image size, this convolutional approach significantly reduces the number of parameters to k×l, making the model more computationally efficient.

The introduction of this weight-sharing strategy led to the development of early convolutional architectures, such as time-delay neural networks (TDNNs), which leveraged this concept for improved learning efficiency and robustness in pattern recognition tasks.

1.2.4 Locality

Next, we consider the second key principle: locality. As discussed earlier, meaningful information for determining the value at location (i,j) should primarily come from its immediate surroundings. This assumption implies that beyond a certain range k or l, the weights Wk,l should be set to zero. Consequently, we can rewrite the convolution operation as
hi,j=∑|k|≤K,|l|≤LWk,lxi−k,j−l+b.
(1.3)

This modification drastically reduces the number of parameters from m2 to (2K+1)(2L+1), where K and L are typically much smaller than m. As a result, we achieve an enormous reduction in the number of learnable parameters—often by four orders of magnitude. This refined structure is the essence of a convolutional layer, a core component of convolutional neural networks (CNNs). Within the deep learning community, the weight matrix W is commonly referred to as a convolution kernel, a filter, or simply the learnable parameters of the layer.

Previously, constructing a single fully connected layer for an image-processing network required billions of parameters. With convolutional layers, we now need only a few hundred, all while preserving the input and hidden representation dimensions. However, this reduction in complexity comes with a trade-off: the learned features become translation invariant, and each layer can only capture local information. Most of the machine learning models rely on inductive biases, when these align with real-world parameters.

1.2.5 Convolutions

Let us briefly revisit the mathematical definition of convolution. In mathematical analysis, the convolution of two functions f and g is defined as
(f∗g)(t)=∫−∞∞f(τ)g(t−τ)dτ.
(1.4)
This operation measures the overlap between f and g when one function is flipped and shifted by t. When working with discrete functions, the integral transforms into a summation. Specifically, for square-summable infinite-dimensional vectors indexed by i, the discrete convolution is given by
(f∗g)i=∑jfjgi−j.
(1.5)
For two-dimensional tensors, this summation extends to indices k and l, resulting in
hi,j=∑k,lWk,lxi−k,j−l.
(1.6)

This formulation closely resembles Equation (1.3), with a key difference: instead of using xi−k,j−l, we use the difference operation. However, this distinction is largely notational, as we can always adjust the formulation to match Equations (1.3) and (1.6). Notably, our original definition in Equation (1.3) more accurately describes a cross-correlation rather than a convolution. This distinction will be further explored in the following section.

1.3 Recurrent Neural Networks (RNNs)

Recurrent neural networks (RNNs) are a class of neural networks specifically designed for sequential data processing. Unlike traditional feedforward networks, RNNs introduce a memory mechanism that allows information to persist across time steps, making them effective for tasks like speech recognition, time series forecasting, and natural language processing. Figure 1.9 illustrates how information flows through time by showing hidden states, inputs, and outputs at multiple time steps. It highlights sequential dependencies and the recurrent connections that define an RNN’s structure.

1.4 Mathematical Representation of RNNs

A standard RNN processes sequential input xt at each time step t and maintains a hidden state ht that captures past information. The update equation for the hidden state is given by
ht=f(Whht−1+Wxxt+b)
(1.7)
where

  • ht is the hidden state at time step t, containing information from previous inputs.

  • ht−1 is the hidden state from the previous time step.

  • xt is the input at time step t.

  • Wh and Wx are weight matrices.

  • b is a bias term.

  • f is an activation function (commonly tanh or ReLU).

The output at each time step is computed as
yt=Wyht+by
(1.8)
where Wy is a weight matrix mapping the hidden state to the output and by is the output bias. Typical RNN architecture is presented in Figure 1.10.

1.8 GPU Acceleration in Deep Learning

1.8.1 Introduction to GPU Acceleration

GPU acceleration refers to the use of a graphics processing unit (GPU) to perform computation more efficiently than is possible with a traditional central processing unit (CPU). GPUs are designed to handle parallel tasks that require high levels of mathematical computation, making them ideal for fields such as deep learning, scientific simulations, and data analysis.

1.8.2 Types of GPU Acceleration

1.8.2.1 General-Purpose GPU (GPGPU)

Initially designed for rendering graphics, GPGPUs have evolved to handle a variety of nongraphical tasks. They leverage the parallel processing capabilities of GPUs for general-purpose computation. This evolution has been driven by the increasing demand for high-performance computing across various fields such as artificial intelligence, scientific simulations, and financial modeling.

GPGPUs differ from traditional CPUs in their architecture. While CPUs are optimized for sequential task execution, GPGPUs excel at handling thousands of parallel threads simultaneously. This parallelism is particularly beneficial for workloads that involve large-scale matrix operations, image processing, and deep learning algorithms.

A major breakthrough in GPGPU computing came with the introduction of programming frameworks such as NVIDIA’s CUDA and OpenCL. These frameworks allow developers to harness GPU power for a wide range of applications beyond graphics processing. By writing specialized code, programmers can offload computationally intensive tasks to the GPU, significantly speeding up execution times compared to CPU-based approaches.

Industries ranging from healthcare to finance have embraced GPGPUs to accelerate data processing and analysis. For example, in medical imaging, GPGPUs enable real-time rendering of complex scans, enhancing diagnostic capabilities. In finance, they are used for high-frequency trading algorithms that require rapid data processing and decision-making.

Despite their advantages, GPGPUs also come with challenges. Efficient utilization requires knowledge of parallel programming, and not all applications can be easily adapted to benefit from GPU acceleration. Additionally, power consumption and heat dissipation remain concerns for large-scale deployments.

1.8.2.3 Dedicated GPUs

Dedicated GPUs, as shown in Figure 1.11, are standalone hardware components with their own memory and processing power, providing significantly higher performance, especially for complex computations such as deep learning training. These GPUs are commonly used in gaming, professional rendering, and scientific computing. Their dedicated VRAM allows them to process large datasets efficiently, avoiding bottlenecks associated with shared memory in integrated solutions (Figure 1.12).

One of the key advantages of dedicated GPUs is their ability to handle intensive workloads such as AI model training, video editing, and real-time 3D rendering. These GPUs come with advanced features like ray tracing, tensor cores for deep learning, and high memory bandwidth, making them essential for high-performance applications.

However, dedicated GPUs consume more power and generate more heat compared to integrated GPUs. This necessitates robust cooling solutions, making them more suitable for desktops and high-end laptops rather than ultra-portable devices.

1.8.2.4 Integrated GPUs

Integrated GPUs are built into the same chip as the CPU, sharing memory and resources, which limits their performance but offers a cost-effective option for less demanding applications. These GPUs are typically found in laptops, ultrabooks, and budget desktops, where power efficiency and space constraints take priority over raw performance. Integrated GPUs are well-suited for everyday computing tasks such as web browsing, media playback, and office applications. Additionally, modern integrated GPUs have improved significantly, enabling casual gaming and basic content creation. However, for tasks requiring high graphical performance, such as 3D rendering and deep learning, dedicated GPUs remain the preferred choice (Figure 1.13).

1.9 Training a Deep Learning Model

Deep learning has revolutionized many areas of machine learning, and one of its most fundamental applications is image classification. A typical introductory problem in deep learning is classifying handwritten digits from the MNIST dataset—a benchmark dataset widely used for training and testing image recognition models. This dataset consists of 60,000 training images and 10,000 test images of handwritten digits (0–9), each representing a 28∗28 grayscale image. The goal is to develop a neural network model that can accurately recognize and classify these digits, mimicking how humans interpret handwritten numbers.

1.9.1 Image Classification Problem

Traditional programming relies on explicitly defining rules and conditions for a program to follow in order to produce the correct output. This approach works well for many structured problems where rules can be clearly stated. However, image classification presents a unique challenge: it requires a program to correctly identify and categorize images it has never seen before. Given the vast variations in images—differences in lighting, orientation, handwriting styles, and distortions—it is nearly impossible for a programmer to manually define a comprehensive set of rules that would work for all possible cases. As a result, traditional programming techniques struggle with image classification tasks.

1.9.3 Data Engineering and Dataset Splits

The process of organizing and preparing data for deep learning is known as data engineering. 1.9.4 Tensors

A tensor is an n-dimensional array that generalizes vectors (1D arrays) and matrices (2D arrays). Tensors are fundamental to deep learning, as modern neural networks are designed to process them efficiently. A 3D tensor can represent pixel data, where dimensions correspond to width, height, and color channels (RGB). GPUs are highly efficient at processing tensors due to their similarity to matrix computations in graphics processing. To use images in deep learning, they must be converted into tensors. TorchVision provides the ToTensor function, which transforms PIL images into tensor representations for neural networks.

1.9.5.1 DataLoaders

A DataLoader defines how data is fed into a deep learning model. Instead of passing the entire dataset at once (which is computationally expensive), mini-batches of data are used. For instance, with a batch size of 32, the model processes 32 images at a time.

  • Training Data: Uses shuffling to randomize data selection, preventing bias

  • Validation Data: Does not require shuffling, as the model is not learning during validation

Choosing an optimal batch size (e.g., 32 or 64) balances efficiency and performance, reducing memory constraints while ensuring effective training.

1.9.6 Creating the Model

1.9.6.1 The Hidden Layer

A hidden layer enhances learning by adding more neurons to process intermediate representations. Similar to the input layer, the hidden layer’s nn.Linear function requires knowledge of the input shape. Since each neuron in the previous layer computes one value, the number of inputs to the hidden layer equals the number of neurons in the preceding layer. The model is structured as follows:

layers = [
    nn.Flatten(),
    nn.Linear(input_size, 512),  # Input Layer
    nn.ReLU(),  # Activation for Input
    nn.Linear(512, 512),  # Hidden Layer
    nn.ReLU()  # Activation for Hidden Layer
]

1.9.6.2 The Output Layer

The output layer consists of 10 neurons, corresponding to the 10 possible digit classes in MNIST. The model assigns higher values to neurons representing the predicted class. Unlike the previous layers, the output layer does not use the ReLU activation function; instead, a loss function will be applied in the next step.

n_classes = 10
layers = [
    nn.Flatten(),
    nn.Linear(input_size, 512),
    nn.ReLU(),
    nn.Linear(512, 512),
    nn.ReLU(),
    nn.Linear(512, n_classes)  # Output Layer
]

This architecture enables effective classification by leveraging multiple layers to extract and refine meaningful features from the input images.

1.9.8 Calculating Accuracy

While the loss function helps in training the model, its numerical values can be difficult to interpret. Therefore, accuracy is often used as an additional metric to evaluate model performance.

To compute accuracy, we compare the number of correctly classified images to the total number of predictions. Since the dataset is processed in batches, accuracy is calculated per batch and accumulated to determine the overall performance.

The total number of predictions is equal to the dataset size, denoted as N, where n represents the batch size:
