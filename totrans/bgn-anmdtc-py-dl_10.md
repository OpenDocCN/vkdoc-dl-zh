## 什么是转换器？

转换器是一类深度学习架构，它彻底改变了自然语言处理（NLP）领域。继基于循环神经网络（RNN）的架构，如 LSTM 模型之后，转换器网络能够扩展到极其大的参数和数据集大小，这得益于它们并行训练和推理的能力。这种并行化能力之所以可能，是因为网络的层依赖于自注意力的概念。

**自注意力**，从高层次来看，是模型学习根据特定被评估的标记，在序列中为标记分配重要性权重的机制。例如，考虑句子“狗吃了些食物，它变得饱了”，假设模型正在尝试评估在这个上下文中“它”的含义。单词“它”可能指的是“狗”或“食物”。模型通过从所有训练中识别出的语言模式，学习在这种情况下将更高的重要性分配给“狗”而不是“食物”，因此更关注“狗”。

在模型操作的数值术语中，当模型评估“它”的嵌入表示以理解序列时，它对“狗”的嵌入表示赋予更高的权重。在非常高的层次上，自注意力基本上意味着模型关注输入的具体细节以理解上下文。由于这个过程可以一次性在整个序列上操作，它能够准确地捕捉长距离依赖关系，避免梯度消失/爆炸，并避免在非常长的序列中丢失上下文。

转换器网络在许多 NLP 任务上优于基于 RNN 的对应物，例如文本分类、问答、情感分析、机器翻译等等。也许最新和最激动人心的转换器模型类型是大型语言模型（LLM）。在大量文本数据上训练的 LLM 执行文本生成、问答、摘要、翻译、代码编写甚至头脑风暴等任务。一些 LLM 可以被调整以作为对话聊天机器人，能够以类似人类的方式执行多个这些任务。在这些 LLM 中最著名和最成功的是 ChatGPT，这个模型非常强大，以至于一些研究人员将其基础模型 GPT4 描述为通往实现通用人工智能的关键点。

虽然我们不会在本章中创建任何有感知的 AI，但我们可以利用转换器模型在基于序列的任务上表现良好的能力来执行时间序列异常检测。与 RNN 相比，转换器模型具有以下优势：

+   **并行化**：转换器模型能够并行处理序列的能力，使其能够更充分地利用 GPU 计算能力，允许它在比基于 RNN 的网络更短的时间内并行训练和进行推理。

+   **长序列**：由于自注意力机制，转换器一次关注序列中的每个标记，无论距离多远。这允许更好地、更准确地建模序列中任何位置的标记之间的关系。

+   **可扩展性**：由于转换器的并行化，我们可以扩展到比基于 RNN 的网络可行的大得多的规模。

+   **梯度消失**：尽管转换器非常深，但它们在块之间使用残差跳跃连接，允许梯度在必要时更深入地通过网络传播，从而稳定训练，通过使网络更深来实现易于扩展。

+   **丰富的特征学习**：转换器可以有多个注意力头，每个注意力头都可以根据输入数据学习一组特定的特征。这有点类似于 CNN 滤波器可以学习不同的核，例如一个滤波器学习水平边缘检测，而另一个滤波器学习垂直边缘检测。拥有多个注意力头使转换器能够更深入、更好地理解给定的输入。

考虑到所有这些优势，很容易理解为什么转换器已经超越了 RNN，尽管一些 RNN 已经被修改以使用注意力机制。

话虽如此，转换器确实有一些缺点：

+   **计算需求**：转换器扩展得很好，但由于参数数量庞大，它们在计算上极其昂贵。它们需要大量的 GPU 内存，这既资源密集又昂贵。

+   **数据饥渴**：变压器也需要大量数据来训练，如果没有适当的计算资源，这可能很麻烦。

+   **过拟合**：变压器有大量参数，这意味着它们的模型容量非常高。虽然这使得它们能够非常强大地执行，但也可能导致对较小数据集的严重过拟合。

+   **顺序推理**：尽管变压器并行执行序列处理，但它们是顺序生成标记的——一次生成一个，使用之前的标记。只是序列处理本身是并行的，而在 LSTM 中仍然是顺序的。

+   **幻觉**：更适用于文本生成任务，变压器可以“产生”听起来合理但实际上是错误的输出。这在倾向于自信地编造错误信息的 LLM 中是明显的。

在高层次上，编码器的任务是学习以某种方式编码输入序列，从而允许对原始输入序列有丰富的理解。然后解码器处理输出嵌入和解码器的编码输出，进行处理，并输出某种类型的值。这可能是文本生成模型中的单词标记，或者回归模型中的实值输出。在图 10-1 以及从该图改编的原始论文中，解码器输出通过线性层和 `softmax()` 层进行处理，以创建确定输出单词标记的概率。

输入嵌入是原始输入标记的表示，旨在捕捉其含义。输入嵌入可以预先初始化为特定的学习嵌入，或者它们可以通过模型在训练过程中学习。输出嵌入也是如此。由于模型没有递归并且并行处理整个序列，变压器无法理解序列中标记的顺序。这就是为什么我们必须包含位置嵌入，它有助于编码序列中所有标记的相对位置。

一个编码器块

编码器块通过一个多头注意力层传递输入，该层负责计算注意力权重，并为每个注意力头学习不同的特征，这些特征通过前馈层进行聚合。所谓的“加和归一化”步骤基本上是将多头注意力层之前的输入加到多头注意力层的输出上，然后对其进行归一化。这被称为 *残差跳跃连接*，因为在反向传播过程中，梯度可以更直接地向上传播网络，如果需要，可以跳过多头注意力层。由于梯度可以跳过多头注意力层，因此避免了梯度消失的问题，因为链式法则会添加更多要乘以梯度的值，并可能减小其幅度（如果这些乘数的大小也小于 1）。这使得 Transformer 可以变得非常深，因为梯度信号可以向上传播更远，而不会减小幅度。

图 10-5

图 10-5 描述了在由各自的线性层输出的 Q、K 和 V 矩阵的条件下，缩放点积注意力层中发生的情况。这些线性层是没有激活函数的神经网络层。

让我们关注一下高层次的内容。Q、K 和 V 可以这样理解：

+   **Q (Query)**: 关于输入中某些部分的“问题”。不同的注意力头可以关注输入的不同部分，从而对输入提出不同的问题。例如：在输入“the animal is eating food”（这只动物正在吃食物）中，“the animal”（这只动物）这个短语最相关的是什么？

+   **K (Key)**: “参考”材料，以特定方式转换输入。当计算 Q、K 的缩放点积并随后应用`softmax()`时，我们得到注意力分数，这些分数“回答”了问题，并给出了对哪些标记需要给予“关注”的重要性的加权。例如：“eating”（吃）可能在输入“the animal is eating food”（这只动物正在吃食物）中具有最高的注意力分数。

+   **V (Value)**: “答案”和输入的不同转换。当与注意力分数相乘时，我们得到缩放点积注意力机制的最终输出嵌入。例如：Q 和 K 告诉我们“eating”（吃）与“the animal”（这只动物）最相关。生成的嵌入是对输入句子的一种解释，专注于“the animal”（这只动物）和“eating”（吃），然后由后续层进一步转换。

请记住，由于存在多个注意力头，每个头都会学会提出不同的问题，最终产生对原始输入更丰富和多样化的理解。当这些模型在大量多样化的数据上进行扩展时，它们可以学会在不同领域之间建立概念和想法的联系，使得像 ChatGPT 这样的 LLM 能够在广泛的领域内产生更有说服力、信息丰富和准确的回答（尽管 LLM 有时会产生自信但事实上不准确的答案）。

回到多头注意力层，每个缩放点积注意力头的输出被连接起来，并通过一个线性层传递。现在，我们应该到达图 10-3 中的加和归一化步骤。将这个多头注意力层的输出加到其输入上并归一化后，我们通过一个前馈层（一个带有激活函数的神经网络层）传递，然后重复另一个加和归一化步骤。

在所有前面的讨论之后，我们已经探讨了编码器块的工作原理。编码器层只是 N 个这样的编码器块的序列，其最终输出是生成的最终编码。

![图片](img/483137_2_En_10_Fig7_HTML.jpg)

流程图如下。遮蔽多头注意力层、加和归一化、多头注意力层、加和归一化、前馈层和加和归一化。最终的编码器输出指向多头注意力层。

图 10-7a

解码器有一个额外的多头注意力层，该层执行交叉注意力（关注编码器最终输出的自注意力）

被遮蔽的多头注意力层执行与编码器中的多头注意力层相同的功能，除了它应用了一个因果掩码，因此注意力是在前一个标记上计算的，而不是在未来的标记上。

对于第二个多头注意力层，这个层没有被遮蔽。查询来自被遮蔽的多头注意力层的输出，但键和值矩阵是使用最终编码器层的输出计算的。这允许在编码器对原始输入序列的表示的上下文中计算注意力分数。

例如，在从英语到法语的语言翻译任务中，编码器可以处理英语句子并输出一些编码。这种编码可以看作是一种“思维”矩阵，代表初始英语句子。在翻译成法语时，模型使用这个思维矩阵来关注英语句子的相关部分。解码器块的其余部分与编码器块中的情况相同。

解码器的最终输出可以根据需要传递到进一步的层。在图 10-1 中，它被传递到一个线性层，然后通过 softmax 得到一个输出标记的概率分布。

### Transformer 推理

使用转换器进行推理是自回归的。以下是在机器翻译设置中的工作方式：

1.  使用编码器对输入句子进行编码。

1.  使用 `<sos>` 作为标记来指示解码器句子的开始。

1.  使用 `<sos>` 和编码，获取下一个单词的概率分布。

    +   使用选择技术选择一个单词——贪婪搜索算法选择概率最高的单词，但您也可以采样前 K 个标记或进行束搜索。

1.  取下一个单词，将其添加到输出序列中，以获得 `<sos>` 和 `<first_output>`，其中 `<first_output>` 指的是所选的第一个单词。

1.  重复此过程，直到达到截止长度或达到序列的结束标记 `<eos>`，表示句子的结束。

这样，你应该对转换器网络的工作原理有一个高级的理解。现在我们可以开始实现转换器并使用它来进行时间序列异常检测。

### 使用转换器进行异常检测

让我们尝试将一个转换器应用于纽约出租车数据集，看看它在时间序列设置中如何执行**无监督异常检测**。就像在前两章中的 LSTM 和 TCN 示例一样，我们将训练转换器以接受时间步长 1 到 T-1 的输入序列，并预测第 T 个时间序列值。再次，时间序列数据的前一半将被设置为训练集，因为它不应该包含后一半中出现的异常。预测误差的第 95 百分位数（由预测值和实际时间序列值之间的平均绝对误差给出）用作确定异常的错误阈值。如果一个预测误差高于此阈值，则将其标记为异常。

只在序列的前半部分进行训练的理由是，在我们用异常预测后半部分之前，教模型正常时间序列值看起来是什么样子。想法是预测值不会匹配异常，产生更高的误差，并被标记为异常。由于异常检测不需要标签，这是一个无监督学习任务。

使用变压器预测识别的异常

图 10-38 显示，相当多的点被标记为异常。最重要的是，模型成功地将感恩节、圣诞节、除夕和类似暴风雪的天气异常预测为异常，以及其他一些点。尽管如此，还有一些点看起来不应该被标记为异常。Transformer 的一个问题是它确实倾向于过拟合，因为它具有很高的模型容量。然而，你可以调整 Transformer 模型的超参数，以及在你的训练循环中添加早期停止或其他机制，以查看你是否可以管理过拟合。

这是一个平衡行为，因为 Transformer 在训练过程中如何拟合训练数据将影响后续的异常检测算法。这是由于 Transformer 预测输出的时间序列的显著偏差将被标记为异常。但如果模型欠拟合训练数据，你可能会错过你应该注意到的异常。

总体来说，这是一个**无监督异常检测**的例子。我们取了原始序列的一部分，训练模型在给定滚动窗口的情况下输出预测，并通过计算一个阈值来识别异常。在这个过程中，没有任何步骤涉及到训练标签。
