人工智能，或 AI，是一个令人兴奋的领域，我写这本书的目的就是将其中的一些兴奋传递给你。我将使用树莓派单板计算机作为主要工具，通过它你可以探索人工智能是如何工作的，从而获得如何将人工智能融入你的项目和应用中的额外见解。

我首先想明确一点：阅读这本书并完成所有项目不会让你成为人工智能领域的专家。这就像一个普通人参加急救课程后，永远不能声称自己是医生或护士一样。成为人工智能专家需要你在多个领域，包括数学、计算机科学、逻辑，甚至哲学，修读许多大学课程——本科和研究生课程。还有一些人工智能专家来自其他兴趣领域，包括音乐和相关艺术。在做出上述陈述后，我确实希望你能理解，通过阅读这本书和其他易于获取的资源，获得人工智能的合理入门是非常可行的。只是你阅读这本书后，不应该试图声称自己是人工智能专家。

接下来，我将讨论为什么树莓派是一个很好的平台来研究人工智能。你应该首先注意到，它本身就是一个非常强大的计算机。当然，它的速度和内存能力可能不如现代的 PC 或 Mac，但它并不慢，尤其是在使用树莓派 3 时。这款型号的时钟速度为 1 GHz，使用四个核心，并拥有 1 GB 的动态 RAM。当你意识到这种性能只需 35 美元（美元）的价格标签时，这相当令人印象深刻。然而，使树莓派对人工智能演示如此有吸引力的关键特性是它是一个微控制器。这意味着你可以根据人工智能事件的结果直接控制事物。微控制器还允许传感器轻松连接到它们，从而为人工智能应用提供与真实世界交互的手段。

虽然 PC 也可以设置成感知和控制，但通常需要昂贵的复杂接口才能实现这些功能。树莓派最初的设计是为了能够以最小的接口需求感知和控制设备，也许更重要的是，以最小的软件需求。PC 软件接口通常非常复杂、昂贵，通常是专有的——这意味着对用户进行更改或修改是一项困难甚至不可能的任务。

我在这本书中使用的树莓派使用的是名为 Jessie 的 Raspian Linux 发行版。这个发行版是完全开源的，可以从树莓派基金会的下载网站免费获取。这是一个非常稳定的操作系统（OS），支持几个极其庞大的开源应用程序仓库。这意味着这本书中使用的所有软件都是免费提供的，并且可以轻松下载到树莓派中。

我在书中的各种演示和项目中使用了多种语言和应用。使用的语言主要是 Python、Prolog 和 Wolfram 语言。每种语言都带来了一些独特的特性，使得书中的演示可以快速且容易地实现。

我在书中使用的主要应用是 Mathematica，这是一个功能齐全的符号处理程序，碰巧也是 Jessie 发行版的一部分。Mathematica 也是一个商业程序，通常需要花费数百美元，但由于 Wolfram 公司及其首席执行官 Stephen Wolfram（CEO）特别慷慨的赠予，它是免费提供的。

我试图通过首先在第一章 1 中介绍人工智能来以逻辑方式安排这本书的结构。向从未听说过人工智能的人解释人工智能是困难的，尽管告诉他们人工智能经常影响他们的日常生活往往令人惊讶。我在第一章中提供了大量的细节，试图定义人工智能以及它在日常生活中的常见应用。很快你就会明白，无论你是否喜欢，人工智能在现代社会中已经变得多么具有侵略性。请注意，我使用“侵略性”一词并非贬义，只是为了指出人工智能在许多领域都有应用，其中一些可能会让你感到惊讶。此外，我还讨论了商业智能（BI）这一主题，因为它与人工智能非常紧密相关，并且通常是人工智能影响大多数人的载体。一些人工智能 [从业者](https://www.google.com/search?newwindow=1&client=safari&rls=en&q=practitioners&spell=1&sa=X&ved=0ahUKEwjjornY-LDQAhVJ64MKHQ74A-wQvwUIGigA) 通常将商业智能简单地称为在商业环境中应用的人工智能。然而，你将了解到它远不止于此。我采用它是因为它是一个有用的简化。

我接下来在第二章 2 中探讨了某些基本的人工智能概念。最初有一些关于基本逻辑结构的讨论，因为它们对于理解推理非常重要，而推理是人工智能的核心基础。接下来讨论了专家知识系统，它们构成了更广泛的知识管理系统（KMS）的主要部分——商业智能（BI）的一个重要组成部分。然后讨论转向机器学习，这是现代人工智能中的一个巨大的研究领域。最后，我在本章的结尾介绍了模糊逻辑（FL），这在后续的书籍项目中得到了充分的展示。

第三章 3 向您展示了如何使用 Prolog 语言实现一个实用的专家系统。我探讨了 Prolog 的一些关键特性，并解释了这种相对专业的语言在实现人工智能概念时是多么有用，而不需要像使用 C/C++或 Java 这样的通用语言那样需要大量的编程支持。在实践演示中使用了简单的控制台问答程序。

第四章专注于使用游戏进行人工智能（AI）。诚然，游戏相当简单；然而，本章的目标仅仅是演示 AI 如何融入游戏逻辑。这些游戏 AI 概念可以很容易地扩展以处理更复杂的游戏。我使用 Python 实现了游戏，这些游戏通过传统的文本控制台界面进行控制。不要期望在本章中看到魔兽世界（WoW）级别的图形，但请放心，WoW 确实在其游戏中使用了 AI。

在第五章中，我回归使用 Prolog 来实现一些模糊逻辑控制，以进行实际项目演示。项目中还集成了简化的专家规则系统。一个使用温度和湿度传感器的树莓派系统将控制虚拟的加热和冷却系统。

第六章介绍了浅层机器学习的概念。创建了一个 Python 程序，其中计算机“学习”你的最喜欢的颜色，并就颜色选择做出“决策”。最后，我通过讨论适应性学习来结束本章，适应性学习在商业智能（BI）中扮演着重要角色。

第七章继续机器学习主题，通过检查使用人工神经网络（ANN）进行机器学习来展开。ANN 是目前最普遍的人工智能方法，用于实现机器学习。我详细讨论了 ANN 的构建过程，然后演示了一个使用 Python 创建的实际神经网络。

机器学习在第八章继续，其中讨论了深度学习。在本章的项目中，我详细讨论了多层人工神经网络（ANN）的功能，包括梯度搜索功能。

第九章包含两个使用多层人工神经网络（ANN）进行深度学习的演示。第一个演示基于 MNIST 训练和测试数据集识别手写数字。第二个演示使用带有树莓派（Raspberry Pi）的 Pi 相机来捕捉手写数字的图像，然后使用之前训练好的 ANN 来确定最接近的匹配。

第十章讨论了进化计算（EC），它包括但不限于进化编程、遗传算法和遗传编程。我提供了几个有趣的演示，突出了一些进化计算的特点，为你提供对这个迷人领域的良好介绍。

第十一章讨论了包含，这是一个基于行为的研究领域。它与人工智能紧密相关。我使用在第七章首次介绍的机器人汽车进行几个演示。你将很快意识到，采用包含行为的机器人可以非常出色地模仿实际的人类行为，从而完成人类思维和运动行为之间的人工智能循环。

我非常自信，在阅读完这本书并复制了大部分——如果不是全部的项目和演示之后，你将对人工智能以及如何将其融入你未来的项目中有一个出色的理解。
