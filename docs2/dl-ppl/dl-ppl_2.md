## 定义数据

**数据**是未组织和未处理的事实：它可能是一个原始数字、图表、图像、文字或声音，来源于观察或测量。通常，数据在本质上来说是静态的，是一组关于事件的事实集合，其中没有固有的意义。

在数据科学的一般领域，理解不同类型的数据有几个原因很重要。不仅因为数据类型将有助于决定清洗和处理的办法——尽管选择最佳方法很重要——而且知道数据是否是非结构化的或可能是定量的，可以让你对所测量的现实世界现象有更多的了解。

因此，我们在本章中讨论的是数据集的*特征*，而不是*整个数据集*。我们将在任何给定时间非常清楚地说明我们指的是哪一个。

## 为什么你应该阅读本章？

在深入到有趣的材料，如深度学习中的模型构建和维护之前，停下来思考我们有什么类型的数据可能看似毫无价值，但这是你进行数据科学所需采取的最重要步骤之一。

考虑一个例子，我们正在查看一组患有某种疾病的人的数据集。在人的数据集中，如果我们考虑数据集中每位患者的年龄，我们不能说有一位年龄为零岁的患者；年龄不能为零或负数。另一个例子是表示性别的**性别**列，它通过一个识别号来表示以节省空间。例如，也许*男性*用 1 表示，而*女性*用 2 表示。如果我们不理解这些数字实际上并不是我们通常所认为的有序数字（其中 2 大于 1，因此女性大于男性），我们将在分析中犯下可怕的错误。

当你使用任何数据字段管道时，你应该应用相同的原理。当给定一个数据集时，直接跳入探索、应用统计模型和研究深度学习在应用中的研究以快速获得结果是很诱人的。然而，如果你不了解你正在处理的数据类型，那么你可能会浪费时间应用对那个数据集无效的模型。

当给定一个新的数据集时，始终建议花大约一个小时（可能更少）来区分以下章节中提到的区分。

## 结构化、半结构化和非结构化数据

你可能对即将到来的数据集的第一个问题就是数据是否结构化。让我们向你展示结构化、半结构化和非结构化数据之间的区别。

+   *结构化* *数据*：易于组织，通常存储在数据库或类似`CSV`和`EXCEL`的平面文件中。结构化数据通常由数值信息组成，是客观的。

一些结构化数据可以是机器生成的，例如来自医疗设备（心率、血压）、制造传感器（每分钟的旋转次数、温度）或网络服务器日志（页面访问次数）的数据。结构化数据也可以是人类生成的：如年龄、邮编和性别等数据。

+   *非结构化* *数据*：随着组织试图利用新的和新兴的数据源，在企业中继续增长其影响力。这些新的数据源主要是由来自社交媒体平台、移动应用程序、位置服务和物联网技术的流数据组成。

大多数以文本形式存在的数据，包括服务器日志、Facebook 帖子及其评论，都是*非结构化*的。此外，即使核苷酸的顺序很重要（例如，ACGTATTGCA），化学核苷酸的序列（例如，ACGTATTGCA）也是*非结构化*的，因为我们无法使用行/列格式来形成序列的描述符。

+   *半结构化* 数据：是一种结构化数据，它不符合与关系数据库或其他数据表形式相关的数据模型的形式结构，但仍然包含标签或其他标记来分离语义元素，并在数据内部强制执行记录和字段的层次结构。因此，它也被称为自描述结构。

半结构化数据可能存在于 JSON 和 XML 格式的文件类型中。

作为数据工程师或深度学习工程师，你将始终倾向于与结构化数据一起工作，尽管`有时也会是半结构化数据`。我们大多数人，作为数据科学家/机器学习工程师，在由列和行组成的数据集上构建统计和机器学习模型，这些列和行使得模型容易跟随其模式，但它们不能处理非结构化数据，因为非结构化数据没有特定的模式或解释。因此，我们无法期望我们的模型在没有适当清理的情况下与这些类型的数据一起工作。

但使`非结构化数据`如此重要的原因是它在互联网上非常普遍；几乎 80%-90%的世界数据是非结构化数据，这表明 80%-90%的世界知识隐藏在其中。这些数据存在于许多形式中，如`推文`、`电子邮件`、`文献`和`服务器日志`。这些通常是未结构化的数据形式。

我们将在后面看到如何从非结构化的自由形式数据集中提取知识，以及如何使用称为*预处理*的预分析技术，这些技术将非结构化数据转换为干净、有序的表格——换句话说，将其转换为结构化数据。

## 整洁数据

整洁数据是将数据集的意义映射到其结构的标准方式。数据集是混乱的还是整洁的，取决于行、列和表如何与观察、变量和类型匹配。在**整洁数据**中：

1.  每个变量形成一列。

1.  每个观察值形成一行。

1.  每种观察单位类型形成一张表。

这是 Codd 的第三范式，但以统计语言的形式提出约束，并将重点放在单一数据集上，而不是关系数据库中常见的许多相关数据集。**混乱数据**是任何其他的数据排列。

整洁数据使得分析师或计算机能够轻松提取所需变量，因为它提供了一种标准的方式来结构化数据集。例如，比较不同版本的怀孕数据：在混乱版本中，你需要使用不同的策略来提取不同的变量。这会减慢分析速度并引起错误。如果你考虑有多少数据分析操作涉及到变量的所有值（每个聚合函数），你就会看到以简单、标准的方式提取这些值是多么重要。整洁数据特别适合像 Python 这样的矢量化编程语言，因为布局确保了来自同一观察的不同变量的值始终成对出现。

虽然变量和观察值的顺序不会影响分析，但良好的排序可以使扫描原始值更容易。一种组织变量的方式是按照它们在分析中的作用：值是由数据收集的设计确定的，还是它们在实验过程中测量的？固定变量描述了实验设计，并且事先已知。计算机科学家通常将固定变量称为*维度*，而统计学家通常用随机变量的下标来表示它们。测量变量是我们实际上在研究中测量的变量。固定变量应该排在前面，然后是测量变量，每个变量都按相关变量连续排列。然后可以根据第一个变量对行进行排序，用第二个和随后的（固定）变量来解决冲突。这是本书中所有表格显示采用的惯例。

## 表格数据

### 定量数据与定性数据

我们之前讨论了数据集的特定特征并对其进行了定义，我们知道结构化数据以表格或矩阵的形式出现，每个都由行和列组成。行代表现实世界的观察或案例研究，例如，某些患者的生物统计学特征。列代表数据字段，例如，患者的性别字段。

每个数据列都有其定义列的特性，例如，如果*没有缺失值*，患者性别可能是男性或女性。因此，这些患者性别的值遵循一些需要定义的表示。

### 注意

列可以命名为*特征*、*列*、*特性*，甚至*变量*。

所有字段遵循两种数据类型之一，定义如下：

+   *定量数据*：这些数据可以使用数字描述，并且可以在集合上进行基本数学运算，包括加法。

+   *定性数据*：这些数据不能使用数字和基本数学来描述。这些数据通常被认为是使用“`自然`”类别和语言来描述的。

### 示例——泰坦尼克号

RMS 泰坦尼克号的沉没是历史上最臭名昭著的船难之一。假设我们正在处理关于哪些人可能幸存的观察，使用以下描述符（特征）。并且每个特征都可以被分类为定量或定性；这种简单的区别可以改变一切。让我们看看每一个：

+   **乘客 ID** - *定量*：观察 ID；通常取数值

+   **是否幸存** - *定性*：生存（0 = 否；1 = 是）

+   **舱位等级** - *定性*：乘客舱位等级（1 = 一等；2 = 二等；3 = 三等）

+   **姓名** - *定性*：乘客姓名

+   **性别** - *定性*：乘客性别

+   **年龄** - *定量*: 乘客年龄

+   **SibSp** - *定量*: 船上兄弟姐妹/配偶的数量

+   **Parch** - *定量*: 船上父母/孩子的数量

+   **票号** - *定性*: 票号

+   **票价** - *定量*: 乘客票价

+   **船舱** - *定性*: 船舱编号

+   **登船港口** - *定性*: 登船港口（C = Cherbourg; Q = Queenstown; S = Southampton）

因此，在我们将每一列分类为定量或定性之后，你可能会想知道为什么这些列属于这个类别。让我再举一个快速例子来进一步说明。

让我们考虑“Name”这一列，它代表乘客姓名。这一列不是以数字的形式表达的，我们无法对乘客的姓名进行数学运算，因此它是*定性*的。现在考虑“Age”这一列，它以数字形式表示乘客年龄。我们可以进行基本的运算，例如将所有乘客的年龄加起来，然后除以乘客总数，以得到乘客的**平均**年龄，因此将“Age”列归入*定量*类别。

### 注意

尽管票号是用数字描述的，但它不是定量的。这是因为你不能谈论所有票号的*总和*或平均票号。这些描述是没有意义的。

几乎每当一个词被用来描述一个特征时，它都是一个*定性*因素。

### 分而治之

定量数据可以进一步分解为*离散*和*连续*量，定义如下：

+   *离散变量*: 是一个通过计数获得值的变量——例如，出勤的学生人数，因为你可以数出某个班级的学生人数。

+   *连续变量*: 是一个通过测量获得值的变量。一个例子是班级中学生的身高，因为一个学生的身高可能是 150 厘米或 150.5 厘米，甚至是 150.09 厘米。一个人的身高或建筑的身高是连续的数字，因为可能存在无限的小数位数。其他连续数据的例子包括时间和温度。

### 设置检查点

到目前为止，我们已经探讨了结构化、半结构化和非结构化数据之间的差异，以及定性和定量特征之间的差异。

这些是数据科学领域中的简单概念，但这些区别可能会对数据科学家对给定数据集的分析产生巨大影响。

### 简要总结

为了简单起见，数据整体上可以是*结构化*的或*非结构化*的，这意味着数据可以采取有组织的行/列——类似表格的视图——结构，具有描述数据集中每一行的独特特征，或者存在于自由形式的状态，通常必须预处理成易于消化的形式。

如果数据采用*结构化*格式，或者说数据是*表格化*的，我们可以将数据集的每一列（*特征*）视为是*定量*或*定性*的。基本上，这一列可以用数学和数字来描述吗？本章的下一部分将数据分解为四个非常具体和详细级别。在每个级别上，我们将应用更复杂的数学规则，从而我们可以获得更直观和可量化的数据理解。

### 数据的四个级别

观察结构化数据，我们理解每一列（*特征*）可以是以下四个级别之一：

+   名称

+   序数

+   间隔

+   比率

随着我们解释这个列表的每个元素，你会看到数据的结构和列的变化；这个列表的每个元素都附带其处理和使用的技术的实践指南。但在我们开始解释这些技术之前，我们将介绍一个小型的统计定义：*中心度量*。

#### 中心度量

中心趋势度量（*中心度量*）是一个值，它试图通过确定数据集的中心位置（作为集合中“典型”值的代表）来描述一组数据。这个*单一数值*描述了数据*倾向于*什么，并且对于每个数据级别，中心度量技术都会变化。因此，让我们开始解释这些级别。

### 备注

有时一个特征的*中心度量*被称为该特征的*平衡点*。

### 名称级别

让我们从最容易理解的一个开始，即名称级别，它由纯粹由**名称**或**类别**描述的数据组成。名称尺度可以简单地称为“*标签*”。基本例子包括*性别*、*国籍*或*物种*。它们不是用数字描述的，因此是**定性**的。以下是一些例子：

+   你的性别处于数据的名称级别。你是*男性*或*女性*。

+   “*你的头发颜色是什么？*”的答案也是名称类别，可能的答案包括“*棕色、黑色、金色、灰色*或*其他。”*

#### 允许的名称数学运算

我们不能在数据的名称级别执行数学运算，除了以下两个例子所示的基本*等式*和*集合成员*函数：

+   成为*数据科学家*等同于身处*科技行业*，但反之则不然。

+   被描述为*正方形*的图形属于*矩形*的描述，但反之则不然。

### 备注

只有两种类别（例如，男/女）的名称尺度子类型被称为*二分法*。

#### 名称的中心度量

为了找到名称数据的*平衡点*，我们通常转向*众数*。

**众数**定义为数据集中出现频率最高的数字。众数在涉及按名称级别测量的分类（定性）数据的情况下最有用。

作为这个例子，考虑我们收集了某所学校所有学生的信息，并列出了他们的性别，是**男**还是**女**，如下所示：

+   男，女，男，男，女，男，女，女，男，女，男

这里的**众数**是男，如果你数一下值，你会发现有六个男性和五个女性。

### 注意

如**均值**和**中位数**这样的中心度量在这个水平上没有意义，因为我们无法对观察值进行排序或甚至将它们相加。

#### 什么是名义水平类型？

名义水平的数据大多是分类性质的，就像我们在上一个例子中看到的男/女一样，因为我们通常只能用文字来描述数据。

虽然名义水平的数据确实可能很有用，但我们必须小心我们可能从中得出的见解。由于只有**众数**作为基本**中心**度量，我们无法得出关于平均观察值的结论。这个概念在这个水平上不存在。只有在下一个水平上，我们才可能开始对我们的观察进行真正的数学运算。

### 有序水平

名义水平并不那么灵活——由于我们无法以任何自然方式对其观察值进行排序，因此我们无法在其观察值上使用大多数数学运算。

因此，我们可以说，可以排序或具有等级的数据属于**有序水平**类别；然而，尽管有序水平为我们提供了等级顺序，或放置一个观察值在另一个观察值之前的能力，但它并不提供观察值之间的相对差异。这意味着虽然我们可以按顺序从第一到最后排列观察值，但我们无法执行如加法或减法等数学运算来获得任何实际意义。

#### 有序水平的例子

每个量表都是测量级别的增量，这意味着每个量表都履行了前一个量表的功能，以及所有调查问题量表，如**李克特量表、语义差异量表、二分量表**等。

例如，一个语义差异量表问题可能是“你对我们的服务满意吗？”答案可能是以下之一：

+   非常不满意 – 1

+   不满意 – 2

+   中立 – 3

+   满意 – 4

+   非常满意 – 5

如您所见，变量的顺序很重要，标签也很重要。非常不满意总是比不满意更差，满意总是比非常满意更差。

### 注意

在有序水平上，变量之间的距离无法计算。描述质量表明与名义量表类似的标记属性，此外，有序量表还具有变量的相对位置。这个量表的起源不存在；因此，没有固定的起点或“真实零点”。

#### 有序水平上的数据是什么样的

正如我们关于名义级别数据的讨论，这里也是一样。序数级别主要是分类性质的，就像我们在上一个例子中看到的 *满意*/*自然*/*不满意*，因为我们通常可以用文字来描述数据在过程中的顺序。

虽然序数级别的数据当然可能很有用，但我们必须小心从它们中得出的见解，正如你所知，顺序很重要。因此，选择错误的中心度量技术可能会影响你的结果，导致事情出错。这个概念在这个级别上不存在。只有在下一个级别，我们才可能开始对我们的观察进行真正的数学运算。

#### 序数允许的数学运算

在序数级别，我们可以对数据执行更多的数学运算，这些运算在名义级别数据中是无法执行的。我们从名义级别继承了所有数学（相等性和集合成员），我们还可以将以下内容添加到允许在序数级别执行的运算列表中：

+   排序

+   比较

*排序* 指的是数据为我们提供的天生顺序；然而，有时这可能很难确定。例如，如果你正在构建一个推荐引擎，你可以考虑使用每个项目的排名来排序项目，让排名更高的项目首先出现在用户面前，或者你可能认为按成本或材料等对这些项目进行排序。这可能会改变数据的顺序，但只要我们在定义顺序方面保持一致，那么它是什么并不重要。

*比较* 是在序数级别允许的另一种新操作。在序数级别，说 *男性* 比女性好或反之亦然是没有意义的。但在序数级别，我们可以进行某些比较。例如，回到之前提到的语义差异量表示例，我们可以说在调查中放置“不满意”比放置“中立”更糟糕。

#### 序数数据的中心度量

在这个级别上，**中位数**是定义数据中心的一种适当方式。然而，**平均数**在这个级别上是不可能的，因为在这个级别上不允许除法和加法。但我们可以使用**众数**，就像在名义级别上做的那样。

这里有一个小例子来阐述中位数的使用。想象一下，我们使用之前示例中的满意度问题进行了一次调查；你将看到以下结果：

```py
5, 2, 5, 2, 4, 1, 2, 3, 1, 5, 4, 3, 4, 5,
```

### 快速回顾和检查

到目前为止，我们已经向您介绍了四个水平中的两个：

+   名义水平

+   有序水平

在名义水平上，我们处理的数据通常使用词汇描述，或者可以说它只是命名（尽管有时带有数字），没有顺序，并且很少使用数学（相等性和集合成员）。在有序水平上，我们有可以用数字描述的数据，并且具有“`自然`”的顺序，允许我们将一个放在另一个前面，并且可以对它们进行比较和排序。

### 间隔水平

**间隔水平**被定义为一种数值尺度，其中变量的顺序是已知的，以及这些变量之间的差异。具有熟悉、恒定和可计算差异的变量使用间隔尺度进行分类。也容易记住这个尺度的基本作用。“*间隔*”表示两个实体之间的距离，这正是间隔尺度帮助实现的。

间隔尺度包含有序尺度的所有属性，除此之外，它还提供了变量之间差异的计算。这个尺度的主要特征是对象之间的等距差异。

### 间隔水平数据的例子

温度是间隔水平数据的绝佳例子。如果一个国家的温度是华氏 100 度，另一个国家的温度是华氏 80 度，那么第一个国家比第二个国家高出 20 度。这个简单的例子在这个水平上比之前的例子提供了更多的操作空间。

+   80 度总是高于 50 度，这两个温度之间的差值与 70 度和 40 度之间的差值相同。

+   此外，0 的值是任意的，因为温度的负值确实存在，这使得摄氏度/华氏度温度尺度成为间隔尺度的经典例子。

### 间隔水平数据的特征

如您所见，这个数据水平由数字表示，这看起来分析起来容易一些，但实际上您必须注意您的变量。这是因为在选择中心趋势测量技术，如**平均值**时，您有很多异常值可能会影响您从其中得出的结论。

### 区间允许的数学运算

我们可以使用在较低级别允许的所有运算（相等性、排序、比较等），以及另外两个显著的运算：

+   加法

+   减法

这两个新运算允许我们以有用的方式表达观察结果。

#### 区间变化的度量

这是一个我们尚未讨论的新度量。在数据科学中，你应该不仅考虑你数据的中心，还要考虑它的变异；提到数据的“分散程度”非常重要。描述这种现象的度量被称为变异度量，或方差。

你可能之前听说过**标准差**。这个概念非常重要，我们应该简要地讨论一下。变异度量——就像标准差——是一个试图描述数据分布程度的数字。与中心度量一起，变异度量几乎可以用两个数字完全描述一个数据集。

##### 标准差

标准差是区间级别及以上的数据最常用的变异度量。可以将标准差视为**数据点到均值的平均距离**。虽然这个描述在技术和数学上是错误的，但它是一个很好的思考方式。标准差的公式可以分解为以下步骤：

1.  计算数据的均值。

1.  对于数据集中的每个数字，从均值中减去它，然后平方它。

1.  计算每个平方差的平均值（方差）。

1.  取第三步得到数值的平方根。这就是标准差。

### 注意

我们想要每个点与平均值之间的“平方差”而不是“实际差”，是因为平方值实际上强调了异常值——那些异常远离的数据点。

### 比率水平

最后，我们将探讨比率水平。在经历了三个不同水平、允许的数学运算级别不同的过程之后，比率水平证明是四个水平中最强的。

我们不仅能够定义顺序和差异，比率水平还允许我们进行乘法和除法。这看起来可能并不值得大惊小怪，但它几乎改变了我们在这个水平上看待数据的方式。

### 图像数据

计算机视觉是人工智能中最热门的话题之一。它在自动驾驶汽车、机器人技术以及各种照片校正应用等多个领域取得了巨大的进步。物体检测的稳步进展每天都在进行。视觉正在向我们展示一般技术和特别是深度学习的未来，因此我们甚至无法想象其所有可能性。

计算机视觉基本上是操纵和处理图像以从中提取特定知识的科学。因此，这种方法处理从灰度到彩色，从二维到四维的各种类型的图像。

图像处理对图像执行一些操作以获得预期的操作。想想当我们开始新的数据分析时我们做了什么。我们进行一些数据预处理和特征工程。图像处理也是如此。我们进行图像处理以操纵图片，从中提取一些有用的信息。我们可以减少噪声，并控制亮度和颜色对比度。

#### 图像类型（2-D，3-D，4-D）

对于 2-D、3-D 和 4-D 数据，图像的流程可能仍然相同，但你必须知道你可能处理的不同类型的数据。你还应该了解并理解你可能必须处理的每个步骤，以应对未来可能遇到的任何图像数据。

##### 2-D 数据

当每个观测值由一个 2-D 矩阵组成时，我们说图像位于二维空间中，而构成整个数据的是三维空间。二维图像很可能是灰度图像；这是直观的，因为在二维空间中的每个像素只表示一个值，这个值可以在 0 到 255 之间。现在让我们将维度空间提升并增加一个维度，看看会发生什么。

##### 3-D 数据

如果你增加维度空间一个维度，那么图像就在三维空间中，这使得每个图像观测值由一个 3-D 张量组成，而整个数据在四维空间中，由一个 4-D 张量组成。由 3-D 组成的图像由三个通道组成，很可能是红、绿、蓝通道，使得这个数据集中的每个图像都是彩色的。图像中的每个像素由三个值组成，每个通道一个值，介于 0 到 225 之间，我们将在后面的第十一章节中看到如何处理这类数据。

##### 4-D 数据

再次，如果你增加维度空间一个维度，那么图像就在四维空间中，这使得每个图像观测值由一个 4-D 张量组成，而整个数据在五维空间中，由一个 4-D 张量组成。由 4-D 组成的图像由三个通道组成，很可能是红、绿、蓝通道，使得这个数据集中的每个图像都是彩色的，并将图像的时间变化作为另一个维度。

例如，生物医学图像通常是体积图像（3-D），有时还包含额外的时维（4-D）和/或多个通道（4-D–5-D）（例如，多序列磁共振图像）。生物医学图像（图 5-7）的变化与自然图像（例如，照片）的变化相当不同，因为临床方案旨在分层图像的获取方式（例如，患者躺在背部，头部不倾斜等）。在我们的分析中，我们旨在检测细微的差异（即，一些小区域表明异常发现）。

## 摘要

你正在处理的数据类型是数据科学和机器学习的一个非常大的领域。它必须在你的大多数分析之前，因为你所拥有的数据类型会影响甚至可能进行的分析类型！每当面对一个新的数据集时，你应该首先问以下问题：

+   数据是有组织的还是无组织的？

例如，我们的数据是否存在于整洁的行/列结构中？

+   每一列是定量还是定性的？

例如，值是数字、字符串，还是代表数量？

+   每一列的数据处于什么数据级别？

例如，值处于名义、有序、区间还是比率级别？

+   如果是图像数据，我们需要使用什么数据增强方法？

+   如果是文本数据，我们将使用什么方法将文本转换为数字？

这些问题的答案不仅会影响您对数据的了解，还会决定您分析的下一步。它们将决定您能够使用的图表类型以及您如何在即将到来的数据模型中解释它们。有时我们不得不从一级转换到另一级，以获得更多的视角。

# 6. 数据整理和预处理

在上一章中，我们定义了数据意味着什么；我们还讨论了数据的类型和层次。因此，我们现在正在开始实际操作数据！在本章中，您将学习如何理解和清洗您的数据集。

## 缺失数据

正如我们所知，当有人将数据集从一个来源转换到另一个来源时，数据丢失是不可避免的。我们无法避免这种损失或阻止其发生，但我们可以修复它或让我们的模型学习这种损失或忽略它，正如我们将在接下来的章节中看到的。因此，现在我们将讨论零、空和空值的神秘组合。

你可能想知道这些之间的区别。我们将通过三个简单的场景来展示这些区别，并定义零、空和空值的含义。

**场景一**：想象一下，你有一个装满硬币的钱包，当我问你一些关于它的问题时，你应该回答它们。

如果我问你钱包里有多少枚硬币，你观察到钱包里没有硬币。问题“钱包里有多少枚硬币？”是无法回答的。没有钱包！答案既不是正值，也不是零值，甚至不是空的。这个值是空值。

**场景二**：几个小时后，你再次查看钱包，发现里面有十枚硬币。太棒了；你现在有了对这个问题的答案。在这种情况下，你可以用这块数据回答这个问题，其值是`10`。

过了一段时间，你再次查看钱包，发现里面已经没有硬币了。每一枚硬币都不见了。你查看、搜索，并发现“钱包里有多少枚硬币？”的答案是零值。所以，你仍然有一份数据，其值现在是`0`。

**场景三**：就在傍晚时分，你从口袋里掏出钱包。当你穿过街道时，你发现你的朋友并问他，“钱包里有什么？”目前，钱包是空的。钱包里什么都没有—*没有硬币，也没有其他东西*。请注意，答案不是零，因为他问的问题不是一个数字问题。答案也不是**空值**，因为我们确实有一个钱包，我们也确实查看了它，但就是没有答案。

### 空值

空值比零更难处理，但在某些情况下很有意义，例如，当处理字符串时。例如，假设我们有一个名为中间名的属性。嗯，我没有中间名，所以我总是留这个字段为空。对于一个真正为空的值填写一个空格或破折号（或编造一些东西）并没有太多意义。空格并不等同于空。在空字符串的情况下，正确的值实际上可能是“`empty`”。

### 使用零而不是空或空值填充缺失数据是否有时有用？

让我们陈述一个事实：总是填充缺失值比留下空或空值要好。你可以以一种巧妙的方式知道缺失值：如果这个变量在 0 和 1 之间有一个均匀分布，并且它在-1 处有一个小峰值，那么-1 实际上是一个缺失值。

根据数据来源，缺失数据的识别方式不同。pandas 总是将缺失值识别为*NaN*。然而，除非数据已经预处理到分析师会遇到缺失值作为*NaN*的程度，否则缺失值可能以问号（?）或零（0）或负一（-1）或空白的形式出现。因此，数据科学家在编写任何机器学习算法之前始终先进行探索性数据分析（EDA）是很重要的。EDA 只是理解并了解我们数据行为的试金石。

### 管理缺失特征

有时数据集可能包含缺失特征，因此可以考虑以下几种选项：

+   删除整个行

+   创建一个子模型来预测这些特征

+   使用自动策略根据其他已知值输入它们

+   第一个选项是最极端的，只有在数据集相当大、缺失特征数量高且任何预测都可能有风险时才应考虑。

+   第二个选项要困难得多，因为它需要确定一个监督策略来训练每个特征的模型，最后预测它们的值。

+   考虑所有利弊，第三个选项可能是最佳选择。

Scikit-learn 提供了`Imputer`类，该类负责使用基于`mean`（默认选择）、`median`或*frequency*-`mode`（最频繁的条目将被用于所有缺失项）的策略来填充空缺。以下代码片段展示了使用三种方法（缺失特征的默认值是*NaN*。然而，你可以通过`missing_values`参数使用不同的占位符）的示例：

## 处理大型数据集

如果你想要加载的数据集太大而无法放入内存，那么你必须将其分成几块（或称为`chunks`）。这种方法有时被称为抽样，我们将在未来的章节中看到一些关于它的内容。之后，你将创建一个批处理深度学习算法，该算法一次只处理数据的一部分/块。使用批处理方法也很有意义，如果你只需要数据的一个样本。多亏了 Python，你实际上可以分块加载数据。这种操作也称为数据流，因为数据集作为一个连续的流动进入`DataFrame`或其他数据结构。与所有之前的案例相反，数据集在独立步骤中已经完全加载到内存中。

使用 pandas，有两种方式来分块和加载文件：

+   第一种方法是通过加载相同大小的数据块；每个块是数据集的一部分，包含所有列和有限的行数，不超过在函数调用中设置的（即`chunksize`参数）。

+   你还可以使用 CSV 包，它提供了两个函数来迭代文件中的小数据块：`reader`和`DictReader`函数。

### 注意

在这种情况下，`read_csv`函数的输出不是一个 pandas DataFrame，而是一个类似迭代器的对象。实际上，要获取内存中的结果，你需要迭代该对象：

```py
iris_df = pd.read_csv('Iris.csv', sep=',', decimal='.',
header=None, index_col=False,
iterator=True,
names=['sepal_length', 'sepal_width',
'petal_length', 'petal_width', 'target'])
iris_df.get_chunk(10).shape
# Outputs
# 10
```

将会有`14`个这样的其他部分，每个部分的形状都是`（10，5）`。加载大型数据集的另一种方法是请求其迭代器。在这种情况下，你可以动态地决定每个`pandas DataFrame`部分的长度（即要获取的行数）：

```py
for chunk in iris_df:
print('SHAPE: ', chunk.shape)
print(chunk, '\n')
```

## 数据增强

正如我们所知，任何深度学习算法或任何神经网络的性能通常都会随着可用数据的数量而提高。因此，你的数据集越大，你就能找到越准确的结果——当然，这要考虑到其他因素，比如为你的问题选择正确的算法。

**数据增强**是一种从现有训练数据中人工创建新训练数据的技术。这是通过对训练数据中的示例应用特定领域的技巧来完成的，从而创建新的和不同的训练示例。

图像数据增强可能是最知名的数据增强类型，它涉及创建属于原始图像同一类别的训练数据集中的图像的变换版本。

变换包括来自图像处理领域的各种操作，如平移、翻转、缩放等等。

目的是通过添加新的、合理的示例来扩展训练数据集。这意味着，模型可能看到的训练集图像的变体。例如，一张猫的图片的水平翻转是有意义的，因为照片可能是从左边或右边拍摄的。而一张猫的图片的垂直翻转则没有意义，并且可能不适合模型，因为模型几乎不可能看到一张颠倒的猫的照片。

因此，很明显，必须仔细选择用于训练数据集的具体数据增强技术，并且要在训练数据集的上下文和问题域知识的基础上进行选择。此外，单独和联合实验数据增强方法可能是有用的，以查看它们是否会导致模型性能的可测量改进，也许可以通过一个小型的原型数据集、模型和训练运行来实现。

现代深度学习算法，如卷积神经网络（CNN），可以学习到对图像位置不变的特征。尽管如此，增强可以进一步帮助这种变换不变的学习方法，并有助于模型学习到对变换也具有不变性的特征，例如从左到右到从上到下的顺序、照片中的光照水平等等。

图像数据增强通常仅应用于**训练数据集**，**不**应用于**验证**或**测试数据集**。这与图像调整大小和像素缩放等数据准备不同；它们必须在与模型交互的所有数据集中一致执行。

因此，让我们开始吧，看看我们能从现有的图像中获得多少图像。首先，我们需要按照以下方式导入我们的包：

### 裁剪和调整大小

此函数从输入图像在 `boxes` 中定义的边界框位置提取裁剪，并调整到由 `crop_and_size` 指定的公共输出大小。

```py
im = tf.expand_dims(image, 0)
crop_and_resize = tf.image.crop_and_resize(im, boxes=[[0.0, 0.0, 0.5, 0.5]], crop_size=[256, 256], box_ind=[0])
show_image(image, tf.squeeze(crop_and_resize, 0), "Crop and Resize")
```

`crop_and_resize` 操作从输入图像张量中提取裁剪区域，并使用双线性采样或最近邻采样（可能改变长宽比）将其调整到由 `crop_size` 指定的公共输出大小。这比 `crop_to_bounding_box` 操作更通用，后者从输入图像中提取固定大小的切片，不允许调整大小或改变长宽比。另一个参数是 `box_ind`，它是每个框的索引；它需要指定或用作指向每个输出裁剪图像的指针，因此如果有多个框，输出将是多张图像。

它返回一个张量，包含在 `boxes` 中定义的边界框位置处的输入图像的裁剪。裁剪的框都调整到固定大小 `= [crop_height, crop_width]`（使用双线性或最近邻插值）。

最后，我们需要进行 tf.squeeze 操作，因为来自 `crop_and_resize` 的结果是 4-D 张量 `[num_boxes, crop_height, crop_width, depth]`。因此，我们需要将其转换为 3-D 张量。

## 分类数据和文本数据

正如我们在前一章中讨论的那样；通常，你会发现自己在处理两种主要类型的数据：

+   数值

+   分类

数值数据，如温度、金额、使用天数或房屋号码，可以由浮点数（如 1.0、-2.3、99.99 等）或整数（如-3、9、0、1 等）组成。数据可以取的每个值与其他值有直接关系，因为它们是可比较的。换句话说，可以说值为 2.0 的特征比值为 1.0 的特征更大（实际上，是两倍）。这类数据定义明确，易于理解，具有等于、大于和小于等二元运算符。

分类数据也称为名义数据和有序数据。分类数据表示一个无法测量的属性，并在有限或无限的值集中取值，通常称为级别。

例如，天气是一个分类特征，因为它在离散集合（晴朗、多云、下雪、雨天和雾天）中取值。其他例子包括包含 URL、IP、你放入电子商务购物车中的项目、设备 ID 等特征。在这类数据上，你不能定义等于、大于和小于的二元运算符，因此，你不能对它们进行排序。

对于分类和数值值来说，布尔值是一个优点。实际上，它们可以被视为分类（特征的缺失/存在）或者，从另一方面来看，作为特征具有展示（已展示，未展示）的概率。由于许多深度学习算法不允许输入为分类数据，布尔特征通常被用来将分类特征编码为数值。

你可以回顾第五章，它定义了数据类型。

### 数据编码

许多深度学习库要求将分类特征/列编码为整数值。尽管 Scikit-learn 中的大多数分类/回归估计量在内部将列转换为整数，但将所有数据特征/列作为整数数组提供被认为是良好的实践，以避免技术故障。

要编码分类变量，我们可以使用与之前讨论的有序特征映射类似的方法。我们需要记住，列不是有序的，并且我们分配给特定字符串标签的整数编号并不重要。因此，我们可以简单地从 `0` 开始枚举类标签：

```py
class_mapping = {label:idx for idx,label in
enumerate(np.unique(titanic_df.Sex))}
class_mapping
# Output
# {'female': 0, 'male': 1}
```

接下来，我们可以使用映射字典将分类特征或列（图 6-31）转换为整数：

```py

titanic_df['Sex'].head()
```

让我们继续前进到另一个更具复杂性和有效性的任务。计数单词是好的，但我们可以通过更多的复杂性从数据中获得更多信息。让我们计算它们的频率。这是一个可以跨不同大小的数据集进行比较的度量。它给出了一个想法，即一个单词是否是停用词（即非常常见的单词，如 a、an、the 或 is）或罕见的、独特的单词。通常，这些单词是最重要的，因为它们能够表征一个实例以及基于这些单词的特征，这些特征在学习过程中非常具有判别性。为了检索每个文档中每个单词的频率，我们使用`TfidfVectorizer`来计算频率矩阵（图 6-37）；尝试以下代码：

```py
from sklearn.feature_extraction.text import TfidfVectorizer
tf_vect = TfidfVectorizer(use_idf=False, norm="l1")
word_freq = tf_vect.fit_transform(twenty_sci_news.data)
word_list = tf_vect.get_feature_names()

```

频率的总和是 1（或由于近似而接近 1）。这是因为我们选择了 L1 范数。在这种情况下，单词频率是一个概率分布函数。有时，增加罕见词和常见词之间的差异是件好事。在这种情况下，你可以使用 L2 范数来归一化特征向量。

将文本数据向量化的一种更有效的方法是使用`Tfidf`。简而言之，你可以将构成文档的单词的词频乘以该单词自身的逆文档频率（即它在文档中出现的次数，或其对数缩放变换）。这对于突出描述每个文档且在数据集中具有强大判别元素的单词非常有用。

`Tfidf`自从计算机开始处理和挖掘文本数据以来获得了很大的流行。大多数搜索引擎和信息检索软件主要使用它来有效地衡量句子相似性和距离，使其成为从用户输入的文本搜索查询中检索文档的优选解决方案。

```py
from sklearn.feature_extraction.text import TfidfVectorizer
tfidf_vect = TfidfVectorizer()
word_tfidf = tfidf_vect.fit_transform(twenty_sci_news.data)
word_list = tfidf_vect.get_feature_names()
for n in word_tfidf[0].indices:
print ('Word "%s" has tf-idf %0.3f' % (word_list[n], word_tfidf[0, n]))
```

到目前为止，对于每个单词，我们已经生成了一个特征。**那么，将几个单词放在一起会怎样呢？**

这正是当你考虑二元组而不是单语素时发生的情况。使用二元组（或泛指 n-gram），一个词的存在或不存在——以及它的邻居——都很重要（也就是说，它附近的词及其排列）。当然，你可以混合单语素和 n-gram，并为每个文档创建一个丰富的特征向量。在一个简单的例子中，让我们测试一下 n-gram 是如何工作的：

```py
count_vect_1_grams = CountVectorizer(ngram_range=(1, 1), stop_words=[], min_df=1)

word_list = count_vect_1_grams.get_feature_names()
print ("Word list = ", word_list)
print ("text_1 is described with", [word_list[n] + "(" + str(word_count[0, n]) + ")" for n in word_count[0].indices])

# text_1 is described with ['we(1)', 'love(1)',
#            'data(1)', 'science(1)']

print ("Word list = ", word_list)
print ("text_1 is described with", [word_list[n] + "(" + str(word_count[0, n]) + ")" for n in word_count[0].indices])
# Outputs

#                  'love data', 'science is', 'we love']
# text_1 is described with ['we love(1)',
#                  'love data(1)', 'data science(1)']
count_vect_1_grams = CountVectorizer(ngram_range=(1, 2))
```

### 注意

当你使用 n-gram 时，特征的数量呈指数级增长。

如果你数据集中的特征太多（富文本数据）（字典可能太丰富，可能有太多的 n-gram，或者计算机可能有限），你可以使用一个降低问题复杂性的技巧（但你应该首先评估性能/复杂性的权衡）。

常见的是使用哈希技巧，其中许多单词（或 n-gram）被哈希，并且它们的哈希值发生冲突（这形成了一个词的桶）。桶是语义上不相关的词的集合，但它们的哈希值发生冲突。正如以下示例中所示，使用`HashingVectorizer()`，你可以决定你想要的单词桶的数量。当然，生成的矩阵反映了你的设置：

```py
from sklearn.feature_extraction.text import HashingVectorizer
hash_vect = HashingVectorizer(n_features=1000)
word_hashed = hash_vect.fit_transform(twenty_sci_news.data)
word_hashed.shape
# Outputs
# (1187, 1000)
```

### 注意

你不能逆转哈希过程（因为它是一个不可逆的摘要过程）。

在这个变换之后，你将不得不直接处理哈希特征。哈希提供了相当多的优势：

+   允许快速将词袋转换为特征向量

    +   哈希桶是我们在这个情况下的特征。

+   容易地容纳数据集中从未见过的单词

+   通过将无关的词组合在同一特征中避免过拟合

### 分词、词干提取和停用词

#### 分词和分词是什么？

**分词**意味着将你的文本分割成最小的有意义的单元。这是任何处理之前的必经步骤。它是将字符串序列分解成单词、关键词、短语、符号和其他称为*标记*的元素的行为。

**标记**可以是单个单词、短语，甚至是整个句子。这实际上就是分词的全部内容，

我们需要讨论这个问题，但我们应该从飞行的摄像机的角度来放大视野，看看会发生什么。这个摄像机将使我们能够看到什么是*词法分析、lexing*或*分词*；这是将字符序列（例如计算机程序或网页中的字符序列）转换为标记序列的过程。

执行词法分析的程序可能被称为 lexer、tokenizer 或 scanner，尽管 scanner 也是 lexer 的第一阶段的术语。

我们多次提到了单词*lexer*。所以，问题是，什么是 lexer？或者更严谨地说，什么是*lexeme*？Lexeme 是源程序中与标记模式匹配的字符序列，并被词法分析器识别为该标记的一个实例。但我们并没有真正以严谨的形式回答“*什么是标记？*”这个问题。标记是一个具有指定和因此被识别的意义的字符串；它由一个标记名称和一个可选的标记值组成的对结构。标记名称是词汇单元的类别，常见的标记名称有：

+   *标识符*：程序员选择的名称

+   *关键字*：编程语言中已经存在的名称

+   *分隔符*（也称为标点符号）：标点符号和成对的分隔符。

+   *运算符*：对参数进行操作并产生结果的符号

+   *字面量*：数字、逻辑、文本、引用字面量

+   *注释*：行，块

在本书的范围内，这些名称并不重要，但我们试图给你一些关于分词基础的信息。然而，我们并不手动完成所有这些；我们将看到一些 Python 包，它们可以帮助我们轻松地进行**分词**和**词干提取**。现在，让我们通过一个例子来更好地了解它。

*输入*：朋友们，罗马人，同胞们，借给我你的耳朵。

*输出*：（朋友们），（罗马人），（同胞们），（借给我），（你的），（耳朵）。

**XML 输出**：

```py

lend

```

在我们看过这个例子之后，我认为我们对分词有了很好的理解。但在我们进入例子之前，我们将看到一个新术语，称为*词干提取*。所以，问题来了，什么是词干提取？

**词干提取**是指将单词缩减到其词根形式。在执行自然语言处理任务时，你将遇到各种场景，在这些场景中，你会发现具有相同词根的不同单词。

例如，“*compute, computer, computing, computed.*”你可能希望将单词缩减到其词根形式以保持一致性。这就是词干提取发挥作用的地方。现在我们可以看到词干提取是什么，但词干提取和词形还原（表 6-1）之间的区别是什么？

表 6-1

##### 什么是 BoW？

我们可能想要对文档进行分类，因此每个文档都是一个“输入”，而类标签则是我们预测算法的“输出”。算法以数字向量作为输入；因此，我们需要将文档转换为固定长度的数字向量。那么在机器学习算法中我们如何实现这一点是不可能的，因此我们需要一个模型，该模型在机器学习中考虑文本文档，丢弃单词中的所有顺序信息，并关注文档中单词的出现。这个 BoW 是我们执行此任务非常高效所需的内容。我们知道 BoW 的定义以及如何使用它。让我们看看它在技术层面上是如何工作的。

# 7. 数据重采样

在统计学中，重采样是从原始数据中重复抽取样本的方法。因此，我们可以说*重采样方法*是从现代统计学中借用的一种工具。这些方法涉及从训练集中重复抽取样本，并在每个样本上训练模型，以获取有关模型的其他信息。

例如，我们可以估计感兴趣模型学习的可变性。我们可以从训练数据中抽取不同的样本，然后每次在给定的样本上训练模型时，我们检查结果之间的差异。这种方法可能使我们能够获得如果我们只使用所有训练数据集一次训练模型时无法获得的信息。

使用重采样方法可能会因计算能力而变得昂贵和昂贵，特别是如果你考虑这些方法多次使用不同的训练数据集子集来训练相同的模型。但如今计算机能力已经增长，这些方法的要求并非不可能。

在本章中，我们将介绍数据重采样中最常用的两种方法：**交叉验证**和**自助法**。这两种方法都非常重要，在许多实际应用中广泛使用，并且是任何机器学习流程的必要部分。例如，**交叉验证**可以用来估计来自训练模型的误差，以检查和评估其性能，或者选择所谓的灵活性水平。**自助法**通常用于给出训练中使用的模型参数的准确性。

+   **注意**：评估模型性能的过程称为**模型评估**。

+   **注意**：选择最佳灵活性水平的过程称为**模型选择**。

## 创建训练集和测试集

当数据集足够大时，将其分为训练集和测试集是一个好习惯；前者用于训练模型，后者用于测试其性能。

你可能会想知道为什么从一开始就要这样做。将数据分为**训练**和**测试**集是评估数据挖掘模型的重要部分。通常，当你将数据集分为训练集和测试集时，大部分数据用于训练，而一小部分数据用于测试。

本章我们将讨论的方法的主要责任是随机采样数据，以确保测试集和训练集相似。通过使用相似的数据进行训练和测试，你可以最小化数据差异的影响，更好地理解模型的特性。

模型经过训练集处理后，通过在测试集上进行预测来测试模型。因为测试集中的数据已经包含了你想要预测的属性的已知值，所以很容易确定模型的猜测是否正确。

总结来说，执行此类操作有两个主要规则：

+   两个数据集都必须反映原始分布。

+   在分割阶段之前，原始数据集必须随机打乱，以避免后续元素之间的相关性。

## 交叉验证

在前一章中，我们说明了**测试错误率**和**训练错误率**之间的区别。使用某种模型预测新观察结果的输出而产生的错误称为**测试错误率**。

你可以看到：

+   **越低**的测试错误率，模型就越好。

+   **越高**的比率，模型就越差。

### 注意

不要忘记测试集**没有**用于训练模型。

训练错误率可以通过将模型应用于其训练中使用的数据集来轻松计算。但是，你会注意到这个错误率通常与测试错误率不同，并且更低，它不是模型的好度量，因为它可能会低估真正的准确性。

由于我们希望对测试错误率有一个良好的估计，我们需要一个大的测试集。但是，如果你无法获得大量的测试数据集怎么办？这里有一些方法可以用来估计测试错误率，使用训练数据集。在下一节中，我们将考虑一种方法，该方法通过保留训练数据的一个小部分来估计测试错误率，并且不在这个子集上训练模型；然后它使用模型来预测这些子集输出。

+   验证集提供的测试错误率估计可能**高度可变**，这取决于选择哪些观测值进入训练集和测试集的过程。

### 留一法交叉验证（LOOCV）

LOOCV 与验证集非常相似，执行相同的任务，但它被创建出来是为了解决验证集的缺点。

与验证集类似，LOOCV 将数据集分为两部分。然而，验证集只包含一个观测（数据集的一个点）。让我们考虑这个点/观测是 (*x*[1]，*y*[1])，并且这个点被用作验证集，而所有其他观测 {(*x*[2]，*y*[2])，…，(*x*[*n*]，*y*[*n*])} 都被用作训练模型（作为训练集）。现在模型从训练集中的 n-1 个观测中学习，并在仅一个观测上验证其学习过程。我们预测 y1 对 x1，因为 (*x*[1]，*y*[1]) 没有用于学习过程（训练模型），并且使用的度量包含了对我们描述为 ![$$ {E}_1={\left({y}_1-\hat{y_1}\right)}² $$](img/484548_1_En_7_Chapter_TeX_IEq1.png) 的第一个点 (*x*[1]，*y*[1]) 的误差，对于回归，以及 ![$$ {E}_1={\left({y}_1\ne \hat{y_1}\right)}² $$](img/484548_1_En_7_Chapter_TeX_IEq2.png) 对于分类。

现在我们几乎提供了一个无偏的测试错误估计。尽管这个误差项是无偏的，但它仍然是一个较差的估计，因为它高度可变；你可以感觉到这一点，因为我们只使用一个观测（*x*[1]，*y*[1]）来验证我们的模型知识。

如果我们想解决这个问题，我们可以简单地通过选择 (*x*[2]，*y*[2]) 作为验证集，并将所有其他观测作为训练集来重复同样的过程。我们在 n-1 个其他观测上训练我们的模型，并计算验证点 *x*[2] 的 *E*[2]。我们再次对 (*x*[3]，*y*[3]) 进行操作，训练模型，验证，并产生 *E*[3]。我们再次对 (*x*[4]，*y*[4]) 进行操作，并产生 *E*[4]，并对其所有数据点/观测进行 N 次操作以产生 *N* 个误差，*E*[1]，…，*E*[*n*]。现在 LOOCV 对测试错误的估计是这些 *N* 个验证集错误率的平均值。

![$$ {CV}_{(N)}=\frac{1}{n}\sum \limits_i^N{E}_i $$](img/484548_1_En_7_Chapter_TeX_Equa.png)

如我们所见，LOOCV 有一些优势，这些优势在验证集方法中是不存在的：

+   LOOCV（留一法）相较于验证集具有更少的偏差，因为我们反复在 *n* - 1 个数据观测上训练模型（这几乎涵盖了整个数据集）。在验证集方法中并不会发生这种情况；在验证集方法中，我们取整个数据集的一个子集作为验证集，并在其他数据子集上训练模型。因此，LOOCV 方法往往不会像验证集方法那样高估测试错误率。

+   每次运行 LOOCV 都会产生相同的结果；与验证集方法不同，训练-测试分割中没有随机性。LOOCV 重复地将每个数据点作为验证集一次，并在最后计算平均错误率。没有生成随机性，因为你没有选择验证集观测；你只是将每个观测作为一次验证，并迭代。


### K 折交叉验证

你可能会认为这种新的方法是对 LOOCV 的升级。K 折交叉验证涉及将数据集随机划分为 k 个相等的子集/组，它们被称为*折*。然后我们将第一个子集（折）作为验证集，并在其他*k* - 1 个折上训练模型。产生的验证误差（让我们称之为*E*[1]）是从验证集（保留的折）计算出来的。我们重复这个过程 k 次；每次，我们取一个不同的折作为验证集，其余*k* - 1 个作为训练集。这个过程产生了*k*个测试误差率的估计，即*E*[1]，…，*E*[*k*]，然后我们取平均值：

![$$ {CV}_{(N)}=\frac{1}{N}{\sum}_{i=1}^N{E}_i $$](img/484548_1_En_7_Chapter_TeX_Equb.png)

你可以看到，LOOCV 是 K 折交叉验证的一个特殊情况，其中 k 等于 N。在实践中，开发者使用 k=5 或 k=10 进行 K 折交叉验证，但大多数开发者由于计算能力的原因，不会使用 k=N，正如我们之前提到的。

## 自举

自举是一种用于有放回抽样数据的统计技术。

目前，我们对自举技术有一个简要的介绍。我们将使用一个真实示例，让你更好地了解使用自举进行数据重采样的情况。还有其他重采样技术，如蒙特卡洛和随机化技术，但在这里我们只讨论自举技术。

自举方法可以用来估计总体中的数量。这是通过反复取小样本，计算统计量，并计算统计量的平均值来完成的。

### 统计学中的自举

自举表示为*x*^∗。让我们开始生成一个自举样本。我们有 N 个观测值，需要从中采样；我们可以从总体分布中获取这些样本。

![img/484548_1_En_7_Chapter/484548_1_En_7_Figa_HTML.jpg](img/484548_1_En_7_Figa_HTML.jpg)

因此，N 个观测值可以表示为 *x*[1] = 2, *x*[2] = 1, *x*[3] = 7, …, *x*[*N*] = 5 等。

从 N 个观测值得到的样本我们可以表示为向量 ![$$ x\left(\underset{\_}{x}\right) $$](img/484548_1_En_7_Chapter_TeX_IEq3.png)，我们可以在该样本中进行任何操作，例如计算均值 ![$$ S=\theta \left(\underset{\_}{x}\right) $$](img/484548_1_En_7_Chapter_TeX_IEq4.png)。这个动作可以不替换进行，或者我们可以进行相同的操作但带替换。替换和不替换有什么区别？替换意味着我们可以在同一观测值中重复相同的值超过一次，而不替换则不能在观测值中重复值超过一次。注意观测值应该是*IID*（独立同分布）。独立同分布意味着没有整体趋势——分布不会波动，样本中的所有项目都来自相同的概率分布。独立意味着样本项目都是独立事件。所以让我们完成 bootstrap。

因此，*N*个观测值可以表示为 *x*[1] = 2, *x*[2] = 1, *x*[3] = 7, …, *x*[*N*] = 5。从 N 个观测值得到的样本可以表示为 Bootstrap *x* (*x*^∗)

我们可以在该样本中进行任何操作，例如计算均值 S = (*x*^∗)，但在这里是带替换的。让我们看看一个可能使事情更简单的图。

### 使用 Bootstrap（带替换重采样）的技巧提示

为了给您提供最佳的 bootstrap 技术使用体验，我们将其总结为步骤，您可以使用这些步骤来创建您自己的数据选择管道。

1.  维护数据结构但重新排列值，外推到总体

1.  该过程可以多次采样每个值，或者根本不采样。

1.  适用于估计非正态数据中的统计参数

1.  具有未知的统计特性（例如，PCA——我们将在下一章讨论它）。

因此，让我们思考一下这种方法是如何工作的，更详细地了解它，并看看我们如何制作这些样本。构建一个样本的过程可以总结如下：

1.  选择样本的大小。

1.  如果样本的大小小于所选大小：

    1.  从数据集中随机选择一个观测值。

    1.  将其添加到样本中。

bootstrap 方法可以通过以下步骤计算总体数量：

1.  选择要执行的 bootstrap 样本数量。

1.  选择样本大小。

1.  对于每个 bootstrap 样本：

    1.  使用选择的大小进行带替换的抽样。

    1.  在样本上计算统计量。

1.  计算计算样本统计量的平均值。

在设置好理论工具箱之后，让我们用一个例子来演示这些统计方法是如何工作的。

现在我们有了我们的数据样本。示例故意演示了相同的值可以在样本中出现零次、一次或多次。在这里，观测值 0.2 出现了两次。然后可以对抽取的样本进行估计计算。

### 回调

我们可以大量使用的其中一个重要模块是`keras.callbacks.Callback()`模块。此 API 允许您指定要监控的指标，例如训练或验证数据集上的损失或准确度。您可以指定是否寻找最大化或最小化分数的改进。最后，您用于存储权重的文件名可以包含变量，如 epoch 编号或指标。Keras 文档定义了。

+   *params*：字典。训练参数（例如，冗余度、批次大小、epoch 数量...）

+   *model*：keras.models.Model 的实例。正在训练的模型的引用

回调方法作为参数接受的日志字典将包含与当前批次或 epoch 相关的数量键。

目前，Sequential 模型类的`fit(...)`方法将在其传递给回调的日志中包含以下数量：

+   `on_epoch_end`: 日志包括 acc 和 loss，以及可选的 val_loss（如果启用了 fit 中的验证），以及 val_acc（如果启用了验证和准确度监控）。

+   `on_batch_begin`: 日志包括大小，当前批次中的样本数量。

+   `on_batch_end`: 日志包括损失，以及可选的 acc（如果启用了准确度监控）。

更多关于回调的信息，您可以在[`faroit.com/keras-docs/1.1.0/callbacks/`](http://faroit.com/keras-docs/1.1.0/callbacks/)的文档中查看。

现在让我们谈谈`on_epoch_end()`函数；正如我们所见，您可以通过使用其参数来使用它。让我们看看一个例子。这个函数在训练中非常重要，而不仅仅是生成器示例中；例如，如果我们训练一个模型，并且这个模型达到了我们满意的准确度，但它仍然有大量的 epoch 会浪费我们的时间，我们就使用`on_epoch_end()`。让我们获取`on_epoch_end()`的定义。

```py
class myCallback(tf.keras.callbacks.Callback):
def on_epoch_end(self, epoch, logs={}):
if(logs.get('acc')>0.6):
print("\nReached 60% accuracy so cancelling training!")
self.model.stop_training = True
```

在这个例子中，模型将在我们获得 60%的准确度后停止训练，这是我们目前满意的结果，因此我们不需要初始化 epoch 数量。

由于我们正在讨论生成器，在完成初始化过程后，现在是时候继续这个流程到一个新的任务中。对提供给分类器的示例顺序进行洗牌是有帮助的，这样批次之间就不会看起来相似。这样做最终会使我们的模型更加鲁棒。

另一种对生成过程至关重要的方法是完成最关键的任务：生成数据批次。负责此任务的私有方法称为`__data_generation`，并接受目标批次 ID 列表作为参数。

```py
def __data_generation(self, list_IDs_temp):
'Generates data containing batch_size samples' # X : (n_samples, *dim, n_channels)
# Initialization
X = np.empty((self.batch_size, *self.dim, self.n_channels))
y = np.empty((self.batch_size), dtype=int)
# Generate data
for i, ID in enumerate(list_IDs_temp):
# Store sample
X[i, ] = np.load('data/' + ID + '.npy')
# Store class
y[i] = self.labels[ID]
return X, keras.utils.to_categorical(y, num_classes=self.n_classes)
```

在数据生成过程中，此代码从每个示例对应的文件`ID.npy`中读取 NumPy 数组。由于我们的代码是多核友好的，请注意，您可以进行更复杂的操作（例如，从源文件中进行计算）而不用担心数据生成成为训练过程中的瓶颈。

此外，请注意，我们使用了`keras.utils.to_categorical`方法将存储在`y`中的数值标签转换为二进制形式——具体来说，是独热向量形式——（例如，在 6 类问题中，第三个标签对应于`[0 0 1 0 0 0]`），这种形式适合分类。

接下来，我们将构建所有这些组件。每次调用都会请求一个介于 0 和总批次数之间的批次数索引，后者在`__len__`方法中指定。

```py
def __len__(self):
'Denotes the number of batches per epoch'
return int(np.floor(len(self.list_IDs) / self.batch_size))
```

你可能遇到过一些这些情况，或者至少你可能考虑过它们：

+   你发现大多数变量都是**相关的**。

+   你失去了耐心，决定在全部数据上运行模型。这返回了**较差的准确度**，你感到非常糟糕。

+   你开始犹豫不决该做什么。

+   你开始考虑一些战略性的方法来**找到**一些重要的变量。

因此，我们需要一种方法来选择重要的变量，换句话说，选择那些可以影响模型并产生良好输出估计的重要变量。

另一个原因是，特征选择和提取这个主题在统计学和数据科学领域非常重要。但是，当为学生们或任何想要学习的人准备课程时，我们发现网上的资源过于技术化，没有完全满足我们的需求，或者提供了相互矛盾的信息。

因此，我们想要整理这个主题的“*是什么*”，“*何时*”，“*如何*”，“*为什么*”，并具体回答这些问题。具体来说，我们想要展示这个主题的合理性，背后的数学原理，一些最佳实践，以及这个主题下每种方法的潜在缺点。

### 什么是降维？

如果你之前处理过很多变量，你知道这可能会给你和你要构建的模型带来一些问题。所以你可能决定选择一些重要的变量来帮助你充分利用数据集，并使模型达到最高的准确度。另一方面，你可能会想要移除所有那些不会为你做这些事情的变量。为了完成所有这些——选择/移除过程——你需要回答以下问题：

+   你是否理解每个变量之间的关系？

+   你是否有如此多的变量，以至于你处于将模型过度拟合到数据的风险中，或者你可能违反了你正在使用的建模策略的假设？

你可能会问这样的问题：*我如何将我收集的所有变量集中关注其中几个？* 从技术角度来说，你想要通过减少特征空间的维度来“降低你的特征空间维度”。这将减少你需要考虑的变量之间的关系，你不太可能过度拟合你的模型。

### 注意

这并不意味着过拟合等问题不再成为关注点——但我们正在朝着正确的方向前进！

有一点令人惊讶的是，降低特征空间的维度被称为*降维*。有许多方法可以实现降维，但这些技术大多分为两大类：

+   特征消除

+   特征提取

**特征消除**正如其名：通过消除特征来减少特征空间。我们可能不会考虑每一个变量，而是只保留我们认为最能预测美国国内生产总值（GDP）的三个变量。特征消除方法的优势包括简单性和保持变量的可解释性。

然而，作为缺点，你从那些你丢弃的变量中得不到任何信息。如果我们只使用去年的 GDP、根据最新的美国社区调查数据中制造业就业人口的比重以及失业率来预测今年的 GDP，我们就错过了那些被丢弃的变量可能对我们的模型做出的贡献。通过消除特征，我们也完全消除了那些被丢弃的变量可能带来的任何好处。

然而，**特征提取**不会遇到这个问题。比如说，我们有一十个独立变量。在特征提取中，我们创建了十个“新”的独立变量，其中每个新独立变量是十个“旧”独立变量中的每一个的组合。然而，我们以特定的方式和顺序创建这些新独立变量，按照它们预测因变量的好坏来排序这些新变量。

你可能会问*降维在哪里发挥作用？* 好吧，我们保留我们想要的尽可能多的新独立变量，但丢弃最不重要的那些。因为我们按照新变量预测因变量的好坏来排序，我们知道哪个变量是最重要的，哪个是最不重要的。但是——这里是关键——因为这些新独立变量是我们旧变量的组合，即使我们丢弃一个或多个这些“新”变量，我们仍然保留了我们旧变量中最有价值的部分！

### 我应该在什么时候使用降维？

1.  你是否想要**减少**变量的数量，但又无法确定哪些是重要的变量，哪些是你认为应该移除的变量？

1.  你是否想要确保你的变量之间是**独立**的，或者没有任何**相关性**存在于任何*预测/输入*变量之间？

1.  你是否关心数据集变量的**可解释性**？

如果你对所有三个问题都回答“是”，那么降维是一个很好的方法。如果你对问题 3 回答“否”，那么**不应该**使用降维。

最后，**如何**和**为什么**取决于每种方法的特性，因此它们将在描述和说明降维方法中的每种方法时得到解答。

## 无监督降维通过主成分分析（PCA）

正如我们解释的那样，我们可以使用特征提取来减少数据集中的特征数量。我们可以通过使用特征提取算法将数据转换或投影到新的特征空间来实现这一点。

在降维的背景下，特征提取可以理解为一种数据压缩方法，其目标是保持大部分相关信息。特征提取通常用于提高计算效率，但也可以帮助减少维度诅咒。

### 注意

如果我们使用非正则化模型工作，使用降维是非常好的，这是因为它减少了数据复杂性；因此，它们是两个不同的问题，它们在相似领域工作。

维度诅咒指的是在分析和组织高维空间中的数据时出现的现象。这些问题的共同主题是，当维度增加时，空间的体积增长速度如此之快，以至于可用的数据变得稀疏。

**PCA**是一种广泛用于不同领域的**无监督线性变换**技术，主要用于降维。PCA 的其他流行应用包括以下内容：

+   股票市场交易中的探索性数据分析和对信号的降噪

+   生物信息学领域中基因组数据和基因表达水平的分析

PCA 帮助我们根据特征之间的相关性在数据中识别模式。简而言之，PCA 旨在找到高维数据中最大方差的方向，并将其投影到一个新的子空间，其维度与原始空间相等或更少。新子空间的正交轴（主成分）可以解释为在约束新特征轴相互正交的条件下，最大方差的方向，如图 8-1 所示。在这里，x1 和 x2 是原始特征轴，而 PC1 和 PC2 是主成分。

![img/484548_1_En_8_Chapter/484548_1_En_8_Fig1_HTML.jpg](img/484548_1_En_8_Fig1_HTML.jpg)

图 8-1

数据集的两个主要维度

如果我们使用主成分分析（PCA）进行降维，我们构建一个 dk 维度的变换矩阵 W。这使我们能够将样本向量 x 映射到一个新的 k 维特征子空间，其维度少于原始的 d 维特征空间：

![x=({x}_1,{x}_2,..., {x}_{d-1},{x}_d)，x 属于 R^d](img/484548_1_En_8_Chapter_TeX_Equa.png)

![下箭头 xW，W 属于 R^d×k](img/484548_1_En_8_Chapter_TeX_Equb.png)

![z=({z}_1,{z}_2,..., {z}_{k-1},{z}_k)](img/484548_1_En_8_Chapter_TeX_Equc.png)

由于将原始 *d*-维数据转换到这个新的 *d*-维子空间（通常 *k* ≪ *d*），第一个主成分具有最大的可能方差，并且所有后续的主成分将具有最大的可能方差，前提是它们与其他主成分不相关（正交）。请注意，*PCA* 方向对数据缩放非常敏感，如果我们想要将特征赋予相同的重要性，并且特征是在不同的尺度上测量的，那么在 PCA 之前我们需要标准化特征。

在更详细地查看降维的 PCA 算法之前，让我们用几个简单的步骤总结一下这种方法：

1.  标准化 *d*-维数据集。

1.  构建协方差矩阵。

1.  将协方差矩阵分解为其特征向量和特征值。

1.  选择与 *k* 个最大特征值相对应的 *k* 个特征向量，其中 *k* 是新特征子空间的维度（*k* ≤ *d*）。

1.  从“顶部” *k* 个特征向量中构建投影矩阵 W。

1.  使用投影矩阵 *W* 将 *d*-维输入数据集 *X* 转换为新的 *k*-维特征子空间。

### 总方差和解释方差

在本小节中，我们将解决 PCA 的前四个步骤：

+   标准化数据

+   构建协方差矩阵

+   获取协方差矩阵的特征值和特征向量

+   按降序排列特征值以对特征向量进行排序

*C* 是对称和正半定的，所以所有特征值都是非负的，但 *每个值的含义是什么？* 之前示例中的协方差矩阵 *C* 是对称的；正如预期的那样，水平方差相当高，而垂直方差较低。此外，其他值接近于零。如果你记得定义，并且为了简单起见，移除均值项，它们代表了一对分量之间的交叉相关。很明显，在我们的例子中，*X* 和 *Y* 是不相关的（它们是正交的），但在现实生活中的例子中，可能会有特征表现出残留的交叉相关。从信息论的角度来看，这意味着知道 *Y* 给我们关于 *X*（我们已知）的一些信息，因此它们共享的信息实际上是成倍的。所以，我们的目标也是在尝试降低其维度的同时去相关 *X*。这可以通过考虑 *C* 的排序特征值并选择最佳的主成分值来实现：

因此，可以将原始特征向量投影到这个新的（子）空间中，其中每个分量都承载部分总方差，并且新的协方差矩阵去相关以减少不同特征之间的无用信息共享（从相关性的角度来说）。在 Scikit-learn 中，有一个 PCA 类，可以非常顺畅地完成所有这些操作：

我建议读者尝试不同的成分数量（使用解释方差数据）以及 `n_components='mle'`，这实现了最佳维度的自动选择（参见 Thomas P. Minka 的“PCA 的自动维度选择”（NIPS, 2000: 598-604））。Scikit-learn 使用 **SVD**（奇异值分解）来解决 **PCA** 问题，这可以在 David Poole 的《线性代数》（Brooks Cole, 2015）一书中详细研究。可以通过参数 `svd_solver` 控制算法，其值有 `'auto'`、`'full'`、`'arpack'` 和 `'randomized'`。

### 非负矩阵分解

当数据集由非负数组成时，可以使用称为 *非负矩阵分解*（NNMF）的良好算法，而不是使用标准的 PCA。该算法基于 Frobenius 范数优化损失函数（在 *W* 和 *H* 上交替）：

![$$ L=\frac{1}{2}{\left\Vert X- WH\right\Vert}_{Frob}², where{\left\Vert A\right\Vert}_{Frob}²=\sqrt{\sum_i{\sum}_j{\left|{a}_{ij}\right|}²} $$](img/484548_1_En_8_Chapter_TeX_Eque.png)

如果 *dim*(*X*) = *n* × *m*，那么 *dim*(*W*) = *n* × *p* 和 *dim*(*H*) = *p* × *m*，其中 *p* 等于请求的组件数量（即`n_components`参数），这通常 *小于* 原始维度 *n* 和 *m*。

最终的重构完全是加性的，并且已经证明它对于通常没有非负元素的图像或文本特别有效。在下面的代码片段中，有一个使用 Iris 数据集（它是非负的）的例子。`init`参数可以取不同的值（参见文档），这些值决定了数据矩阵最初是如何处理的。对于非负矩阵，一个随机选择是仅进行缩放（不执行 SVD）：

```py
from sklearn.datasets import load_iris
from sklearn.decomposition import NMF
iris = load_iris()
iris.data.shape
# Output
(150L, 4L)
nmf = NMF(n_components=3, init="random", l1_ratio=0.1)
Xt = nmf.fit_transform(iris.data)
nmf.reconstruction_err_
# Output
1.8819327624141866
iris.data[0]
# Output
array([ 5.1, 3.5, 1.4, 0.2])
Xt[0]
# Output
array([ 0.20668461, 1.09973772, 0.0098996 ])
nmf.inverse_transform(Xt[0])
# Output
array([5.10401653, 3.49666967, 1.3965409, 0.20610779])
```

NNMF，与其他分解方法一起，对于更高级的技术，如推荐系统和主题建模，将非常有用。NNMF 对其参数非常敏感（特别是初始化和正则化），因此我建议阅读原始文档以获取更多信息：[`http://scikit-learn.org/stable/modules/generated/sklearn.decomposition.NMF.html`](http://scikit-learn.org/stable/modules/generated/sklearn.decomposition.NMF.html)。

## 稀疏 PCA

Scikit-learn 提供了不同的 PCA 变体，可以解决特定问题。我确实建议阅读原始文档。然而，我想提到`SparsePCA`，它允许在提取主成分的同时利用数据的自然稀疏性。如果你考虑必须被分类的手写数字或其他图像，它们的初始维度可以相当高（一个 10x10 的图像有 100 个特征）。然而，应用标准 PCA 只选择平均最重要的特征，假设每个样本可以使用相同的组件重建。简化来说，这相当于：

![公式：\( {y}_R={c}_1{y}_{R1}+{c}_2{y}_{R2}+\dots +{c}_{\left(g-1\right)}{y}_{R\left(g-1\right)}+{c}_g{y}_{Rg} \)](img/484548_1_En_8_Chapter_TeX_Equf.png)

另一方面，我们可以始终使用有限数量的组件，但不需要由密集投影矩阵给出的限制。这可以通过使用稀疏矩阵（或向量）来实现，其中非零元素的数量相当低。这样，每个元素都可以使用其特定的组件重建（在大多数情况下，它们将是最重要的），这可以包括通常由密集 PCA 丢弃的元素。前面的表达式现在变为：

![公式：\( {y}_R=\left[{c}_1{y}_{R1}+{c}_2{y}_{R2}+\dots +{c}_{\left(g-1\right)}{y}_{R\left(g-1\right)}+{c}_g{y}_{Rg}\right]+ ZEROTERM \)](Images/484548_1_En_8_Chapter_TeX_Equg.png)

![公式：\( ZEROTERM=\left(0\bullet {y}_{R\left(g+1\right)}+0\bullet {y}_{R\left(g+2\right)}+\dots +0\bullet {y}_{R\left(g+n-1\right)}+0\bullet {y}_{R(gn)}\right) \)](img/484548_1_En_8_Chapter_TeX_Equh.png)

在这里，非零分量已经被放入了第一个块（它们的顺序与之前的表达式不同），而所有其他的零项已经被分开。从线性代数的角度来看，现在的向量空间具有原始维度。然而，利用稀疏矩阵的强大功能（由`scipy.sparse`提供），Scikit-learn 可以比传统的 PCA 更高效地解决这个问题。

下面的代码片段展示了具有 60 个分量的稀疏 PCA。在这种情况下，它们通常被称为原子，稀疏程度可以通过*L1 范数*正则化来控制（更高的 alpha 参数值会导致更稀疏的结果）。这种方法在分类算法中非常常见，将在下一章中讨论：

```py
from sklearn.decomposition import SparsePCA
spca = SparsePCA(n_components=60, alpha=0.1)
X_spca = spca.fit_transform(digits.data / 255)
spca.components_.shape
# Output
(60L, 64L)
```

想要了解更多关于 SciPy 稀疏矩阵的信息，请访问[`https://docs.scipy.org/doc/scipy-0.18.1/reference/sparse.html`](https://docs.scipy.org/doc/scipy-0.18.1/reference/sparse.html)。

## 潜在狄利克雷分配（LDA）

LDA 是两个描述的缩写：第一个描述与 NLP 相关，第二个描述与数据分析相关。因此，NLP 中的 LDA 被称为**潜在狄利克雷分配**，而在数据分析中被称为**线性判别分析**。但这可能有点奇怪。我们是在谈论 NLP 中的 LDA 还是数据分析中的 LDA？正如我们将看到的，我们需要谈论 NLP 中的 LDA。但为什么不说一个热门话题，比如线性判别分析以及它与 PCA 之间的区别？PCA 是**主成分分析**的缩写。在本节中，让我们从潜在狄利克雷分配开始。

### 潜在狄利克雷分配（LDA 在 NLP 中）

在我们进入 LDA 的定义之前，我们首先需要了解这种技术的根源——主题建模。目标是识别最能描述一组文档的主题。这些主题只会在主题建模过程中出现，这个过程被称为潜在或无监督学习技术。由于 LDA 是主题建模中流行的技术，让我们给它下一个定义。LDA 想象了一个固定的话题集合。每个话题代表一组单词。LDA 的目标是以一种方式将所有文档映射到话题，使得每个文档中的单词大部分被这些想象中的话题所捕获。

让我们用这个定义，并尝试更简单地重新解释它。你有一组话题；这些话题有一组单词，这些单词分布了话题；每个文档有一组话题。LDA 需要通过单词将集合 X 中的这些话题映射到集合 Y 中的文档。LDA 背后的主要思想是每个文档都可以由话题的分布来描述，每个话题都可以由单词的分布来描述。所以让我们举一个例子或创建一个图像来帮助定义 LDA。

正如我们所说，潜在狄利克雷分配是一种自动发现这些文档包含的主题的技术。假设你有一组以下句子：

+   我吃鱼和蔬菜。

+   鱼是宠物。

+   我的猫咪喜欢吃鱼。

给定前面的句子，LDA 可能会将红色单词归类到主题*F*，我们可能会将其标记为“食物”。同样，蓝色单词可能会被归类到另一个单独的主题*P*，我们可能会将其标记为“宠物”。LDA 将每个主题定义为单词袋，并且你必须根据你的判断来标记这些主题。

如我们所见，LDA 在按单词级别对数据进行分类时有两个好处：

1.  我们可以通过词频推断每个句子的内容分布，例如，如果我们看到一个句子中只有一个 F，那么它将是 100% 主题 F。让我们看看我们示例的统计数据。

    +   *句子 1*：100% 主题 F

    +   *句子 2*：100% 主题 P

    +   *句子 3*：33% 主题 P 和 67% 主题 F

1.  我们可以推导出每个词在给定主题中的比例。例如，主题 F 可能包含以下比例的词：

    +   40% 吃，40% 鱼，20% 蔬菜，…

        现在我们已经获得了一些理解，让我们通过一些关于 LDA 如何工作的步骤来亲自动手；它分为三个步骤。

        为了说明这些步骤，想象你现在正在发现文档中的主题而不是句子。想象你有两个包含以下单词的文档（图 8-12）。

        ![img/484548_1_En_8_Chapter/484548_1_En_8_Fig12_HTML.jpg](img/484548_1_En_8_Fig12_HTML.jpg)

        图 8-12

        两份文档的表格：X 和 Y

1.  步骤 1：你告诉算法你认为有多少个主题。你可以使用有根据的估计（例如，以前分析的结果）或者简单地尝试错误。在尝试不同的估计时，你可能选择生成你希望的可解释性主题，或者产生最高统计确定性（即对数似然）的那个。在我们之前的例子中，主题的数量可能仅通过目测文档就能推断出来。

1.  步骤 2：算法将每个词分配到一个临时主题。主题分配是临时的，因为它们将在步骤 3 中更新。临时主题以半随机的方式分配给每个词（确切地说，是根据狄利克雷分布）。这也意味着如果一个词出现两次，每个词可能被分配到不同的主题。请注意，在分析实际文档时，功能词（例如，“the”，“and”，“my”）被移除并且不会被分配到任何主题。

1.  步骤 3：算法将检查并更新主题分配，遍历每份文档中的每个词。对于每个词，其主题分配将基于两个标准进行更新：

    +   该词在主题中的普遍程度如何？

    +   主题在文档中有多普遍？

那么，步骤三中的这两个标准是如何工作的？

首先，该词在主题中的普遍程度如何？由于“鱼”这个词在两份文档中占剩余主题 F 词的近一半，但在剩余主题 P 词中占 0%，随机选择的“鱼”更有可能关于主题 F（图 8-13）。

![img/484548_1_En_8_Chapter/484548_1_En_8_Fig13_HTML.jpg](img/484548_1_En_8_Fig13_HTML.jpg)

其次，该词在主题中的普遍程度如何？由于文档 Y 中的词以 50-50 的比例分配给主题 F 和主题 P，剩余的“鱼”词似乎同样可能关于任一主题。

在这个例子之后，我们现在知道它是如何工作的。为了适应这个算法，我们仍然需要一些关于 LDA 的数学信息，但本书的目标只是对算法有一个良好的理解。如果我们需要提供一些数学知识，它将在最后一部分。因此，我们有一个 LDA 的代码示例，我们将继续讨论 PCA 与 LDA。

## LDA 与 PCA 的比较

如我们在上一节中关于 NLP 中的 LDA（**潜在狄利克雷分配**）所讨论的，现在是时候讨论 LDA（**线性判别分析**）了，但与 PCA（**主成分分析**）一起。LDA 与 PCA 的比较是机器学习中最为常见的比较之一。我们将以简单的方式定义它，以便那些没有强大数学背景的人也能理解。我们只需要您了解其背后的基本思想。

再次，什么是主成分分析或 PCA？PCA 是一种降维方法，通常用于通过将大量变量转换为一个包含大部分信息的较小集合来降低大型数据集的维度。

那么，线性判别分析是什么？LDA 是一种降维技术，用作机器学习和模式分类应用中的预处理步骤。

现在我们需要了解降维技术的含义。降维技术的主要目标是通过对特征进行转换，从高维空间到低维空间，通过去除冗余和依赖的特征来减少维度。

这个比较的关键是监督特征：PCA 使用相关性，旨在无监督学习；LDA 使用分类。

*PCA 做什么？*

+   它旨在找到解释数据中最大方差（包括误差和变量内的方差）的成分。与 LDA 不同，它不考虑类别成员（即无监督），当此类信息不可用时使用。重要的是，LDA 和 PCA 都不需要任何关于变量之间关系的先验概念，并且得到的成分不能根据潜在结构来解释。

*LDA 做什么？*

+   当有此类先验信息时（即监督），这种方法识别出成分（即观察变量的线性组合），这些成分最大化了类别分离（即类间方差）。例如，您有一个包含指定每个观察类别变量的训练集。

但当我们比较 PCA 和 LDA 时，我们需要通过图表来观察它们在数据中的表现——它们如何作用于数据。通过数据可视化，我们可以看到数据，我们的模型如何与之反应，或者我们的工具如何改变我们的数据，以及我们应该如何调整模型参数以获得最佳的数据模型。我们将在另一章中讨论数据可视化。让我们可视化数据，以获得更多关于 PCA 和 LDA 的直观理解（图 8-14 和 8-15）。

## ZCA 白化

ZCA 白化是 PCA 的一部分。ZCA 代表**零相位成分分析**，这是一种用于去相关（白化）数据（图像）的白化变换。

那么，*什么是白化变换*？

**白化变换**或**球化变换**是一种线性变换，它将具有已知协方差矩阵的随机变量向量转换成一组新变量，其协方差是单位矩阵，这意味着它们是不相关的，并且每个都具有方差。这种变换被称为“白化”，因为它将输入向量转换为白噪声向量。

它用于数据增强，那么我们来看看我们如何使用它。正如我们所知，ZCA 是一种图像增强技术，让我们先谈谈数据增强。什么是数据增强？

深度学习神经网络的性能通常会随着可用数据的数量而提高。

数据增强是一种从现有训练数据中人工创建新训练数据的技术。这是通过对训练数据中的示例应用特定领域的技巧来完成的，从而创建新的和不同的训练示例。图像数据增强可能是最知名的数据增强类型，它涉及创建属于原始图像同一类别的训练数据集中的图像的变换版本。变换包括来自图像处理领域的各种操作，如平移、翻转、缩放等等。目的是通过新的、合理的示例来扩展训练数据集。这意味着训练集图像的变体，这些变体很可能是模型会看到的。

例如，一张猫的图片水平翻转可能是有意义的，因为照片可能是从左边或右边拍摄的。而一张猫的图片垂直翻转则没有意义，考虑到模型不太可能看到一张颠倒的猫的照片，这很可能是不合适的。

因此，很明显，必须仔细选择用于训练数据集的具体数据增强技术，并且要在训练数据集的上下文中以及了解问题领域的情况下进行选择。此外，单独和联合实验数据增强方法可能很有用，以查看它们是否会导致模型性能的可测量改进，可能使用一个小型的原型数据集、模型和训练运行。现代深度学习算法，如卷积神经网络（CNN），可以学习到对图像位置不变的特性。尽管如此，增强可以进一步帮助这种变换不变的学习方法，并有助于模型学习到对变换也具有不变性的特征，例如从左到右到从上到下的顺序、照片中的光照水平等。

但要理解图像数据增强通常仅应用于**训练数据集**，而不是验证或测试数据集。这与数据准备，如图像缩放和像素缩放不同；它们必须在所有与模型交互的数据集中一致执行。一些数据增强技术包括

+   水平和垂直平移增强

+   水平和垂直翻转增强

+   随机旋转增强

+   随机亮度增强

+   随机缩放增强

+   ZCA 白化

大多数这些算法都在第六章的“数据增强”部分进行了考虑和讨论。

因此，我们看到各种数据增强技术，但我们将专注于 ZCA 白化。图像的白化转换是一种线性代数操作，它减少了像素图像矩阵中的冗余。图像中的冗余减少旨在更好地突出图像中的结构和特征，以便学习算法能够更好地识别。

通常，图像白化使用 PCA 技术执行。最近，一种称为 ZCA 的替代方法显示出更好的结果，导致转换后的图像保留了所有原始维度，并且与 PCA 不同，结果图像仍然看起来像它们的原始图像。

你可以通过将`zca_whitening`参数设置为 True 来执行 ZCA 白化转换。

## 摘要
