## 什么是 RNN？

你在书中已经看到了几种类型的神经网络，所以你知道神经网络的较高层次表示看起来像图 8-5。

![图片](img/483137_2_En_8_Fig5_HTML.png)

水平流程图。输入传递到神经网络以生成输出。

图 8-5

神经网络的较高层次表示

显然，神经网络处理输入并产生输出，并且它可以在具有不同特征的各种输入数据上工作。然而，一个需要注意的关键点是，这个神经网络没有事件（输入）发生时间的概念，只知道输入已经到来。那么，对于长时间内作为流进入的事件（输入）会发生什么？图 8-5 中所示的神经网络如何处理事件的趋势、事件的季节性等问题？它如何从过去学习并将其应用于现在和未来？循环神经网络通过逐步构建神经网络，将来自先前时间戳的信号引入当前网络来尝试解决这个问题。图 8-6 描述了一个 RNN。

![图 8-6](img/483137_2_En_8_Fig6_HTML.png)

循环神经网络的流程图包括输入 T 1 到 T n 和输出 T 1 到 T n。循环神经网络从上到下连接。

图 8-6

循环神经网络

你可以看到，RNN 是一个具有多个层或阶段的神经网络。每个阶段代表一个时间，T；T+1 时的 RNN 将 T 时的 RNN 视为一个信号。每个阶段将其输出传递给下一个阶段。从阶段传递到下一个阶段的隐藏状态是 RNN 工作得如此之好的关键，这个隐藏状态类似于某种记忆保持。RNN 层（或阶段）作为编码器，在处理输入序列并返回其自身内部状态时起作用。这个状态作为下一阶段解码器的输入，该解码器被训练来预测目标序列的下一个点，给定目标序列的先前点。具体来说，它被训练将目标序列转换为相同的序列，但未来一个时间步长偏移。

在训练 RNN 时，使用反向传播，就像在其他神经网络中一样，但在 RNN 中还有一个时间维度。在反向传播中，我们计算损失相对于每个参数的导数（梯度）。使用这些信息（损失），我们可以将参数向相反方向移动，目标是最小化损失。由于我们在时间上移动，我们可以在每个时间步长都有一个损失，我们可以将时间上的损失相加以得到每个时间步长的损失。这等同于时间上梯度的求和。

由常规神经网络节点构成的先前 RNN 的问题在于，当我们试图模拟由大量其他值分隔的序列值之间的依赖关系时，时间步 T 的梯度依赖于 T-1 的梯度，T-1 的梯度又依赖于 T-2 的梯度，依此类推。这导致最早梯度对贡献越来越小，因为我们沿着时间步移动，梯度链变得越来越长。这就是所谓的梯度消失问题。这意味着那些早期层的梯度会越来越小，因此网络无法学习长期依赖。RNN 因此变得有偏见，只能处理短期数据点。LSTM 网络是解决 RNN 中这个问题的方法之一。

## 什么是 LSTM？

LSTM 网络是一种循环神经网络。正如前文所述，RNN 是一种试图模拟时间或序列相关行为的神经网络，例如语言、股票价格、天气传感器等。这是通过将时间 T 时刻神经网络层的输出反馈到时间 T+1 时刻同一网络层的输入来实现的。LSTM 在 RNN 的基础上进行了扩展，增加了一个记忆组件，旨在帮助将时间 T 时刻学习到的信息传播到未来的 T+1、T+2 等时刻。主要思想是 LSTM 可以在选择性更新状态的同时，忘记之前状态中的不相关部分，然后输出与未来相关的状态部分。这如何解决 RNN 中的梯度消失问题呢？好吧，现在我们正在丢弃状态的一部分，更新状态的一部分，并向前传播状态的一部分，这样我们就不再像在 RNN 中那样有一个长长的反向传播链。因此，LSTMs 比典型的 RNNs 要高效得多。

传统循环神经网络（RNN）倾向于记住所有东西，包括不必要的输入，这导致它们无法从长序列中学习。相比之下，长短期记忆网络（LSTMs）会选择性地记住重要的输入，这使得它们能够处理短期和长期依赖。LSTMs 通过使用三个重要的门在隐藏状态和细胞状态之间释放信息来实现这一点。一个常见的 LSTM 单元由一个细胞、一个输入门、一个输出门和一个遗忘门组成。细胞在任意时间间隔内记住值，而三个门调节信息流入和流出细胞的信息流。

更详细的 LSTM 架构如图 8-12 所示。使用的关键函数是 tanh 和 sigmoid 激活函数。F[t] 是遗忘门，I[t] 是输入门，O[t] 是输出门。

![](img/483137_2_En_8_Fig12_HTML.jpg)

长短期记忆模型的结构包括 c 下标 t 减 1，h 下标 t 减 1，LSTM 单元，和 c 下标 t 加 1，h 下标 t 加 1。输入是 x 下标 t 减 1，x 下标 t，和 x 下标 t 加 1。输出是 o 下标 t 减 1，o t，和 o t 加 1。

图 8-12

一个详细的 LSTM 网络图（来源：[`commons.wikimedia.org/wiki/File:Long_Short-Term_Memory.svg`](https://commons.wikimedia.org/wiki/File:Long_Short-Term_Memory.svg)）

遗忘门是 LSTM 阶段的第一部分，它决定了从前一个阶段应该记住或忘记多少信息。这是通过将前一个隐藏状态 hT-1 和当前输入 xT 通过 sigmoid 函数传递来实现的。

输入门通过使用 sigmoid 函数和 tanh 函数帮助决定传递到当前阶段的信息量。

输出门控制隐藏状态将保留多少信息并传递到下一个阶段。同样，当前状态通过 tanh 函数传递。

仅作信息参考，以下是一个带有遗忘门的 LSTM 单元前向传递方程的紧凑形式，其中初始值是 c0 = 0 和 h0 = 0，下标表示时间步：

![$$ {F}_t=\sigma \left({W}_f\bullet \left[{h}_{t-1},{x}_t\right]+{b}_f\right) $$](img/483137_2_En_8_Chapter/483137_2_En_8_Chapter_TeX_Equa.png)

![$$ {I}_t=\sigma \left({W}_i\bullet \left[{h}_{t-1},{x}_t\right]+{b}_i\right) $$](img/483137_2_En_8_Chapter/483137_2_En_8_Chapter_TeX_Equb.png)

![$$ {\overset{\sim }{C}}_t=\mathit{\tanh}\left({W}_C\bullet \left[{h}_{t-1},{x}_t\right]+{b}_C\right) $$](img/483137_2_En_8_Chapter/483137_2_En_8_Chapter_TeX_Equc.png)

![$$ {C}_t={F}_t\ast {C}_{t-1}+{I}_t\ast {\overset{\sim }{C}}_t $$](img/483137_2_En_8_Chapter_TeX_Equd.png)

![$$ {O}_t=\sigma \left({W}_o\bullet \left[{h}_{t-1},{x}_t\right]+{b}_o\right) $$](img/483137_2_En_8_Chapter/483137_2_En_8_Chapter_TeX_Eque.png)

![$$ {h}_t={O}_t\ast \mathit{\tanh}\left({C}_t\right) $$](img/483137_2_En_8_Chapter_TeX_Equf.png)

在这里，F[t]，I[t]，O[t]，和 C[t] 分别对应图 8-12 中的对应部分。![$$ {\overset{\sim }{C}}_t $$](img/483137_2_En_8_Chapter_TeX_IEq1.png)是 tanh 门的输出。

以下列表总结了所有变量：

+   X[t]：LSTM 单元的输入向量。

+   F[t]：遗忘门的激活向量。

+   I[t]：输入/更新门的激活向量。

+   O[t]：输出门的激活向量。

+   h[t]：隐藏状态向量，也称为整个 LSTM 单元的输出向量。

+   c[t]：细胞状态/上下文向量。

+   W, b：权重矩阵和偏置向量。每个 W 矩阵和 b 向量组合及其独特的下标代表一组必须在训练期间学习的可学习参数。

对于 *σ*，这是 sigmoid 激活函数。

## 用于异常检测的 LSTM

本节介绍了使用时间序列数据作为示例的一些用例的 LSTM 实现。有几个不同的时间序列数据集可供使用，以尝试使用 LSTM 检测异常。它们都有时间戳和可以轻松在 Python 中绘制的值。

基本想法是在正常的时间序列数据序列上训练一个 LSTM。在这种情况下，序列的前半部分将是我们训练集，序列的后半部分将是测试集。我们将数据集处理成 T 个时间步长的滚动窗口，其中前 T-1 个时间步长将被传递到 LSTM 中，以预测第 T 个时间步长的值。

然后，我们将使用 LSTM 预测整个时间序列序列，包括训练和测试分割。在预测每个点时，我们将使用平均绝对误差计算 LSTM 的预测输出和实际真实输出之间的损失。为了预测异常，我们将设置预测误差阈值，如果超过此阈值，则表示自动标记为异常。我们使用 95% 分位数的训练误差，或最高 5% 的误差。

此阈值将使用 top n^(th) 百分比训练预测误差来设置。然后，将此边界应用于其余的测试数据，这次，任何高于此边界的预测误差都将被计为异常。我们使用 95^(th) 分位数的训练误差，或最高 5% 的误差。

只在序列的前半部分进行训练的理由是，在我们用异常值预测后半部分之前，让模型了解正常时间序列值的样子。想法是预测值不会与异常值匹配，产生更高的误差，并被标记为异常。由于异常检测不需要标签，这是一个无监督学习任务。这个通用公式是我们将在几个不同的时间序列数据集上应用，以无监督方式执行异常检测。

序列中有 10,320 个数据点，你的目标是找到异常值。这意味着你试图找出数据点何时异常。如果你可以根据直到 T-1 的历史数据预测时间 T 的数据点，那么你就有了一种方法来比较预期值和实际值，以查看你是否在时间 T 的预期值范围内。如果你预测 2015 年 1 月 1 日需要 y_pred 数量的出租车，那么你可以将这个 y_pred 与实际的 y_actual 进行比较。y_pred 和 y_actual 之间的差异给出了误差，当你得到序列中所有点的误差时，你最终得到一个仅包含误差的分布。

为了实现这一点，你将使用 Keras 的序列模型。该模型由两个 LSTM 层组成，最后的 LSTM 层是输出层。第一个 LSTM 层接受时间序列数据作为输入，并学习如何随时间学习值。最后的 LSTM 层是输出层，它将接受前一个 LSTM 层的输出并将其转换为对下一个时间步的单个预测。然后，你对这个 LSTM 层应用 sigmoid 激活函数，使得最终输出介于 0 和 1 之间。

如上图所示，你使用了 LSTM 层。让我们看看 LSTM 层函数的详细信息，包括所有可能的参数（来源：[`www.tensorflow.org/api_docs/python/tf/keras/layers/LSTM`](https://www.tensorflow.org/api_docs/python/tf/keras/layers/LSTM)）：

```py
tf.keras.layers.LSTM(
units,
activation='tanh',
recurrent_activation='sigmoid',
use_bias=True,
kernel_initializer='glorot_uniform',
recurrent_initializer='orthogonal',
bias_initializer='zeros',
unit_forget_bias=True,
kernel_regularizer=None,
recurrent_regularizer=None,
bias_regularizer=None,
activity_regularizer=None,
kernel_constraint=None,
recurrent_constraint=None,
bias_constraint=None,
dropout=0.0,
recurrent_dropout=0.0,
return_sequences=False,
return_state=False,
go_backwards=False,
stateful=False,
time_major=False,
unroll=False,
**kwargs
)
```

以下是对参数的总结：

+   `units`: 正整数，表示输出空间的维度。

+   `activation`: 要使用的激活函数。默认：双曲正切函数 (`tanh`)。如果你传递 `None`，则不应用激活函数（即，“线性”激活：`a(x) = x`）。

+   `recurrent_activation`: 用于循环步骤的激活函数。默认：sigmoid (`sigmoid`)。如果你传递 `None`，则不应用激活函数（即，“线性”激活：`a(x) = x`）。

+   `use_bias`: 布尔值（默认 `True`），表示层是否使用偏置向量。

+   `kernel_initializer`: `kernel` 权重矩阵的初始化器，用于输入的线性变换。默认：`glorot_uniform`。

+   `recurrent_initializer`: `recurrent_kernel` 权重矩阵的初始化器，用于循环状态的线性变换。默认值：`orthogonal`。

+   `bias_initializer`: 偏置向量的初始化器。默认值：`zeros`。

+   `unit_forget_bias`: 布尔值（默认 `True`）。如果为 `True`，则在初始化时将 `1` 添加到遗忘门的偏置。将其设置为 `True` 还会强制 `bias_initializer="zeros"`。这在 Jozefowicz 等人的研究中被推荐。

+   `kernel_regularizer`: 应用于 `kernel` 权重矩阵的正则化函数。默认值：`None`。

+   `recurrent_regularizer`: 应用于 `recurrent_kernel` 权重矩阵的正则化函数。默认值：`None`。

+   `bias_regularizer`: 应用于偏置向量的正则化函数。默认值：`None`。

+   `activity_regularizer`: 应用于层输出（其“激活”）的正则化函数。默认值：`None`。

+   `kernel_constraint`: 应用于 `kernel` 权重矩阵的约束函数。默认值：`None`。

+   `recurrent_constraint`: 应用于 `recurrent_kernel` 权重矩阵的约束函数。默认值：`None`。

+   `bias_constraint`: 应用于偏置向量的约束函数。默认值：`None`。

+   `dropout`: 介于 `0` 和 `1` 之间的浮点数。用于输入线性变换的单元丢弃比例。默认值：`0`。

+   `recurrent_dropout`: 介于 `0` 和 `1` 之间的浮点数。用于循环状态线性变换的单元丢弃比例。默认值：`0`。

+   `return_sequences`: 布尔值。表示是否返回输出序列中的最后一个输出，或整个序列。默认值：`False`。

+   `return_state`: 布尔值。表示是否返回最后一个状态以及输出。默认值：`False`。

+   `go_backwards`: 布尔值（默认 `False`）。如果为 `True`，则反向处理输入序列并返回反转的序列。

+   `stateful`: 布尔值（默认 `False`）。如果为 `True`，则每个样本在批次中索引 i 的最后一个状态将用作下一个批次中索引 i 的样本的初始状态。

+   `time_major`: `inputs` 和 `outputs` 张量的形状格式。如果为 `True`，输入和输出将具有形状 `[timesteps, batch, feature]`，而在 `False` 的情况下，它将是 `[batch, timesteps, feature]`。使用 `time_major = True` 会稍微高效一些，因为它避免了 RNN 计算开始和结束时的转置。然而，大多数 TensorFlow 数据是批次主序，因此默认情况下此函数接受输入并以批次主序形式发出输出。

+   `unroll`: 布尔值（默认 `False`）。如果为 `True`，网络将被展开，否则将使用符号循环。展开可以加快 RNN 的速度，尽管它通常更占用内存。展开仅适用于短序列。
