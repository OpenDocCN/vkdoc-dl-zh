# 6. 超参数调优

超参数可以松散地定义为在训练过程中不发生变化的参数。例如，FFNN 中的层数、每层的神经元数量、激活函数、学习率^(1)等等。

本章探讨了寻找最佳超参数以从模型中获得最佳结果的问题。进行这一过程被称为超参数调优。我们首先描述什么是黑盒优化问题，以及这些类别的问题如何与超参数调优相关。我们探讨了处理这类问题的三种最常见方法：*网格搜索、随机搜索*和*贝叶斯优化*。通过示例，你将了解在什么条件下哪种方法有效。本章结束时，你将看到如何使用这些技术使用真实世界数据集（Zalando 数据集）来调整深度模型。本章的目标不是给你一个所有可能的 Python 库的概述，以帮助你进行超参数调优，但到本章结束时，你应该知道什么是超参数调优，为什么它很重要，以及它是如何工作的，至少在其最基本的形式上。

## 黑盒优化

超参数调优的问题只是更一般类型问题的一个子类：*黑盒优化*。一个黑盒函数 *f*(*x*) 是一个

![f(x):{\mathbb{R}}^n\to \mathbb{R}](img/463356_2_En_6_Chapter_TeX_Equa.png)

是一个解析形式未知的函数。黑盒函数可以评估以获得其定义的所有 *x* 值的值，但无法获得其他信息，如梯度。通常，当我们试图在给定某些约束的情况下找到 *f*(*x*) 的最大值或最小值时，我们谈论黑盒函数的全局优化（有时称为黑盒问题）。以下是一些这类问题的例子：

+   寻找给定机器学习模型的最佳超参数，以最大化所选优化指标

+   寻找一个函数的最大值或最小值，这个函数只能通过数值或我们无法查看的代码来评估。在某些行业环境中，存在非常复杂的遗留代码，有一些函数必须基于其结果进行最大化。

+   寻找最佳的石油钻探地点。你的函数是你能找到多少石油，而你的 *x* 是你的位置。

+   在难以建模的情况下寻找最佳参数组合，例如在太空中发射火箭时。如何优化燃料量、火箭每个阶段的直径、精确轨迹等等。

这是一个非常迷人的问题类别，它产生了智能的解决方案。我们将探讨其中的三个：网格搜索、随机搜索和贝叶斯优化。

为什么寻找神经网络最佳超参数是一个黑盒问题？因为我们无法计算网络输出相对于超参数的信息，例如 FFNN 中的层数，尤其是在使用复杂优化器或自定义函数时。我们需要其他方法来找到最佳超参数，以最大化选择的优化指标。请注意，如果我们能获得梯度，我们可以使用梯度下降等算法来找到最大值或最小值。

注意

我们的黑盒函数 *f* 将是我们的神经网络模型（包括优化器、成本函数形式等），它以超参数作为输入，输出我们的优化指标。在这种情况下，*x* 将是包含超参数的数组。

问题可能看起来相当简单，为什么不尝试所有可能性呢？好吧，这可能在我们在过去章节中看到的例子中是可能的，但如果你正在处理一个问题，并且训练模型需要一周时间，这可能是一个挑战。由于通常你会有几个超参数，尝试所有可能性将不可行。让我们通过一个例子来更好地理解它。假设我们正在训练一个具有多个层的神经网络模型。我们可能决定考虑以下超参数，以查看哪种组合效果最好：

+   学习率：假设我们想尝试 *n* · 10^(−4) 的值，其中 *n* = 1, …, 10²（100 个值）

+   正则化参数：0，0.1，0.2，0.3，0.4 和 0.5（六个值）

+   优化器选择：GD，RMSProp 或 Adam（三个值）

+   隐藏层数量：1，2，3，5 和 10（五个值）

+   隐藏层中的神经元数量：100，200 和 300（三个值）

考虑你将需要训练你的网络

![100×6×3×5×3=27000](img/463356_2_En_6_Chapter_TeX_Equb.png)

如果你想测试所有可能的组合。如果你的训练需要五分钟，你将需要 13.4 周的计算时间。如果训练需要数小时或数天，这变得不可能。如果训练需要一天，例如，你需要 73.9 年才能尝试所有可能性。大多数超参数的选择将来自经验；例如，你可以安全地始终使用 Adam，因为它是更好的优化器（在几乎所有情况下）。但你将无法避免尝试调整其他参数，如隐藏层的数量或学习率。你可以通过经验（就像优化器一样）或一些聪明的算法来减少你需要组合的数量，正如你将在本章后面看到的。

### 关于黑盒函数的注意事项

黑盒函数通常分为两大类：

+   **廉价函数**：可以评估数千次的函数

+   **昂贵函数**：通常只能评估几次的函数，通常少于 100 次

如果黑盒函数成本低，那么优化方法的选择并不关键。例如，我们可以通过数值计算来评估与*x*相关的梯度，或者简单地通过在大量点上评估函数来搜索最大值。如果函数成本高，我们需要更智能的方法。其中之一是*贝叶斯优化*，我们将在本章后面讨论，以给你一个关于这些方法如何工作以及它们复杂性的概念。

注意

神经网络几乎总是，尤其是在深度学习领域，是成本较高的函数。

对于成本较高的函数，我们需要找到用尽可能少的评估次数解决问题的方法。

## 超参数调整问题

在我们探讨如何找到最佳超参数之前，我们需要快速回顾一下神经网络，并讨论在深度模型中我们可以调整什么。通常，当谈到超参数时，初学者只会想到数值参数，例如学习率或正则化参数。记住，以下也可以调整以查看是否能得到更好的结果：

+   **训练轮数**：有时简单地延长网络的训练时间会得到更好的结果。

+   **优化器的选择**：你可以尝试选择不同的优化器。如果你使用的是普通梯度下降，你可以尝试 Adam，看看是否能得到更好的结果。

+   **正则化方法的选择**：如前所述，有几种应用正则化的方法。尝试不同的方法可能值得尝试。

+   **激活函数的选择**：尽管过去章节中用于隐藏层神经元的激活函数是 ReLU，但其他函数可能效果更好。例如尝试 sigmoid 或 Swish 可能会帮助你获得更好的结果。

+   **层数和每层的神经元数量**：尝试不同的配置。例如尝试具有不同神经元数量的层。

+   **学习率衰减方法**：尝试不同的学习率衰减方法（如果你没有使用已经具备这种功能的优化器）。

+   **小批量大小**：调整小批量的大小。当你有少量数据时，可以使用批量梯度下降。当你有大量数据时，小批量更有效率。

+   权重初始化方法。

让我们将我们可以在模型中调整的参数分为以下三个类别：

1.  参数是连续的实数，换句话说，可以假设任何值。例如：学习率和正则化参数。

1.  参数是离散的，但理论上可以假设无限多个值。例如：隐藏层的数量、每层的神经元数量或训练轮数。

1.  参数是离散的，只能假设有限的可能值。例如：优化器、激活函数和学习率衰减方法。

对于类别 3，除了尝试所有可能性之外，没有太多可做的事情。它们通常会完全改变模型，因此无法模拟它们的影响，因此测试成为唯一的选择。这也是经验可能最有帮助的类别。众所周知，Adam 优化器几乎是最佳选择，因此您可以在一开始就将精力集中在其他地方。对于类别 1 和 2，这要困难一些，您需要使用一些智能方法来找到最佳值。

## 样本黑盒问题

为了尝试解决一个黑盒问题，让我们创建一个“假”的黑盒问题。给定公式，找到函数 *f*(*x*) 的最大值

![$$ {\displaystyle \begin{array}{l}g(x)=\cos \frac{x}{4}-\sin \frac{x}{4}-\frac{5}{2}\cos \frac{x}{2}+\frac{1}{2}\sin \frac{x}{2}\\ {}h(x)=-\cos \frac{x}{3}-\sin \frac{x}{3}-\frac{5}{2}\cos \frac{2}{3}x+\frac{1}{2}\sin \frac{2}{3}x\\ {}f(x)=10+g(x)+\frac{1}{2}h(x)\end{array}} $$](img/463356_2_En_6_Chapter_TeX_Equc.png)

假装不知道公式本身。公式将让您看到结果，但让我们假装它们是未知的。您可能会想知道为什么我们使用如此复杂的公式。我们想要有一个具有几个极大值和极小值的函数，以展示方法在非平凡示例上的工作方式。*f*(*x*) 可以使用以下 Python 代码实现

```py
def f(x):
tmp1 = -np.cos(x/4.0)-np.sin(x/4.0)
-2.5*np.cos(2.0*x/4.0)+0.5*np.sin(2.0*x/4.0)
tmp2 = -np.cos(x/3.0)-np.sin(x/3.0)
-2.5*np.cos(2.0*x/3.0)+0.5*np.sin(2.0*x/3.0)
return 10.0+tmp1+0.5*tmp2
```

### 网格搜索

我们首先考虑的方法是网格搜索，这也是最不“智能”的方法。网格搜索只是简单地以固定间隔尝试函数，并查看对于哪个 *x*，函数 *f*(*x*) 取得最高值。在这个例子中，我们想要找到在两个 *x* 值 *x*[*min*] 和 *x*[*max*] 之间的函数 *f*(*x*) 的最大值。我们做的只是取 *x*[*min*] 和 *x*[*max*] 之间等间距的 *n* 个点，并在这些点上评估函数。我们定义一个点向量

![$$ \boldsymbol{x}=\left({x}_{min},{x}_{min}+\frac{\varDelta x}{n},\dots, {x}_{min}+\left(n-1\right)\frac{\varDelta x}{n}\right) $$](img/463356_2_En_6_Chapter_TeX_Equd.png)

其中我们定义 Δ*x* = *x*[*max*] - *x*[*min*]。然后我们在这些点上评估函数 *f*(*x*)，得到一个包含值的向量 ***f***

![$$ \boldsymbol{f}=\left(f\left({x}_{min}\right),f\left(\ {x}_{min}+\frac{\varDelta x}{n}\right),\dots, f\left({x}_{min}+\left(n-1\right)\frac{\varDelta x}{n}\right)\right) $$](img/463356_2_En_6_Chapter_TeX_Eque.png)

最大值 ![$$ \left(\overset{\sim }{x},\overset{\sim }{f}\right) $$](img/463356_2_En_6_Chapter_TeX_IEq1.png) 的估计将如下：

![$$ \overset{\sim }{f}=\underset{0\le i\le n-1}{\max }{f}_i $$](img/463356_2_En_6_Chapter_TeX_Equf.png)

假设最大值在 ![$$ i=\overset{\sim }{i} $$](img/463356_2_En_6_Chapter_TeX_IEq2.png) 处找到，我们也将有

![$$ \overset{\sim }{x}={x}_{min}+\frac{\overset{\sim }{i}\Delta x}{n} $$](img/463356_2_En_6_Chapter_TeX_Equg.png)

现在，正如您可能想象的那样，您使用的点越多，您的最大值估计就越准确。问题是，如果 *f*(*x*) 的评估成本很高，您将无法测试您可能想要的那么多点。您需要在点数和精度之间找到一个平衡。让我们考虑一个使用我们之前描述的函数 *f*(*x*) 的例子。让我们考虑 *x*[*max*] = 80 和 *x*[*min*] = 0，并取 *n* = 40 个点。我们将有 ![$$ \frac{\Delta x}{n}=2 $$](img/463356_2_En_6_Chapter_TeX_IEq3.png)。我们可以用以下代码在 Python 中轻松创建向量 ***x***：

```py
gridsearch = np.arange(0,80,2)
```

数组 `gridsearch` 将看起来像这样


小贴士

网格搜索仅在黑盒函数便宜时才有效。为了获得好的结果，通常需要大量的采样点。

为了确保您真的得到了最大值，您应该减小步长 Δ*x*，或者增加采样点的数量，直到找到的最大值不再显著变化。在先前的例子中，如图 6-4 右侧的图所示，当我们的步长 Δ*x* 小于大约 2.0 时，或者换句话说，当采样点的数量大于或大约等于 40 时，我们确信我们接近最大值。请记住，40 在一开始看起来可能是一个小数字，但如果 *f*(*x*) 评估您深度学习模型的指标，并且训练需要例如两小时，您将需要 3.3 天的计算机时间。

在深度学习领域，通常两小时并不足以训练一个模型，因此在开始长时间网格搜索之前进行快速计算。此外，请注意，在进行超参数调整时，您是在一个多维空间中移动（您不是只优化一个参数，而是许多参数），因此所需的评估数量会迅速增加。

让我们来看一个快速示例。假设您决定可以承担 50 次对您的黑盒函数的评估。如果您决定想要尝试以下超参数：

+   优化器（RMSProp、Adam 或普通 GD）（三个值）

+   训练轮数（1000、5000 或 10000）（三个值）

您已经看到了九个评估。那么您能尝试多少个学习率的值呢？只有五个！并且只有五个值，您不太可能接近最优值。这表明网格搜索仅适用于成本较低的黑盒函数是可行的。记住，时间往往不是唯一的问题。例如，如果您正在使用 Google Cloud 平台来训练您的网络，您将按秒支付您使用的硬件。也许您有很多时间可用，但成本可能会很快超出预算。

### 粗到细优化

仍然有一个优化技巧可以帮助网格或随机搜索。它被称为“粗到细优化”。假设我们想在 *x*[*min*] 和 *x*[*max*] 之间找到 *f*(*x*) 的最大值。我们将探讨随机搜索的想法，但它与网格搜索以相同的方式工作。以下步骤提供了你需要遵循的算法。

1.  在区域 *R*[1] = (*x*[*min*], *x*[*max*]) 中进行随机搜索。用 (*x*[1], *f*[1]) 标记找到的最大值。

1.  考虑围绕 *x*[1]，*R*[2] = (*x*[1] − *δx*[1]，*x*[1] + *δx*[1]) 的较小区域（其中 *δx*[1] 我们稍后讨论），并在该区域内进行随机搜索。当然，假设真实最大值位于该区域内。用 (*x*[2], *f*[2]) 标记你在这里找到的最大值。

1.  在 *x*[2] 附近重复步骤 2，在用 *R*[3] 标记的区域，*δx*[2] 小于 *δx*[1]，并在此步骤中用 (*x*[3], *f*[3]) 标记你找到的最大值。

1.  再次在 *x*[3] 附近重复步骤 2，在我们用 *R*[4] 标记的区域，*δx*[3] 小于 *δx*[2]。

1.  按照相同的方式继续进行，直到区域 *R*[*i* + 1] 中的最大值 (*x*[*i*], *f*[*i*]) 不再发生变化。

通常只需要一两次迭代，但理论上你可以进行大量的迭代。这个方法的问题是你不能确定你的真实最大值实际上位于你的区域 *R*[*i*] 中。但如果它确实如此，这种优化有一个很大的优点。

让我们考虑我们进行标准随机搜索的情况。如果我们想使采样点之间的平均距离为 *x*[*max*] - *x*[*min*] 的 1%，如果我们决定只进行一次随机搜索，我们就需要大约 100 个点，因此我们必须进行 100 次评估。现在让我们考虑我们刚才描述的算法。我们可以从区域 *R*[1] = (*x*[*min*], *x*[*max*]) 中的十个点开始，其中我们用 (*x*[1], *f*[1]) 标记我们找到的最大值。然后让我们取 ![$$ 2\delta x=\frac{x_{max}-{x}_{\mathrm{min}}}{10} $$](img/463356_2_En_6_Chapter_TeX_IEq12.png) 并在区域 *R*[2] = (*x*[1] - *δx*, *x*[1] + *δx*) 中使用十个点。在 (*x*[1] - *δx*, *x*[1] + *δx*) 区间内，我们将有平均距离为 *x*[*max*] - *x*[*min*] 的 1%，但我们只采样了函数 20 次，而不是 100 次，所以减少了五倍！例如，让我们用以下代码在 *x*[*min*] = 0 和 *x*[*max*] = 80 之间用十个点采样我们之前使用的函数

```py
np.random.seed(5)
randomsearch = np.random.random([10])*80
x = 0
m = 0.0
for i, val in np.ndenumerate(f(randomsearch)):
if (val > m):
m = val
x = randomsearch[i]
```

这给我们提供了最大位置和值 *x*[1] = 69.65 和 *f*[1] = 14.89，所以还不错，但不如真实的精确——69.18 和 15.027。现在让我们在区域 *R*[2] = (*x*[1] - 4, *x*[1] + 4) 附近采样十个点。

```py
randomsearch = x + (np.random.random([10])-0.5)*8
x = 0
m = 0.0
for i, val in np.ndenumerate(maxim(randomsearch)):
if (val > m):
m = val
x = randomsearch[i]
```

这给我们提供了结果 69.189 和 15.027。仅用函数的 20 次评估就得到了相当精确的结果。如果我们用 500 个（比我们刚才做的多 25 倍）采样点进行简单的随机搜索，我们得到 *x*[1] = 69.08 和 *f*[1] = 15.022。这个结果展示了这个技巧如何真正有用。但记住风险：如果你的最大值不在你的区域 *R*[*i*] 中，你将永远找不到它，因为你仍在处理随机数。所以，总是选择相对较大的区域 (*x*[*i*] - *δx*[*i*], *x*[*i*] + *δx*[*i*]) 来确保它们包含你的最大值是一个好主意。

### 贝叶斯优化

#### Nadaraya-Watson 回归

要理解贝叶斯优化，我们必须看看一些新的数学概念。让我们从 1964 年的 Nadaraya-Watson 回归开始，这是一个想法[2]。基本思想相当简单。给定一个未知的函数 *y*(*x*) 和给定 *N* 个点 *x*[*i*] = 1, …, *N*，我们用 *y*[*i*] = *f*(*x*[*i*]) 表示 *i* = 1, …, *N* 时函数在不同 *x*[*i*] 处的值。Nadaraya-Watson 回归的思想是，我们可以使用以下公式在未知点 *x* 上评估未知函数 *f*：

![$$ y(x)=\sum \limits_{i=1}^N{w}_i(x){y}_i $$](img/463356_2_En_6_Chapter_TeX_Equh.png)

其中 *w**i* 是根据以下公式计算的权重

![$$ {w}_i(x)=\frac{K\left(x,{x}_i\right)}{\sum \limits_{j=1}^NK\left(x,{x}_j\right)} $$](img/463356_2_En_6_Chapter_TeX_Equi.png)

其中 *K*(*x*, *x*[*i*]) 被称为 ***核函数***。请注意，根据权重的定义，我们有

![$$ \sum \limits_{i=1}^N{w}_i(x)=1 $$](img/463356_2_En_6_Chapter_TeX_Equj.png)

在文献中，你可以找到几个核函数，但我们感兴趣的是高斯核，通常称为 *径向基函数* (RBF)

![$$ K\left(x,{x}_i\right)={\sigma}²{e}^{-\frac{1}{2{l}²}{\left\Vert x-{x}_i\right\Vert}²} $$](img/463356_2_En_6_Chapter_TeX_Equk.png)

参数 *l* 使高斯形状更宽或更窄。*σ* 通常是你数据的方差，但在这个情况下，它不起作用，因为权重被归一化到 1。这是贝叶斯优化的基础，你将在后面看到。

#### 高斯过程

在讨论高斯过程之前，我们必须定义什么是随机过程。当我们为任何点 *x* ∈ *ℝ*^(*d*) 分配一个随机变量 *f*(*x*) ∈ *ℝ* 时，我们称之为 *随机过程*。如果一个随机过程对于任何有限数量的点，它们的联合分布是正态的，那么它就是高斯过程。这意味着对于 ∀*n* ∈ *ℕ*，以及对于 ∀*x*[1]，…，*x*[*n*] ∈ *ℝ*^(*d*)，那么向量

![$$ \boldsymbol{f}\equiv \left(\begin{array}{c}f\left({x}_1\right)\\ {}\vdots \\ {}f\left({x}_n\right)\end{array}\right)\sim \mathcal{N} $$](img/463356_2_En_6_Chapter_TeX_Equl.png)

其中，我们用符号表示向量分量遵循正态分布，用 ![$$ \mathcal{N} $$](img/463356_2_En_6_Chapter_TeX_IEq13.png) 表示。记住，具有高斯分布的随机变量被称为正态分布。正态分布的概率分布由以下函数给出

![$$ g\left(x\ \right|\ \mu, {\sigma}²\Big)=\frac{1}{\sqrt{2\pi {\sigma}²}}{e}^{\frac{{\left(x-\mu \right)}²}{2{\sigma}²}} $$](img/463356_2_En_6_Chapter_TeX_Equm.png)

其中 *μ* 是分布的均值或期望，*σ* 是标准差。从现在开始，我们将使用以下符号

![f 的平均值公式](img/463356_2_En_6_Chapter_TeX_Equn.png)

并且随机值的协方差将由 *K* 表示

![协方差公式](img/463356_2_En_6_Chapter_TeX_Equo.png)

字母 *K* 的选择是有目的的。在以下内容中，我们将假设协方差将具有高斯形状，并且我们将使用之前定义的径向基函数（RBF）作为 *K*。

#### 稳定过程

为了简化，我们只考虑稳定过程。一个随机过程是稳定的，如果其联合概率分布不随时间变化。这意味着均值和方差在时间上的平移也不会改变。我们还将考虑一个其分布只依赖于点之间相对位置的过程。这导致以下条件

![K(x1,x2) 和 Var[f(x)] 的公式](../images/463356_2_En_6_Chapter/463356_2_En_6_Chapter_TeX_Equp.png)

*请注意，为了应用我们描述的方法，您必须将数据转换为稳定状态，例如消除时间序列中的季节性或趋势。*

#### 高斯过程预测

现在我们已经到达了有趣的部分：给定向量 ***f***，我们如何估计任意点 *x* 处的 *f*(*x*)？由于我们处理的是随机过程，我们将估计未知函数取给定值 *f*(*x*) 的概率。从数学上讲，我们将预测以下量

![p(f(x) | f) 的公式](img/463356_2_En_6_Chapter_TeX_Equq.png)

或者换句话说，给定由所有点 *f*(*x*[1])，…，*f*(*x*[*n*]) 组成的向量 ***f***，得到值 *f*(*x*) 的概率。

假设 *f*(*x*) 是一个具有 *m* = 0 的平稳高斯过程，预测可以表示为

![p(f(x) | f) = p(f(x), f(x1), ..., f(xn)) / p(f(x1), ..., f(xn)) = N(f(x), f(x1), ..., f(xn) | 0, ~C) / N(f(x1), ..., f(xn) | 0, C) 的公式](img/463356_2_En_6_Chapter_TeX_Equr.png)

其中，![$$ \mathcal{N}\left(f(x),f\left({x}_1\right),\dots, f\left({x}_n\right)\ \right|\ 0,\overset{\sim }{C}\Big) $$](img/463356_2_En_6_Chapter_TeX_IEq14.png) 表示在平均值为 0 且协方差矩阵 ![$$ \overset{\sim }{C} $$](img/463356_2_En_6_Chapter_TeX_IEq15.png) 的 *n* + 1 × *n* + 1 维度上计算的正态分布，因为我们有 *n* + 1 个点在分子中。推导过程较为复杂，基于几个定理，例如贝叶斯定理。更多细节可以参考 C.B. Do [3] 的（高级）解释。为了理解贝叶斯优化是什么，我们可以简单地使用公式而不进行推导。*C* 将具有 *n* × *n* 的维度，因为我们只有 *n* 个点在分母中。

我们有

![$$ C=\left(\begin{array}{ccc}K(0)&amp; K\left({x}_1-{x}_2\right)&amp; \dots \\ {}K\left({x}_2-{x}_1\right)&amp; \ddots &amp; \vdots \\ {}\vdots &amp; \dots &amp; K(0)\end{array}\right) $$](img/463356_2_En_6_Chapter_TeX_Equs.png)

And

![$$ \overset{\sim }{C}=\left(\begin{array}{cc}K(0)&amp; {\boldsymbol{k}}^T\\ {}\boldsymbol{k}&amp; C\end{array}\right) $$](img/463356_2_En_6_Chapter_TeX_Equt.png)

其中我们定义了

![$$ \boldsymbol{k}=\left(\begin{array}{c}K\left(x-{x}_1\right)\\ {}\vdots \\ {}K\left(x-{x}_n\right)\end{array}\right) $$](img/463356_2_En_6_Chapter_TeX_Equu.png)

可以证明^(2)，两个正态分布的比值仍然是一个正态分布，这意味着我们可以写出

![$$ p\left(f(x)\ \right|\ \boldsymbol{f}\left)=\mathcal{N}\left(f(x)\ \right|\ \mu, {\upsigma}²\right) $$](img/463356_2_En_6_Chapter_TeX_Equv.png)

With

![$$ {\displaystyle \begin{array}{l}\mu ={\boldsymbol{k}}^T{C}^{-1}\boldsymbol{f}\\ {}{\sigma}²=K(0)-{\boldsymbol{k}}^T{C}^{-1}\boldsymbol{k}\end{array}} $$](img/463356_2_En_6_Chapter_TeX_Equw.png)

*μ* 和 *σ* 的确切形式的推导相当长，超出了本书的范围。基本上，现在我们知道，平均而言，我们的未知函数将在 *x* 处取 *μ* 的值，具有 *σ* 的方差。现在让我们看看这种方法在 Python 中的实际应用。让我们定义我们的核 *K*(*x*)

```py

return sigm**2*np.exp(-1.0/2.0/l**2*(x**2))
```

![$$ f(x)={x}²-{x}³+3+10x+0.07{x}⁴ $$](img/463356_2_En_6_Chapter_TeX_Equx.png)

+   我们使用 `mu = np.dot(np.dot(np.transpose(k), np.linalg.inv(C)), f_)` 和标准差来计算 *μ*。

+   我们以相反的顺序重新应用我们为了使我们的过程平稳所做的所有转换，简单地将 *f*(*x*) 的平均值加回到评估的代理函数 `ybayes = np.asarray(ybayes_)+np.average(f1)`。

让我们停下来思考一下为什么我们要做所有这些。其背后的想法是找到一个 *代理函数* ![$$ \overset{\sim }{f} $$](img/463356_2_En_6_Chapter_TeX_IEq17.png)，它近似我们的函数 *f*，并且具有以下性质

![近似最大值函数公式](img/463356_2_En_6_Chapter_TeX_Equy.png)

*这个代理函数必须具有另一个非常重要的属性：它必须易于评估。这样，我们就可以轻松地找到* ![近似函数公式](img/463356_2_En_6_Chapter_TeX_IEq18.png) *的最大值，通过使用前面的属性，我们将得到 f 的最大值，根据假设这是昂贵的。*

正如你所看到的，知道我们是否有足够的数据点来找到最大值的正确值可能非常困难。毕竟，根据定义，我们对我们黑盒函数的外观没有任何想法。那么我们如何解决这个问题呢？

该方法背后的主要思想可以通过以下算法来描述：

1.  从少量随机选择的样本点开始（点的数量将取决于你的问题）。

1.  使用这组点来获取一个代理函数，如前所述。

1.  使用我们稍后讨论的特定方法向集合中添加一个点，并重新评估代理函数。

1.  如果使用代理函数找到的最大值继续变化，继续添加点，如步骤 3 中所述，直到最大值不再变化或你用完时间或预算，无法进行更多评估。

如果步骤 3 中提到的这种方法足够聪明，你将能够相对快速和准确地找到最大值。

#### 获取函数

如何选择前一小节中提到的步骤 3 中的附加点？想法是使用一个获取函数。算法以这种方式工作：

1.  选择一个函数（我们稍后会看到一些可能性），称为“获取函数”。

1.  然后选择一个附加点 *x*，在该点获取函数达到最大值。

我们可以使用几种获取函数。我们只描述我们将使用的一个，以了解这种方法是如何工作的，但还有几个你可能想检查的，包括熵搜索、改进概率、期望改进和上界置信度。

##### 上界置信度（Upper Confidence Bound, UCB）

在文献中，你可以找到这种获取函数的两个变体。我们可以将函数写成

![上界置信度函数公式](img/463356_2_En_6_Chapter_TeX_Equz.png)

其中我们用 ![$$ \mathbbm{E}\overset{\sim }{f}(x) $$](img/463356_2_En_6_Chapter_TeX_IEq19.png) 表示我们在问题中具有的 *x*-范围上代理函数的“期望”值。期望值不过是函数在给定 *x* 范围内的平均值。*σ*(*x*) 是我们在点 *x* 处使用我们的方法计算的代理函数的方差。我们选择的新点是 *a**UCB* 最大的点。*η* > 0 是一个权衡参数。*此获取函数基本上选择方差最大的点*。再次查看图 6-15。该方法会选择方差较大的点，因此尽可能远离我们已有的点。这样，近似通常会越来越好。UCB 获取函数的另一种变体如下

![$$ {\overset{\sim }{a}}_{UCB}(x)=\overset{\sim }{f}\left(\mathrm{x}\right)+\eta \sigma (x) $$](img/463356_2_En_6_Chapter_TeX_Equaa.png)

这次，获取函数将在选择代理函数最大值周围的点和其方差最大的点之间进行权衡。第二种方法最适合快速找到 *f* 的良好近似，而第一种方法则倾向于在整个 *x* 范围内给出 *f* 的良好近似。下一节将解释这些方法是如何工作的。

##### 示例

让我们从本章开头描述的复杂三角函数开始，考虑 *x* 范围 [0, 40]。我们的目标是找到其最大值并近似该函数。为了方便代码，让我们定义两个函数：一个用于评估代理函数，另一个用于评估新点。为了评估代理函数，我们可以使用以下函数

```py
def get_surrogate(randompoints):
ybayes_ = []
sigmabayes_ = []
for x in xsampling:
f1 = f(randompoints)
sigm_ = np.std(f_)**2
f_ = (f1-np.average(f1))
k = K(x-randompoints, 2.0, sigm_ )
C = np.zeros([randompoints.shape[0],
randompoints.shape[0]])
Ctilde = np.zeros([randompoints.shape[0]+1,
randompoints.shape[0]+1])
for i1,x1_ in np.ndenumerate(randompoints):
for i2,x2_ in np.ndenumerate(randompoints):
C[i1,i2] = K(x1_-x2_, 2.0, sigm_)
Ctilde[0,0] = K(0, 2.0)
Ctilde[0,1:randompoints.shape[0]+1] = k.T
Ctilde[1:,1:] = C
Ctilde[1:randompoints.shape[0]+1,0] = k
mu = np.dot(np.dot(np.transpose(k),
np.linalg.inv(C)), f_)
sigma2 = K(0, 2.0, sigm_)
- np.dot(np.dot(np.transpose(k),
np.linalg.inv(C)), k)
ybayes_.append(mu)
sigmabayes_.append(np.abs(sigma2))
ybayes = np.asarray(ybayes_)+np.average(f1)
sigmabayes = np.asarray(sigmabayes_)
return ybayes, sigmabayes
```

此函数的代码与我们之前章节中的示例相同，但它被封装在一个函数中，该函数返回包含在 `ybayes` 数组中的代理函数，以及包含在 `sigmabayes` 数组中的 *σ*²。此外，我们还需要一个函数来使用获取函数 *a**UCB* 评估新点。我们可以通过以下函数获得它

```py
def get_new_point(ybayes, sigmabayes, eta):
idxmax = np.argmax(np.average(ybayes)
+eta*np.sqrt(sigmabayes))
newpoint = xsampling[idxmax]
return newpoint
```

为了使事情更简单，我们在函数外部定义包含我们想要的所有 *x* 值的数组。让我们从六个随机选择的点开始。为了了解这种方法的表现，让我们先定义一些东西

### 对数尺度上的采样

有一个最后的微小细节需要讨论。有时你可能会发现自己处于想要尝试一个参数可能值的大范围的情况，但你从经验中知道，最佳值可能在一个特定的范围内。假设你想要找到你模型的学习率最佳值，你决定测试从 10^(-4)到 1 的值，但你知道，或者至少期望，你的最佳值位于 10^(-3)和 10^(-4)之间。现在假设你正在使用网格搜索，并且采样了 1,000 个点。你可能认为你有足够的点，但你将会得到

+   在 10^(-4)和 10^(-3)之间没有 0 个点

+   在 10^(-3)和 10^(-2)之间有 8 个点

+   在 10^(-1)和 10^(-2)之间有 89 个点

+   在 1 和 10^(-1)之间有 899 个点

你在不太有趣的范围内得到很多点，而在你想要的地方却没有。图 6-24 显示了点的分布。注意，*x*轴使用了对数尺度。你可以清楚地看到，对于学习率更大的值，你得到的点更多。

记住，784 代表图像像素的灰度值（像素大小为 28 × 28 像素）。

现在我们需要修改数据以获得每个图像的“平坦化”版本，这意味着一个包含 754 个像素的数组，而不是 28x28 像素的矩阵。

以下行调整矩阵的维度

```py
data_train = trainX.reshape(60000, 784)
data_test = testX.reshape(10000, 784)
```

最后，让我们将输入归一化，以便它不是从 0 到 255（灰度值）的值，而是只有从 0 到 1 的值。使用这段代码做这个非常简单

```py
data_train_norm = np.array(data_train/255.0)
data_test_norm = np.array(data_test/255.0)
```

在我们开始开发我们的网络之前，我们需要解决一个最终问题。在执行多类分类任务时，标签必须以不同的形式提供。执行此操作的 Python 代码非常简单：

```py
labels_train = np.zeros((60000, 10))
labels_train[np.arange(60000), trainY] = 1
labels_test = np.zeros((10000, 10))
labels_test[np.arange(10000), testY] = 1
```

现在我们已经准备好了所需的数据。关于完整的讨论，请参阅第三章，在那里我们花费了大量时间准备这个数据集的数据。现在让我们继续到模型部分。让我们从简单的事情开始。作为一个指标，让我们在这个例子中使用准确率，因为数据集是平衡的。让我们考虑一个只有一层的网络，看看有多少个神经元能提供最佳的准确率。

在这个例子中，我们的超参数将是隐藏层中的神经元数量。基本上，我们需要为超参数（隐藏层中的神经元数量）的每个值构建一个新的网络并对其进行训练。我们需要两个函数：一个用于构建网络，另一个用于训练它。为了构建模型，我们可以定义以下函数

```py
def build_model(number_neurons, learning_rate):
# create model
model = keras.Sequential()
# add first hidden layer and set input dimensions
model.add(layers.Dense(number_neurons, input_dim = 784,
activation = 'relu'))
# add output layer
model.add(layers.Dense(10, activation = 'softmax'))
# compile model
model.compile(loss = 'categorical_crossentropy',
optimizer = tf.keras.optimizers.SGD(
learning_rate = learning_rate),
metrics = ['categorical_accuracy'])
return model
```

你应该理解这个函数，因为我们已经在书中多次使用了这段代码。这个函数有一个输入参数，`number_neurons`，它将包含，正如其名称所示，隐藏层中的神经元数量。训练模型的函数看起来像这样

```py
def train_model(number_neurons, learning_rate, number_epochs,
mb_size):
# build model
model = build_model(number_neurons, learning_rate)
# train model
history = model.fit(
data_train_norm, labels_train,
epochs = number_epochs, verbose = 0,
batch_size = mb_size,
callbacks = [tfdocs.modeling.EpochDots()])
# test model
train_loss, train_accuracy = model.evaluate(
data_train_norm, labels_train,
verbose = 0)
test_loss, test_accuracy = model.evaluate(
data_test_norm, labels_test,
verbose = 0)
return train_accuracy, test_accuracy
```

你已经多次看到一个非常类似的功能。主要部分应该是清晰的。你将发现一些新的东西。首先，我们在函数内部构建模型，使用

```py
model = build_model(number_neurons, learning_rate)
```

此外，我们还评估了训练集和开发集上的准确率，并将这些值返回给调用者。这样我们就可以为隐藏层中神经元数量的几个值运行循环，并得到准确率。请注意，这次函数有一个额外的输入参数：`number_neurons`。我们需要将这个数字传递给构建模型的函数。

假设我们选择了以下参数：批量大小 = 50，训练 100 个周期，学习率 *λ* = 0.001，并在隐藏层中使用 15 个神经元构建我们的模型。

然后我们运行模型

```py
train_accuracy, test_accuracy = train_model(15, 0.001, 100, 50)
print(train_accuracy)
print(test_accuracy)
```

对于训练集，我们得到 0.8549 的准确率，对于开发集，我们得到 0.8396 的准确率。我们能做得更好吗？嗯，我们可以从网格搜索开始

```py
nn = [1, 5, 10, 15, 25, 30, 50, 150, 300, 1000, 3000]
for nn_ in nn:
train_accuracy, test_accuracy = train_model(nn_, 0.001, 100, 50)
print('Number of neurons:', nn_, 'Acc. Train:', acc_train, 'Acc. Test', acc_test)
```

```py
neurons_ = np.random.randint(low = 35, high = 60.0, size = (10))

learning_ = 10**r
mb_size_ = np.random.randint(low = 20, high = 80, size = 10)
epochs_ = np.random.randint(low = 40, high = 100, size = (10))
```

```py
for i in range(len(neurons_)):

```

## 关于径向基函数的简要说明

在结束本章之前，让我们讨论一下关于径向基函数的一个小问题

![公式](img/463356_2_En_6_Chapter_TeX_Equab.png)

理解参数 *l* 的作用非常重要。在我们的例子中，我们选择了 *l* = 2，但我们没有讨论为什么。选择一个过小的 *l* 将使获取函数在已有点周围形成非常狭窄的峰值，正如你在图 6-27 的左侧图中可以看到。大的 *l* 值将对获取函数产生平滑效果，正如你在图 6-27 的中间和右侧图中可以看到。

![图 6-27](img/463356_2_En_6_Fig27_HTML.png)

图 6-27

改变径向基函数中的 *l* 参数的影响

通常，避免过小或过大的 *l* 值是一个好的实践，以便能够使方差在已知点之间平滑变化，如图 6-27 中 *l* = 1 的情况。一个非常小的 *l* 将使点之间的方差几乎保持不变，因此算法几乎总是选择点之间的中间点，正如你可以从获取函数中检查的那样。而选择一个过大的 *l* 将使方差变小，因此难以使用。正如你在图 6-27 中 *l* = 5 所见，获取函数几乎保持不变。典型值大约是 1 或 2。
