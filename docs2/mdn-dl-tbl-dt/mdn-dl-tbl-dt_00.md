前言 1

表格数据是现实世界中人工智能最常见的数据类型，但直到最近，表格数据问题几乎完全是通过基于树的模型来解决的。这是因为基于树的模型在表示表格数据时效率很高，具有内在的可解释性，并且训练速度快。此外，传统的深度神经网络（DNN）架构不适合表格数据——它们过度参数化，导致它们很少找到表格数据流形的最佳解。

那么，深度学习能为表格数据学习提供什么？主要吸引力是显著的预期性能提升，尤其是在大型数据集上——正如在过去十年中在图像、文本和语音中所展示的那样。此外，深度学习还提供了许多其他好处，例如进行多模态学习的能力、无需特征工程、跨不同领域的迁移学习、半监督学习、数据生成和多任务学习——所有这些都导致了在其他数据类型上显著的 AI 进步。

这促使我们开发了 TabNet，这是第一个专门为表格数据设计的深度神经网络架构。类似于其他数据类型的典型架构，TabNet 从头到尾进行训练，无需特征工程。它使用序列注意力在每个决策步骤中选择要推理的特征，从而在利用学习容量对最显著特征进行学习的同时，实现了可解释性和更好的学习效果。重要的是，我们能够证明 TabNet 优于或与基于树的模型相当，并且能够通过无监督预训练实现显著的性能提升。这导致了大量关于表格数据深度学习的研究工作。

重要的是，深度学习在表格数据上的改进不仅限于学术数据集，还扩展到了现实世界中的大规模问题。在我为谷歌云工作的一部分工作中，我观察到 TabNet 在许多组织的数十亿规模的真实世界数据集上实现了显著的性能提升。

当安德烈联系我分享这本书时，我很高兴地写了一篇前言。这是我遇到的第一本专门介绍将深度学习应用于表格数据的书。这本书对只了解一点 Python 的人都很友好，提供了有帮助的大量代码示例和笔记本。它在介绍基础知识的同时，也讨论了更前沿的研究，包括可解释性、数据生成、鲁棒性、元优化和集成。总的来说，这是一本非常适合数据科学家类型的人开始学习表格数据深度学习的优秀书籍。我希望它能鼓励并使更多的人能够成功地将深度学习应用于表格数据的实际应用。

前言 2

几乎三十年前，支持向量机（SVMs）的诞生给机器学习范式带来了论文的狂潮。SVMs 的影响深远，触及了许多研究领域。从那时起，数据复杂性和硬件技术都得到了多倍的增长，需要更复杂的算法。这种需求促使我们开发了最先进的深度学习网络，如卷积神经网络（CNNs）。

人工智能（AI）工具如深度学习获得了动力，并扩展到了自动驾驶汽车等行业。高性能图形处理单元（GPU）加速了深度学习模型的计算。因此，更深层的网络得以发展，推动了工业创新和学术研究。

通常，视觉数据占据了深度学习技术空间，而表格数据（非视觉）在这个领域被大量忽视。许多研究领域，如生物学、材料科学、医学、能源和农业，都产生了大量的表格数据。

本书试图探索使用深度学习技术对表格数据分析这一较少被考察领域的探索。最近，一些令人兴奋的研究已经完成，这可能会引领我们进入一个广泛应用深度学习技术于表格数据的新时代。

Andre 和 Andy 无疑触及了这个维度，并为新手和专家读者全面提出了必要且相关的作品，使他们能够理解。他们描述了连接表格数据和深度学习技术的核心模型。此外，他们还构建了详细且易于遵循的编程语言代码。提供了 Python 脚本和必要数据的链接，供任何人重新实现这些模型。

尽管最现代的深度学习网络在许多应用上表现非常令人鼓舞，但它们也有其局限性，尤其是在处理表格数据时。例如，在生物科学中，多组学数据等表格数据具有巨大的维度，但样本却非常少。深度学习模型具有迁移学习的能力，这可以在大规模的表格数据上使用，以挖掘隐藏信息。

Andre 和 Andy 的工作值得称赞，他们把这一关键信息汇集在一个单一的作品中。这是 AI 探险家必读之作！

引言

深度学习已经成为人工智能的公众和私人面孔。当人们在聚会上与朋友闲聊人工智能，在街头与陌生人交谈，以及在工作中与同事讨论时，几乎总是围绕着那些令人兴奋的模型展开，这些模型可以生成语言、创作艺术、合成音乐等等。庞大的、设计复杂的深度学习模型为这些令人兴奋的机器能力提供了动力。然而，许多从业者却合理地反对深度学习的科技炒作。虽然深度学习是“酷”的，但它绝对不是建模的全部和终结。

虽然深度学习无疑在图像、文本和音频等专门、高维数据形式上占据主导地位，但普遍的看法是，它在表格数据上的表现相对较差。因此，那些对深度学习有反感，甚至有怨恨的人就在表格数据上提出他们的论点。过去和现在，发表被接受的深度学习论文似乎是微不足道的，甚至是科学上可疑的修改——这是对深度学习研究文化的一项抱怨——但现在，在这个少数派中，抨击“新晋新一代数据科学家”过于迷恋深度学习，反而吹捧比较经典的基于树的方法作为表格数据的“最佳”模型。你会在任何地方找到这种观点——在醒目的研究论文中，在面向 AI 的社交媒体上，在研究论坛和博客文章中。确实，反文化常常与主流文化一样流行，无论是嬉皮士还是深度学习。

这并不是说没有好的研究支持基于树的方法优于深度学习——当然有。¹ 但这种微妙的研究往往被误解，并被当作普遍规律，那些对深度学习有反感的人常常会坚持与许多推进深度学习状态的人相同的错误教条：在一般定义良好的限制范围内获得的结果，并任意地将其外推，不顾这些限制。

那些主张基于树模型而非深度学习模型的人最明显的短视之处在于问题域空间。对表格型深度学习方法的常见批评是，它们似乎像是“技巧”，一次性方法，偶尔有效，而不是可靠的高性能树方法。沃尔珀特和麦克雷迪的经典“无免费午餐定理”提出的智力问题是：当我们遇到关于普遍优越性的声明时，无论是性能、一致性还是其他指标，这种优越性是“在问题空间的哪个子集上普遍存在的”？

被广泛报道的研究调查和更多非正式调查中使用的，展示深度学习在表格数据上成功的数据集是常见的基准数据集——森林覆盖数据集、希格斯玻色子数据集、加利福尼亚住房数据集、葡萄酒质量数据集等等。这些数据集，即使评估到数十个，无疑也是有限的。提出所有数据形式中，表格数据最为多样化的观点并不无道理。当然，我们必须承认，与更同质化的基准数据集相比，在表现不佳的多样化数据集上进行评估性调查研究要困难得多。然而，那些将此类研究的发现吹捧为对神经网络在表格数据上能力广泛判断的人，却忽略了机器学习模型应用在表格数据领域中的广泛性。

随着从生物系统中可获取的数据信号的增多，生物数据集的特征丰富性从一二十年前的情况显著增加。这些表格数据集的丰富性揭示了生物现象的巨大复杂性——从局部到全球的多个尺度上的复杂模式，以无数种方式相互作用。深度神经网络几乎总是用于模拟代表复杂生物现象的现代表格数据集。另一方面，内容推荐，这是一个需要仔细和高度建模能力的复杂领域，或多或少普遍采用深度学习解决方案。例如，Netflix 报告称，在实施深度学习后，“我们的推荐在离线和在线指标上都有大幅提升”。同样，一篇展示深度学习作为 YouTube 推荐背后范式重构的谷歌论文写道：“与谷歌其他产品领域相结合，YouTube 已经经历了根本性的范式转变，转向使用深度学习作为解决几乎所有学习问题的通用解决方案。”^(2) 同样，一篇展示深度学习作为 YouTube 推荐背后范式重构的谷歌论文写道：“与谷歌其他产品领域相结合，YouTube 已经经历了根本性的范式转变，转向使用深度学习作为解决几乎所有学习问题的通用解决方案。”^(3)

如果我们愿意寻找，我们可以找到更多例子。许多表格数据集包含文本属性，例如包含文本评论以及以表格形式表示的用户和产品信息的在线产品评论数据集。最近的房屋列表数据集包含与标准表格信息（如面积、浴室数量等）相关的图像。或者，考虑包含公司信息表格形式以及捕获时间序列数据的股票价格数据。如果我们还想添加这表格数据和时间序列数据中的前十大金融头条新闻来预测股票价格呢？据我们所知，基于树的模型无法有效解决任何这些多模态问题。另一方面，深度学习可以用来解决所有这些问题。（本书将探讨这三个问题以及更多问题。）

事实上，自 2000 年代和 2010 年代初以来，数据已经发生了变化，那时许多基准数据集被用于研究深度学习和基于树模型之间的性能差异。表格数据比以往任何时候都更细粒度和复杂，捕捉了极其复杂的现象。在表格数据的情况下，深度学习作为一个无结构、稀疏和随机成功的方 法显然是不正确的。

然而，原始的监督学习不仅仅是建模表格数据中的一个单一问题。表格数据通常很嘈杂，我们需要方法来去噪或以其他方式开发对噪声鲁棒的方法。表格数据通常也在不断变化，因此我们需要能够轻松适应新数据的模型。我们经常也会遇到许多具有根本相似结构的不同数据集，因此我们希望能够将知识从一个模型转移到另一个模型。有时表格数据不足，我们需要生成真实的新数据。或者，我们也希望能够开发出非常鲁棒和泛化的模型，而数据集非常有限。据我们所知，基于树的模型要么无法完成这些任务，要么在完成这些任务时存在困难。另一方面，神经网络经过对表格数据从计算机视觉和自然语言处理领域的适应后，可以成功地完成以下所有任务。

当然，对神经网络存在一些重要的合法的普遍反对意见。

一种这样的反对意见是可解释性——即深度神经网络比树模型更难以解释。可解释性是一个特别有趣的想法，因为它更多的是人类观察者的属性，而不是模型本身的属性。在处理数百个特征的情况下，梯度提升模型真的比在相同数据集上训练的多层感知器（MLP）更本质地可解释吗？树模型确实可以构建易于理解的单一特征分割条件，但仅仅是这样本身并不是非常有价值。此外，许多或大多数流行的树集成模型，如梯度提升系统，并不直接建模目标，而是建模残差，这使得直接的可解释性更加困难。我们更关心的是特征之间的相互作用。为了有效地把握这一点，树模型和神经网络都需要外部可解释性方法来将决策的复杂性简化为关键方向和力量。因此，在复杂数据集上，树模型是否比神经网络模型更本质地可解释尚不清楚^(4)^(5)。

第二个主要的反对意见是调整神经网络元参数的繁琐性。这是神经网络的一个不可调和的特性。考虑到可能的配置和架构、训练课程、优化过程等多种方法，神经网络在本质上更接近于理念，而不是真正的具体算法。需要注意的是，这比表格数据领域中的计算机视觉和自然语言处理领域更为突出，并且已经提出了缓解这种影响的方法。此外，需要注意的是，基于树模型也有大量的元参数，通常需要系统性的元优化。

第三个反对意见是神经网络无法有效地预处理数据，以反映特征的有效含义。流行的基于树模型能够以被认为对异构数据更有效的方式解释特征。然而，这并不妨碍将深度学习与预处理方案结合使用，这可以使神经网络在访问表达性特征方面与基于树模型处于同等地位。我们在书中花费了大量篇幅介绍不同的预处理方案。

所有这些都是为了挑战基于树模型优于深度学习模型，甚至基于树模型始终或普遍优于深度学习模型的观念。这并不是要表明深度学习普遍优于基于树模型。为了明确，以下是一些声明：

+   深度学习在某个定义明确的领域问题上工作得非常成功，正如基于树的方法在另一个定义明确的领域问题上工作得非常成功。

+   深度学习可以解决许多基于原始表格监督学习之外的问题，这些问题基于树的方法无法解决，例如建模多模态数据（除了表格数据外，还包括图像、文本、音频等其他数据），去噪噪声数据，跨数据集迁移知识，在有限的数据集上成功训练，以及生成数据。

+   深度学习确实存在一些困难，如可解释性和元优化。然而，在两种情况下，基于树模型都遭受了程度相似的问题（在深度学习至少在一定程度上会成功的足够复杂的情况下）。此外，我们可以尝试通过结合预处理管道与深度模型等措施来弥补神经网络的弱点。

本书的目标是通过提供将深度学习应用于表格数据问题的理论和工具来证实这些主张。这种方法是尝试性的和探索性的，特别是在考虑到这项工作的许多新颖性质的情况下。我们不期望您接受所展示的一切或大多数事物的有效性或成功。本书更像是对广泛思想的处理，其主要的教学习目标是曝光。

本书面向两个受众（尽管如果您觉得自己不属于任何一个，您也绝对欢迎）：经验丰富的表格数据怀疑论者和寻求将深度学习应用于其领域的领域专家。对于前者，我们希望您会发现本书中提出的方法和讨论，无论是原创的还是从研究中综合而来的，至少是引人入胜的，值得思考，即使不是令人信服的。对于后者，我们以提供足够介绍必要工具和概念的方式构建了本书，并希望我们的讨论能帮助您构建专业领域的模型。

## 书的组织结构

本书共分为 12 章，分为三个部分。

第一部分，“机器学习和表格数据”，包括第一章，“经典机器学习原理和方法”，以及第二章，“数据准备和工程”。本部分介绍了与本书后续内容成功相关的机器学习和数据概念。

+   第一章，“经典机器学习原理和方法”，涵盖了重要的机器学习概念和算法。本章展示了几个基础机器学习模型和表格深度学习竞争对手的理论和实现，包括梯度提升模型。它还讨论了经典学习与深度学习之间的桥梁。

+   第二章，“数据准备和工程”，广泛介绍了操纵、管理、转换和存储表格数据（以及您可能需要用于多模态学习的其他形式的数据）的方法。本章讨论了 NumPy、Pandas 和 TensorFlow 数据集（原生和自定义）；分类、文本、时间和地理数据的编码方法；归一化和标准化（及其变体）；特征转换，包括通过降维；以及特征选择。

第二部分，“应用深度学习架构”包含以下章节：3，“神经网络和表格数据”；4，“将卷积结构应用于表格数据”；5，“将循环结构应用于表格数据”；6，“将注意力应用于表格数据”；以及 7，“基于树的深度学习方法。”这一部分构成了本书的大部分内容，展示了各种神经网络架构在其“原生应用”中的功能，以及它们如何以直观和非直观的方式应用于表格数据。第 3、4 和 5 章分别围绕深度学习的三个成熟（甚至“传统”）领域——人工神经网络（ANNs）、卷积神经网络和循环神经网络——及其与表格数据的相关性展开。第六章和第七章共同涵盖了将深度学习应用于表格数据的两个最突出的现代研究方向——注意力/转换方法，这些方法从建模跨单词/标记关系和建模跨特征关系的相似性中汲取灵感，以及基于树的神经网络方法，这些方法试图以某种方式模仿神经网络形式中基于树的模型的结构或能力。

+   第三章，“神经网络和表格数据”，涵盖了神经网络理论的基础——多层感知器、反向传播推导、激活函数、损失函数和优化器——以及用于实现神经网络的 TensorFlow/Keras API。本章还讨论了比较高级的神经网络方法，如回调、批量归一化、dropout、非线性架构和多输入/多输出模型。本章的目标是提供理解神经网络的重要理论基础，以及开始实现用于建模表格数据的实用神经网络所需的工具。

+   第四章，“将卷积结构应用于表格数据”，首先演示了卷积和池化操作的底层机制，然后介绍了标准卷积神经网络在图像数据中的应用。本章以三种方式探讨了将卷积结构应用于表格数据：多模态图像和表格数据集、一维卷积和二维卷积。这一章对生物应用特别相关，因为生物应用经常采用本章中的方法。

+   第五章，“将循环结构应用于表格数据”，与第四章类似，首先展示了三种循环模型变体——“普通”循环层、长短期记忆（LSTM）层和门控循环单元层——如何捕捉输入中的序列属性。循环模型被应用于文本、时间序列和多模态数据。最后，提出了直接将循环层应用于表格数据的推测性方法。

+   第六章，“将注意力应用于表格数据”，介绍了注意力机制和 Transformer 模型系列。注意力机制被应用于文本、多模态文本和表格数据，以及表格上下文中。详细讨论并实现了四篇研究论文——TabTransformer、TabNet、SAINT（自注意力和跨样本注意力 Transformer）和 ARM-Net）。

+   第七章，“基于树的深度学习方法”，主要关注研究，并聚焦于三种主要的基于树的神经网络类别：受树启发/模仿的神经网络，这些神经网络试图在神经网络的结构或结构中复制树模型的特征；堆叠和提升神经网络；以及蒸馏，即将树结构知识转移到神经网络中。

第三部分，“深度学习设计和工具”，包含第八章，“自编码器”；第九章，“数据生成”；第十章，“元优化”；第十一章，“多模型排列”；以及第十二章，“神经网络可解释性”。本部分通过相对较短的章节展示了如何使用和理解神经网络，而不仅仅是监督建模的原始任务。

+   第八章，“自编码器”，介绍了自编码器架构的特性，并展示了它们如何用于预训练、多任务学习、稀疏/鲁棒学习和去噪。

+   第九章，“数据生成”，展示了变分自编码器和生成对抗网络如何在数据有限的环境中应用于表格数据的生成。

+   第十章，“元优化”，展示了如何使用 Hyperopt 进行贝叶斯优化来自动化包括数据编码管道和模型架构在内的元参数的优化，以及神经网络架构搜索的基本原理。

+   第十一章，“多模型排列”，展示了如何动态地组合和堆叠神经网络模型以提升性能或评估实时/“实时”模型预测质量。

+   第十二章，“神经网络可解释性”，介绍了三种方法，既包括模型无关的方法，也包括模型特定方法，用于解释神经网络的预测结果。
