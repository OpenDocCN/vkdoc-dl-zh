# 10. 集成规划和学习

在这本书中，先分别研究主题，然后再一起学习它们，已经成为一个反复出现的主题。我们首先在第三章中研究了基于模型的算法。在这个设置中，我们知道代理所操作的世界的模型动态。代理使用模型动态的知识以及贝尔曼方程，首先执行评估/预测任务来学习状态或状态-动作值。然后，它通过改进策略来获得最佳行为，这被称为*策略改进/策略迭代*。一旦你知道了模型，你就可以提前规划来执行评估/改进步骤。这被称为*规划阶段*。

第四章开始探索无模型制度。*无模型*意味着你不知道模型。通过与之交互来学习模型。这被称为*学习*。在无模型设置中，你学习了蒙特卡洛（*MC*）和时间差分（*TD*）方法。我比较了使用这两种方法的优缺点。接下来，你使用*n 步*和*资格痕迹*将*MC*和*TD*结合成一个单一的综合方法。

第五章至第七章将第四章中的 MC 和 TD 方法扩展到非表格、连续空间、大规模问题，使用函数逼近和深度学习。第三章至第七章中涵盖的所有方法都称为*基于值的方法*，其中学习状态或状态-动作值，然后使用这些学习到的状态/状态-动作值进行策略改进。

第八章重点介绍了一种替代方法，即*基于策略的方法*。这种方法涉及直接学习最优策略，而不需要通过学习状态/动作值的中介步骤。你学习了强化学习、演员-评论家，以及 TRPO 和 PPO。最初，这给人一种基于值和基于策略的方法是不同的印象。然而，就像在资格痕迹下将*MC*和*TD*结合在一起一样，第九章将基于值的 Q 学习和策略学习结合起来，在演员-评论家（AC）方法如 DDPG、TD3 和 SAC 下实现两者的最佳结合。

本章继续重新组合不同的方法。具体来说，本章将基于模型的策略和基于模型的方法结合起来，使算法更强大、更高效地采样。这种方法利用了两者中的最佳之处，是本章的主要重点。

你还将更详细地研究探索-利用困境，以超越仅仅盲目遵循ε-贪婪策略。你将查看更简单的设置，以获得对探索-利用困境的更深入理解。这将由对“引导式前瞻性树搜索方法”的深入研究所跟随，称为*蒙特卡洛树搜索* (MCTS)。

最后，你将查看 AlphaGo 和 AlphaZero 方法，作为 MCTS 应用的例子。

## 基于模型的强化学习

什么是无模型 RL？在无模型 RL 中，你不知道模型。相反，通过让智能体与环境交互来学习价值函数和/或策略，即通过经验学习。这正是你在本书的第四章到第九章中看到的内容。相比之下，这里的基于模型的 RL 是通过让智能体与环境交互来学习模型，再次通过经验学习。学习到的模型用于规划价值函数和政策——这与你在第三章中看到的内容类似。第三章假设了对模型的先验知识，并假设它是完美的。然而，在基于模型的 RL 中，你学习模型然后使用该知识。然而，知识可能是不完整的；换句话说，你可能不知道确切的转移概率或完整的奖励分布。你学习到作为智能体与环境交互时体验到的系统动力学的一部分。你基于有限的交互建立这种知识，因此没有涵盖所有可能性的详尽交互，模型的知识是不完整的。

我在这里所说的“模型”是什么意思？这意味着要估计转移概率 *P*(*s*[*t* + 1]| *s*[*t*], *a*[*t*]) 和奖励 *R*(*s*[*t*], *a*[*t*])。智能体与世界/环境交互，形成对世界的印象。直观上，它可以表示如图 10-1 所示。

就像任何其他常规监督机器学习设置一样，你可以从代理与其环境交互中收集的前一个样本中学习。

+   学习 *s*[*t*]，*a*[*t*] → *r*[*t* + 1] 是一个具有类似均方损失的回归问题。

+   学习转移动态 *s*[*t*]，*a*[*t*] → *s*[*t* + 1] 是一个密度估计问题。你可以学习一个离散分类分布、高斯分布或高斯模型参数的混合模型。损失可以是 KL 散度损失。

如果你将学习扩展到贝叶斯学习，你还可以对模型的不确定性进行推理——你对学习到的模型转移和奖励函数有多确定或不确定。贝叶斯学习方法不仅产生一个点估计，还产生一个估计的整个分布，这使你能够对估计的强度进行推理。估计的狭窄概率分布意味着大部分概率集中在峰值周围；也就是说，你对估计有很强的信心。相反，估计的广泛分布反映了估计的不确定性更高。

然而，生活中没有免费的午餐。这种两步方法，首先学习模型的不完美表示，然后使用这种不完美的表示来规划或找到最优策略，引入了两种错误来源。首先，学习到的模型动态可能不准确。其次，从不完美的模型中学习价值函数可能有其自身的误差。

学习到的模型可以表示为“查找表”（类似于你在第二章和第三章中看到的），线性期望模型或线性高斯模型。还可以有更复杂的模型表示，如高斯过程模型或深度信念网络模型。这取决于问题的性质、数据收集的容易程度等因素。选择正确的表示需要领域专业知识。让我们看看学习查找表模型的一个简单例子。

你只从 *A* 到 *B* 看到了一个转换，并且获得了奖励 0。因此，你得出结论 *P*(*B*| *A*) = 1，*R*(*状态* = *A*) = 0。请注意，在前面的例子中，我没有明确显示动作以保持简单。你可以将其视为在每个状态下采取随机动作，或者你可以将其视为马尔可夫奖励过程（MRP）而不是完整的马尔可夫决策过程（MDP）。

+   *参数化*:

## 使用学习模型进行规划

一旦你了解了模型，你可以使用它来执行第三章节中的价值或策略迭代进行规划。在这些方法中，你使用贝尔曼方程进行一步滚动。

然而，学习模型还可以有另一种用途。你可以从中采样，并使用这些样本在 *MC* 或 *TD* 学习方法下进行操作。在这种情况下，执行 *MC* 或 *TD* 风格的学习时，智能体不会与真实环境交互。相反，它与其从过去经验中近似的环境 *模型* 进行交互。在某种程度上，这仍然被称为 *规划*。你正在使用学习到的模型进行规划，而不是从与真实世界的直接交互中进行规划。

请记住，你学习的模型并不精确。它基于智能体与环境的部分交互。它通常不是完整或详尽的。我在前一个章节中谈到了这一点。学习模型中的错误会影响学习的质量。算法根据你形成的模型进行优化，而不是真实世界的模型。这可能会限制学习的质量。如果你的学习模型信心不高，你可以回到之前看到的模型无关的强化学习方法。或者，你可以使用贝叶斯方法来推理模型不确定性。贝叶斯方法不是我在这本书中更多涉及的内容。感兴趣的读者可以参考各种高级强化学习文本和论文，以了解更多关于它的信息。

到目前为止，你已经看到基于模型的强化学习提供了学习模型的好处，从而使得学习更加样本高效。然而，它以模型估计不准确为代价，这反过来又限制了学习的质量。有没有一种方法可以将基于模型和无模型的学习结合到一个单一的统一框架中，并利用两种方法的优势？这就是你接下来要看的。

## 集成学习和规划（Dyna）

你可以有两种经验：一种是在智能体与真实环境交互时获得下一个状态和奖励的真实经验，另一种是模拟经验，其中智能体使用它所学习的模型来创建更多的模拟经验。模拟经验更容易且成本更低，尤其是在机器人领域。快速的机器人模拟器可以产生可能与现实世界不完全匹配的样本，但允许你比机器人的真实世界交互更快地模拟和生成智能体行为。然而，模拟可能不够精确，这就是为什么从真实世界经验中学习有帮助。我们可以将图 10-1 的图表改为包括这一步骤，如图 10-3 所示。

在 Dyna 中，智能体与环境（行动）交互以产生真实经验。这些真实经验用于学习世界模型，并直接改进价值/策略，就像无模型的强化学习设置一样。这与你在第 [4](http://dx.doi.org/10.1007/978-1-4842-6809-4_4) 章节到第 [8](http://dx.doi.org/10.1007/978-1-4842-6809-4_8) 章节中看到的方法相同。从现实世界交互中学习的模型用于生成模拟转换。这些模拟转换用于改进价值/策略。这被称为步骤规划，因为你正在使用世界模型来生成经验。你可以通过使用模拟经验多次进行“行动”的每一步来执行规划步骤，并从真实世界中生成新的样本。

```py
def train_agent(env, agent, episode_cnt = 10000,
tmax=10000, anneal_eps=True):
episode_rewards = []
for i in range(episode_cnt):
G = 0
state,_ = env.reset()
for t in range(tmax):
action = agent.get_action(state)
next_state, reward, done, truncated, _ = env.step(action)
done = done or truncated
agent.update(state, action, reward, next_state, done)
G += reward
if done:
episode_rewards.append(G)
# to reduce the exploration probability epsilon over the
# training period.
if anneal_eps:
agent.epsilon = agent.epsilon * 0.99
break
# add the experience to agent’s buffer (i.e. agent’s model estimate)
agent.buffer[(state,action)] = (next_state, reward, done)
state = next_state
# plan n steps through simulated experience
for j in range(agent.n):
state_v, action_v = random.choice(list(agent.buffer))
next_state_v, reward_v, done_v = agent.buffer[(state_v,action_v)]
agent.update(state_v, action_v, reward_v, next_state_v, done_v)
return np.array(episode_rewards)
```

## Dyna Q+

Dyna Q+ 是一个典型的探索/利用困境的例子，在所有强化学习设置中都需要仔细思考。如果你开始早期利用知识（即模型或策略），你可能会错过了解更好路径的机会。智能体贪婪地对其已经学会的内容感到满意。另一方面，如果智能体过度探索，即使它已经找到了最优解，它也会浪费时间查看次优路径/选项。不幸的是，智能体没有直接知道它已经达到了最优策略，因此它需要使用其他启发式方法/方式来平衡探索/利用困境。

在 Dyna Q+ 中，你通过向观察到的状态之外的状态添加奖励来鼓励探索未见过的状态。这个额外的奖励项鼓励探索在现实世界中一段时间没有访问过的状态。

在模拟规划部分，你需要在奖励中添加 ![κ√τ](img/502835_2_En_10_Chapter_TeX_IEq2.png)；也就是说，奖励 *r* 变为 ![r+κ√τ](img/502835_2_En_10_Chapter_TeX_IEq3.png)，其中 κ 是一个小的常数，τ 是从问题中提到的过渡在现实世界探索中首次出现的时间。这鼓励智能体尝试这些过渡并有效地发现环境中的变化，同时有一些由 κ 控制的延迟。在图 10-7 中，智能体最初学会通过左侧的开口。它按照修订后的条款进行探索，但发现到达目标唯一的方式是通过左侧的开口。然而，当通过在右侧打开新的开口来改变环境后，智能体在随后的对这个迷宫部分的探索中会发现新的开口，并最终发现这是到达目标的一条更短的路径。智能体将修正其最优行为，跟随右侧的开口。随着时间的推移，最后一次访问网格的一部分，奖励中的 ![κ√τ](img/502835_2_En_10_Chapter_TeX_IEq4.png) 项增长。在某个阶段，它可能增长得如此之大，以至于它掩盖了当前最优行为的奖励，迫使智能体再次探索未访问的部分。

## 探索与利用

在强化学习中，你总是需要在使用当前知识和探索更多以获得新知识之间取得平衡。最初你对世界（环境）的了解几乎没有或非常少。你进行大量探索，并开始改进你对世界的信念。随着你的信念不断改善和加强，智能体开始利用这种信念，同时逐渐减少其探索。

到目前为止，你已经看到了不同形式的探索。第一次遇到探索是在ε-greedy 策略的形式中，其中智能体根据其当前信念以概率(1 - *ε*)采取最佳动作，并以概率*ε*随机探索。随着学习的进展，你逐渐减少*ε*。你在所有的 DQN 方法中都看到了这一点。如果你查看第 6 和 7 笔记本，你会注意到一个`epsilon_schedule`函数，如列表 10-3 所示。在学习的开始，你从ε=1 开始，在一段时间内减少到 0.05，以在训练结束时进行少量探索。这被称为*epsilon 减少计划*。减少可以是线性的或遵循任何其他曲线，类似于你对α（学习率）所做的事情。在监督学习中，作为随机梯度下降的一部分，你根据称为*学习率计划*的计划来变化α。有各种方法来变化α。如果你有ε-greedy 策略，你可以使用这些方法中的大多数来变化ε。

```py

return start_eps + (end_eps-start_eps)*min(step, final_step)/final_step
```

让我们先描述你将使用的设置。考虑多臂赌博机有 *K* 个动作。每个动作有一个固定的概率 θ[*k*]，*k* ∈ [1\. . *K*]，产生奖励 1，以及一个 (1 − θ[*k*]) 的概率产生奖励 0。最佳动作是 *k*（小写 *k*）的值，它具有最大的成功概率。

![$$ {\theta}^{\ast }={\theta}_k $$](img/502835_2_En_10_Chapter_TeX_Equn.png)

列表 10-4 展示了赌博机的代码。你将 *K* 存储为 `self.n_actions`，并将所有 θ[*k*] 存储为 `self._probs.`。`pull` 函数接受 *k* 动作作为输入，根据动作的概率分布（即 θ[*k*]）返回奖励 1 或 0。`optimal_reward` 函数返回 `self._probs.` 的 `max` 值。

```py
class Bandit:
def __init__(self, n_actions=5):
self._probs = np.random.random(n_actions)
self.n_actions = n_actions
def pull(self, action):
if np.random.random() > self._probs[action]:
return 0.0
return 1.0
def optimal_reward(self):
return np.max(self._probs)
```

### Epsilon Greedy Exploration

这与你在前几章中看到的 *ϵ*-贪婪策略类似。智能体尝试不同的动作，并形成对赌博机环境不同动作的未知实际成功概率 θ[*k*] 的估计 ![$$ {\hat{\uptheta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq5.png)。它以 (1 − *ϵ*) 的概率采取动作 *k*，其中 ![$$ {\hat{\uptheta}}^{\ast }=\underset{k}{\max }{\hat{\uptheta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq6.png)。换句话说，它利用到目前为止获得的知识，根据 ![$$ {\hat{\theta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq7.png) 估计采取它认为最好的动作。此外，它以概率 *ϵ* 采取随机动作以进行探索。

让我们将 *ϵ* 设置为 0.01 作为探索概率，并在整个实验中保持其恒定。列表 10-5 展示了实现此策略的代码。首先，你将 `RandomAgent` 定义为基类。有两个数组用于存储每个动作的成功和失败结果的累积计数。你将这些计数存储在数组中：`self.success_cnt` 和 `self.failure_cnt`。

`update(action, reward)` 函数根据采取的动作和结果更新这些计数。`reset()` 函数重置这些计数。这样做是为了运行相同的实验多次，并绘制平均值以平滑方差。`get_action()` 函数是实现探索策略的地方。它将根据策略类型使用不同的方法来平衡探索与利用。对于 `RandomAgent`，动作总是从可能动作集中随机选择。

接下来，你在 `EGreedyAgent` 类中实现 *ε*-探索代理。这是通过扩展 `RandomAgent` 并重写 `get_action()` 函数来实现的。正如解释的那样，以概率 (1 − *ϵ*) 选择具有最高概率的动作，以概率 *ϵ* 选择随机动作。

```py
class RandomAgent:
def __init__(self, n_actions=5):
self.success_cnt = np.zeros(n_actions)
self.failure_cnt = np.zeros(n_actions)
self.total_pulls = 0
self.n_actions = n_actions
def reset(self):
self.success_cnt = np.zeros(n_actions)
self.failure_cnt = np.zeros(n_actions)
self.total_pulls = 0
def get_action(self):
return np.random.randint(0, self.n_actions)
def update(self, action, reward):
self.total_pulls += 1
if reward == 1:
self.success_cnt[action] += 1
else:
self.failure_cnt[action] += 1
class EGreedyAgent(RandomAgent):
def __init__(self, n_actions=5, epsilon=0.01):
super().__init__(n_actions)
self.epsilon = epsilon
def get_action(self):
estimates = self.success_cnt / (self.success_cnt+self.failure_cnt+1e-12)
if np.random.random() < self.epsilon:
return np.random.randint(0, self.n_actions)
else:
return np.argmax(estimates)
```

### 上置信界探索

本节探讨了称为 *上置信界* (UCB) 的策略。在此方法中，代理尝试不同的动作并记录成功次数 (α[*k*]) 或失败次数 (β[*k*])。

它计算了成功概率的估计值 ![$$ {\hat{\theta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq8.png)，这些值将用于选择具有最高成功估计值 ![$$ {\theta}^{\ast }={\hat{\uptheta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq9.png) 的动作。这是动作 *k* 的“利用”部分。

![exploit_k=hat{theta}_k=frac{alpha_k}{alpha_k+beta_k}](img/502835_2_En_10_Chapter_TeX_Equo.png)

它还计算了有利于最少访问动作的探索需求，如下所示：

![explore_k=sqrt{frac{2.log t}{alpha_k+beta_k}}](img/502835_2_En_10_Chapter_TeX_Equp.png)

其中 *t* 是到目前为止所采取的总步数。通过探索和利用来定义每个动作 *k* 的分数，如公式 10-6 所示。

![$$ scor{e}_k= explor{e}_k+ explo{t}_k=\frac{\alpha_k}{\alpha_k+{\beta}_k}+\uplambda \sqrt{\frac{2.\log t}{\alpha_k+{\beta}_k}} $$](img/502835_2_En_10_Chapter_TeX_Equ6.png)

(10-6)

然后，智能体选择具有最大 *score*[*k*] 值的行动 *k*。在方程 10-6 中，λ 控制了探索与利用的相对重要性。你将在代码中使用 λ = 1。

UCB 中的方法是计算一个上置信界（上置信区间），以确保你有 0.95（或任何其他置信水平）的概率，实际估计值在 UCB 值内。随着试验次数的增加，以及你反复采取特定的行动，估计的不确定性会降低，方程 10-6 中的分数将接近估计值 ![$$ {\hat{\theta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq10.png)。尚未访问过的行动将具有 *score* = ∞，因为在方程 10-6 中，α[*k*] + β[*k*] = 0，使得分数无限。UCB 已证明具有接近最优的增长界限。我不会深入到这个数学证明。你可以参考关于多臂老虎机问题的先进文本。

现在，你将了解 UCB 智能体的实现。列表 10-6 提供了代码。正如解释的那样，你再次扩展了列表 10-5 中所示的 `RandomAgent` 类，并重写 `get_action` 函数以根据 UCB 方程 10-6 选择行动。请注意，为了避免除以零，在 `self.success_cnt+self.failure_cnt (`*α*[*k*] + *β*[*k*]) 出现的分母中添加了一个小的常数 `1e-12`。

```py
class UCBAgent(RandomAgent):
def get_action(self):
exploit = self.success_cnt / (self.success_cnt+self.failure_cnt+1e-12)
explore = np.sqrt(2*np.log(np.maximum(self.total_pulls,1))/ \
(self.success_cnt+self.failure_cnt+1e-12))
estimates =  exploit + explore
return np.argmax(estimates)
```

### 汤普森抽样探索

本节探讨了最终策略，即汤普森抽样。这是基于选择一个行动的想法，即它具有最大化预期奖励的概率。虽然 UCB 基于频率主义概率观念，但汤普森抽样基于贝叶斯思想。

初始时，你对哪种行动更好没有任何先验知识，也就是说，哪种行动具有最高的成功概率 θ[*k*]。因此，你形成了一个初始信念，即所有 *θ*[*k*] 值在范围 (0, 1) 内均匀分布。这在贝叶斯术语中被称为 *先验*。你用 Beta 分布^(2) 表示这一点，即 ![$$ {\hat{\uptheta}}_k\sim \mathrm{Beta}\left(\upalpha =1,\upbeta =1\right) $$](img/502835_2_En_10_Chapter_TeX_IEq11.png)。在第一次迭代中，智能体使用 Beta 分布 ![$$ {\hat{\uptheta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq12.png) 为每个行动 *k* 抽样一个值，并选择具有最大抽样值 ![$$ {\hat{\uptheta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq13.png) 的行动 *k*。

接下来，它执行动作（即调用`pull`函数执行所选动作）并观察结果（成功或失败）。根据结果，它更新所选动作*k*的后验分布![$$ {\hat{\theta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq14.png)。随着每一步中不同动作*k*的成功和失败次数更新，代理继续更新其在每一步*k*中对特定![$$ {\hat{\theta}}_k $$](img/502835_2_En_10_Chapter_TeX_IEq15.png)的信念。这是在贝叶斯框架下所有动作*k*∈*K*的未知值θ[*k*]的后验分布。

![$$ {\hat{\theta}}_k\sim Beta\left(\alpha ={\alpha}_k+1,\beta ={\beta}_k+1\right)\kern0.75em \forall k\in \left[1..K\right] $$](img/502835_2_En_10_Chapter/502835_2_En_10_Chapter_TeX_Equ7.png)

(10-7)

这个循环会继续。随着你通过多个步骤，α[*k*]和β[*k*]，动作*k*的成功和失败次数增加。Beta 分布变得更窄，峰值在α[*k*]/(α[*k*] + β[*k*])的均值附近。

现在看看 Thompson 采样在代码中的实现，如列表 10-7 所示。和之前一样，你扩展`RandomAgent`类并重写`get_action`方法来实现 Beta 分布方程 10-7。

```py

estimates = np.random.beta(self.success_cnt+1, self.failure_cnt+1)
return np.argmax(estimates)
```

```py
def get_regret(env, agent, n_steps=10000, n_trials=10):
score = np.zeros(n_steps)
optimal_r = env.optimal_reward()
for trial in range(n_trials):
agent.reset()
for t in range(n_steps):
action = agent.get_action()
reward = env.pull(action)
agent.update(action, reward)
score[t] += optimal_r - reward
score = score / n_trials
score = np.cumsum(score)
return score
```

## 决策时规划和蒙特卡洛树搜索

Dyna 使用规划的方式被称为 *背景规划*。模型用于生成模拟样本，然后将其输入到算法中，就像使用真实样本一样。这些额外的模拟样本有助于进一步改进价值估计，超过使用现实世界交互所做的估计。当在，比如说，状态 *S*[*t*] 中选择行动时，在这个决策时没有进行规划。

另一种规划方法是开始和结束规划时处于状态 *S*[*t*]，以便向前看，有点像使用到目前为止学到的模型在你的脑海中演绎各种场景，并使用这个规划来决定在时间 *t* 要采取的行动 *A*[*t*]。一旦采取了行动 *A*[*t*]，代理发现自己处于新的状态 *S*[*t* + 1]。代理再次向前看，规划并迈出一步。这个周期继续进行。这被称为 *决策时规划*。根据可用的时间，代理可以从当前状态规划得更远/更深，那么这个规划在决定在时间 *t* 采取什么行动时将更加强大和有帮助。因此，当不需要快速响应时，决策时规划最有帮助。在像国际象棋和围棋这样的棋类游戏中，决策时规划在决定采取什么行动之前可以向前看十几步。然而，如果需要快速响应，那么背景规划最为合适。

这些向前搜索算法通过向前看选择最佳可能行动。它们从代理当前处于的状态 *S*[*t*] 开始构建可能的选项搜索树。给定状态-行动对中的可能选项可能取决于代理从先前交互中学到的 MDP 模型，或者它可能基于代理一开始收到的 MDP 模型。例如，在像国际象棋和围棋这样的棋类游戏中，游戏的规则是明确定义的，因此你知道环境在完全、精确的细节。图 10-11 展示了一个搜索树的例子。

![](img/502835_2_En_10_Fig11_HTML.jpg)

图 10-11

然而，如果分支因子很高，你无法构建一个穷举搜索树。在围棋游戏中，平均分支因子约为 250。即使你构建一个三层的穷举树，第三层将有 250³ = 15,625,000 个节点（1500 万个节点）。而且为了充分利用远见，你将需要深入到两三个层次以下。这就是为什么需要形成一个基于模拟的搜索树而不是穷举搜索树。你使用 *rollout policy* 来模拟多个轨迹，并形成动作价值的 MC 估计。这些估计用于在根状态 *S*[*t*] 选择最佳动作。这有助于改进策略。在采取最佳动作后，代理发现自己处于一个新的状态，然后它从该新状态重新开始模拟树。这个循环会继续。尝试的 rollout 轨迹数量取决于代理在决定动作时的时间限制。

MCTS 是决策时间规划的例子。MCTS 与之前描述的完全一致，只是增加了一些增强功能来累积价值估计并指导搜索到树的最有回报的部分。MCTS 的基本版本包括四个步骤。

1.  *选择/树遍历*：从根节点开始，使用树策略遍历树并达到一个叶节点，即一个未完全扩展的节点。

1.  *扩展*：向叶节点添加子节点（动作）并从中选择一个。

1.  *模拟/rollout*：从之前选择的子节点开始玩游戏，直到终止。使用固定的 rollout 策略来玩，这也可以是一个随机 rollout 策略。

1.  *反向传播*：通过跟随（回溯）遍历的路径，使用 playout 的结果更新从终止状态到根节点路径上的所有节点。

循环会重复多次，始终从当前根节点开始，直到搜索时间耗尽或达到固定的迭代次数。这被称为 MCTS 的“任何时间算法”，因为它可以在任何时候停止，并且总是会返回一个有效的搜索答案。如果增加分配的时间，MCTS 返回的搜索质量将提高。此时，根据累积的统计数据选择价值最高的动作。图 10-12 展示了 MCTS 的示意图。

## MCTS 的示例用途

在我总结这一章之前，本节简要讨论了最近和著名的 MCTS 应用案例——AlphaGo 和 AlphaZero，它们帮助设计了一个算法，使计算机在围棋游戏中击败了人类专家。本节还探讨了使用 MCTS 的 AlphaFold，它有助于预测大型蛋白质复合物的结构。它还简要提到了 MCTS 最近被应用的各个领域。

### AlphaGo

AlphaGo 由 Deep Mind 于 2016 年^(3)设计，旨在在围棋游戏中击败人类专家。标准的搜索树会从一个给定位置测试所有可能的情况。然而，具有高分支因子的游戏不能使用这种方法。如果一个游戏具有分支因子 *b*（即任何位置的合法移动数是 *b*）和总游戏长度（即深度或直到结束的连续移动数）是 *d*，那么就有 *b*^(*d*) 个可能的移动序列。评估所有选项对于任何中等规模的棋盘游戏都是不可行的。考虑一下具有(*b* ≈ 35, *d* ≈ 50)的国际象棋游戏或具有(*b* ≈ 250, *d* ≈ 150)的围棋游戏，穷举搜索是不可能的。但可以通过两种通用方法来减少有效的搜索空间。

+   **您可以通过位置评估来减少深度**。在扩展搜索树一定深度后，将叶节点的状态替换为近似评估函数 *v*(*s*) ≈ *v*^∗(*s*)。这种方法适用于像国际象棋、跳棋和五子棋这样的棋盘游戏，但由于围棋的复杂性，它不适用于围棋。

+   **您可以通过从策略** ***p***(***s***)** 中采样动作来减少节点的完全展开的树宽度**。蒙特卡洛滚动不进行分支；它们从叶节点开始播放一个场景，然后使用平均回报作为 *v*(*s*) 的估计。您在 MCTS 中看到了这种方法，其中滚动值被用来估计叶节点的 *v*(*s*)，并通过使用修订的 *v*(*s*) 估计进一步改进了策略。

在这篇关于 AlphaGo 的学术论文中，作者将 MCTS 与深度神经网络相结合，实现了超人类的表现。他们将棋盘图像作为 19×19 像素的图像传递给卷积神经网络（CNN），并使用它来构建棋盘位置的表示，即 *状态*。

他们使用专家人类动作进行监督学习，并学习了两条策略，即 *p*(*s*)。

+   **SL 策略网络** (*p*[*σ*])：作者使用监督学习来学习策略网络 *p*[*σ*]。这是使用 CNN 风格的网络训练的，输入状态 (*s*) 为 19x19 的棋盘位置，棋盘位置基于专家人类动作。

+   **滚动策略** (*p*[*π*])：使用相同的数据训练了另一个网络，称为快速策略 *p*[*π*]，它在 MCTS 的“滚动”阶段充当滚动策略。原始 MCTS 使用随机滚动策略。然而，如前所述，如果它非常快，您可以使用策略模型。

接下来，作者训练了一个强化学习（RL）网络 *p*[*ρ*]，从 SL 策略网络初始化，试图改进 SL 策略网络以获得更多胜利，即使用策略优化来提高目标，即获得更多胜利。这与您在第八章中学到的策略优化类似。

最后，还训练了另一个网络*v*[*θ*]来预测 RL 策略网络*p*[*ρ*]通过自我博弈的状态*v*(*s*)的值。图 10-13 显示了神经网络训练的高级图。

### AlphaFold 与 MCTS

AlphaFold^(8)是由 DeepMind 开发的一个 AI 系统，它可以从氨基酸序列预测蛋白质的 3D 结构。它通常达到与实验相媲美的准确性。AlphaFold 可以以非常高的准确性预测单链和多链蛋白质的结构。然而，准确性随着链数的增加而降低，可用的 GPU 内存限制了可以预测的蛋白质复合物的大小。

### MCTS 在其他领域的应用

MCTS 的一些其他用途如下：

+   化学合成，正如你在之前的 AlphaFold 子节中看到的那样。

+   规划：在物流和机器人技术中的最优规划中，自动化规划是 MCTS 算法在游戏之外的主要应用领域之一。规划问题通常被表述为 MDP。MCTS 也已被用于机器人的规划。已提出 MCTS 的分布式变体用于多机器人场景。压缩形式的树定期发送到其他机器人，这导致策略空间上的联合分布更新。

+   安全性：在攻击者-防御者场景中确定安全部队的最佳巡逻时间表。蒙特卡洛树搜索（MCTS）在安全游戏（SG）中的应用是最近的。本质上，SG 使用博弈论方法来找到安全部队（如警察、情报机构和保安）的最有效巡逻时间表。他们在保护一组潜在目标免受攻击者（如罪犯、恐怖分子和走私者）攻击的同时这样做。由于全球恐怖主义威胁的增加，SG 的重要性及其应用最近有所增加。

+   调度：将蒙特卡洛树搜索（MCTS）与额外的启发式方法相结合是解决大型计算问题的一种常见方法。MCTS 已被提出用于生成多模式资源受限多项目调度问题的初始解决方案。然后通过精心设计的超启发式方法进行局部搜索对这些解决方案进行修改。

+   车辆路径：运输问题，如容量限制车辆路径问题（CVRP），是通常涉及规划和调度的问题的特殊类别。
