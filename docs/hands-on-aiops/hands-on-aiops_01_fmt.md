# 前言

在开启我们的 AIOps 之旅之前，让我们先简要讨论一下自动化及其在过去十年中的演变。

技术领域的自动化被定义为一个系统，在该系统中，流程或任务可以在最少的人工监督和操作下完成。人类一直在自动化各种任务，最终发明了减少人力投入并提高效率的机械机器。如今，像制造业这样过去部分自动化的领域，正在转向 3D 打印，这完全自动化了制造过程；然而，设计要制造什么仍然属于人类领域。

因此，自动化减少了人力投入，并使用机器或软件来完成可定义、可重复的任务。

在 IT 领域，人类执行着各种任务，包括构想新产品或应用程序、开发将需求转化为可用软件的代码、部署基础设施和应用程序，并在其生命周期内保持更新。

IT 团队在从软件开发到运营的各个领域都广泛使用了自动化；然而，这在很大程度上是孤立的，并且没有采用正式的自动化系统或方法。IT 团队使用脚本、运行手册自动化工具、作业调度系统和机器人流程自动化系统来自动化他们的任务。这些工具提高了效率，并减少了运营 IT 环境所需的人力。

随着云计算和 DevOps 原则的日益普及，基础设施即服务和平台即服务环境的配置以及应用程序的部署也已实现自动化。这不仅实现了任务自动化和效率提升，还带来了敏捷性和速度，使企业能够通过根据客户和市场反馈快速更改功能和特性，来调整和适应不断变化的市场需求。

IT 运行在三大支柱之上：流程、人员和技术。要实现自动化，需要了解这些支柱之间的相互关系。人员使用定义的流程来处理技术，而通过自动化，我们本质上是在自动化人员当前用于运营环境的流程。然而，随着技术格局的变化以及人工智能和机器学习能力的日益普及和成熟，我们现在可以审视当前的流程，并制定新的流程，以利用这些技术提供的变革能力。当前的流程是基于当时最先进的技术建立的，这些流程随后定义了人类应如何在其中操作以执行任务并实现目标；但是，随着技术格局的剧烈变化，流程也需要改变和适应。例如，随着云计算的普及，IT 流程需要改变和适应，顺序的、非自动化的流程需要改变，以适应自动配置等新功能。2013 年，我谈到了容量管理将如何在云计算世界中发生巨大变化，以及需要针对云成本管理制定新的程序。其中一些概念已扩展到涵盖整个财务运营领域，统称为*云 FinOps*。

同样，IT 运营自动化一直以来都是孤立存在的。人们使用脚本、监控工具、运行手册自动化、配置和部署自动化、RPA 工具，以及通过 ITSM 工具实现自动化服务管理流程。然而，所有这些都发生在不同的领域，并通过孤立的集成进行整合。

在技术方面，人工智能和机器学习技术已成为主流，并被广泛应用于面向业务和面向客户的应用程序的各个方面，从网站到搜索引擎再到协作和通信工具。AI 迅速占领了 IT 世界；然而，IT 在采用这些技术方面却姗姗来迟。虽然 IT 团队正在使用这些技术为客户创建具有 AI 能力的新应用程序，但他们自己的内部系统仍然使用着较旧的技术，并运行在过去十年中基本未受影响的流程和系统上。

通过 DevOps 和敏捷运动，这种变革性的变化已经改变了应用程序构建、测试和部署的方式，并且开发价值流中的大多数任务都已实现自动化和集成，从而使组织能够迈向持续交付和持续部署。与 DevOps 带来的变革类似，随着 AIOps 的兴起，我们将见证一场变革性的变化，它将彻底改变过去 IT 运营的运行方式。旧的流程、系统、工具和工作方式将被 AIOps 的运营方式所取代，并有助于实现 NoOps 的愿景，即运营工作在没有人工监督和干预的情况下，以自动化方式无缝、无中断地进行。

当今企业在自动化方面的成熟度参差不齐。大多数企业尚未达到较高的自动化成熟度，部分原因在于流程变更、打破组织架构壁垒以及部署新技术都是复杂且耗时的任务。在一些组织中，数字化和云计算项目已成为大规模转型的一部分；`AIOps` 及相关技术和流程变革正在推动数字化的全面转型。采用 `AIOps` 的一个关键障碍是该领域缺乏全面的流程和技术指导。现有的指导十分有限，且大多聚焦于供应商试图作为一站式解决方案推销的产品，声称能让组织实现跨越式发展。通过本出版物，我们旨在提供切实可行的实践指导，说明组织应如何从源头采纳这些变革，以及需要避免哪些陷阱。实施 `AIOps` 并非部署一个事件关联工具，而是要将人工智能和算法融入其中，并实现运维各环节的自动化。

目前，大多数组织已实现单个任务的自动化和某些流程的自动化。部分组织还实现了复杂端到端流程的自动化。然而，一个端到端的自维持自动化模型仍然缺失。自动化少数流程与自动化所有可自动化内容之间的转变，人们尚未启动。在大多数组织中，先自动化少数流程，再将自动化扩展到跨团队、跨职能和跨部门，仍是一项进行中的工作。

`AIOps` 不仅仅是技术或流程的变革，更是一种文化变革。在这种文化中，人类与人工智能协同工作，相互支持和增强彼此的能力，以实现效率和可扩展性——这是 IT 团队梦寐以求却一直未能达成的目标。`NoOps` 的承诺之所以未能实现，是因为组织未能对 `AIOps` 采取整体且端到端的方法，而是依赖以孤岛式工具形式部署的快速修复方案。

在自动化成熟度曲线中处于较高水平的企业中，大多数可完全或部分自动化的活动都已实现自动化，并且由人工智能驱动决策，而非人类通过仪表盘和分析来确定下一步行动。这是一种思维模式的转变：人类必须接受人工智能系统在某些领域的优势地位，并放弃我们物种固有的控制欲。IT 团队习惯于确定性以及基于规则的系统，这些系统主导了 IT 领域数十年。而基于人工智能的系统是概率性的，而非确定性的；因此，一些企业虽然部署了 `AIOps` 系统，却将其配置得与基于规则的系统无异。接受概率性系统而非逐步的基于规则的系统是一种思维转变，人们需要接受实施此类系统所带来的风险，并建立相应的控制措施，以确保运行基于人工智能的系统的最佳实践；将人类对某些操作的控制纳入流程变革的一部分。

企业也在不断演进，调整其运营模式以符合敏捷原则；它们开始整合应用和基础设施团队，并采用站点可靠性工程等新实践，使运维支持能够支撑高可靠性、高可用性的关键任务系统和互联网规模的应用。`AIOps` 是基于站点可靠性工程的运维中的关键支柱。Apress 于 2021 年出版的《*Hands On Guide to AgileOps*》一书提供了关于如何在 IT 运维中采用敏捷方法的实践指导。

本书将带你深入了解 `AIOps` 的各个方面，包括人工智能和机器学习算法，并演示企业 `AIOps` 平台如何实现 `AIOps` 的某些功能。你将能够利用 `AIOps` 流程和平台提供的能力，将组织的运维成熟度提升到更高水平。我们将在配套网站 [`www.AgileInfraOps.com`](http://www.agileinfraops.com) 上持续提供更新的内容和指导，随着流程和技术的发展，你可以在该网站找到相关文章和最佳实践。

## 引言

本书是一本实践指南，旨在详细理解 `AIOps`，并协助在组织中实施其技术和流程。

本书阐述了 IT 行业对 `AIOps` 的需求、其架构以及促成其演进的技术。读者将能够掌握 `AIOps` 的核心理论概念，并通过实施利用机器学习技术的实践示例和练习来深入理解 `AIOps`。本书还提供了关于如何在企业中搭建 `AIOps` 以及如何避免陷阱以成功完成实施的指导。此外，本书还解释了 `AIOps` 在 `SRE` 和 `DevOps` 模型中的作用，并详细说明了 `AIOps` 如何赋能关键的 `SRE` 原则。

这本实践指南还提供了多个 `AIOps` 用例的实现，并附带了可运行的示例代码，帮助你更好地理解构成 `AIOps` 核心的一些基础原理和技术。

本书分享了在企业中建立 `AIOps` 实践和系统的最佳实践、流程和指南，以及衡量成果并持续演进的方法。
