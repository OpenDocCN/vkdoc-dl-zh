## 大型语言模型的演进步骤

本节回顾了安德雷·马尔可夫和克劳德·香农在 20 世纪初的基础性工作，他们引入了 n-gram 和信息论等概念，为语言建模奠定了基础。同时，本节也评述了诺姆·乔姆斯基在 1956 年提出的乔姆斯基语法层级理论，该理论指出了有限状态语法的局限性，并倡导在 NLP 中使用上下文无关语法。

# 马尔可夫、香农与语言模型

语言模型的起源可追溯至 20 世纪早期和中期**安德雷·马尔可夫**^(¹) **与克劳德·香农**^(²)的贡献。马尔可夫为我们如今所称的 n 元语法奠定了数学基础。有趣的是，n 元语法的概念最早出现在马尔可夫的早期著作中。安德雷·马尔可夫是深入研究语言模型的最早科学家之一，尽管当时"语言模型"这一术语尚未被创造出来。

考虑条件概率`p(w1|w0) = p(w1)`。不同的语言模型采用不同的方法来计算条件概率，记为`p(wi|w1, w2, ..., wi-1)`。获取并应用语言模型的过程被称为**语言建模**。n 元语法模型是一种基础模型类型，它假设任意位置的单词仅受其前面`n – 1`个位置的单词影响。本质上，该模型是一个**n – 1 阶马尔可夫链**。

1906 年，马尔可夫开始研究马尔可夫链，最初探索了一个仅有两个状态及它们之间相应转移概率的简单模型。他证明，通过根据转移概率在这些状态之间移动，转移到每个状态的频率最终会收敛到期望值。在随后的几年里，马尔可夫扩展了该模型，并证明这一结论在更普遍的情境下也成立。

为了说明其模型的实用性，马尔可夫于 1913 年将其应用于亚历山大·普希金的诗体小说《叶甫盖尼·奥涅金》。通过移除空格和标点符号，并将前 20000 个俄文字母分类为元音和辅音，马尔可夫得到了一个代表小说元音和辅音分布的序列。利用手动计数方法，马尔可夫计算了元音和辅音之间的转移概率，并用所得数据验证了最简单马尔可夫链的特性。

有趣的是，马尔可夫链最初的应用领域正是语言，马尔可夫的研究是对**语言模型最简单形式**的基础性探索。

时间来到 20 世纪中期，**克劳德·香农**建议使用马尔可夫过程来模拟自然语言。他利用 n 阶马尔可夫链，开发了一个能够描述字母序列（包括单词和句子）概率的统计模型。从数学角度来看，香农的方法涉及统计长度为 n 的字符序列（即 n 元语法）的出现频率。

1948 年，克劳德·香农以其开创性论文《通信的数学理论》革新了信息论领域。这篇论文标志着熵和交叉熵等关键概念的开端，以及对 n 元语法模型的探索。（香农在约翰·冯·诺依曼的建议下，从统计力学中借用了"熵"这一术语。）

在此背景下，熵表示概率分布固有的不确定性，而交叉熵则概括了一个概率分布相对于另一个概率分布的不确定性。值得注意的是，熵是交叉熵的下界。具体来说，当单词序列长度趋于无穷时，可以定义语言的熵，假设该熵是一个可以从语言数据中估计的常数值。

香农的贡献为评估语言模型提供了一个有价值的工具。如果一个语言模型在预测单词序列方面比另一个更准确，那么它将表现出更低的交叉熵。这一见解为评估语言建模提供了稳健的框架。需要认识到，语言模型不仅限于自然语言，还包括形式语言和半形式语言。

马尔可夫和香农的开创性工作为语言建模的多种方法铺平了道路，包括基于规则的系统、基于神经网络的系统，以及最近基于注意力机制的**基于 Transformer 的预训练大语言模型**（LLMs）。

## 乔姆斯基与语言模型

1956 年，诺姆·乔姆斯基^(³)提出了**乔姆斯基层级文法**，这是一个旨在表示**语言句法**的框架。乔姆斯基强调了有限状态文法（包括 n 元语法模型）在充分描述自然语言方面的局限性。

乔姆斯基的理论认为，语言由有限或无限的句子集合组成，其中每个句子是从有限词汇表中抽取的有限单词序列。根据乔姆斯基的说法，文法由一组能够生成语言中所有句子的产生式规则定义。不同的文法产生不同复杂程度的语言，形成一个**层级结构**。

能够生成有限状态机可接受的句子的文法称为有限状态文法或正则文法，而能够生成非确定性下推自动机可接受的句子的文法则称为上下文无关文法。有限状态文法被包含在上下文无关文法之中。

构成有限马尔可夫链或 n 元语法模型基础的文法与有限状态文法一致。然而，有限状态文法在生成具有语法关系的英语句子（例如 (i) 和 (ii) 所示的句子）方面存在局限性。

*   如果 S1，那么 S2。
*   要么 S3，要么 S4。
*   要么如果 S5，那么 S6，要么如果 S7，那么 S8。

虽然这些关系理论上可以无限组合以形成正确的英语表达（如示例 iii），但有限状态文法无法解释所有此类组合。乔姆斯基认为，用有限状态文法（包括 n 元语法模型）来描述语言存在重大限制。相反，他主张使用上下文无关文法来更有效地建模语言。这一影响导致在随后的几十年里，上下文无关文法在自然语言处理（NLP）中得到更广泛的应用。尽管乔姆斯基的理论在当代 NLP 中可能不再具有显著影响力，但它仍保留着重要的科学价值。

## 基于规则的语言模型

本节概述了第一个聊天机器人 ELIZA，它由约瑟夫·韦森鲍姆于 1966 年创建。ELIZA 基于规则的对话模拟方法标志着一个重要的里程碑，尽管它缺乏对语言细微差别的真正理解。这个时代关注基于规则的聊天机器人程序，并催生了几个类似的系统，包括 PARRY、ALICE 和 CLEVER。

# 首个聊天机器人：`ELIZA`

语言模型的探索可追溯至 20 世纪 50 年代，当时研究人员致力于研究基于规则的语言处理系统。早期的尝试因需要手动构建语法规则而受阻。

1966 年，约瑟夫·维森鲍姆^(⁴)推出了“`ELIZA`”程序（图 1-1），这标志着一次重大突破。`ELIZA` 是最早能够通过聊天和问答与人类互动的计算机程序之一。作为一个模拟罗杰斯式心理治疗师的程序，`ELIZA` 通过文本对话与用户交流，堪称对话模拟的开创性范例。然而，`ELIZA` 的交互严重依赖预定义模式，缺乏对语言细微差别的真正理解。

`ELIZA` 于 1966 年诞生于麻省理工学院，使用 `LISP-MAD` 语言编写。它利用知识数据库，在响应用户查询时识别排名最高的关键词，然后通过脚本提供的转换规则对句子模式进行处理。一个流行的脚本示例名为 `DOCTOR`。`ELIZA` 的成功激励了研究人员扩展这一思路，并开发其他基于规则的聊天机器人系统。

在 `ELIZA` 问世后的一段时间里，研究人员对基于规则的聊天机器人程序给予了显著关注。随后几年中，出现了多个类似系统，如 `PARRY`、`ALICE`、`CLEVER`、`SHRDLU` 等。这些早期的基于规则聊天机器人系统的特点是简单，且依赖手动编写的规则和脚本中定义的句子模式。因此，这些程序的知识库和功能仅限于预定义的规则和信息。

在随后的几十年里，随着更复杂的基于规则系统的发展，语言理解能力的提升取得了进展。尽管有所进步，但这些系统仍难以应对人类语言的复杂性，无法捕捉上下文细微差别并适应多样化的语言表达。研究人员日益认识到，必须从僵化的基于规则方法转向能够从数据中学习和泛化的模型。这一转变催生了统计语言处理技术，如 n-gram 和隐马尔可夫模型，为更精细的语言分析打开了大门。

这一区别也凸显了早期基于规则的聊天机器人程序与基于神经网络和大语言模型（LLM）的当代先进模型之间的显著差异。另一类早期语言模型依赖于信息检索（IR）技术。在这种方法中，聊天机器人通过匹配预构建对话对数据库中的模式来生成回复。以 20 世纪 80 年代末的 `CleverBot` 为例的基于信息检索的聊天机器人，受益于词频-逆文档频率（`TF-IDF`）、余弦相似度以及词级别的状态空间模型等技术。

## 统计语言处理

尽管语言建模领域的一些早期设计和模型在当今先进的架构中已显过时，但某些基础技术已成为当代语言模型及更广泛机器学习方法中不可或缺的组成部分。自然语言处理的关键构建模块包括 n-gram、词袋（`BOW`）和 `TF-IDF`。

### N-gram

n-gram 模型在 20 世纪 90 年代和 21 世纪初的引入，标志着统计语言建模的关键进步。这些模型基于一个简单而强大的概念，通过检查序列中前一个词来评估某个词出现的可能性。

尽管本质简单，n-gram 模型为理解语言中的上下文提供了一种关键机制。它们表示数据样本中 n 个相邻项目的序列，例如句子中的单词。通过关注词与词之间的局部关系，这些模型开始捕捉构成有意义语言表达的内在依赖关系。

尽管简单，n-gram 在基于前 n-1 个词预测第 n 个词的概率方面发挥了关键作用。这一概念在基础语言建模和更复杂模型的发展中都至关重要。

### 词袋模型（BOW）

语言建模中的另一项基础技术是词袋模型（BOW）。这种简单的方法根据文档中单词的频率，将语言元素表示为数值。本质上，BOW^(⁶) 利用词频来创建固定长度的向量，用于表示文档。

尽管简单，但 BOW 一直是一种基础性的向量化或嵌入技术。现代语言模型通常建立在先进的词嵌入和分词技术之上。

BOW 模型提供了一种将文档转换为数值形式的方法，这是在机器学习算法中使用文档之前的一个必要步骤。在任何自然语言处理任务中，这种初始转换都是必不可少的，因为机器学习算法无法直接处理原始文本；因此，我们必须将文本转换为数值表示，这个过程被称为`文本嵌入`。

文本嵌入涉及两种主要方法：词向量和文档向量。在词向量方法中，文本中的每个单词都被表示为一个向量（一个数字序列），然后整个文档被转换为一组这些词向量的序列。相反，文档向量将整个文档嵌入为一个单一的向量，与单个词嵌入相比，这种方法简化了过程。

此外，它确保了所有文档都被嵌入为相同的大小，这对于通常需要固定大小输入的机器学习算法来说非常方便。例如，对于一个包含 1000 个单词的词汇表，文档被表示为一个 1000 维的向量，其中每个条目表示对应词汇在文档中出现的频率。

虽然这种技术对于复杂任务可能有所局限，但它对于较简单的分类问题效果很好。其简单性和易用性使其成为嵌入一组文档并应用各种机器学习算法的一个有吸引力的选择。BOW 模型易于实现且执行迅速。

与其他通常需要专门领域知识或大量预训练的嵌入方法不同，这种方法避免了此类复杂性，甚至无需手动特征工程。它基本上可以开箱即用。然而，其有效性仅限于那些不依赖于理解词语上下文细微差别的相对简单的任务。

词袋模型的典型应用是嵌入文档以用于分类器训练。分类任务涉及将文档归类到多个类型中，该模型的特征对于垃圾邮件过滤、情感分析和语言识别等任务特别有效。

例如，可以根据“立即行动”和“紧急回复”等关键短语的频率来识别垃圾邮件，而情感分析则可以使用“无聊”和“糟糕”与“美丽”和“壮观”等词语来辨别积极或消极的语气。此外，在检查词汇时，语言识别也变得简单直接。

一旦文档被嵌入，就可以将它们输入到分类算法中。常见的选择包括朴素贝叶斯分类器、逻辑回归或决策树/随机森林 `–` 与更复杂的神经网络解决方案相比，这些选项相对容易实现和理解。

### TF-IDF（词频-逆文档频率）

TF-IDF^(⁷) 是另一种利用词频来评估一个词在给定文档集合中相关性的统计度量。与词袋模型相比，TF-IDF 更为先进，因为它使用两种不同的度量标准来更精确地量化词与文档之间的关系。

TF-IDF 应用于信息检索和文本挖掘，特别是在文档中搜索关键词等任务中。它也是自然语言处理应用和语言模型中的一个有价值的工具。词频-逆文档频率（TF-IDF）同样是一种广泛使用的统计方法，用于衡量一个词在单个文档中相对于整个文档集合（称为语料库）的重要性。

在文本向量化过程中，文本文档中的单词被转换为重要性分数，而 TF-IDF 是最常见的评分方案之一。本质上，TF-IDF 通过将词频（TF）与逆文档频率（IDF）相乘来为一个词打分。

词频（TF）衡量一个词在文档中出现的频率，相对于该文档的总词数。

![](img/613844_1_En_1_Figa_HTML.png)

另一方面，逆文档频率（IDF）评估语料库中包含该词的文档所占的比例。对于仅出现在一小部分文档中的术语，例如技术行话，其 IDF 值要高于在所有文档中都出现的常见词，如“a”、“the”和“and”。

![](img/613844_1_En_1_Figb_HTML.png)

一个词的 TF-IDF 分数由其 TF 和 IDF 分数相乘得出。简而言之，当一个词在特定文档中频繁出现，但在其他文档中很少出现时，它就具有高重要性。这种由 TF 衡量的文档内部普遍性与由 IDF 衡量的文档间稀有性之间的平衡，产生了 TF-IDF 分数，指示了该词对于语料库中某个文档的重要性。

![](img/613844_1_En_1_Figc_HTML.png)

TF-IDF 应用于各种自然语言处理任务，包括搜索引擎，用于对查询的文档相关性进行排序。它还应用于文本分类、文本摘要和主题建模。

需要注意的是，计算 IDF 分数存在不同的方法。计算通常使用以 10 为底的对数，尽管一些库可能选择自然对数。此外，在分母上加 1 是一种常见的做法，以防止除以零。

### 向量空间模型与状态空间模型

值得注意的是，词袋（BOW）模型被认为是一种基础的词嵌入技术，它将词语与向量关联起来。然而，将词语嵌入向量空间的广泛应用取得了显著进展，尤其是在 21 世纪初的突破之后，以及随后在 2013 年和 2014 年分别出现的`Word2Vec`^(⁸)和`GloVe`。词嵌入技术已成为语言建模中不可或缺的一部分，并被其他各种模型广泛采用。

向量空间模型和状态空间模型是语言建模中的基本概念。向量空间模型涉及用代数方法将语言元素表示为嵌入到特定向量空间中的向量。

向量空间由向量组成，这些向量是词语、句子甚至文档的数值表示。虽然基本向量（如地图坐标）只有两个维度，但自然语言处理中使用的向量可能包含数千个维度。

将词语表示为向量的概念（通常称为词嵌入或词向量化）自 20 世纪 80 年代以来就已存在。^(⁹)这简化了评估词语之间的相似度或搜索查询与文档相关性的过程。余弦相似度通常用于衡量向量之间的相似度。通过使用非二元词项权重的线性代数，向量空间模型能够计算两个对象（例如查询和文档）之间的连续相似度，从而实现部分匹配。

**注意**

**状态空间模型**

*尽管向量空间模型已在自然语言处理（NLP）领域得到深入研究和广泛应用，但最近 NLP 领域对状态空间模型的兴趣激增。尽管如此，状态空间模型已在包括信号处理和控制理论在内的多个领域的序列建模中证明了其价值。鉴于状态空间模型在处理长序列方面的有效性，这些模型为在语言建模中构建创新架构以及改进现有技术提供了广阔前景。*

### 神经语言模型——大语言模型的崛起

2001 年，约书亚·本吉奥及其同事引入了最早的神经语言模型之一，标志着语言建模新时代的开端。值得注意的是，本吉奥^(¹⁰)、杰弗里·辛顿和扬·勒昆因其开创性贡献荣获 2018 年 ACM 图灵奖，他们被广泛认为使深度神经网络成为计算领域的关键组成部分。

传统的 n-gram 模型在学习能力上存在局限性。常规方法是通过平滑方法从语料库中估计条件概率`p(wi|wi-n+1, wi-n+2, ···, wi-1)`。然而，随着`n`值的增加，模型参数以`O(V^n)`的速度呈指数增长，其中`V`代表词汇量大小。这种指数增长，加上训练数据的稀疏性，阻碍了模型参数的准确学习。

本吉奥等人的神经语言模型通过两项关键改进增强了 n-gram 模型。首先，它引入了一个称为词嵌入的实值向量来表示单个词语或词语组合。与词语的“独热向量”表示（其中对应元素为 1，其余为 0）相比，词嵌入的维度显著降低。

作为一种“分布式表示”形式，词嵌入与独热向量相比，具有更高的效率、更强的泛化能力、鲁棒性和可扩展性。其次，该语言模型被构建为神经网络，从而大幅减少了模型中的参数数量。

人工神经网络（ANN）已成为自然语言处理以及各种机器学习和人工智能模型开发中广泛采用的方法。

在本吉奥及其同事的贡献之后，涌现了大量关于词嵌入和神经语言建模的技术，并从不同角度引入了改进。

21 世纪后期见证了一个关键转变，其标志是对神经网络（尤其是深度学习技术的出现）重新燃起的兴趣。利用循环神经网络（RNN）和长短期记忆网络（LSTM）^(¹¹)处理序列数据，在推动语言理解和生成方面发挥了关键作用。这一变革时期出现了显著的应用，尤其是在机器翻译领域。谷歌在 2016 年推出的“神经机器翻译”系统就是这一进步的例证，它通过利用深度学习技术进行语言翻译，超越了传统方法。

这一时期的一个显著里程碑是 2013 年由托马斯·米科洛夫及其在谷歌的团队发布的`Word2Vec`模型。该模型通过学习基于词语在大型文本数据集中上下文使用的分布式词表示，彻底改变了语言表示方式。`Word2Vec`模型为捕捉词语之间的语义关系奠定了基础，将语言理解提升到了超越单纯统计关联的水平。

#### 循环神经网络

循环神经网络（RNN）（图 1-2）代表了继基本统计模型和基于规则的模型之后语言模型的关键进步。RNN 起源于 20 世纪 80 年代，在处理序列和时间序列数据方面发挥了重要作用。虽然 RNN 促进了语言模型的发展，但它在处理长期依赖关系^(¹²)方面面临挑战，例如梯度消失和梯度爆炸问题。研究人员一直在寻求改进的解决方案，以解决这些局限性并增强处理长序列的能力。

在循环神经网络（RNN）领域，一个关键方面是中间表示或状态的概念。词语之间的相互依赖关系通过 RNN 模型中状态之间的关系得以体现。虽然模型的参数在不同位置是共享的，但生成的表示会根据其各自位置而有所不同。

![](img/613844_1_En_1_Fig2_HTML.jpg)

**图 1-2** RNN 层的基本架构

在 RNN 的每个位置，每一层都存在一个中间表示，它体现了到该位置为止的词语序列的“状态”。当前位置当前层的中间表示由前一位置同一层的中间表示以及当前位置下一层的中间表示共同决定。当前位置的最终中间表示在计算下一个词语的概率时起着关键作用。

# 长短期记忆网络

长短期记忆网络（LSTM）是一种深度学习序列神经网络，旨在解决信息保留的挑战。作为循环神经网络（RNN）的一种特殊形式，LSTM 有效解决了传统 RNN 遇到的梯度消失问题，这是机器学习算法中的一个重要难题。`Hochreiter 和 Schmidhuber`^(¹³) `开发了 LSTM 以克服传统 RNN 固有的局限性`，提出了一种提升机器学习模型性能的解决方案。LSTM 模型的实现可通过 Python 中的`Keras`库来完成。

举例来说，想象一下在观看视频或听一首歌时回忆起之前的场景，或者在读书时记起前几章的情节。类似地，RNN 通过保留先前信息并利用其处理当前输入来运作。然而，由于梯度消失问题，RNN`在记住长期依赖关系方面存在局限`。而 LSTM 则专门设计用于规避这些与长期依赖相关的挑战。

与传统神经网络不同，长短期记忆网络（LSTM）集成了反馈连接，使其能够处理整个数据序列，而非单个数据点。这一独特能力使其在理解和预测序列数据（包括时间序列、文本和语音）中的模式方面表现出色。

在人工智能和深度学习领域，LSTM 已成为一种强大工具，通过从序列数据中提取有价值的见解，为各个领域的突破做出了贡献。

从广义上讲，LSTM 的运作方式与 RNN 单元非常相似。LSTM 网络的内部机制包含三个关键组件，每个组件都有不同的功能：

- 遗忘门
- 输入门
- 输出门

第一部分决定来自上一时间戳的信息是否相关，应予以保留，还是被视为无关而丢弃。第二部分专注于从输入中学习新信息。最后，在第三部分中，来自当前时间戳的更新信息被传递到下一个时间戳，构成 LSTM 循环中的一个时间步。

这三个组件也被称为`门`。这些门控制着信息进出记忆单元或 LSTM 单元的流动。遗忘门、输入门和输出门调节着这些流动。一个 LSTM 单元由这些门以及一个记忆单元或 LSTM 单元组成，可以类比为`传统前馈神经网络`中的一层神经元，其中每个神经元都包含一个隐藏层和一个当前状态。

与基本 RNN 类似，LSTM 具有一个隐藏状态，记为上一时间戳的`H(t-1)`和当前时间戳的`H(t)`。此外，LSTM 还包含一个单元状态，分别用`C(t-1)`和`C(t)`表示上一时间戳和当前时间戳。隐藏状态被称为`短期记忆`，而单元状态则被称为`长期记忆`。

**示例：** *考虑两个以句号分隔的句子：“约翰是个好人，”接着是“另一方面，艾莉森很坏。”当我们从第一个句子过渡到第二个句子时，LSTM 网络利用遗忘门，可以恰当地忽略关于约翰的信息，并意识到焦点已转移到艾莉森身上。*

> **注意**
>
> **双向 LSTM**
>
> *超越 LSTM，双向 LSTM 是一种循环神经网络（RNN）架构，它以前向和后向两个方向处理输入数据。与仅考虑从过去到未来信息流的传统 LSTM 不同，双向 LSTM 融入了* `未来到过去的上下文`，增强了其捕捉双向依赖关系的能力。
>
> *该架构由两个 LSTM 层组成——一层用于前向处理，另一层用于后向处理——在需要全面理解输入序列的任务中尤其有效，例如* `情感分析`、`机器翻译`和`命名实体识别`。双向 LSTM 从两个方向整合信息，增强了其捕捉`长期依赖关系`的能力，从而在复杂的序列数据中实现更准确的预测。

# 门控循环单元

门控循环单元（GRU）由 Cho 等人于 2014 年提出，^(¹⁴) 是缓解 RNN 中梯度消失问题的另一种方法。GRU 与 LSTM 网络相似，通过引入额外的门来处理梯度挑战。然而，GRU 仅使用两个门（`更新门`和`重置门`），并且缺少 LSTM 中存在的额外单元。因此，GRU 拥有更简单的架构和更少的参数，从而训练速度更快。尽管如此，LSTM 和 GRU 各有其独特的优势和劣势，使其适用于不同的应用场景。值得注意的是，`在数据具有长期依赖关系的情况下，LSTM 网络可能表现更优`。

# 序列到序列语言模型

语言模型发展中的一个重要里程碑是序列到序列（`seq2seq`）模型的诞生。这类语言模型的主要推动力在于解决机器翻译的挑战。与机器翻译相关的任务通常涉及解决序列到序列或序列建模问题。在这些场景中，目标是创建能够将一个领域的序列映射到另一个领域对应序列的模型，例如将句子从英语翻译成意大利语。

设计`seq2seq`模型^(¹⁵)的一种常见策略是采用编码器-解码器架构，其中编码器和解码器都基于循环神经网络（RNN）架构，如`LSTM`或`GRU`。编码器处理输入序列，生成输入序列的向量表示（或称上下文向量）。解码器随后利用这个上下文向量生成输出序列。

尽管`Seq2Seq`模型在机器翻译任务中广受欢迎，但它们也表现出局限性，这主要是由于它们依赖于基于 RNN 的架构（如`LSTM`或`GRU`）。`Seq2Seq`架构的一个显著缺陷是它们无法辨别序列中不同部分的重要性。换句话说，它们在处理序列时，无法识别特定片段是否比其他片段更重要。

这一缺陷促使研究人员探索如何让模型有选择性地关注序列中的关键部分。这一探索催生了基于注意力的语言模型，并最终引出了 Transformer 架构。

> **注意**  
> **条件语言模型**  
> 语言模型具有双重用途：它们可以计算语言（词序列）的概率，也可以生成语言本身。在后一种情况下，通常会通过从语言模型中随机采样来生成自然语言句子或文章。众所周知，在大型数据集上训练的`LSTM`语言模型具备生成高度自然句子的能力。  
>  
> 语言模型概念的一个扩展是条件语言模型，它评估在特定条件下词序列的条件概率。当这个条件是另一个词序列时，任务就变成了通常所说的序列到序列问题。这个问题在机器翻译、文本摘要和生成式对话等任务中显而易见。或者，如果条件是图像，挑战就变成了将图像转换为词序列，例如图像描述任务。  
>  
> 条件语言模型广泛应用于各种任务：  
> - 在机器翻译中，这些系统将句子从一种语言转换为另一种语言，同时保留语义。  
> - 在对话生成中，系统对用户的输入生成回复，构成一轮对话。  
> - 文本摘要涉及将长文本转换为简洁的表示，以概括原始内容的精髓。  
>  
> 这些模型的条件概率分布所体现的语义因具体应用而异，并通过每个应用上下文中的数据驱动学习获得。

## 基于注意力的语言模型

基于注意力的语言模型是一种神经网络架构，通过关注输入中每个输出最相关的部分来处理序列数据。它们使用自注意力等注意力机制来权衡句子中不同词语的重要性，从而比传统模型更有效地捕捉长距离依赖关系和上下文。这种方法显著提升了自然语言处理任务（如翻译和文本生成）的性能。

### 注意力机制

几十年来，研究人员一直致力于开发人工系统，并将人类视觉系统和大脑机制的元素融入这些系统。2014 年，研究人员朝着实现这一目标迈出了重要一步，他们引入了一个基本而巧妙的概念：在编码器-解码器架构中加入注意力机制^(¹⁶)。

其关键主张是，序列中元素的相对重要性可以在序列处理过程中被编码并分配权重。注意力技术提出了一种对齐模型来计算分数，然后通过`softmax`函数生成权重。随后，通过编码器隐藏状态的加权和得出上下文向量。

注意力机制的整合是推动人工神经网络以及更广泛的机器学习模型发展的关键概念。值得注意的是，它为自然语言处理（NLP）领域的多个重大突破提供了灵感，包括著名的 Transformer 架构。而 Transformer 架构又为各种创新开发铺平了道路，例如视觉 Transformer（`ViT`）、`BERT`模型和`GPT`模型，最终催生了像`ChatGPT`这样强大的聊天机器人程序。

### Transformer 架构

语言模型发展中的一个关键突破是 2017 年 Transformer 架构的引入。Transformer 架构从编码器-解码器架构中汲取灵感，其强大能力源于注意力机制，以及词嵌入和位置编码等其他组件。

Transformer 架构与早期基于 RNN 的架构之间的主要区别在于其非序列化的数据处理方式。值得注意的是，Transformer 巧妙地结合了词嵌入和位置编码技术，从而实现了并行数据处理。这一特性有助于提升 Transformer 的效率，使其能够受益于并行化并加速训练过程。

在 2017 年之前，循环网络，特别是长短期记忆网络（LSTM），是语言建模的主流架构。LSTM 拥有理论上无限的上下文大小，并通过双向方法整合过去和未来的词语。尽管具有理论上的优势，但其实际收益有限，并且循环结构导致了更高的训练成本和时间消耗，尤其是在 GPU 上实现并行化方面面临挑战。正是由于这些挑战，Transformer 才得以取代 LSTM。

Transformer 利用注意力机制，模型通过学习根据上下文相关性为词语分配权重。与循环模型（其中最近的词语对预测下一个词语的影响最为直接）相比，注意力机制允许当前上下文中的所有词语都可被访问，从而使模型能够辨别哪些词语需要优先处理。

在一篇标题恰如其分的论文《注意力就是一切》^(¹⁷)中，谷歌研究人员介绍了 Transformer 序列到序列架构。这种革命性的模型摒弃了循环连接，在文本生成过程中将其自身输出作为上下文。这种设计选择使得训练过程易于并行化，从而能够将模型和训练数据扩展到前所未有的规模。

对于分类任务，来自 Transformer 的双向编码器表示^(¹⁸)（BERT）成为了首选模型。在文本生成领域，重点转向了模型规模的扩展。

Transformer 架构被广泛认为是人工智能（AI）的基础模型，对人工智能的各个领域都产生了深远影响，包括自然语言理解（例如 BERT）、计算机视觉（例如 ViT）和语音识别。

本质上，Transformer 架构是一种通用的神经计算机，可应用于各种下游 NLP 技术和多模态 AI 应用。它特别适合部署在能够处理文本、图像和视频等多种数据类型的多模态大语言模型中。

### 大语言模型（LLM）

人工神经网络的突破性进展以及自然语言处理模型的进步，催生了各种大语言模型（LLM），它们在 NLP 任务中展现出卓越的能力。这些 LLM 的突出例子包括 OpenAI 的 ChatGPT 和 GPT-4、Google 的 Bard、Meta 的 LLaMA、Anthropic 的 Claude 以及 BloombergGPT。

大多数大语言模型的能力和成功可归因于语言模型发展过程中的先前创新。基于编码器-解码器模型并融合了注意力机制的 Transformer 架构，是一项为强大 LLM 铺平道路的重大进步。

其他促成因素包括利用大规模数据集进行模型训练。虽然 LLM 在问答和文本摘要等特定 NLP 任务中已展现出显著效果，但它们在许多方面仍有待进一步改进。

### 多模态学习时代

多模态深度学习是指探索计算机算法，通过从包含多种模态的多样化数据集中学习来提升其性能。

在机器学习领域，多模态深度学习是一个专门的子领域，专注于训练人工智能模型以分析和建立不同类型数据（称为模态）之间的联系。这些模态通常包括图像、视频、音频和文本。通过整合不同的模态，深度学习模型可以更全面地理解其周围环境，因为某些线索是特定模态所独有的。

为了说明这一点，考虑情感识别任务；虽然人脸部的视觉线索（视觉模态）至关重要，但一个人声音的语调和音高（音频模态）携带着关于其情绪状态的重要信息，这些信息可能仅从面部表情中无法明显看出，即使它们是同步的。

虽然仅处理单一模态的单模态模型已经过广泛研究，并在计算机视觉和自然语言处理等领域取得了令人瞩目的成果，但它们存在固有的局限性。因此，对多模态模型的需求日益增长。

多模态模型通常利用深度神经网络，尽管早期研究也整合了其他机器学习模型，如隐马尔可夫模型（HMM）或受限玻尔兹曼机（RBM）。

在多模态深度学习领域，常见的模态通常包括视觉元素（如图像和视频）、文本信息以及听觉组件（包括语音、声音和音乐）。然而，一些不太常见的模态也发挥着作用，例如 3D 视觉数据、深度传感器数据和 LiDAR 数据，特别是在自动驾驶汽车等应用中。

尽管如此，最常见的组合涉及四种主要模态：

-   图像 + 文本
-   图像 + 音频
-   图像 + 文本 + 音频
-   文本 + 音频

#### 多模态学习挑战

多模态深度学习致力于解决五个基本挑战，这些挑战目前是活跃的研究领域。解决或改进其中任何一个挑战都有可能推动多模态 AI 研究和应用领域的发展。

第一个挑战是表示，涉及将来自不同模态的数据编码为向量或张量。创建能够封装原始数据语义信息的有效表示，对于机器学习模型的成功至关重要。然而，难点在于以能够最佳利用其协同作用的方式从异构数据中提取特征。充分利用不同模态之间的互补性，同时避免包含冗余信息，同样至关重要。

多模态表示可分为两种类型：

1.  **联合表示：** 每种单独的模态经过编码，随后被放入一个共享的高维空间。这种直接的方法可能有效，尤其是当模态具有相似性质时。
2.  **协调表示：** 单独的模态被独立编码，但通过施加约束来协调它们的表示。例如，它们的线性投影应表现出最大相关性。这种方法有助于在保持协调性的同时整合不同的模态。

# 融合

融合任务涉及整合来自两种或多种模态的信息以进行预测。有效融合视频、语音和文本等不同模态，由于多模态数据的异质性，这构成了一个挑战。

融合异质信息是多模态研究的一个基本方面，伴随着众多挑战。实际问题包括处理格式、长度和非同步数据的差异。理论挑战则围绕确定最优融合技术展开，涵盖诸如拼接或加权求和等简单操作，以及诸如 Transformer 网络或基于注意力的循环神经网络（`RNNs`）等更复杂的机制。

在早期融合与晚期融合之间进行选择也是一个考量点。早期融合在特征提取后立即进行整合，采用前述机制。相比之下，晚期融合仅在每个单模态网络产生预测后才进行整合，通常采用投票方案、加权平均或其他技术。混合融合技术也被提出，它结合了早期融合和单模态预测器的输出。

# 对齐

对齐涉及识别不同模态之间的直接关系。当代多模态学习研究致力于创建模态不变的表征，这意味着当不同模态表示相似的语义概念时，它们的表征在潜在空间中应彼此接近。例如，句子“她跳进了游泳池”、一张游泳池的图片以及溅水声的音频信号，在表征空间流形中应具有邻近性。

# 翻译

翻译任务涉及将一种模态映射到另一种模态，探索如何在保持语义的前提下，将一种模态（例如文本）转换为另一种模态（例如视觉）。翻译本质上是开放性和主观的，缺乏确定的答案，这增加了任务的复杂性。

当前的多模态学习研究涉及构建用于模态翻译的生成模型。最近的例子包括`DALL-E`和其他文本到图像模型，这展示了促进文本与视觉模态之间翻译的生成模型的发展。

# 协同学习

多模态协同学习旨在将从一种或多种模态学到的知识迁移到涉及其他模态的任务中。这种方法在目标任务资源匮乏、模态缺失或部分缺失、或存在噪声数据的情况下尤为关键。

翻译可以作为协同学习的一种方法，以促进知识从一种模态迁移到另一种模态。神经科学见解也表明，人类可能通过翻译方法进行协同学习。患有心盲症（无法在脑海中形成图像）的个体在记忆测试中表现较差。相反，那些擅长创建此类映射（例如在文本/听觉与视觉模态之间进行翻译）的个体，在记忆测试中表现出更强的性能。这突显了跨模态转换表征的能力是人类认知和记忆的一个关键方面。

# 多模态学习如何运作

多模态神经网络通常融合多个单模态神经网络。例如，在一个视听模型中，有两个不同的单模态网络：一个专门处理视觉数据，另一个处理音频数据。每个单模态网络都通过一个称为编码的步骤独立处理其输入。在单模态编码之后，从每个模型中提取的信息需要被有效地融合在一起。

已经提出了各种融合技术，从简单的拼接到引入注意力机制。融合多模态数据的过程是决定成功与否的关键因素。在融合过程之后，一个最终的“决策”网络接收融合后的编码信息，并为最终任务进行训练。

本质上，多模态架构通常包含三个组成部分：

1.  单模态编码器，分别对不同的模态进行编码，通常每个输入模态对应一个编码器
2.  一个融合网络，负责在编码阶段合并从每个输入模态中提取的特征
3.  一个分类器，接收融合后的数据并生成预测

这些组件通常被称为编码模块、融合模块和分类模块。

# 多模态深度学习的应用

多模态深度学习整合并处理来自文本、图像和音频等多种数据源的信息，正在彻底改变众多领域。通过利用不同模态的力量，这种方法增强了 AI 系统的能力，从而产生更准确、更稳健的解决方案。在本节中，我们将探讨`LLM`技术如何应用于不同行业及其产生的重大影响。

## 图像描述

图像描述涉及为给定的图像生成简短的文本描述，利用包含图像和文本描述的多模态数据集。它通过将视觉表征转换为文本表征来解决翻译挑战。模型需要理解图像的语义，检测关键对象、动作和特征。此任务也可以扩展到视频描述。

图像描述模型在为图像提供文本替代方案方面很有用，惠及盲人和视障用户。

## 图像检索

图像检索涉及基于检索键在大型数据库中查找相关图像。此任务也称为基于内容的图像检索（`CBIR`），它利用深度学习多模态模型，提供了具有更强能力的更广泛解决方案，通常减少了对传统标签的依赖。图像检索在日常场景中很常见，例如搜索引擎结果显示与查询相关的图像。

## 文本到图像生成

文本到图像生成是一种流行的多模态学习应用，它解决了翻译挑战。像 OpenAI 的`DALL-E`和 Google 的`Imagen`这样的模型，基于简短的文本描述创建图像，提供了文本语义的视觉表示。这些模型应用于图像编辑、平面设计和数字艺术灵感。

## 情感识别

情感识别是一个多模态数据集优于单模态数据集的应用案例，它涉及分析视频、文本和音频数据以识别情绪状态。整合脑电图数据等传感器数据进一步增强了多模态输入。虽然多模态数据集传达了更多信息，但训练多模态网络也带来了挑战，有时会导致性能相比单模态对应物有所下降。理解这些困难对于有效实施至关重要。

总之，多模态深度学习具有多样化的应用，增强了计算机视觉系统在各种任务中的能力。

# 理解自然语言处理基础

自然语言处理（`NLP`）属于计算机科学领域，具体而言是人工智能（`AI`）这一更广泛范畴下的分支，旨在让计算机具备类似人类的理解能力，能够理解书面文本和口头语言。

`NLP`融合了计算语言学（涉及基于规则的人类语言建模）与统计学、机器学习和深度学习模型。这些技术相结合，使计算机能够处理人类语言（无论是文本形式还是口头交流），并全面理解其含义，包括说话者或作者的意图和情感。

`NLP`驱动着计算机程序的发展，使其能够执行诸如将文本从一种语言翻译成另一种语言、响应语音指令、以及快速总结大量文本内容（甚至是在实时场景下）等任务。你很可能已经通过语音操控的 GPS 系统、数字助手、语音转文字转录软件、客服聊天机器人以及其他日常便利工具与`NLP`有过交互。然而，`NLP`在企业解决方案中也扮演着日益重要的角色，助力优化业务运营、提升员工生产力并简化关键业务流程。

自然语言处理（`NLP`）在人工智能领域占据核心地位，是连接人类交流与计算机理解的关键纽带。它是一个跨学科领域，赋予计算机理解、分析和生成人类语言的能力，促进人与机器之间的无缝交互。`NLP`的重要性在其广泛的应用中显而易见，从自动化客户支持到实时语言翻译，不一而足。

## 自然语言处理究竟是什么？

自然语言处理（`NLP`）是人工智能的一个分支，专注于计算机与使用自然语言的人类之间的交互界面。其主要目标是教导计算机处理和分析海量的自然语言数据。

`NLP`涵盖的任务是让机器能够以既有价值又有意义的方式理解、解读和生成人类语言。以开发`ChatGPT`等先进语言模型而闻名的`OpenAI`，强调了`NLP`在构建能够理解、响应和生成文本的智能系统中的关键作用，从而使技术更加用户友好和易于访问。

## NLP 如何运作？

`NLP`使计算机能够以类似于人类理解的方式理解自然语言。无论是口语还是书面语，自然语言处理都利用人工智能接收现实世界的输入，对其进行处理，并以计算机可理解的格式提取含义。就像人类拥有各种感官器官（例如用于听力的耳朵和用于视力的眼睛）一样，计算机依赖程序进行阅读，依赖麦克风捕获音频数据。此外，正如人类利用大脑处理感官输入一样，计算机利用特定程序处理通过各自输入方式获得的数据。在此处理的某个阶段，输入会被转换成计算机可以解读的代码。

自然语言处理的过程主要包含两个关键阶段：数据预处理和算法开发。

数据预处理涉及对文本数据进行准备和精炼，使其适合机器分析。这个预处理步骤对数据进行优化，使其呈现为适合机器处理的格式，并揭示出算法可以有效利用的独特文本特征。

## NLP 的构成要素

自然语言处理并非单一、统一的方法，而是由多个构成要素组成，每个要素都对语言的全面理解有所贡献。`NLP`试图理解的主要组成部分包括句法、语义、语用和话语。

### 句法

句法涉及单词和短语的组织方式，以在一种语言中构建结构良好的句子。在句子“The boy played with the ball”中，句法需要仔细分析该句子的语法结构，确保其符合英语语法规则，包括主谓一致和正确的词序。

### 语义

语义关注理解单词的含义，以及它们在句子中如何共同传达意义。在句子“The panda eats shoots and leaves”中，语义有助于根据词义和上下文辨别熊猫是吃植物（竹笋和叶子），还是进行暴力行为（开枪）然后离开（离开）。

### 语用

语用涉及在不同语境中理解语言，确保根据情境、说话者的意图和共享知识得出预期的含义。当有人说“Can you pass the salt?”时，语用涉及识别这是一个请求，而不是询问对方传递盐的能力，并在用餐语境中解读说话者的意图。

### 话语

话语侧重于在句子层面之上分析和解读语言，考虑句子在文本和对话中如何相互关联。在一次对话中，一个人喊道“I'm freezing”，另一个人回答“I'll close the window”，话语涉及理解这两句话之间的连贯性，认识到第二句话是对第一句话中隐含请求的回应。

理解这些构成要素对于涉足`NLP`领域的人来说至关重要，因为它们构成了`NLP`模型解读和生成人类语言的基础。

### NLP 任务

人类语言充满了复杂性，这给开发能够从文本或语音数据中准确辨别其意图的软件带来了巨大挑战。同形异义词、同音异义词、讽刺、习语、隐喻、语法和用法上的例外，以及句子结构的变化，都只是人类语言复杂性的冰山一角。尽管人类可能需要多年才能掌握这些细微差别，但程序员必须从一开始就在自然语言驱动的应用程序中灌输准确识别和理解它们的能力，才能使这些应用程序真正有效。

为了以促进计算机理解的方式解构人类文本和语音数据，人们采用了各种 NLP 任务。其中一些任务包括：

- **语音识别**，也称为语音转文本，涉及将语音数据可靠地转换为文本形式。对于依赖语音命令或对语音查询提供响应的应用程序来说，语音识别是一项基本要求。增加语音识别挑战性的是人们的说话方式——语速快、词语连读、重音和语调变化、口音各异，并且常常使用不正确的语法。

- **词性标注**，也称为语法标注，需要根据特定单词或文本片段的使用方式和上下文来确定其语法分类。例如，词性标注可以识别出“make”在“I can make a paper plane”中是动词，而在“What make of car do you own?”中是名词。

- **词义消歧**涉及在一个词具有多种含义时选择其正确的含义，通过语义分析过程来确定在给定语境中最恰当的解释。例如，词义消歧有助于区分动词“make”在“make the grade”（达到）和“make a bet”（下注）中的不同含义。

- **命名实体识别 (NER)**，也称为 NEM，将单词或短语识别为具有特定角色的实体。NER 可以识别“Kentucky”是一个地点，或“Fred”是一个男性名字。

- **共指消解**涉及识别两个词是否指向同一实体，通常用于确定代词的指代对象（例如，“她”指代“玛丽”）。它还可以扩展到识别文本中的隐喻或习语，例如当“bear”指的不是动物，而是一个高大、多毛的人时。

- **情感分析**致力于从文本中提取主观方面，例如态度、情绪、讽刺、困惑和怀疑。

- **自然语言生成**，通常被视为语音识别或语音转文本的对应技术，其核心任务是将结构化信息转换为人类语言。

### 文本预处理与特征工程

在分析和理解人类语言的领域中，自然语言处理 (NLP) 运用了一系列技术和方法。下面将重点介绍一些 NLP 中常用的基础技术。

#### 分词

这个过程涉及将文本分割成单个单词、短语、符号或其他有意义的单元，这些单元被称为词元。分词服务于一个基本目标：以保留上下文且机器可理解的形式呈现文本。通过将文本转换为词元，算法能够有效地识别模式。

这种模式识别能力至关重要，因为它使机器能够解释和响应人类输入。例如，当遇到单词“running”时，机器不会将其视为一个孤立的实体；相反，它会将其视为一组可以分解并提取含义的词元。

分词方法可能因所需的粒度级别和任务的特定要求而异。这些方法涵盖了一个范围，从将文本分解为单个单词，到将其分解为字符甚至更小的单元：

- **单词分词：** 此方法将文本分解为离散的单词，是最普遍且有效的方法。它对于像英语这样具有明确单词边界的语言效果极佳。

- **字符分词：** 在字符分词中，文本被分割成单个字符。这种方法对于缺乏明确单词边界的语言或需要精细分析的任务（如拼写纠正）很有价值。

- **子词分词：** 子词分词在单词分词和字符分词之间取得平衡，将文本分割成比单个字符大但比完整单词小的单元。例如，“Chatbots”可以被分词为“Chat”和“bots”。这种方法在含义源于组合较小单元的语言中，或在处理 NLP 任务中标准词汇表中未出现的单词时，尤其有用。

##### 句法分析

在自然语言处理 (NLP) 中，句法分析指的是分析句子或文本片段的语法结构，以确定其句法和语法元素的过程。句法分析的主要目标是理解句子中单词和短语之间的层级关系，识别它们在句子结构中的功能。

**句法分析涉及以下关键任务：**

- **分词：** 文本首先被分割成单个单词或词元。

- **词性标注：** 每个词元都被标注其语法词性（例如，名词、动词、形容词），这有助于确定每个单词在句子中的作用。

- **依存句法分析：** 此步骤识别句子中单词之间的语法关系，确定哪些单词依赖于或修饰其他单词。它创建了一个称为依存句法分析树的树状结构，其中单词是节点，由代表语法关系的带标签的边连接。

句法分析在 NLP 中对于各种应用至关重要，例如机器翻译、信息检索、情感分析和问答系统。它使计算机能够理解句子的句法结构，从而能够从文本中提取有意义的信息，并生成连贯的自然语言响应。

##### 词形还原

此技术旨在将单词还原为其基本形式或词根形式，有助于将具有共同词根的各种单词形式分组。词形还原旨在找到单词的基本形式或词典形式，即“词元”，同时保留单词的实际含义。它涉及考虑单词的词性（例如，名词、动词、形容词）和上下文，以执行更准确的转换。

词形还原的结果是可以在词典中找到的有效单词，使其适用于单词的可解释性和意义至关重要的任务。例如，单词“running”将被词形还原为“run”，而“better”将被词形还原为“good”。

##### 词语分割

词语分割涉及从给定的文本字符串中提取单个单词的过程。例如，当一个人将手写文档扫描到计算机中时，可以使用算法来分析页面并识别单词边界，通常由单词之间的空格表示。

##### 词义消歧

词义消歧是一种用于确定单词在给定语境中预期含义的技术。例如，让我们分析句子“The pig is in the pen.”。在此语境中，单词“pen”可以有多种含义。采用此方法的算法可以辨别出，在此例中，“pen”指的是一个围起来的动物围栏，而不是其另一种含义，即书写工具。

# 句子边界检测

句子切分，或称句子边界检测，涉及在长篇文本中识别并标记句子边界的任务。例如，当自然语言处理算法处理文本“I played the game. I called Bill”时，它能识别出句号是分隔这两个句子的标记，从而有效地将它们分开。

# 词素分割

词素分割将单词分解为称为词素的更小单位。例如，单词“untestably”会被分解为`[[un[[test]able]]ly]`，算法会识别出`un`、`test`、`able`和`ly`为不同的词素。这项技术在机器翻译和语音识别等应用中尤其有价值。

# 词干提取

词干提取旨在分离出单词的词根形式，特别是那些包含屈折变化的单词。例如，在句子“The boy played with the ball”中，算法可以识别出单词“played”的词根形式是`play`。当用户想要分析文本中某个特定单词（包括其各种变位形式）的所有出现情况时，此功能非常有用。算法会识别出这些变位形式本质上代表同一个单词，尽管它们的拼写有所不同。

运用这种文本处理技术对于解决数据稀疏性和标准化词汇问题非常有价值。它不仅有助于减少冗余（因为屈折词及其词干通常表达相同含义），还能使 NLP 模型在屈折词与其对应的词干之间建立联系。这种联系增强了模型对这些词在类似语境中如何被使用的理解。

词干提取算法通过识别屈折词中常见的前缀和后缀，并将其从单词中截断来运作。有时，这可能会产生并非实际单词的词干。因此，尽管这种方法无疑具有优势，但它也并非没有固有的局限性。

# 命名实体识别（NER）

命名实体识别（NER）是自然语言处理（NLP）领域中的一个关键子任务，属于信息抽取的范畴。NER 致力于将命名实体分类到预定义的类别中，涵盖多种实体，如人名、组织、地理位置、医疗代码、时间表达、数量、货币价值等等。在 NLP 领域，理解这些实体至关重要，因为它们通常包含了给定文本中最重要的信息。

命名实体识别（NER）作为一个重要的中间环节，连接非结构化文本与结构化数据，使机器能够浏览大量文本内容，并挖掘出经过系统分类的有价值数据片段。通过从浩瀚的词汇中识别出特定实体，NER 彻底改变了文本数据处理和利用的方式。

NER 的核心任务是仔细扫描非结构化文本，将不同的片段识别为命名实体，然后将它们分配到预定义的类别中。这种将原始文本转化为组织化信息的过程增强了数据的实用性，有助于执行数据分析、信息检索和知识图谱构建等各种任务。

## 实体如何被识别

命名实体识别（NER）的内部工作机制可以分解为几个连续的阶段：

- **分词：** 在深入实体识别之前，文本会先经过分词处理，即将其分割成标记，这些标记可以是单词、短语甚至整个句子的形式。
- **实体识别：** 下一步是识别文本中潜在的命名实体。此过程依赖各种语言规则或统计方法来检测实体。它包括识别诸如人名中的大写模式（例如“Bill Gates”）或遵循特定格式（例如日期）等模式。
- **实体分类：** 一旦识别出实体，它们就会被系统地分类到预定义的类别中，例如`Person`、`Organization`或`Location`。这种分类通常通过基于标注数据集训练的机器学习模型来实现。例如，“Bill Gates”会被归类为`Person`，而“Microsoft”则会被指定为`Organization`。
- **上下文分析：** NER 系统通常会考虑上下文环境以提高准确性。例如，在句子“Apple released a new iPhone”中，上下文线索帮助系统将`Apple`识别为一个组织，而不是一种水果。
- **后处理：** 在初始识别和分类阶段之后，可能会采用后处理技术来优化结果。这可以包括解决歧义、合并多标记实体，或利用知识库来增强实体信息。

# NLP 与特征工程

NLP 中的特征提取是一个关键过程，它将原始文本内容转换为机器学习模型更易处理的数值表示。NLP 中有多种特征提取方法，每种方法都有其独特的优势和局限性。对于数据科学家来说，精通这些不同的技术并理解何时有效应用它们至关重要。

这一步在 NLP 中至关重要，因为特征工程在自然语言处理（NLP）应用中扮演着重要角色。它涉及定义表征 NLP 任务的特征，塑造机器学习算法理解和处理文本数据的方式，并且极大地影响着为特定任务设计的模型的性能。

## 词嵌入与语义理解

词嵌入是自然语言处理中的一个关键概念，它将词语转换为实值向量以进行文本分析。NLP 的这一进步显著增强了计算机理解文本的能力。这是深度学习的一大飞跃，能够有效解决复杂的 NLP 挑战。

自然语言处理面临的一个主要挑战是，它无法根据上下文解释具有多重含义的词语。上下文语义分析在澄清此类歧义、提升基于文本的 NLP 应用的精确度方面起着关键作用。让我们探讨一下 NLP 中消歧的重要性。

本节将深入探讨这两个概念。

# 词嵌入

在词嵌入中，单词和文档都会被转换为数值向量。这种转换确保了含义相似的词拥有相似的向量表示。其技术能够在压缩的维度空间中捕捉语义，相比传统的、更复杂的图嵌入模型（如 `WordNet`），提供了一种更快速的替代方案。

随后，机器学习模型会利用这些向量，以保持文本的语义和句法完整性。转换后的数据由 NLP 算法处理，这些算法能够高效地解读这些表示。

**此类别下通常有三种类型的向量：**

- 计数向量
- TF-IDF 向量
- 共现向量

词嵌入能够训练诸如 `GRU`、`LSTM` 和 `Transformers` 等先进的深度学习模型。这些模型在多种 NLP 任务中表现出色，包括情感分析、命名实体识别和语音识别。

词嵌入的有效性提升了机器学习在 NLP 中的普及度，使其成为开发者们非常青睐的领域。例如，一个 50 维的词嵌入可以表示 50 个不同的特征。许多人会选择使用预先训练好的词嵌入模型，如 `Flair`、`fastText` 和 `spaCy` 等。

**常见的词嵌入技术（其中一些已在本书中介绍）如下：**

- 词频-逆文档频率（`TF-IDF`）
- 词袋模型（`BOW`）
- `Word2Vec` – 捕捉语义信息
- `GloVe` – 用于词表示的全局向量
- `BERT` – 来自 Transformer 的双向编码器表示

## 优势

1.  词嵌入擅长把握单词的含义和上下文，从而提高文本分析和预测的准确性。
2.  与传统的词袋模型相比，它们提供了一种更有效且可扩展的词表示方法。
3.  词嵌入可以通过大量文本数据进行优化，使其能够辨别复杂的语言细微差别和联系。

## 局限性

1.  词嵌入的有效性可能在某些文本类型或语言中受到限制。
2.  它们可能无法完全捕捉所有含义和上下文的细微差别，因为它们依赖于数据中的统计模式，可能无法准确表示词之间真实的语义联系。
3.  词嵌入可能需要大量的计算资源和内存，尤其是在处理较大数据集时。

## 语义理解

机器通常难以理解单词、句子和文档中的内在含义。诸如词义消歧和含义识别等技术可以提高机器对语言的理解能力。

**语义分析的组成部分包括：**

-   词法分析，涉及扫描字符流、识别词素并将其转换为机器可读的标记。
-   语法分析，按顺序连接词素，应用形式语法进行词性标注。
-   句法分析，检查句法并执行语法规则，有助于在单词和句子层面进行上下文理解。
-   语义分析结合这些元素来辨别词义并解释句子结构，使机器能够像人类一样理解。

## 语义分析在 NLP 中的重要性

从语言数据中提取商业智能对许多组织来说至关重要但又充满挑战，尤其是在分析非结构化数据时。这一障碍常常阻碍专注于语言密集型任务的人工智能项目。

无论是推文、领英帖子还是网站评论，这些交流信息都可能包含关键信息。成功捕获并解读这些数据对于获得竞争优势至关重要。然而，捕获数据比理解其内容要简单得多，尤其是在大规模处理时。

在 NLP 中，基本技术可以识别单词，但语义提供了上下文含义（例如区分“苹果”是指水果还是公司）。

人类语言理解几乎是自动的，这得益于我们的神经网络。我们根据自身的语言知识和文本概念来解读文本。然而，机器无法自然地模仿这一过程。

某些技术给人一种理解文本的错觉，它们通常依赖于关键词匹配或基于频率的方法，这在计算语言学或统计 NLP 中很常见。这些方法存在局限性，因为它们并非真正寻求理解含义。

相比之下，语义分析是在文本分析中实现高准确性的关键。例如，在文本摘要（将长文本压缩成简洁摘要）中，其有效性取决于机器对语言的理解程度。

## 语义系统中的语义分析

语义系统整合了实体、概念、关系和谓词，丰富了语言上下文，使机器解读更准确。语义分析从语言中提取含义，是语义系统增强机器解读能力的基础。

**支持语言理解的语义分析关键要素包括：**

-   **下义关系：** 一个通用术语
-   **同形异义：** 拼写相同但含义不同的词汇项
-   **一词多义：** 拼写相同且含义相关的词汇项
-   **同义关系：** 拼写不同但含义相似的词汇项
-   **反义关系：** 含义相反的词汇项
-   **部分整体关系：** 一个术语与一个更广泛实体之间的关系

理解这些概念对于旨在提取见解和信息、支持自动化处理以及聊天机器人等对话系统的 NLP 应用至关重要。

# 使用 Python 进行情感分析与文本分类

情感分析，也称为意见挖掘，是一种自然语言处理（NLP）技术，用于确定文本中表达的情感、情绪基调或主观意见。NLP 中的一个关键应用是文本分类，它涉及对文本进行标记或分类。让我们来回顾一下每种技术。

### 情感分析

情感分析的主要目标是自动评估文本所传达的情感或情绪状态，判断其是正面、负面、中性，还是更细微的情感，如喜悦、愤怒、悲伤或惊讶。

**NLP 中情感分析的关键方面包括以下几点：**

- **情感分类：** 情感分析通常涉及将文本归入一个或多个情感类别。最常见的类别如下：
    - **正面：** 表示有利的情感，例如满意、快乐或赞同。
    - **负面：** 表示不利的情感，包括不满、失望或批评。
    - **中性：** 表示缺乏强烈情感或持有平衡观点。
    - **混合或中性：** 一些情感分析系统也提供混合或细微的情感类别，以捕捉更复杂的情绪。
- **文本处理：** 情感分析算法通过分词（将文本拆分为单词或短语）、移除停用词，并应用词干提取或词形还原等技术来规范化文本，从而对文本进行预处理。
- **特征提取：** 情感分析系统从文本中提取特征，这些特征可能包括单词、短语甚至句子结构，这些都能指示情感倾向。

这些技术中的每一种都在使计算机能够处理和理解人类语言方面发挥着关键作用，它们是更高级 NLP 应用的基础构建模块。

### 文本分类

在当今日益数字化的世界中，理解和处理海量数据至关重要。自然语言处理（NLP）作为计算机科学中一个充满活力且具有影响力的领域，正满足了这一需求。

例如，电子邮件可以被分类为垃圾邮件或非垃圾邮件，推文可以被分类为正面或负面，文章可以被分类为与特定主题相关或不相关。本指南旨在介绍文本分类的基础知识，为您开发自己的模型铺平道路。

### 文本分类的优势

**文本分类具有多种应用，在以下几个领域发挥着重要作用：**

- **文档分类：** 自动化文档排序过程，有助于高效的信息检索。
- **结果预测：** 预测未来事件，例如根据新闻文章预测股市反应。
- **模式发现：** 揭示隐藏趋势，例如在营销活动中识别客户偏好。

### 文本分类的变体

主要有两种类型：监督式和无监督式。监督式分类涉及在预先标记的数据上训练的模型，而无监督式分类不使用标签，而是从数据本身学习，根据相似性识别类别。监督式分类通常能获得更高的准确性，但需要更多资源。无监督式分类虽然精度较低，但在没有标记数据时非常有用。

### 文本分类的机制

文本分类涉及根据文本内容对其进行分类。此过程通常从定义一组类别或分类开始，然后识别指示这些类别的关键词或短语。随后，文本被手动或自动分配到这些类别中。最后一步涉及分析分类后的数据，分析范围可以从简单的计数到复杂的机器学习分析。

### 文本分类面临的挑战

由于文本数据的非结构化和噪声特性，文本分类面临着挑战。文本数据的高维性以及机器需要深度理解语言的要求增加了复杂性。为了更深入的理解，通常需要采用情感分析或主题建模等高级 NLP 技术。

### 文本分类的应用

文本分类服务于多种目的，例如检测垃圾邮件、分析情感、对新闻进行分类以及识别主题。作为一种监督式学习任务，它需要一个带标签的数据集来训练模型，以便准确地对文本进行分类。这使其成为各个领域中不可或缺的工具，能够简化流程并为大量文本数据提供洞察。

**新闻文本分类示例：**

```
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression

# Load the dataset
news = pd.read_csv('news.csv')

# Split the data into training and testing sets

# Create a TfidfVectorizer to convert text to numerical features

X_train = vectorizer.fit_transform(X_train)
X_test = vectorizer.transform(X_test)

# Train a logistic regression model
model = LogisticRegression()
model.fit(X_train, y_train)

# Evaluate the model on the test set
score = model.score(X_test, y_test)
print('Accuracy:', score)
```

### 探索自然语言处理（NLP）中的方法

让我们深入探讨自然语言处理领域中一些常用的方法。

#### 监督式 NLP

监督式 NLP 方法涉及使用一个输入和相应输出都已标记或已知的数据集来训练软件。最初，程序处理大量已建立的数据集，学习为任何未知输入生成正确的输出。例如，公司利用监督式 NLP 来训练工具，根据预定义的标签对文档进行分类。

#### 无监督式 NLP

无监督式 NLP 依赖统计语言模型，在给定未标记的输入时预测模式。文本消息中的自动补全功能就是一个例子，它通过观察用户的输入来建议句子中合适的词语。

#### 自然语言理解

自然语言理解（NLU）是 NLP 的一个子集，专注于解读隐藏在句子中的含义。NLU 使软件能够辨别不同句子中的相似含义，或理解具有多种解释的词语。

### 自然语言生成

自然语言生成（NLG）致力于以类似人类交流的方式，根据特定关键词或主题来构建对话文本。例如，一个具备 NLG 能力的智能聊天机器人，能够像人类客服人员一样与客户互动。

### 统计自然语言处理、机器学习与深度学习

最早的 NLP 应用是人工构建的、基于规则的系统，能够执行特定的 NLP 功能。然而，它们难以适应日益增长的例外情况以及不断扩大的文本和语音数据量。

这时，统计自然语言处理应运而生，它将计算机算法与机器学习和深度学习模型相结合。它能够自动提取、分类和标记文本及语音数据中的组成部分，并为这些组成部分的潜在含义分配统计概率。

在当代，基于卷积神经网络（`CNN`s）和循环神经网络（`RNN`s）的深度学习模型和学习技术，使 NLP 系统能够在运行过程中逐步“学习”，从大量未经处理、非结构化且未标记的文本和语音数据集中提取出越来越精确的含义。

### NLP 面临的挑战

尽管取得了进展，但由于人类语言的复杂性和微妙性，自然语言处理（NLP）仍面临着几个严峻的挑战。以下是其中一些突出的 NLP 挑战：

*   **歧义性：** 人类语言常常存在歧义，即一个词语有多种含义。这种复杂性给 NLP 模型在不同语境中准确判断词语含义带来了巨大障碍。
*   **语境：** 准确理解依赖于把握词语使用的语境。NLP 在有效解读语言使用背后的语境细微差别方面，一直面临挑战。
*   **精确性：** 传统计算要求人类使用一种精确、清晰且结构化的编程语言进行交流。与此形成鲜明对比的是，人类语言往往缺乏精确性，常因俚语、地域方言和社会语境等因素而充满歧义。
*   **语气与语调：** NLP 在语气、语调及语义分析方面仍有待完善。检测诸如讽刺等元素颇具挑战性，因为这需要理解词语选择和语境。此外，口语中微妙但重要的语气变化可能逃过 NLP 算法的识别。单词重音或音节强调对句子含义的影响，以及不同口音导致的语音语调变化，都给算法解析带来了更多复杂性。
*   **语言用法的演变：** 语言的动态特性，即其使用模式不断演变，给 NLP 带来了持续的挑战。尽管语言遵循既定规则，但这些规则并非一成不变，会随时间推移而改变。当前相关的计算规则，可能会随着现实世界语言的演变而过时。

### 区分 NLP 基础与 LLM 能力

区分 NLP 的基本原理与大语言模型（LLM）的卓越能力至关重要。传统 NLP 涵盖了多种技术，从依赖人工构建规则的基于规则的系统，到利用概率模型和机器学习算法从数据中学习模式的统计方法。这些传统 NLP 技术旨在针对特定的 NLP 任务（如命名实体识别、机器翻译或情感分析）进行设计和优化。

相比之下，LLM 是 NLP 模型的一个子集，其特点是采用深度学习架构，通常包含多层人工神经网络。这种深度学习方法使 LLM 能够从庞大的数据语料库中学习，并将其知识泛化到广泛的任务中。与传统 NLP 技术不同，LLM 无需为每个特定任务进行显式编程或特征工程。相反，它们可以直接从数据中学习复杂的关系和模式，因此具有非凡的多功能性。

然而，这种多功能性是有代价的。LLM 计算密集，需要大量资源进行训练和部署。此外，它们对数据的需求量很大，依赖海量文本数据才能达到最佳性能。这些因素在计算基础设施和数据可用性方面带来了挑战，在实际应用中使用 LLM 时必须仔细权衡。

理解传统 NLP 技术与 LLM 之间的区别，对于为特定任务选择最合适的方法至关重要。当任务定义明确且所需数据有限时，传统 NLP 技术可能更合适。另一方面，LLM 提供了无与伦比的多功能性，并有潜力处理更广泛的任务，尽管其计算需求和数据要求更高。

### 传统 NLP 技术与 LLM 的对比

传统 NLP 技术与 LLM 之间的根本区别在于它们各自的方法和能力范围。传统 NLP 方法通常专为特定任务而设计，例如机器翻译、情感分析或命名实体识别。这些方法依赖于为每个特定任务精心设计的特征和算法。虽然这种方法对于定义明确的任务可能非常有效，但通常需要大量人工来设计必要的特征和算法。

相比之下，LLM 采用了一种更广泛、数据驱动的 NLP 方法。LLM 并非针对特定任务进行显式编程，而是在海量文本数据上进行训练，并学习识别语言中的模式和关系。这使得它们能够执行广泛的任务，甚至包括那些未经明确训练的任务。例如，LLM 可用于文本生成、问答、摘要，甚至创意写作。

然而，这种多功能性是有代价的。LLM 需要的计算资源和数据远多于传统 NLP 方法。训练 LLM 通常需要使用大规模数据集和强大的硬件，例如图形处理单元（GPU）。此外，LLM 可能难以控制，有时会生成无意义或有偏见的输出。

尽管存在这些挑战，LLM 仍代表了 NLP 领域的重大突破，并在许多应用中迅速推进了技术发展水平。随着这些模型的不断发展和改进，预计它们将在我们的生活中扮演越来越重要的角色。

虽然 NLP 的基础为处理语言任务提供了必要的核心技术和基础知识，但 LLM 预示着一次突破性的飞跃。LLM 在广泛的任务中提供了无与伦比的通用性和卓越的性能。然而，重要的是要承认 LLM 有其自身独特的要求和挑战。
