# 致谢

**来自 Luc：**

永不言败；这是我学到的教训。大约四年前，当我完成我的第一本书时，我确信我再也不会写书了。当 Oracle 发布他们的数字助手时，不知何故，这种情况发生了变化。我立即并且仍然相信这个产品的力量以及它为我们的客户带来的附加价值。

我分享知识的渴望和对新技术的热情占据了上风，让我改变了主意，并诱使我再次写作。

感谢 Lonneke Dikmans，她对这个项目表示了支持，并建议我组建一个合著者团队来实现这一目标。没有 Mascha 和 Ardhendu 的帮助，我永远无法做到这一点，他们为这个项目投入了大量的精力、热情和幽默。感谢你们两位！没有你们，这本书永远不会被写出来。

特别感谢 Oracle 的团队，他们不仅帮助进行了技术审阅，还在某些时候提供了早期访问权限和技术挑战方面的指导。感谢 Frank、Grant、Martin、Rohit 以及所有在某个时候提供过帮助的人，即使你们自己不知道。

最后，拥抱和亲吻我的妻子 Judith 以及孩子 Olivier 和 Lisanne，他们在写作过程中再次不得不与我一起生活。这一定很困难，但你们从未停止支持我。谢谢！

**来自 Ardhendu：**

我是在做梦还是真的发生了？！今天很难意识到自己成为了一名作者。对您说“谢谢”太微不足道了，**Luc**。没有您的指导和引领，我不可能做到。还有 **Mascha**，总是鼓励我并指导我。我要感谢 **Lonneke Dikmans** 和 **Ronald van Luttikhuizen**，感谢他们在本书写作过程中给予的持续鼓励、支持和指导。

最重要的是，我要感谢我的父母，没有他们，我不可能取得今天的成就。最后，我要感谢我亲爱的妻子和我的小公主；你们一直是我灵感的源泉。

特别感谢我们的出版商为我们提供了这个平台。

**来自 Mascha：**

从没想过我真的会这样做，与我的两位优秀的 eProseed 同事 Luc 和 Ardhendu 一起成为合著者并出版一本书。没有他们，我无法实现这一目标。作为首次作者，他们一路为我提供了最好的指导、反馈和建议。

谢谢你，Luc，邀请我合作实现这一目标；没有你，我永远不会有这个机会来撰写关于编码数字助手另一面的内容，即用户方面。作为一名用户体验设计师，我总是把用户放在首位。我们为**客户** **销售**和**开发**数字助手，但我们为他们的**用户** **创造**它们。

我还要感谢 Lonneke Dikmans 和 Ronald van Luttikhuizen 鼓励和支持我，并多次表达他们对这本书的热情。

我感谢 Hans Kemp 让我能够教授年轻的用户体验专业人员关于对话式用户体验设计方法，这使我能够完善我的章节。

我无比感激我亲爱的丈夫，他告诉任何想听的人，他的妻子与她的两位同事合著了一本书——她的处女作——这本书已经真正出版并因此可以出售 ☺。

最后，我要感谢我亲爱的父母，他们为我感到热情和自豪，相信我，并总是用“如果有人能写作，那就是你”这句话来安慰我。
