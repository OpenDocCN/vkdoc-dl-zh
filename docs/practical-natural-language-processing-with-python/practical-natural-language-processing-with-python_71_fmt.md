# 第 5 章 虚拟助手中的自然语言处理

**机器人**：您好，我能为您做些什么？

**客户**：我想预订一个座位。

**机器人**：好的，您想找哪个位置？

**客户**：我在东大街，所以想找附近的地方。

**机器人**：您喜欢哪种菜系？

**客户**：高级印度餐厅。

**机器人**：太好了，我知道今晚 30 点可以在`Punjabi Grill`为您订位。

**客户**：完美。

**机器人**：祝您愉快，很高兴为您服务。

**图 5-29.** 多轮对话

在图 5-29 的示例中，如果用户说“想在东大街订一张两人桌”，那么机器人应该足够“聪明”，跳过位置问题，直接询问菜系问题。如果所有信息都已提供，它应该直接进行预订。这是通过为意图中的不同实体保留槽位并在对话过程中更新槽位来实现的。如果槽位已填，则跳过该问题。

有时在对话中，用户可能不回答问题，而是改变意图，或者返回并纠正之前问题的答案。图 5-30 给出了一个示例。

**机器人**：您好，我能为您做些什么？

**客户**：我想预订一个座位。

**机器人**：好的，您想找哪个位置？

**客户**：我在东大街，所以想找附近的地方。

**机器人**：您喜欢哪种菜系？

**客户**：我想更改位置。

**机器人**：我看到您更改了位置。

**客户**：是的。

**机器人**：您更喜欢哪个位置？

**客户**：我改变计划了，稍后会打电话。

**机器人**：谢谢，祝您愉快。

**图 5-30.** 意图切换

在“加粗”的行中，用户改变了他们的意图或提供了无效的回复。机器人工作流程应该足够智能，以更改这些回复。这也可以通过编码器-解码器架构来实现，在该架构中，您可以训练一个大型的成对回复语料库。

## 多语言机器人

对话代理的一个理想领域是多语言机器人。鉴于英语人口仅占世界人口的 20%，机器人需要适应多种语言。这带来了挑战，因为多语言 NLP 的研究工作仍然非常有限。然而，解决方言机器人问题是自然语言机器人在全球取得成功的关键。

人机交互一直是一个长期的需求。我们在最近看到了一些显著的成就。我们不再对会说话或回答问题的机器感到着迷。很快，我们甚至不会意识到我们正在与机器交谈，并将其视为日常生活中的任何其他互动。
