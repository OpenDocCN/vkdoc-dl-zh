## 构建对话机器人的更多细节

### 单轮对话 vs. 多轮对话

图 5-28 展示了单轮对话的示例。单轮对话机器人在许多组织中得到了成功应用。如您所见，这是一个问答对。如果您能正确理解客户的意图，就能给出正确的回复。

**客户**
我的余额是多少

**机器人**
截至 5 月 3 日，您的账户余额为 XXX 美元。

**客户**
我在哪里可以找到最近的自动取款机。

**机器人**
根据您当前的位置，最近的自动取款机列表在此链接中。

**图 5-28.** 单轮对话

然而，在多轮对话中，存在多个问答对。机器人需要理解用户的“意图”以及一些更细粒度的实体级元数据。例如，对于餐厅预订——机器人需要理解这是一个与预订相关的问题（意图），同时还要理解地点、菜系、人数、用户的时间偏好（实体）。在机器人能够继续解决问题之前，需要收集这些信息。用户有时可能会直接陈述问题（“我想订个桌”），或者提供部分信息（“我想在 XYZ 餐厅订一个 4 人的桌”）。机器人可能需要理解一组实体值，而用户可能只陈述了其中的一部分。然后，机器人识别缺失的信息并进行收集。参见图 5-29。
