# 6. 度量分析

让我们考虑我们在第三章中分析的问题，我们对 Zalando 数据集进行了分类。在我们进行所有工作的过程中，我们做了一个强烈的假设，但没有明确说出来：我们假设所有的观测值都被正确标记。我们无法确定这一点。为了进行标记，需要一些人工干预，因此，一定数量的图像肯定被错误分类，因为人类并不完美。这是一个重要的启示。考虑以下场景：在第三章中，我们使用我们的模型实现了大约 90%的准确率。有人可能会尝试不断提高准确率，但何时停止尝试才是合理的？如果你的标签在 10%的情况下是错误的，那么无论你的模型多么复杂，它都无法以非常高的准确率泛化到新的数据，因为它已经为许多图像学习了错误的类别。我们花了很多时间检查和准备训练数据，例如进行归一化，但我们从未花时间检查标签本身。我们还假设所有类别都有相似的特征。（我将在本章后面讨论这究竟意味着什么，目前，对这一概念有一个直观的理解就足够了。）如果某些类别的图像质量比其他类别差怎么办？如果不同类别的像素灰度值与零不同的数量有显著差异怎么办？我们也没有检查是否有某些图像是完全空白的。在这种情况下会发生什么？正如你可以想象的那样，我们无法手动检查所有图像，试图检测这些问题。假设我们有数百万张图像，手动分析肯定是不可能的。

我们需要在我们武器库中添加一种新武器，以便能够发现这样的案例，并能够了解模型的表现。这个新武器是本章的重点，我称之为“度量分析”。在领域内，人们经常将这一系列方法称为“错误分析”。我发现这个名字非常令人困惑，尤其是对于初学者来说。*错误*可能指很多事物：Python 代码错误、方法、算法中的错误、优化器选择中的错误等等。你将在本章中看到如何获得关于你的模型表现和你的数据质量的基本信息。我们将通过评估你在数据中可以推导出的不同数据集上的优化度量来实现这一点。

你之前已经看到了一个基本示例。你应该记得，我们讨论了回归问题，在 *MSE*[*train*] ≪ *MSE*[*dev*] 的情况下，我们处于过拟合的状态。我们的指标是均方误差（MSE），在训练和 dev 两个数据集上评估它，并比较这两个值，可以告诉你模型是否过拟合。我将在本章中扩展这一方法，以便你能从你的数据和模型中提取更多信息。

## 人类水平的表现与贝叶斯误差

在我们用于监督学习的大多数数据集中，必须有人对观测结果进行标记。以我们拥有已分类图像的数据集为例。如果我们要求人们对所有图像进行分类（想象这是可能的，无论图像数量多少），获得的准确率永远不会是 100%。有些图像可能太模糊而无法正确分类，人们会犯错误。例如，如果 5%的图像由于模糊度无法正确分类，那么我们必须预期人们可以达到的最大准确率将始终低于 95%。

让我们考虑一个分类问题。首先，让我们定义一下我们所说的“错误”一词。在本章中，“错误”一词将用来表示以下数量，用 *ϵ* 表示：

![ε = 1 - 准确率](img/463356_1_En_6_Chapter_TeX_Equa.png)

例如，如果我们使用一个模型达到了 95%的准确率，我们将有 *ϵ* = 1 - 0.95 = 0.05，或者用百分比表示，*ϵ* = 5%。

一个有用的概念是理解人类水平的表现，它可以定义为以下内容：

*人类水平的表现（定义 1）*：一个人在执行分类任务时所能达到的最小错误 *ϵ* 值。我们将用 *ϵ*[*hlp*] 来表示它。

让我们设计一个具体的例子。假设我们有一组 100 张图片。现在假设我们要求三个人对这 100 张图片进行分类。想象一下，他们获得了 95%，93%和 94%的准确率。在这种情况下，人类水平的表现准确率将是 *ϵ*[*hlp*] = 5%。请注意，其他人可能在这个任务上做得更好，因此，始终考虑我们得到的 *ϵ*[*hlp*] 的值始终是一个估计值，并且只能作为指导。

现在我们让事情变得复杂一些。假设我们正在处理一个问题，医生需要将 MRI 扫描分为两类：有癌症迹象和无癌症迹象。现在假设我们计算未受过训练的学生获得 15%，有几年经验的医生获得 8%，有经验的医生获得 2%，以及有经验的医生*群体*获得 0.5%的 *ϵ*[*hlp*]。那么 *ϵ*[*hlp*] 是什么？你应该始终选择你能得到的最低值，原因我将在后面讨论。

我们现在可以扩展 *ϵ*[*hlp*] 的定义，增加第二个定义。

*人类水平的表现（定义 2）*：人们或*群体*在执行分类任务时所能达到的最小错误值 *ϵ*。

### 注意

你不必决定哪个定义是正确的。只需使用给你提供最低 *ϵ*[*hlp*] 值的那个定义。

现在，我将简要谈谈为什么我们必须选择我们能得到的最低 *ϵ*[*hlp*] 值。假设在 100 张图片中，有 9 张太模糊而无法正确分类。这意味着任何分类器能达到的最低错误率是 9%。任何分类器能达到的*最低错误率*被称为贝叶斯错误。我们将用 *ϵ*[*Bayes*] 来表示这一点。在这个例子中，*ϵ*[*Bayes*] = 9%。通常，*ϵ*[*hlp*] 非常接近 *ϵ*[*Bayes*]，至少在人类擅长完成任务，如图像识别的任务中是这样。人们常说，人类水平的表现错误是贝叶斯错误的代理。通常，知道 *ϵ*[*Bayes*] 是不可能的或非常困难的，因此从业者使用 *ϵ*[*hlp*]，假设两者很接近，因为后者更容易（相对）估计。

请记住，只有在个人（或一组个人）以与分类器相同的方式进行分类时，比较这两个值并假设 *ϵ*[*hlp*] 是 *ϵ*[*Bayes*] 的代理才有意义。例如，如果两者都使用相同的图像进行分类，这是可以的。但在我们的癌症例子中，如果医生使用额外的扫描和分析来诊断癌症，比较就不再公平了，因为人类水平的表现将不再是贝叶斯错误的代理。医生拥有更多的数据，显然会比只有图像输入的模型更准确。

### 注意

只有在人类和模型以相同的方式进行分类的情况下，*ϵ*[*hlp*] 和 *ϵ*[*Bayes*] 才会接近。所以，在假设人类水平的表现是贝叶斯错误的代理之前，总是要检查这一点。

在处理模型时，你还会注意到，只需付出相对较小的努力，你就可以达到相当低的错误率，并且通常（几乎）达到 *ϵ*[*hlp*]。在超过人类水平的表现（在几个案例中，这是可能的）之后，进步往往变得非常非常缓慢，如图 6-1 所示。

![img/463356_1_En_6_Chapter/463356_1_En_6_Fig1_HTML.jpg](img/463356_1_En_6_Fig1_HTML.jpg)

图 6-1

可达到的精度典型值与投入时间的对比。一开始，通过机器学习很容易就达到相当高的精度，通常能达到 ϵ[hlp]。这一点在图中通过线条直观地表示出来。从那以后，进步往往变得非常缓慢。

只要你的算法错误率大于 *ϵ*[*hlp*]，你就可以使用以下技术来获得更好的结果：

+   从人类或群体中获得更好的标签，例如，在我们的例子中，从医生群体中获取医疗数据。

+   从人类或群体中获得更多标记数据。

+   进行良好的度量分析，以确定获得更好结果的最佳策略。你将在本章中学习如何做到这一点。

一旦你的算法性能超过人类水平，你就不能再依赖那些技术了。因此，了解这些数字，决定如何获得更好的结果是很重要的。以我们的 MRI 扫描为例，我们可以通过依赖与人类无关的来源来获得更好的标签，例如，在 MRI 日期几年后检查诊断，这时通常可以清楚地判断患者是否已经发展出癌症。或者，例如，在图像分类的情况下，你可能会决定自己从特定类别中获取几千张图像。这通常是不可能的，但我想要清楚地表达这个概念：你可以通过其他方式获得标签，而不是要求人类执行与你的算法相同类型的任务。

### 注意

对于人类在图像识别等任务中表现优异的任务，人类水平性能是贝叶斯误差的良好代理。对于人类非常糟糕的任务，性能可能远远低于贝叶斯误差。

## 关于人类水平性能的短篇故事

我想给你讲一个关于安德烈·卡帕西（Andrej Karpathy）在尝试估计特定案例中的人类水平性能时所做的故事。你可以在他的博客文章（一篇很长的文章，但我建议你读一读）中阅读整个故事，网址是[`goo.gl/iqCbC0`](https://goo.gl/iqCbC0)。让我总结一下他做了什么，因为这非常有助于理解人类水平性能的真正含义。卡帕西在 2014 年参与了 ILSVRC（ImageNet 大规模视觉识别挑战赛）（[`goo.gl/PCHWMJ`](https://goo.gl/PCHWMJ)）。任务由 120 万张图像（训练集）组成，分为 1000 个类别，包括动物、抽象物体如螺旋、场景等。结果在 dev 数据集上进行了评估。GoogleLeNet（由谷歌开发的一个模型）达到了惊人的 6.7%的错误率。卡帕西想知道人类的表现会如何。

这个问题比乍看之下要复杂得多。因为所有的图像都是由人类分类的，所以*ϵ*[*hlp*] = 0%不是吗？实际上，并非如此。事实上，这些图像最初是通过网络搜索获得的，然后通过询问人们*二元*问题进行过滤和标记，例如，这是钩子吗？图像是按照 Karpathy 在他的博客文章中提到的二进制方式收集的。人们没有被要求为每张图像分配一个类别，从 1000 个类别中选择，因为算法是这样做的。你可能认为这只是个技术细节，但标签方式的差异使得正确评估*ϵ*[*hlp*]变得相当复杂。因此，Karpathy 开始着手开发一个由左边的图像和右边的 1000 个类别及其示例组成的 Web 界面。你可以在图 6-2 中看到一个界面的例子。你可以尝试这个界面（我建议你这样做），以了解这样一个任务有多么复杂。尝试界面的用户经常错过类别并犯错误。达到的最佳错误率约为 15%。因此，Karpathy 做了每个科学家在其职业生涯的某个时刻都必须做的事情：他让自己无聊至极，并亲自进行了仔细的标注，有时单张图像就需要 20 分钟。正如他在博客文章中所说，他这样做只是为了科学。他能够达到惊人的*ϵ*[*hlp*] = 5.1%，比当时最好的算法高出 1.7%。他列出了 GoogLeNet 比人类更容易犯错的错误来源，例如图像中多个对象的问题，以及人类比 GoogLeNet 更容易犯错的错误来源，例如具有巨大粒度的类别问题（例如，狗被分为 120 个不同的子类别）。

## 偏差

现在我们从指标分析开始：一系列程序，通过评估你的优化指标在不同数据集上的表现，为你提供有关你的模型表现以及数据好坏的信息。

### 注意

指标分析包括一系列程序，通过查看你在不同数据集上评估的优化指标，为你提供有关你的模型表现以及数据好坏的信息。

首先，我们必须定义第三个错误：在训练数据集上评估的错误，用*ϵ*[*train*]表示。

我们想要回答的第一个问题是，如果我们的模型不够灵活或复杂，无法达到人类水平的表现。或者换句话说，我们想知道我们的模型相对于人类水平的表现是否有很高的偏差。

为了回答前面的问题，我们可以做以下操作：从我们的训练数据集*ϵ*[*train*]中计算错误，然后计算|*ϵ*[*train*] − *ϵ*[*hlp*]|。如果这个数字不是小的（大于几个百分比），那么我们面临的是偏差（有时称为可避免的偏差），也就是说，我们的模型太简单，无法捕捉到我们数据的真实细微差别。

让我们定义以下量

![ΔεBias=|εtrain-εhlp|](img/463356_1_En_6_Chapter_TeX_Equb.png)

Δ*ϵ*[*Bias*] 越大，我们的模型偏差就越大。在这种情况下，你希望训练集上的表现更好，因为你知道你可以从训练数据上做得更好。（我们稍后会讨论过拟合的问题。）以下技术可以用来减少偏差：

+   更大的网络（更多的层或神经元）

+   更复杂的架构（例如卷积神经网络）

+   增加训练模型的时间（更多的 epoch）

+   使用更好的优化器（如 Adam）

+   进行更好的超参数搜索（在第七章节中介绍）

你还需要了解其他一些事情。知道 *ϵ*[*hlp*] 和降低偏差以达到它是两件非常不同的事情。假设你知道你问题的 *ϵ*[*hlp*]。这并不意味着你必须达到它。可能的情况是，你正在使用错误的架构，但你可能没有开发更复杂网络所需的技能。甚至可能达到所需错误水平所需的努力会非常巨大（从硬件或基础设施的角度来看）。始终牢记你的问题需求。始终尝试理解什么足够好。对于一个用于识别癌症的应用，你可能希望尽可能多地投资以实现尽可能高的准确率：你不希望让某人回家几个月后才发现癌症的存在。另一方面，如果你构建一个从网络图像中识别猫的系统，你可能发现比 *ϵ*[*hlp*] 更高的错误率是完全可接受的。

## 训练集过拟合

在前几章中，我们详细讨论了另一个问题，即训练数据的过拟合。你将记得在第五章中，当我们执行回归时，我们看到了过拟合的一个极端案例，其中*MSE*[*train*] ≪ *MSE*[*dev*]。在分类问题中也是如此。让我们用*ϵ*[*train*]表示我们的模型在训练数据集上的错误，用*ϵ*[*dev*]表示在 dev 数据集上的错误。然后我们可以说，如果我们训练集的*ϵ*[*train*] ≪ *ϵ*[*dev*]，那么我们就是在过拟合训练集。让我们定义一个新的量

![$$ \Delta {\epsilon}_{overfitting\ train}=\left|{\epsilon}_{train}-{\epsilon}_{dev}\right| $$](img/463356_1_En_6_Chapter_TeX_Equc.png)

使用这个量，我们可以说，如果我们训练数据集的Δ*ϵ*[*overfitting train*]超过几个百分点，那么我们就是在过拟合训练数据集。

让我们总结一下到目前为止我们已经定义和讨论的内容。我们有三处错误：

+   *ϵ*[*train*]：我们的分类器在训练数据集上的错误

+   *ϵ*[*hlp*]：人类水平的表现（如前几节所述）

+   *ϵ*[*dev*]：我们的分类器在 dev 数据集上的错误

使用这三个量，我们已经定义了

+   Δ*ϵ*[*Bias*] = |*ϵ*[*train*] − *ϵ*[*hlp*]|：衡量训练数据集与人类水平表现之间的“偏差”程度

+   *Δϵ*[*overfitting train*] = |*ϵ*[*train*] − *ϵ*[*dev*]|：衡量训练数据集过拟合的程度

此外，到目前为止，我们已经使用了两个数据集

+   *训练数据集*：我们用来训练模型的那个数据集（你现在应该知道了）

+   *Dev 数据集*：我们用来检查训练数据集过拟合的第二个数据集

现在假设我们的模型有偏差并且略微过拟合了训练数据集，这意味着我们有一个Δ*ϵ*[*Bias*] = 6%和*Δϵ*[*overfitting train*] = 4%。现在我们的 MAD 变成了图 6-5 中所示的内容。

![img/463356_1_En_6_Chapter/463356_1_En_6_Fig5_HTML.jpg](img/463356_1_En_6_Fig5_HTML.jpg)

图 6-5

我们两个问题的 MAD 图：训练数据集的偏差和过拟合

如同你在图 6-5 中看到的，你可以快速了解我们面临问题的相对严重性，并决定你想先解决哪一个。

通常当你对训练数据集进行过拟合时，这通常被称为方差问题。当这种情况发生时，你可以尝试以下技术来最小化这个问题：

+   为你的训练集获取更多数据

+   使用正则化（查看第五章，对该主题的完整讨论）

+   尝试数据增强（例如，如果你正在处理图像，你可以尝试旋转、平移等）

## 如何分割您的数据集

现在，我想简要讨论如何在一般和深度学习环境中分割您的数据。

但“分割”究竟是什么意思呢？嗯，如前所述，您需要一个观察结果集来使模型学习，您称之为训练集。您还需要一个观察结果集，它将构成您的开发集，以及一个称为测试集的最终集。通常，您会看到 60% 的观察结果用于训练集，20% 的观察结果用于开发集，以及 20% 的观察结果用于测试集。通常，这些类型的分割以以下形式表示：60/20/20，其中第一个数字（60）指的是构成训练集的整个数据集的百分比，第二个（20）指的是构成开发集的整个数据集的百分比，最后一个（20）指的是构成测试集的百分比。在书籍、博客或文章中，您可能会遇到“我们将分割数据集 80/10/10”之类的句子。您现在有了对这个意思的解释。

通常，在深度学习领域，您将处理大型数据集。例如，如果我们有 *m* = 10⁶，我们可以使用 98/1/1 这样的分割。请记住，10⁶ 的 1% 是 10⁴——这是一个很大的数字！记住，开发/测试集必须足够大，以便对模型的性能有很高的信心，但不是不必要的巨大。此外，您希望尽可能多地保存观察结果用于训练集。

### 注意

当决定如何分割您的数据集时，如果您有大量的观察结果（例如，10⁶ 或更多），您可以分割数据集为 98/1/1 或 90/5/5。因此，一旦您的开发集和测试集达到合理的大小（取决于您的问题），您就可以停止。在决定如何分割数据集时，请记住您的开发/测试集必须有多大。

现在请记住，如您所知，大小并不是一切。您的开发集和测试集应该代表您的训练数据集和问题。让我们创建一个例子。让我们考虑前面提到的 ImageNet 挑战。在那里，您想要将图像分类到 1000 个不同的类别中。为了知道您的模型在开发集和测试集中的表现，您需要为每个类别在每个集中有足够的图像。如果您决定只为开发集或测试集选择 1000 个观测值，您将无法得到任何合理的结论，因为如果所有类别都在开发集中得到代表，您每个类别将只有一个观测值。您应该决定为每个类别至少选择 100 个图像来构建您的开发集和测试集，构建两个数据集（开发集和测试集），每个数据集总共包含 10⁵个观测值（记住我们有 1000 个类别）。在这种情况下，低于这个数字是不合理的。这不仅与深度学习环境相关，也与机器学习的一般情况相关。您应该始终尝试构建一个反映与训练集中观测值相同分布的开发/测试集。为了理解我的意思，以 MNIST 数据集为例。让我们用以下代码加载数据集（就像我们之前做的那样）：

例如，如果你有一个包含三个类别的数据集，每个类别有 1000 个观察值，那么这个数据集具有完美的类别分布，但如果你在类别 1 中只有 100 个观察值，在类别 2 中有 10,000 个观察值，在类别 3 中有 5000 个观察值，那么我们谈论的就是不平衡类别分布。你不应该认为这是一种罕见的情况。假设你必须构建一个识别欺诈信用卡交易模型的模型。可以安全地假设这些交易在整个可用的交易量中只占一个非常小的百分比。

### 注意

在分割你的数据集时，你必须非常注意，不仅要关注每个数据集中观察值的数量，还要注意哪些观察值进入每个数据集。请注意，这个问题不仅与深度学习相关，而且在机器学习中也普遍重要。

要详细讨论如何处理不平衡数据集将超出本书的范围，但了解它们可能产生的后果非常重要。在下一节中，我将向你展示如果你向神经网络馈送不平衡数据集会发生什么，以便你获得具体理解的可能性。在本节的最后，我将提供一些在这种情况下应该做什么的建议。

## 不平衡类别分布：可能发生的情况

因为我们要讨论如何分割我们的数据集以执行度量分析，所以掌握不平衡类别分布的概念以及如何处理它非常重要。在深度学习中，你会发现你经常需要分割数据集，如果你以错误的方式这样做，你应该意识到可能会遇到的问题。让我给你一个具体的例子，说明如果你错误地这样做，事情可能会变得多么糟糕。

我们将使用 MNIST 数据集，我们将使用单神经元进行基本的逻辑回归（就像我们在第二章节中所做的那样）。让我们快速再次看看如何加载数据和准备数据。我们将以类似的方式做，除了我会指出的某些修改。首先，我们加载数据


如果你不懂代码，请回顾第二章节以获取更多细节。我期望你现在已经很好地理解了这个简单的模型，因为我们已经多次看到它了。接下来，我们定义运行模型的功能（你已经在前几章中见过它好几次了）。

```py
def run_logistic_model(learning_r, training_epochs, train_obs, train_labels, debug = False):
sess = tf.Session()
sess.run(init)
cost_history = np.empty(shape=[0], dtype = float)
for epoch in range(training_epochs+1):
sess.run(training_step, feed_dict = {X: train_obs, Y: train_labels, learning_rate: learning_r})
cost_ = sess.run(cost, feed_dict={ X:train_obs, Y: train_labels, learning_rate: learning_r})
cost_history = np.append(cost_history, cost_)
if (epoch % 10 == 0) & debug:
print("Reached epoch",epoch,"cost J =", str.format('{0:.6f}', cost_))
return sess, cost_history
```

### 注意

在复杂问题中分割你的数据集时，你必须特别注意你数据集中观察值的数量，以及你选择的观察值和类别的分布。

在结束本节之前，让我给你一些建议，关于如何处理不平衡的数据集。

+   *改变你的度量标准*：在上一个例子中，你可能想用其他东西代替准确度，因为那可能会误导。你可以尝试使用混淆矩阵，例如，或者其他度量，如精确度、召回率或 F1。检查你的模型表现的一个重要方法，并且我强烈建议你学习的方法是 ROC 曲线，它将极大地帮助你。

+   *与欠采样数据集一起工作*。例如，如果你有 1000 个属于类别 1 的观察值和 100 个属于类别 2 的观察值，你可能会创建一个新的数据集，其中包含类别 1 中的 100 个随机观察值和类别 2 中的 100 个观察值。然而，这种方法的问题是你通常会有更少的数据来训练你的模型。

+   *与过采样数据集一起工作*。你可以尝试做相反的事情。你可以将上面提到的类别 2 中的 100 个观察值简单地复制 10 次，最终在类别 2 中得到 1000 个观察值（有时称为重采样）。

+   *尝试在观察较少的类别中获取更多数据*：这并不总是可能的。在欺诈信用卡交易的情况下，你不能四处生成新数据，除非你想去坐牢…

## 精确度、召回率和 F1 度量

让我们看看一些其他在处理不平衡数据集时非常有用的指标。考虑以下例子。假设我们正在进行一些测试以确定受试者是否患有某种疾病。想象一下，我们有 250 个测试结果。考虑以下混淆矩阵（你应该从我们之前的讨论中知道这是什么）：

![预测: NO & 预测: YES](img/463356_1_En_6_Chapter_TeX_Eque.png)

我们将用 *N* 表示测试结果的总数，在这种情况下 *N* = 250。我们将使用以下术语：

+   *真阳性 (tp)*：预测为是的测试，且受试者确实患有疾病

+   *真阴性 (tn)*：预测为否的测试，且受试者没有疾病

+   *假阳性 (fp)*：预测为是，但受试者没有疾病

+   *假阴性 (fn)*：预测为否，但受试者确实患有疾病

这在视觉上可以表示为以下内容：

![预测: NO & 预测: YES](img/463356_1_En_6_Chapter_TeX_Equf.png)

让我们也用 *ty* 表示真正患有疾病的患者数量，在这个例子中，*ty* = 10 + 150 = 160，以及用 *tno* 表示没有疾病的患者数量，在这个例子中，*tno* = 75 + 15 = 90。在例子中，我们会得到

![tp=150](img/463356_1_En_6_Chapter_TeX_Equg.png)

![fp=15](img/463356_1_En_6_Chapter_TeX_Equi.png)

![fn=10](img/463356_1_En_6_Chapter_TeX_Equj.png)

我们可以将几个指标表示为之前讨论的术语的函数。例如：

+   **准确度**: (*tp* + *tn*)/*N*，我们的测试正确的频率

+   **误分类率**: (*fp* + *fn*)/*N*，我们的测试出错频率。注意，这等于 1 − *accuracy*。

+   **灵敏度/召回率**: *tp*/*ty*，当受试者患有疾病时，测试真正预测为是的频率

+   **特异性**: *tn*/*tno*，当受试者没有疾病时，我们的测试预测为否的频率

+   **精确度**: *tp*/(*tp* + *fp*)，相对于所有获得的阳性结果，预测受试者患有疾病的测试正确比例

所有这些数量都可以用作指标，具体取决于你的问题。让我们创建一个例子。假设你的测试应该预测一个人是否患有癌症。在这种情况下，你想要的是尽可能高的灵敏度，因为检测疾病很重要。但与此同时，你也希望特异性要高，因为没有比在需要治疗时让人回家不治疗更糟糕的事情了。

让我们更仔细地看看精确度和召回率。高精确度意味着当你说你某人是患病时，你是正确的。但你不知道有多少人真的患病，因为这个数量仅由你的测试结果定义。精确度是衡量你的测试表现的一个指标。高召回率意味着你能够识别出样本中的所有患病者。让我给你举一个例子，以便使这个观点更加清晰。假设我们有 1000 人。只有 10 人患病，990 人健康。假设我们想要识别健康的人（这很重要），我们构建了一个测试，如果某人健康就返回“是”，并且**总是**预测人们是健康的。混淆矩阵看起来是这样的：

## 不同分布的数据集

现在，我想讨论另一个术语问题，这将帮助你理解深度学习领域的一个常见问题。你经常会听到这样的句子：“这些集合来自不同的分布。”这句话并不总是容易理解。以两个由专业 DSLR 拍摄的图片组成的数据集为例，另一个由质量可疑的手机拍摄的图片组成。在深度学习领域，我们会将这两个集合描述为来自不同的分布。但这句话的真正含义是什么？这两个数据集因各种原因而不同：图像的分辨率、不同质量镜头导致的模糊度、颜色的数量、焦点的质量，可能还有更多。所有这些差异通常被理解为*分布*。让我们看看另一个例子。我们可以考虑两个数据集：一个是白猫的图片，另一个是黑猫的图片。在这种情况下，我们也是在谈论不同的分布。当你在一个集合上训练模型并希望将其应用于另一个集合时，这会成为一个问题。例如，如果你在一个白猫的图片集合上训练模型，你可能在黑猫的数据集上表现不会很好，因为你的模型在训练过程中从未见过黑猫。

### 注意

当谈论来自不同分布的数据集时，通常意味着两个数据集中的观测值具有不同的特征：黑白猫、高分辨率和低分辨率图像、用意大利语和德语录制的语音等等。

因为数据非常宝贵，人们常常试图从不同的来源创建不同的数据集（训练集、开发集等）。例如，你可能会决定在一个由网络上的图片组成的集合上训练你的模型，并用你用智能手机拍摄的图片组成的集合来检查它的效果。尽可能多地使用数据看起来是个好主意，但这可能会给你带来很多麻烦。让我们看看在真实案例中会发生什么，这样你可能会对做类似事情的结果有所体会。

当在数据集上训练模型时，通常你会得到与训练集中类似观测值的好结果。但你怎么能找出你是否存在这样的问题呢？有一个相对简单的方法来做这件事：扩展我们的 MAD 图。让我们看看如何做。

假设你有一个训练数据集和一个开发数据集，其中观测值具有不同的特征（来自不同的分布）。你所做的是从训练集中创建一个小子集，称为 train-dev 数据集，最终得到三个数据集：一个来自同一分布的训练集和 train-dev 集（观测值具有相同的特征）以及一个开发集，其观测值在某种程度上不同，正如我之前所讨论的。你现在所做的是在你的训练集上训练你的模型，然后在这三个数据集上评估你的误差 *ϵ*：*ϵ*[*train*]，*ϵ*[*dev*]，和*ϵ*[*train* − *dev*]。如果你的训练集和开发集来自同一分布，那么 train-dev 集也是如此。在这种情况下，你应该期望 *ϵ*[*dev*] ≈ *ϵ*[*train* − *dev*]。如果我们定义

![$$ \Delta {\epsilon}_{train- dev}={\epsilon}_{train- dev} $$](img/463356_1_En_6_Chapter_TeX_Equz.png)

我们应该期望 *Δϵ*[*train* − *dev*] ≈ 0。如果训练集（和 train-dev 集）以及开发集来自不同的分布（观测值具有不同的特征），我们应该期望 Δ*ϵ*[*trian* − *dev*] 很大。如果我们考虑我们之前创建的 MNIST 示例，我们实际上有 Δ*ϵ*[*train* − *dev*] = 0.437，或 43.7%，这是一个巨大的差异。让我们回顾一下你应该如何确定你的训练集和你的开发集（或测试集）的观测值具有不同的特征（来自不同的观测值）。

1.  将你的训练集分成两部分——一个用于训练，另一个我们称之为 train 集——以及一个较小的我们称之为 train-dev 集。

1.  在 train 集上训练你的模型。

1.  在 train、dev 和 train-dev 三个集合上评估你的误差 *ϵ*。

1.  计算量 Δ*ϵ*[*train* − *dev*]。如果它很大，这将提供强有力的证据表明原始训练集和开发集来自不同的分布。

注意，你不需要创建条形图，就像我这里所做的那样。技术上，你只需要四个数字，就可以得出相同的结论。

### 注意

一旦你有了 MAD 图表（或简单地数字），解释它将给你提示，告诉你应该尝试做什么以获得更好的结果，例如更高的准确率。

你可以尝试以下技术来解决集合之间的数据不匹配问题：

+   你可以进行手动错误分析，以了解集合之间的差异，然后决定下一步做什么（在章节的最后部分，我将给你一个例子）。这很耗时，通常也相当困难，因为一旦你知道差异是什么，可能很难找到解决方案。

+   你可以尝试使训练集更像你的开发/测试集。例如，如果你正在处理图像，并且测试/开发集具有较低的分辨率，你可能会决定降低训练集中图像的分辨率。

如同往常，没有固定的规则。只需意识到这个问题，并考虑以下内容：你的模型将从你的训练数据中学习特征，因此当应用于完全不同的数据时，它（通常）表现不佳。始终获取反映你希望模型工作的数据的训练数据，而不是相反。

## K-Fold 交叉验证

现在我想要用另一种非常强大且任何机器学习从业者都应该知道的技术来结束这一章（不仅限于深度学习领域）：k 折交叉验证。这项技术是解决以下两个问题的方法：

+   当你的数据集太小，无法将其划分为训练和开发/测试集时应该做什么

+   如何获取你的指标方差的信息

让我们用伪代码来描述这个想法。

1.  将你的完整数据集划分为 k 个大小相等的子集：*f*[1]，*f*[2]，…，*f*[*k*]。这些子集也被称为折。通常这些子集不会重叠，这意味着每个观测值只出现在一个折中。

1.  对于从 1 到 *k* 的 `i`：

    +   在除了 *f*[*i*] 之外的所有折上训练你的模型

    +   在折 *f*[*i*] 上评估你的指标。折 *f*[*i*] 将是迭代 *i* 中的开发集

1.  评估你的指标在 *k* 个结果上的平均和方差

*k* 的典型值是 10，但这取决于你的数据集大小和问题的特性。

记住，我们关于如何划分数据集的讨论也适用于这里。

### 注意

当你创建你的折时，你必须注意它们要反映你原始数据集的结构。例如，如果你的原始数据集有 10 个类别，你必须确保你的每个折都包含所有 10 个类别，并且比例相同。

虽然这可能是一种处理小于最佳大小的数据集的非常有吸引力的技术，但实现起来可能相当复杂。但是，正如你很快就会看到的，检查你在不同折上的指标将给你关于训练数据集可能过度拟合的重要信息。

让我们在真实数据集上尝试一下，看看如何实现它。请注意，你可以很容易地在 sklearn 库中实现 k 折交叉验证，但我会从头开始开发，以便向你展示后台发生的事情。每个人（好吧，几乎所有人）都可以从网上复制代码来实现 sklearn 中的 k 折交叉验证，但不是很多人能解释它是如何工作的或理解它，因此能够选择正确的 sklearn 方法或参数。作为一个数据集，我们将使用与第二章中相同的：只包含数字 1 和 2 的简化 MNIST 数据集。我们将使用一个神经元进行简单的逻辑回归，以便代码易于理解，并让我们专注于交叉验证部分，而不是其他与此不相关的实现细节。本节的目标是让你了解 k 折交叉验证是如何工作的以及为什么它是有用的，而不是如何以尽可能少的代码行实现它。

正是我们需要的。现在我们准备好构建我们的网络了。我们将使用一个神经元网络进行逻辑回归，使用 sigmoid 激活函数。

```py
import tensorflow as tf
tf.reset_default_graph()
X = tf.placeholder(tf.float32, [n_dim, None])
Y = tf.placeholder(tf.float32, [1, None])
learning_rate = tf.placeholder(tf.float32, shape=())
#W = tf.Variable(tf.zeros([1, n_dim]))
W = tf.Variable(tf.random_normal([1, n_dim], stddev= 2.0 / np.sqrt(2.0*n_dim)))
b = tf.Variable(tf.zeros(1))
y_ = tf.sigmoid(tf.matmul(W,X)+b)
cost = - tf.reduce_mean(Y * tf.log(y_)+(1-Y) * tf.log(1-y_))
training_step = tf.train.AdamOptimizer(learning_rate = learning_rate, beta1 = 0.9, beta2 = 0.999, epsilon = 1e-8).minimize(cost)
init = tf.global_variables_initializer()
```

在这里，我们使用了 Adam 优化器，但梯度下降同样有效。这是一个非常简单的情况。我们将使用我们熟知的函数来训练模型。

```py
def run_logistic_model(learning_r, training_epochs, train_obs, train_labels, debug = False):
sess = tf.Session()
sess.run(init)
cost_history = np.empty(shape=[0], dtype = float)
for epoch in range(training_epochs+1):
sess.run(training_step, feed_dict = {X: train_obs, Y: train_labels, learning_rate: learning_r})
cost_ = sess.run(cost, feed_dict={ X:train_obs, Y: train_labels, learning_rate: learning_r})
cost_history = np.append(cost_history, cost_)
if (epoch % 200 == 0) & debug:
print("Reached epoch",epoch,"cost J =", str.format('{0:.6f}', cost_))
return sess, cost_history
```

到这一点，我们不得不遍历折叠。记住我们开始时的伪代码？选择一个折叠作为开发集，并在所有其他连接的折叠上训练模型。对所有折叠都这样进行。代码可能看起来像下面这样。（它有点长，所以花几分钟时间理解它。）在代码中，我添加了注释来指示我们正在讨论哪个步骤，因为你会找到一个相应的编号列表来解释步骤。

```py
train_acc = []
dev_acc = []
for i in range (foldnumber): # Step 1
# Prepare the folds - Step 2
lis = []
ylis = []
for k in np.delete(np.arange(foldnumber), i):
lis.append(X_train[k])
ylis.append(y_train[k])
X_train_ = np.concatenate(lis, axis = 1)
y_train_ = np.concatenate(ylis, axis = 1)
X_train_ = np.asarray(X_train_)
y_train_ = np.asarray(y_train_)
X_dev_ = X_train[i]
y_dev_ = y_train[i]
# Step 3
print('Dev fold is', i)
sess, cost_history = run_logistic_model(learning_r = 5e-4,
training_epochs = 600,
train_obs = X_train_,
train_labels = y_train_,
debug = True)
# Step 4
correct_prediction=tf.equal(tf.greater(y_, 0.5), tf.equal(Y,1))
accuracy = tf.reduce_mean(tf.cast(correct_prediction, tf.float32))
print('Train accuracy:',sess.run(accuracy, feed_dict={X:X_train_, Y: y_train_, learning_rate: 5e-4}))
train_acc = np.append( train_acc, sess.run(accuracy, feed_dict={X:X_train_, Y: y_train_, learning_rate: 5e-4}))
correct_prediction=tf.equal(tf.greater(y_, 0.5), tf.equal(Y,1))
accuracy = tf.reduce_mean(tf.cast(correct_prediction, tf.float32))
print('Dev accuracy:',sess.run(accuracy, feed_dict={X:X_dev_, Y: y_dev_, learning_rate: 5e-4}))
dev_acc = np.append( dev_acc, sess.run(accuracy, feed_dict={X:X_dev_, Y: y_dev_, learning_rate: 5e-4}))
sess.close()
```

代码遵循以下步骤：

1.  对所有折叠（在这种情况下，从 1 到 10）进行循环，使用变量 `i` 从 0 到 9 进行迭代。

1.  对于每个 `i`，使用折叠 `i` 作为开发集，并将所有其他折叠连接起来作为训练集。

1.  对于每个 `i`，训练模型。

1.  对于每个 `i`，评估两个数据集（训练集和开发集）的准确性，并将值保存在两个列表中：`train_acc` 和 `dev_acc`。

让我们也检查一下网络是如何学会识别数字的。你可能会记得，我们神经元的输出是

![$$ \widehat{y}=\sigma (z)=\sigma \left({w}_1{x}_1+{w}_2{x}_2+\dots +{w}_{n_x}{x}_{n_x}+b\right) $$](img/463356_1_En_6_Chapter_TeX_Equaa.png)

其中 *σ* 是 Sigmoid 函数，*x*[*i*] 对于 *i* = 1, …, 784 是图像像素的灰度值，*w*[*i*] 对于 *i* = 1, …, 784 是权重，*b* 是偏置。记住，当 ![$$ \widehat{y}&gt;0.5 $$](img/463356_1_En_6_Chapter_TeX_IEq1.png) 时，我们将图像分类为类别 1（因此，数字 2），如果 ![$$ \widehat{y}&lt;0.5 $$](img/463356_1_En_6_Chapter_TeX_IEq2.png)，我们将图像分类为类别 0（因此，数字 1）。现在从第二章节中对 Sigmoid 函数的讨论中，你会记得当 *z* ≥ 0 时，*σ*(*z*) ≥ 0.5，而对于 *z* < 0，*σ*(*z*) < 0.5。这意味着我们的网络应该以这种方式学习权重，对于所有的 1，我们都有 *z* < 0，而对于所有的 2，*z* ≥ 0。让我们看看这真的是否如此。在图 6-15 中，你可以看到一个数字 1 的绘图，从中你可以找到经过 600 个 epoch 训练后（并在达到 98% 的准确率后）我们网络的权重 *w*[*i*]（实线）以及像素 *x*[*i*] 的灰度值重新缩放到最大值为 0.5（虚线）。注意每次 *x*[*i*] 很大时，*w*[*i*] 都是负的。当 *w*[*i*] > 0 时，*x*[*i*] 几乎为零。显然，结果 ![$$ {w}_1{x}_1+{w}_2{x}_2+\dots +{w}_{n_x}{x}_{n_x}+b $$](img/463356_1_En_6_Chapter_TeX_IEq3.png) 将是负的，因此 *σ*(*z*) < 0.5，网络将识别图像为 1。在图像中，我放大了这部分以使这种行为更加明显。
