# 机器学习与谷歌

谷歌在人工智能和机器学习领域投入了巨资。谷歌是一家数据公司，其使命是——整理全球信息，并使其人人可访问。

谷歌在每一款产品中都运用了机器学习算法。想想 Gmail 中的垃圾邮件过滤器（区分垃圾邮件与非垃圾邮件）、YouTube 上的视频推荐、用于翻译其他语言文本的谷歌翻译、谷歌搜索结果的相关性排序、谷歌助手等等。它的应用如此普遍，以至于我们习以为常。这也意味着每一位谷歌工程师都接受过机器学习培训。

谷歌在极其庞大的数据上应用机器学习，这需要强大的基础设施。例如，从卫星图像中识别道路、预测广告竞价的点击率等等。没错，你可以在自己的笔记本电脑上训练一个机器学习模型。但要处理海量数据，就需要让你的电脑连续运行数周甚至数月。这需要大量的数据存储和计算能力。正因如此，谷歌在全球拥有众多数据中心——那些装满计算机机架的大型建筑，能够并行处理数据。你无需等待数周或数月；你添加的机器越多，训练速度就越快。借助云技术，这一切甚至可以在几分钟内完成。

谷歌还以其框架`TensorFlow`而闻名。这是一个著名的用于创建机器学习模型的（Python）框架，被众多数据科学家和数据工程师使用。它是 GitHub 上最受欢迎的开源项目之一，由谷歌员工杰夫·迪恩创建。尽管该框架是开源的，世界各地的开发者和公司都为其做出贡献，但谷歌仍有一个专门的大型团队致力于改进其代码库。

## 关于谷歌云

谷歌云（原名 Google Cloud Platform/GCP）是谷歌的公有云提供商，提供用于部署、构建和运行应用程序的计算资源——通过快速且安全的谷歌基础设施，从全球各地的数据中心交付存储、计算和服务。它属于谷歌，但这并不意味着你的数据归谷歌所有。谷歌云是谷歌提供的按使用量付费的商业企业服务。正如已签署的谷歌云条款与条件所述，你是数据的所有者。谷歌可以处理你的数据，但不能也不会将其用于自身。

当你构建一个聊天机器人时，通常这并不意味着你只使用一个对话式 AI 工具`Dialogflow`。就像构建一个网站一样，你可能需要更多资源。想想看：需要一个地方来托管你的聊天机器人，将数据存储在数据仓库或数据库中，并且你可能还想使用额外的机器学习模型来检测 PDF 文件的内容或文本的情感倾向。

截至撰写本文时，谷歌云拥有超过 200 款产品。其中包括面向开发者的计算、存储、网络、数据分析和机器学习产品，例如用于识别图像（`Vision AI`）、视频（`Video AI`）、文本（`Natural Language API`）、语言（`Translate API`）和音频（`Speech-to-Text`/`Text-to-Speech API`）的机器学习 API。最后，还有供数据科学家训练模型以及用于身份和安全管理的机器学习工具。这就像乐高积木。通过将这些资源层层堆叠，你将构建出一个产品。

## 开源

谷歌信奉透明，通过开源软件和构建开发者社区来践行这一理念。谷歌在开源开发平台 GitHub 上拥有超过 28 万次提交，自 2016 年以来的项目贡献超过 1.5 万个。这些项目包括广受欢迎的谷歌开源项目，如 Android、Chromium、V8 JavaScript 引擎、WebKit、Angular、TensorFlow、Kubernetes、Istio 和 Go 语言。我相信你很可能认识其中几个。除了这些诞生于谷歌的产品，谷歌还为其他流行的开源项目和标准做出了贡献。想想 HTML5、Linux 内核、Python 语言、MySQL、GCC（GNU 编译器套件）、Spinnaker 等等。谷歌撰写了大量行业研究白皮书，这些白皮书启发了社区和其他大型软件产品，例如后来用于创建 Hadoop 的 MapReduce。

许多优秀的谷歌开源产品始于 20 世纪 90 年代末/21 世纪初。当许多个人或公司还在考虑构建一个简单的网页时，谷歌已经需要维护全球规模最大、访问量最高的网站（谷歌搜索引擎），并且这个网站还必须具备可扩展性。谷歌工程师为解决高可维护性/可扩展性问题而创建的产品，后来成为了开源软件的基础。例如，内部的 Borg 容器编排系统在开源世界中演变成了 Kubernetes。

而在开源世界中行之有效的做法，谷歌又通过将这些产品运行在谷歌云上，带回了企业世界。
