
# 假负例——被遗漏的请求

**假负例**是指意图存在，但聊天机器人未能检测到，从而触发了回退意图的结果。当你的机器人已构建了某个特定意图，但未能识别出替代的措辞或术语时，就会发生被遗漏的请求错误。

例如，当你的虚拟助手有一个名为`Block credit card`的意图，并且它使用训练短语`I want to block my credit card.`进行训练。我可以针对上述场景创建测试数据，用户话语为：`Stop the credit card right now.` 我期望它匹配`Block credit card`意图；然而，虚拟助手检测到了回退意图。因此，这是一个假负例。意图是存在的，但未能匹配上。

你可以将类似上述的测试用例存储在 BigQuery 中，这样每次通过 Dialogflow API 对智能助手进行更改时，都可以重新运行该场景。你需要收集的数据字段包括`用户话语`、`预期意图名称`、`检测到的意图名称`以及`结果`，即`FN`。
