# 真负例——不支持的请求

**真负例**是指聊天机器人正确地将用户话语映射到回退意图的结果。当用户提出机器人设计范围之外的问题时，就会发生不支持的请求错误。但机器人不会返回错误答案或系统错误，而是返回一条友好的回退消息。

例如，当你的虚拟助手有一个名为`Global fallback`的意图，该意图在无法匹配任何意图时被激活。我可以针对上述场景创建测试数据，用户话语为：`Can I buy casino tokens?` 这是一个银行聊天机器人，而不是赌场聊天机器人。我期望它匹配`Global fallback`意图，而 Dialogflow 也确实返回了`Global fallback`意图。因此，这是一个真负例。它的行为完全符合设计预期。

你可以将类似上述的测试用例存储在你的数据仓库中，这样每次通过 Dialogflow API 对智能助手进行更改时，都可以重新运行该场景。你需要收集的数据字段包括`用户话语`、`预期意图名称`、`检测到的意图名称`以及`结果`，即`TN`。
