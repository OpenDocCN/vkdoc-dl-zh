### 设计和实现代理

在本节中，让我们回顾一下代理设计和实现的各个方面，包括定义目标、理解核心概念、管理输入和输出、执行任务以及使用工具和工具包。了解这些将帮助您创建高效且有效的代理。

#### 定义代理的目标

让我们花点时间讨论构建代理最关键的一个方面：定义其目标。

要定义代理的目标，请问自己以下问题：

1. **我希望我的代理解决什么问题？**
   - 您是否在构建一个聊天机器人来协助客户处理咨询？
   - 您是否需要代理来分析数据并提供见解？
   - 您的代理是否负责根据特定提示生成内容？

2. **我的代理是否需要与其他代理协作？**
   - 任务是否需要多个专门的代理协同工作？
   - 不同的代理是否会处理复杂问题的不同方面？
   - 协作能否带来更全面或更高效的解决方案？

3. **我的代理应该执行哪些任务来实现其目标？**
   - 您应该首先将问题分解为更小、可管理的任务。
   - 确定代理需要采取的具体行动来解决问题。
   - 您应该仔细考虑每个任务的输入和输出要求。

4. **我的代理需要哪些工具和资源？**
   - 您必须确定代理将与之交互的库、API 和外部服务。
   - 评估是否需要集成任何特定领域的知识或数据库。
   - 确定与代理目标一致的语言模型或预训练模型。

假设您想创建一个代理业务应用，帮助用户找到给定城市中最好的餐厅。您的代理目标可以定义如下：

**目标**：餐厅推荐代理将帮助用户根据他们的美食偏好和预算限制，在指定城市发现评价最高的餐厅。该代理将通过对话界面与用户交互，了解他们的需求，并提供个性化的餐厅推荐以及相关信息，如评分、评论和联系方式。

有了这个目标，您现在可以概述代理需要执行的任务：

1. 理解用户输入并提取相关信息（城市、美食、预算）。
2. 与餐厅数据库或 API 交互以获取餐厅数据。
3. 应用过滤和排名算法，根据用户偏好选择最佳餐厅。
4. 生成自然语言响应，呈现推荐的餐厅及其详细信息。
5. 处理后续问题，并根据用户要求提供额外信息。

现在您有了明确的目标和一组任务，可以开始收集必要的工具和资源。在这种情况下，您可能需要以下内容：

- 用于自然语言理解和生成的语言模型或对话式 AI 框架
- 用于访问餐厅信息的餐厅数据库或 API
- 用于数据操作、过滤和排名的库
- 与外部服务集成以获取评论、评分和联系方式

### 概念

好的，让我们讨论一下代理背后的核心概念，并探索使代理运转的关键组件：

1. **`AgentAction`**
   - `AgentAction` 是一个数据类，表示代理应采取的行动。它就像是代理下一步行动的蓝图。
   - 它有两个重要属性：
     - `tool`：这是代理应调用的工具的名称。可以将其视为代理将使用的特定技能或能力。
     - `tool_input`：这是应提供给工具的输入。就像为代理提供有效执行行动所需的信息。

2. **`AgentFinish`**
   - `AgentFinish` 表示代理完成任务并准备将输出返回给用户时的最终结果。
   - 它包含一个 `return_values` 键值映射，其中保存了最终的代理输出。这是代理的响应或解决方案存储的地方。
   - 通常，`return_values` 映射包含一个 `output` 键，其中包含一个表示代理最终响应的字符串。

3. **中间步骤**
   - 中间步骤就像是代理在当前代理运行中对先前操作及其相应输出的记忆。
   - 它们对于将信息传递给未来的迭代至关重要，允许代理知道它已经完成了哪些工作并在此基础上继续。
   - 中间步骤表示为一个元组列表，其中每个元组包含一个 `AgentAction` 及其对应的输出。
   - 中间步骤的类型是 `List[Tuple[AgentAction, Any]]`。使用 `Any` 类型作为输出以提供最大的灵活性，因为输出可能因使用的工具而异。在大多数情况下，输出是一个字符串，但也可能是其他类型。
