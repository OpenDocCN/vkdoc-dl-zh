# 第 9 章 构建不同类型的代理

## 代理类型

LangChain 提供了多种代理类型，每种类型都有其独特的特性和能力，你可以根据具体需求和所使用的模型进行选择。

## 选择代理类型的标准

首先，我们来谈谈预期的模型类型。有些代理是专门为聊天模型设计的，这类模型接收消息并输出消息。

其他代理则针对语言模型（LLM）进行了优化，这类模型接收字符串并输出字符串。主要区别在于所使用的提示策略。

虽然你可以将不同类型的代理与模型类型混合搭配，但坚持使用预期的模型类型可能会获得最佳效果：

**聊天历史支持**：某些代理类型能够处理聊天历史，使其成为聊天机器人应用的理想选择。这些代理可以维护上下文，提供无缝的对话体验。另一方面，不支持聊天历史的代理类型更适合单一任务或一次性交互。

**多输入工具**：某些代理类型能够处理需要多个输入的工具，而其他代理则专为单一输入的简单工具设计。如果你正在使用需要多个参数的复杂工具，请确保选择支持多输入工具的代理类型。

**并行函数调用**：并行函数调用是一项强大的功能，允许代理同时调用多个工具，从而显著加快某些任务的执行速度。然而，这种能力对 LLM 来说更具挑战性，因此并非所有代理类型都支持。在决定并行函数调用是否必不可少时，请考虑你的具体需求和任务的复杂性。

**模型参数**：某些代理类型对模型参数也有特定要求。例如，某些代理利用了 OpenAI 的函数调用功能，这需要额外的模型参数。如果某个代理类型没有必需的模型参数，则意味着所有操作都通过提示处理。

## LangChain 代理类型

现在你已经对代理是什么以及它们与链的区别有了扎实的理解，让我们更仔细地看看撰写本文时 LangChain 代理的精彩世界。

### 1. API 工具调用

- 预期模型类型：聊天

- 支持聊天历史：✅

- 支持多输入工具：✅

- 支持并行函数调用：✅

- 必需模型参数：`tools`

- 使用时机：如果你使用的是工具调用模型

### 2. OpenAI 工具

- 预期模型类型：聊天

- 支持聊天历史：✅

- 支持多输入工具：✅

- 支持并行函数调用：✅

- 必需模型参数：`tools`

- 使用时机：[旧版] 如果你使用的是较新的 OpenAI 模型（1106 版本及以后）。建议改用通用工具调用代理

### 3. OpenAI 函数

- 预期模型类型：聊天

- 支持聊天历史：✅

- 支持多输入工具：✅

- 支持并行函数调用：✅

- 必需模型参数：`functions`

- 使用时机：[旧版] 如果你使用的是 OpenAI 模型，或经过函数调用微调且暴露与 OpenAI 相同`functions`参数的开源模型。建议改用通用工具调用代理

### 4. XML

- 预期模型类型：LLM

- 支持聊天历史：✅

- 支持多输入工具：✅

- 支持并行函数调用：✅

- 必需模型参数：无

### 5. 结构化聊天

- 支持多输入工具：✅

- 必需模型参数：无

- 使用时机：如果你需要支持具有多个输入的工具

### 6. JSON 聊天

- 预期模型类型：聊天

- 支持聊天历史：✅

- 支持多输入工具：❌

- 支持并行函数调用：❌

- 必需模型参数：无

### 7. ReAct

- 预期模型类型：LLM

- 支持聊天历史：❌

- 支持多输入工具：❌

- 支持并行函数调用：❌

- 必需模型参数：无

- 使用时机：如果你使用的是简单模型

- 预期模型类型：LLM

- 支持聊天历史：❌

- 支持多输入工具：❌

- 支持并行函数调用：❌

- 必需模型参数：无

- 使用时机：如果你使用的是简单模型且只有一个搜索工具

以下是 LangChain 中可用的不同代理类型的快速概览：

1. **零样本反应代理**：如果你想处理各种任务而无需任何预先训练或微调，可以使用此代理。你只需为其提供一组工具和一个语言模型，它就会根据给定的查询动态生成响应。它能够适应任何情况。

2. **对话代理**：顾名思义，你将此代理用于对话式 AI 应用，例如聊天机器人或虚拟助手。它可以进行来回对话，维护上下文，并根据对话历史提供相关响应。

3. **结构化工具代理**：有时，你希望代理以结构化方式使用一组特定的工具或操作。在这种情况下，你可以使用结构化工具代理来定义预定的操作和工具序列，从而指导代理的行为。

4. **MRKL 代理**：MRKL（带逻辑回归的平均倒数排名）代理是 LangChain 家族中一个强大的新成员。它结合使用搜索算法和逻辑回归来对给定查询最相关的工具进行排序和选择。你可以使用它来优先选择最佳行动方案。

## 零样本反应代理示例

零样本反应代理是一种灵活且适应性强的代理，它使用大型语言模型（LLM）根据工具的描述和用户输入，从一组可用工具中决定使用哪个工具：

- **零样本**：代理没有经过关于如何具体使用每个工具的预先训练。它会在使用时即时学会如何使用它们。

- **ReAct**：这是指导代理决策的底层框架。它是一个推理和行动的循环，允许代理动态选择最适合任务的工具。

使用此代理的优势在于，你无需为每个工具创建复杂的规则或链，也不需要预先训练。

然而，其性能在很大程度上取决于工具的描述以及 LLM 理解描述的能力。因此，这意味着你需要使用一个推理能力强的 LLM，例如`gpt-4`或`gpt-3.5-turbo`。

第二个缺点是它没有记忆。你可以通过在初始化代理时传递`ConversationBufferMemory`对象来解决这个限制。还有一种`conversational-react-description`代理类型，它使用了记忆。
