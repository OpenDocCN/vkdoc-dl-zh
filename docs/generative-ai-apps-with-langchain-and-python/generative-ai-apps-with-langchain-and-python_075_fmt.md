### 使用提示模板的优势

在进入下一节关于示例选择器的内容之前，让我们回顾一下上一个示例中使用提示模板的一些优势：

- **一致性**：确保输入到语言模型的内容遵循一致的结构和格式。这也使语言模型能够以更可预测、更有条理的方式理解和处理输入。

- **可复用性**：提示模板是可复用的，因为它与调用模型的代码相分离。因此，同一个模板可以跨多个模型以及代码中的其他位置使用。

- **可定制性**：你可以通过在提示模板中使用占位符或变量来定制输入到语言模型的内容。在运行时，你可以动态填充相关信息。

- **改进的微调**：你可以引导语言模型生成符合你期望的格式、风格或内容的响应。实际上，提示模板充当了一种微调形式，让你能够根据模板的结构和内容来控制语言模型的输出。

- **可维护性**：将提示模板与代码分离，可以提高代码库的可读性和可维护性。

## 示例选择器

有时，你需要给大语言模型一点指导，才能获得你想要的特定输出。这时就需要用到示例选择器了！

你可以使用示例选择器向大语言模型提供示例，帮助它理解期望的输出格式和模式。通过向模型展示相关示例，你可以引导它生成更准确、更恰当的响应。这种技术被称为**少样本学习**。示例选择器能帮助你挑选出最适合当前任务、能最好地指导语言模型的示例。

### 为提示选择正确的示例

示例选择器接收输入变量，并选出一组最适合给定输入的示例。

想象一下，你正在构建一个根据用户偏好生成个性化锻炼计划的系统。假设你有一个针对不同用户偏好（如健身水平和目标）的示例锻炼计划数据集。当用户提供他们的偏好时，你希望选择最相关的示例包含在提示中。

以下是示例选择器的一个简化实现。

### 定义 `BaseExampleSelector` 类

首先，你必须定义示例选择器的接口：

```python
class BaseExampleSelector(ABC):
    """用于选择要包含在提示中的示例的接口。"""

    @abstractmethod
    def select_examples(self, input_variables: Dict[str, str]) -> List[dict]:
        """根据输入选择要使用的示例。"""

    @abstractmethod
    def add_example(self, example: Dict[str, str]) -> Any:
        """向存储中添加新示例。"""
```

这是一个抽象基类，为我们的示例选择器定义了接口。

你需要在继承自 `BaseExampleSelector` 的类中实现抽象方法 `select_examples`。它接收一个输入变量字典，并返回一个所选示例的列表。

你可以通过实现 `add_example` 方法向示例选择器的存储中添加新示例。

请注意，你需要从 `abc` 模块导入 `ABC` 和 `abstractmethod`。你还应该从 `typing` 模块导入必要的类型以进行类型提示。
