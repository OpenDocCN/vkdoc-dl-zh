## LangChain 框架组件

到现在，你一定已经清楚 LangChain 是一个可以帮助你开发 LLM 应用程序的强大框架。你已经看到了不少关于如何使用 LangChain 库调用 LLM 的例子。你还在第 1 章中简要了解了一些组件。让我们进一步探索 LangChain 框架。我们有专门的章节来介绍这些组件，所以现在我们只进行高层次的讨论。

### 开发

我们在第 1 章已经了解到 LangChain 提供了以下组件：

- **模型 I/O**：模型输入/输出（I/O）是关于如何格式化和管理进出语言模型的数据。

- **提示**：提示是你可以用来将数据输入语言模型以指导其如何生成响应的特定格式。你可以将它们视为帮助你创建问题以将对话引导到所需方向的工具。这种方法让你可以控制流程，并确保响应与你的特定需求相关。

- **聊天模型**：聊天模型是处理类似聊天交互的专用接口。你可以使用这些模型将聊天消息作为输入，并以对话格式获取响应。你可以使用它们来构建聊天机器人和类似的交互式应用程序。

- **LLM（大型语言模型）**：LLM 是复杂的 AI 模型，主要用于理解和生成类似人类的文本。它们擅长文本生成、问答和信息提取等任务。虽然它们的核心功能围绕文本，但 LLM 可用于广泛的应用程序，从创意写作到数据分析和问题解决。

**注意**：最近的进展已将 LLM 的能力扩展到文本处理之外。通过各种扩展，LLM 现在可以：

- 生成和操作图像

- 处理和创建音频内容

- 协助代码生成和分析

- 执行基本推理任务

- 这些扩展使 LLM 成为跨行业多样化多模态应用程序的强大工具。

- **检索**：检索组件是一项非常重要的进步，它允许你将应用程序特定的数据与语言模型连接起来，这对于检索增强生成（RAG）等应用程序非常理想。你可以使模型能够从庞大的数据集中提取相关信息，以开发相关且有意义的响应。

- **文档加载器**：文档加载器是从各种来源导入数据并将其格式化为“文档”以供应用程序稍后处理的工具。你可以使用此功能加载文本文件、从数据库提取数据或从 Web 拉取内容。我将在第 7 章中讨论这一点。

- **文本分割器**：你可以使用文本分割器将大块文本分割成更小、更易于管理的部分。这不仅使处理更容易，而且通过专注于最相关的文本部分来提高模型的性能。

- **嵌入模型**：嵌入模型是一个迷人的概念，它允许你将文本块转换为向量表示。这些向量以数字格式捕获文本的本质，你可以使用它们通过大型数据集执行自然语言搜索。

- **数据存储与检索**：LangChain 支持多种高效存储和检索数据的选项：

  1. **向量存储**：这些是专门的数据库，管理向量表示，用于对非结构化数据进行自然语言搜索。你可以特别使用它们来构建复杂的搜索引擎和推荐系统，以快速提供相关结果。

  2. **图数据库**：对于高级用户，图数据库提供了一种替代方案，在某些场景下可能更合适，例如处理高度连接的数据时。LangChain 支持与图数据库的集成，允许进行复杂的关系查询。

  3. **传统数据库**：对于结构化的数据和精确的查询，传统的 SQL 数据库仍然是一个强大的选择。

这些选择取决于你的具体用例、数据结构和查询需求。LangChain 的灵活性允许你为项目集成最合适的存储解决方案。

**超越向量存储**：虽然向量存储擅长在大型非结构化数据集中进行快速、相关的搜索，但你也应该考虑将图数据库用于高度连接的数据场景。LangChain 两者都支持，因此你必须根据项目需求选择最合适的。在此处了解有关 LangChain 中图数据库集成的更多信息：[`python.langchain.com/docs/use_cases/graph/`](https://python.langchain.com/docs/use_cases/graph/)。

- **检索器**：检索器比向量存储更通用一些。它们是你可以用来基于非结构化查询获取文档的接口。你可以将它们视为你的个人数据检索器，根据你当前的需求去获取信息。

- **组合**：组合组件就像是 LangChain 的架构师。你可以使用它们组合各种系统和 LangChain 原语来构建复杂的功能。例如，你可能正在将文本分割器与嵌入模型链接起来，或者创建一个成熟的 AI 助手。使用这个组合模块，你可以精确地根据你的需求定制应用程序，并优化其效率和效果。

- **工具**：你可以使用工具提供的接口来允许语言模型与外部系统交互。使用工具，你可以扩展应用程序的能力，例如连接到数据库或第三方 API。

- **代理**：代理是 LangChain 中的决策者。你可以使用它们分析高级指令，并决定使用哪些工具以自动化的方式实现期望的结果。它们确保你的应用程序在正确的时间自动使用正确的资源。有关 LangChain 代理的详细处理，请参阅第 8-10 章。

- **链**：链是 LangChain 模块化设计的基础。它们是各种组件的组合，这些组件作为构建块协同工作。你可以混合搭配这些组件来定制应用程序的行为。下面是一个示例，只是为了让你了解如何使用链：
