# 第 2 章 使用 LangChain 集成 LLM API

## 直接使用 LLM API 与使用 LangChain 的对比

对于希望构建生成式 AI 应用的初学者来说，LangChain 提供了一种简化且强大的方法。虽然你当然可以直接使用 LLM API，但 LangChain 在简单性、灵活性和可扩展性方面提供了显著优势。在本节中，让我们探讨为什么 LangChain 是大多数开发者的推荐选择，尤其是对于那些刚开始使用 LLM 的开发者。我还会简要提及直接使用 API 时遇到的挑战，以便提供背景信息，但本书的重点将是利用 LangChain 的能力来构建高效、可扩展的 AI 应用。

让我们看看 LLM API 的一些挑战，以及 LangChain 如何帮助解决这些问题。

### 开发复杂性

当直接使用 LLM API 时，你需要深入了解每个模型的细节和局限性，这需要陡峭的学习曲线。

**LangChain 解决方案：** 正如我们之前讨论的，LangChain 通过提供一个统一的接口，让你能够无缝地与多个 LLM 交互，从而抽象了直接进行 API 调用的复杂性。这个抽象层意味着你可以花更少的时间处理 LLM API 的细节，而将更多精力放在设计正确的提示和集成有价值的数据源上，无需担心底层模型。
