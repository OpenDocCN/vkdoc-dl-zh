## 第 1 章 LangChain 与 LLM 简介

应用设计的好坏将决定其完成后的可扩展性、可维护性和性能。这与传统软件开发生命周期（SDLC）非常相似，区别在于传统 SDLC 更侧重于客户端-服务器交互、数据库设计和面向服务架构等各个方面。

**搭建开发环境**：这是关键环节，你将开始准备开发环境，包括必要的软件、工具以及对 LLM API 的访问权限。

**实现 LangChain 组件**：现在进入激动人心的部分——编码！你将开始集成 LangChain 的组件，如模型、数据连接和智能体，以构建你设计的功能。本书将重点讲解这一步骤，并提供实用示例和指导。此阶段与传统软件编程的不同之处在于，你并非为软件功能编码，而是为 LLM 交互编码。

**整合数据源**：在此阶段，你将与外部数据集成，为应用提供所需的上下文和相关性，如工作流示意图（图 1-4）中“整合数据源”步骤所示。这一阶段强调应用需要从数据库、API 或实时数据流等多种来源拉取数据。此阶段与传统 SDLC 的不同之处在于，你还需要关注如何使用数据来训练和微调 LLM。

## 第 1 章 LangChain 与 LLM 简介

**使用 LLM 进行训练/测试**：如有必要，你可能需要使用 LangChain 对 LLM 进行训练或微调。确保对应用进行彻底测试，以验证其满足指定需求。这与传统测试的不同之处在于，你不仅要检查错误，还要确保 LLM 生成正确且上下文恰当的响应。

**迭代与优化**：利用测试反馈对应用的设计和功能进行迭代。重点优化 LLM 性能、调整 LLM 配置、微调 LangChain 设置、增强可用性，甚至使用新数据重新训练模型。

**准备部署**：现在是时候为部署最终确定应用了。这可能涉及获取权限、最终测试以及准备部署脚本。

**部署应用**：工作流示意图（图 1-4）中高亮显示的“部署应用”阶段，标志着将 LLM 应用部署到实际使用环境的重要时刻。此时你需要为应用上线做好准备，确保其稳健、安全，并能处理用户交互。你需要关注选择合适的主机环境、支持增长的可扩展性，以及实施用于持续性能评估的监控工具。
