# 第 7 章 使用检索增强生成（RAG）构建高级问答与搜索应用

你可以根据 LangChain 中多种检索器的独特特性和用例来使用它们，因此理解这些检索器将有助于你为特定需求选择最佳方案：

1. **向量存储检索器**：如果你刚开始接触并希望寻找快速简便的方法，`向量存储检索器` 是你的首选。这是最简单的方法，你可以为每段文本创建嵌入向量，从而轻松找到相似内容。

2. **父文档检索器**：当你的文档包含大量独立索引但可一起检索的细小信息片段时，可以使用 `父文档检索器`。它会为每个文档索引多个数据块，并根据嵌入向量找到最相似的数据块。真正有用的是，它不会返回单个数据块，而是检索整个父文档以提供更全面的结果。

3. **多向量检索器**：有时，你可能希望从文档中提取你认为比文本本身更相关的额外信息进行索引。`多向量检索器` 允许你为每个文档创建多个向量，从而灵活捕捉内容的不同方面。例如，你可以仅基于文档摘要或假设性问题创建向量。

4. **自查询检索器**：你是否遇到过用户提出的问题更适合通过元数据而非文本相似度来检索文档的情况？`自查询检索器` 使用语言模型（LLM）将用户输入转化为两部分：用于语义查找的字符串和用于应用的元数据过滤器。当问题更关注文档的元数据而非内容时，这一功能极为有用。

5. **上下文压缩检索器**：有时，检索到的文档可能包含大量干扰语言模型的无关信息。`上下文压缩检索器` 通过添加后处理步骤，仅从检索到的文档中提取最相关的信息，从而为你排忧解难。这可以通过嵌入向量或语言模型实现，确保你的语言模型始终聚焦于关键内容。

LangChain 还提供了其他高级检索类型，例如 `时间加权向量存储检索器`、`多查询检索器`、`集成检索器` 和 `长上下文重排序检索器`。每种检索器都有其独特的优势和用例，使你能够根据具体需求定制检索流程。
