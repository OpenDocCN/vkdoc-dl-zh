# 第 7 章 使用检索增强生成（RAG）构建高级问答与搜索应用

以下是实现这种组合方法的方式。

#### 关键词搜索

首先，使用关键词搜索来筛选包含相关术语的文档。这一步在处理大型数据集和快速识别潜在匹配项时非常有用。

#### 文本嵌入

对筛选后的文档应用文本嵌入，以捕获其语义含义。使用这些嵌入根据与查询的相关性对筛选结果进行排序。

这种组合确保你能受益于关键词搜索的速度和精确性，同时也能利用嵌入提供的语义深度。

你可以将文本嵌入视为文本的向量形式的魔法表示。它们捕获了词语和句子的语义含义，帮助你以有意义的方式进行比较。这就像为高维空间中的每一段文本分配坐标，相似的文本彼此更接近。

在使用文本嵌入模型时，你将使用 LangChain 中的 `Embeddings` 类。它提供了一个标准接口，用于与各种嵌入模型提供商（如 OpenAI、Cohere 和 Hugging Face）进行交互。你可以从超过 25 种不同的嵌入提供商和方法中进行选择，范围从开源到专有 API。无论你选择哪个提供商，`Embeddings` 类都能为你提供帮助。最棒的是，你可以使用此接口轻松地在模型之间切换，从而轻松找到适合你需求的模型。
