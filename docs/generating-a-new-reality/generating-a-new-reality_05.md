## 理解深度学习的学习内容

深度学习的数学绝非易事，并且在许多方面，它曾是这项技术数十年间难以逾越的主要障碍。直到自动微分技术出现，深度学习才真正得以发展。在此之前，人们需要花费数小时来为最简单的网络调整数学计算。即使是我们前几章学过的那些最简单的网络，也可能需要数天甚至数周才能把数学逻辑理清。

如今，借助自动微分以及像 `PyTorch` 或 `TensorFlow` 这样的框架，我们可以在几分钟内构建出强大的网络。更重要的是，我们可以更快地教授和赋能人们使用深度学习。你不再需要博士学位就能训练网络，事实上，就连小学生进行深度学习实验也已是司空见惯。

不幸的是，好事总伴随着坏事。自动微分让我们可以将深度学习系统视为一个黑盒。这意味着我们可能缺乏对网络学习方式的微妙理解，从而可能忽略系统中明显的问题，更糟的是，可能会因为缺乏理解而长期挣扎于解决某些问题。

幸运的是，理解深度学习系统学习方式的数学原理可以大大简化。虽然向一个五岁孩子解释这些概念可能很困难，但任何能看懂图表和简单方程的人，很可能都能理解下一节中的概念。

微积分帮助我们理解方程或函数的变化率。这对于从发射宇宙飞船到建造更好的建筑，当然也包括深度学习，都非常有用。理解为什么使用微积分来逼近深度学习中的函数也很重要，这将在下一节中介绍。



#### 微积分的局限性

如果你学过微积分，就会了解它的局限性。其中之一是，我们用微积分进行求导的所有函数都必须是连续的。图 3-4 展示了一些带有所谓*不连续点*区域的函数示例。这些区域中的函数会阻碍深度学习系统学习或逼近该函数。

![../images/502181_1_En_3_Chapter/502181_1_En_3_Fig4_HTML.jpg](img/502181_1_En_3_Fig4_HTML.jpg)

图 3-4

不连续函数的示例

在生成式建模中，你可以将不连续点视为数据中的间隙或真空。但不要将这里的“间隙”与空间或距离混淆。我们在此使用术语*间隙*，指的是函数中没有任何样本值的断裂处。而术语*空间*、*潜在空间*或*隐藏空间*则用于定义连续数据集中的未知区域。

通常，当我们在生成的输出中看到黑色区域时，这表示输入数据存在极限或不连续性问题。

理解连续数据与不连续数据之间的区别需要一些时间，有时这种区别可能并不明显。这里的关键要点是，函数逼近存在局限性，而这些局限性通常由数据本身决定，有时需要网络在不连续的数据上逼近一个连续函数。

现在，回到极限问题，让我们了解极限如何影响网络学习或逼近数据。在我们的练习示例中，网络的极限是 1 和 10。如果参考图 3-3，我们可以看到当 `X`<1 时，函数变得平坦，考虑到极限为 1，这是合理的。这意味着当 `X` 小于 0 时，我们的网络将函数逼近限制为 `y`=5。

然而，不太明显的是，为什么当极限为 10 时，我们的函数逼近在大于 7 的值上会失败。要理解这个问题出现的原因，我们需要了解深度学习如何利用微积分来逼近函数，这将在下一节中介绍。

#### 深度学习爬山法

任何深奥的数学概念核心，总有一个直观的解释来说明其“为什么”。不幸的是，大多数经典数学课程常常忽略了“为什么”的解释。相反，它们依赖于学生通过进行证明或死记硬背方程来自己发现其中的直觉。这对崭露头角的数学家来说效果很好，但往往会压垮其他人。因此，我们将始终更关注数学的“为什么”，而不是具体的“怎么做”。

微积分是我们在深度学习系统中用来平衡网络权重/参数的工具。回想一下，微积分通过理解系统的变化率来工作。在深度学习中，微积分使我们能够确定需要调整网络权重的变化量以及调整的位置。

微积分和梯度优化或梯度下降并不是我们用来寻找网络权重的唯一工具。它目前是最有效的，但深度学习系统已经并将继续使用其他机器学习方法。如果你有兴趣了解更多，可以搜索 *深度学习 PSO* 或 *深度学习 GA*。

图 3-4 展示了一个一维函数，其中有一个小球位于位置 1。我们可以将深度学习系统的目标视为引导这个球在函数上移动。更进一步，我们希望这个球能够尽可能紧密地跟随该函数。球能够跟随函数路径的效果越好，我们的网络预测或生成内容的效果就越好。

请注意，在图 3-4 中，我们移动球的量是通过自动微分和我们的优化方法计算得出的，再乘以学习率（`alpha`）。这在图中被标记为“梯度 X alpha”。梯度的量可以通过选择不同的优化方法和/或调整超参数来改变。调整优化器，无论是随机梯度下降还是 Adam，都是我们稍后会涉及的内容。

这就引出了 *alpha*，即学习率或学习参数。这个超参数允许我们控制网络在将球映射到函数时使用的调整程度。在图 3-5 和我们之前的例子中，我们应用了较高的学习率。正如你所见，这通过球移动到位置 2 然后到位置 3 的运动来体现。注意球在第三个位置是如何远高于函数线的。这就是导致我们前一个例子中出现问题的原因。

从中得出的主要结论是，在映射函数时，学习率通常是我们最敏感的超参数。学习率，即 *alpha*，几乎总是你为网络调整和理解的首要参数。`Alpha` 也是调整网络训练速度的关键。当 `alpha` 过大时，网络会超出函数范围。相反，将 `alpha` 保持得非常小可以让球很好地学习函数。然而，这可能会以巨大的计算性能为代价。



## 学习率与过拟合/欠拟合

考虑一个学习率为`.001`的情况，与值为`.00001`的情况进行比较。假设我们可以用较小的学习率在 10 分钟内训练一个网络。较小的学习率（小 100 倍）由于训练迭代次数的增加，很可能需要 100 倍的时间，将训练时间增加到 10 × 100 = 1000 分钟，即超过 16 小时。这一切都源于一个更小但不幸的是效率更低的学习率。

### 过拟合与欠拟合

通常，当新手设计网络时，他们的第一个假设是神经元越多意味着结果越好。不幸的是，由于我们将在后续章节中理解的多种原因，情况很少如此。现在，我们想关注网络规模本身如何导致对函数的过拟合或欠拟合。

你可能经常听到深度学习从业者提到他们的网络对数据过拟合或欠拟合。他们真正的意思是网络对匹配或分类该数据的函数过拟合或欠拟合。

图 3-8 展示了我们之前的示例与函数`x2 + 5`的对比图，以查看我们实际匹配结果的程度。如你所见，我们匹配得并不好——事实上，考虑到我们网络的规模，匹配得相当差。然而，网络规模并非全部问题所在。网络规模和数据量几乎是相辅相成的。

这意味着当我们的模型逼近函数`x2 + 5`时，数据之间存在大片区域。我们将这些区域称为*潜在空间*或*隐藏空间*。再次强调，这不应与数据中的间隙、鸿沟或真空混淆。数据中的这种隐藏空间代表了函数中缺少任何实际训练数据的区域。这意味着网络必须在这些数据点之间逼近值。

事实证明，生成建模的一个重要部分是理解如何从这种潜在空间中提取结果，正如我们将在本书中看到的那样。这与我们之前尝试从函数逼近练习中提取结果的方式并无不同。我们之前的假设可能是减少训练点的数量，但现在我们可以看到，这增加了潜在空间的大小，导致模型对函数过拟合。

当模型逼近一个函数但错误地填充或逼近潜在空间时，就会发生过拟合。在我们之前的函数逼近示例中，情况显然如此，这是由两个问题造成的。第一个问题是数据不足，导致网络估计了大面积的潜在空间。第二个问题是网络规模过大，拥有过强的能力来估计数据中的空间。因此，它基本上是在编造值，这可能成为一个问题。

当网络规模很大时，它拥有很大的能力，在这种情况下是过大的能力。大型网络可以记忆各种形式的数据，包括图像或视频。这对于针对特定数据的网络可能有用，但在生成建模以及大多数其他机器学习学科中，我们的目标始终是泛化。

泛化始终是我们构建模型时要追求的目标。这意味着我们输入模型的数据需要分布良好。分布良好的数据为我们提供了数据之间均匀的潜在空间。在我们之前的示例中，数据被分布到 10 个区间中，值从 1 到 10。然而，正如我们所看到的，我们的数据分布得不够好。



## 泛化与过拟合、欠拟合

泛化也意味着我们的网络应该只保持所需的大小，不多也不少。如果网络过大，它往往会过拟合，或用不正确的值填充潜在空间。同样，过小的网络会导致欠拟合。欠拟合通常是模型中潜在空间或间隙过大的结果。

## 构建变分自编码器

在前面的示例练习中，我们探讨了网络如何执行函数逼近，并发现了离散区间内的潜在空间。我们发现，通过增加数据采样区间的数量，可以减少未知的潜在空间，并改进模型训练。这种技术对于简单的函数效果良好，但无法扩展到更复杂的问题。

理解问题和数据本身的更好方法是使用统计学。统计学使我们能够理解模型和数据中的潜在空间。反过来，统计学可以为我们提供模型的摘要或概览，以及我们生成新数据的方式。

统计学是数据科学、深度学习和生成式建模的基础。在数据科学中，我们使用统计学来决定使用哪些数据。对于深度学习，统计学用于衡量模型的效率，而生成式建模则利用统计学来理解模型正在学习什么。

现在，在我们深入探讨生成式建模的统计学和潜在空间的理解之前，先看一个实际运行的示例会很有帮助。这个示例的训练也需要一些时间，所以在训练的同时，你可以阅读下一节中的理论知识。

在下一个练习中，我们将深入构建一个变分自编码器（VAE）。VAE 在结构和功能上类似于自编码器，不同之处在于它学习编码部分的方式。回想一下，在 AE 中，模型学习如何对数据的潜在表示进行编码，然后再解码回原始数据。而在 VAE 中，模型学习的是潜在编码的分布方式。我们暂时不必担心理解分布部分，而是直接进入练习 3-4，看看代码是如何工作的。

```
class ConvVAE(nn.Module):
    def __init__(self, image_channels=3, h_dim=1024, z_dim=32):
        super(ConvVAE, self).__init__()
        self.encoder = nn.Sequential(
            nn.Conv2d(image_channels, 32, kernel_size=4, stride=2),
            nn.ReLU(),
            nn.Conv2d(32, 64, kernel_size=4, stride=2),
            nn.ReLU(),
            nn.Conv2d(64, 128, kernel_size=4, stride=2),
            nn.ReLU(),
            nn.Conv2d(128, 256, kernel_size=4, stride=2),
            nn.ReLU(),
            Flatten()
        )
        self.fc1 = nn.Linear(h_dim, z_dim)
        self.fc2 = nn.Linear(h_dim, z_dim)
        self.fc3 = nn.Linear(z_dim, h_dim)
        self.decoder = nn.Sequential(
            UnFlatten(),
            nn.ConvTranspose2d(h_dim, 128, kernel_size=5, stride=2),
            nn.ReLU(),
            nn.ConvTranspose2d(128, 64, kernel_size=5, stride=2),
            nn.ReLU(),
            nn.ConvTranspose2d(64, 32, kernel_size=6, stride=2),
            nn.ReLU(),
            nn.ConvTranspose2d(32, image_channels, kernel_size=6, stride=2),
            nn.Sigmoid(),
        )
```

## 使用 VAE 学习分布

在 VAE 中，模型的学习方式不仅仅是理解中间或潜在空间的编码，而是理解编码本身的分布。通过理解编码的分布或变异性，我们可以学习如何映射潜在编码并生成那些隐藏空间。

图 3-13 并排展示了 AE 和 VAE。在图中，`x` 代表输入，`x'` 代表生成的输出。同样地，`z` 代表我们数据的潜在编码或中间表示。在 VAE 中，有一个细微的差别：我们不学习编码本身，而是学习该编码的分布或变异性。然后，我们使用这些学习到的参数来采样一个 `z` 表示，并将其输入到解码器中。

![../images/502181_1_En_3_Chapter/502181_1_En_3_Fig15_HTML.jpg](img/502181_1_En_3_Fig15_HTML.jpg)

在图 3-15 中，我们可以看到 `μ` 和 `σ2` 的不同变化如何改变正态分布的形状，其中均值始终代表分布的中心，而标准差代表分布的离散程度或多样性。

通常，我们使用统计学来发现数据是如何分布的，或者它代表了什么样的变异性。使用 VAE，我们仍然在学习数据的参数是什么，只不过现在我们关注的数据空间变成了该数据的隐藏或潜在空间表示。与尝试学习一组训练图像的整个潜在空间相比，这具有三个优势。

首先，通过学习嵌入空间，数据量或数据的维度可以大大减少。这使得我们的 VAE 能够以更简单的形式学习特征的表示方式，从而让我们能够建模一个简单的正态分布。更复杂的数据形式可能需要 VAE 来建模更复杂的分布或分布集合。

第二个优势使我们能够从这个学习到的潜在空间映射或生成新的输出。我们只需要理解和控制这些分布参数，就能得到新颖的输出。

第三个优势（但可能不那么重要）是确认我们只需学习输入数据分布并将其转换为正态参数，就能生成输出。AE 和 VAE 之间的一个关键架构差异是发生在模型中间的采样操作。这个操作本质上将编码器和解码器模型解耦，为我们提供了更好的复用能力和进一步的生成式建模机会。

3. 跳转到类定义代码块以及此处显示的 `encode` 和 `decode` 函数：

```
def bottleneck(self, h):
    mu, logvar = self.fc1(h), self.fc2(h)
    z = self.reparameterize(mu, logvar)
    return z, mu, logvar
def encode(self, x):
    h = self.encoder(x)
    z, mu, logvar = self.bottleneck(h)
    return z, mu, logvar
def decode(self, z):
    z = self.fc3(z)
    z = self.decoder(z)
    return z
def forward(self, x):
    z, mu, logvar = self.encode(x)
    z = self.decode(z)
    return z, mu, logvar
```

4. 注意网络如何通过 `bottleneck` 函数限制生成 `mu` 和 `logvar`（方差参数）。然后使用这些参数从分布中采样一个 `z` 或编码表示。接着在 `forward` 函数中，我们可以看到 `encode` 如何用于生成这些参数以及采样的 `z`，而 `z` 被输入到解码器中，作为用于生成新输出的编码。

5. 接下来，我们再次向下滚动到 `loss_fn`，如下所示：

```
def loss_fn(recon_x, x, mu, logvar):
    BCE = F.binary_cross_entropy(recon_x, x, size_average=False)
    KLD = -0.5 * torch.mean(1 + logvar - mu.pow(2) - logvar.exp())
    return BCE + KLD, BCE, KLD
```

6. 在这种情况下，自定义损失函数使用两种方法来测量学习到的分布与观察到的分布之间的散度。第一种技术称为*二元交叉熵*（BCE），它测量原始输入图像与重建图像之间的差异。这与我们在普通 AE 中测量损失的方式并无不同。第二种技术称为 Kullback Leibler 散度（KLD），用于测量数据分布之间的实际差异。

学习数据如何分布，并将其映射到一个已知或未知函数的概念，是深度学习生成式建模的核心。在本书后续内容中，我们将反复使用这一概念。在下一节中，我们将探讨理解数据分布如何让我们能够操控输出。

## 结论

要构建可运行且实用的生成模型，我们需要理解深度学习网络是如何学习的。我们需要了解网络如何逼近函数，以及这对网络学习的内容有何影响。在此基础上，我们详细探讨了各种超参数如何影响网络的学习方式和学习内容，并进一步学习了如何在玩具示例上调整这些超参数。

接下来，我们从调整超参数转向理解它们如何让我们控制模型中的潜在空间或隐藏空间。学会控制潜在空间的大小使我们能够更好地训练模型。然后，我们进阶到构建变分自编码器，这种模型不仅通过理解潜在空间来学习，还通过理解映射到该潜在空间的分布参数来学习。

最后，基于将参数映射到潜在空间的知识，我们可以严格地根据这些参数构建输出。这使我们能够详细了解模型学到了什么，以及它对输入数据的敏感程度。虽然本章我们并未着手解决模型的问题，但未来我们一定会研究其他系统来改进它。
