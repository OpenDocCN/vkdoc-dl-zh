# 10. Azure 机器学习

问：为什么埃德在闹鬼的丛林里被袭击了？

答：因为其他人都足够理智，不会随便进入随机森林。

这是一个以我们合著者为代价的机器学习笑话，但它很好地引出了诸如随机森林之类的定制化算法。（随机决策森林通过对决策树中的任务进行分类，来输出均值或平均预测/回归结果。埃德的报复方式是他来解释这些笑话。）到目前为止，在本书中，你已经对认知服务进行了广泛的回顾，这些服务以托管且组织良好的方式提供了一组预构建的算法、模型和配方。这些服务能够加速机器学习开发，并使数据科学家能够更快速、更有效地交付产品和服务，而无需重复造轮子。

然而，现实世界的业务用例存在于数据科学和机器学习领域，而不仅仅是使用预构建的 API。你必须能够构建自己的模型，尝试新的数据集，增强现有模型，并创建自己的算法。请放心，Microsoft Azure 拥有广泛的能力来满足这些定制化需求。Azure 机器学习是微软 AI 生态系统不可或缺的一部分，它帮助机器学习工程师、AI 爱好者和专业人士以及公民数据科学家实现智能解决方案。端到端的机器学习生命周期不仅包括构建、训练和部署机器学习模型，还包括长期维护它们。你必须监控模型漂移和衰减，根据需求进行扩展，并完成挑战者测试，以确保最佳模型胜出。Azure 机器学习平台有助于整合这些多样化的服务，以支持完整的机器学习生命周期。

本章将帮助你了解 Azure 机器学习生态系统产品，以及如何训练应用程序在没有显式编程的情况下进行学习，这正是机器学习的目标。到目前为止，你主要看到了 Azure 认知服务的实际应用，但在本章中，我们将回顾整个 Azure 机器学习生态系统，为你提供认知服务之外的广阔图景。你将学习：

*   Azure 机器学习堆栈和产品

*   使用 Azure 机器学习设计器通过无代码方式构建模型

*   使用 Python 构建 ML 模型并使用 Azure 笔记本进行部署

*   使用 Azure 机器学习部署和测试模型

让我们开始吧。

1.  **端点** – 机器学习模型需要被使用，有几种方法可以实现。部署机器学习模型的一种流行方式是通过 Web 服务，该服务包含一个或多个端点。端点是一个地址，可以通过定义良好的协议（例如基于 HTTP 的 RESTful API）访问和调用模型。（*REST* 代表表述性状态转移。）

2.  **Azure 笔记本** – Azure 笔记本为数据科学家提供了一个集成的工作环境，使其能够在 Azure 工作区内访问文件、文件夹、存储和计算资源。我们在第 6 章中看到了 Jupyter 笔记本，Azure 笔记本与之类似；它们只是使用了 Azure 的计算和存储资源，我们稍后将进行探讨。Azure 笔记本提供了一个绝佳的平台，可以在一个地方协作处理数据集、模型并进行实验。参见图 10-4 中的笔记本。

**图 10-5** Azure 机器学习计算目标

1. **自动化机器学习** – 自动化机器学习（简称 AutoML）是一种应用机器学习技术，为问题找出最佳模型的能力。这包括自动特征工程、神经架构搜索和超参数优化。Azure AutoML 消除了寻找最佳超参数时的猜测工作，并能以无代码的方式帮助你找到最适合使用的模型。

2. **数据集** – 对于机器学习从业者来说，管理机器学习资产（如模型和数据集）可能是一个具有挑战性的问题。借助 Azure 数据集，你可以轻松地在多个机器学习工作流中共享和跟踪数据，同时数据的一个副本会保留在存储中。例如，无需担心连接性和连接字符串即可访问数据。

3. **试验** – 试验和可重复性是*数据科学*之所以成为科学的关键，它能够创建数据集和算法并进行试验。Azure 机器学习的“试验”部分会跟踪你的机器学习试验和运行情况，以便你可以重放管道。

4. **管道**^(³⁶) – *管道*可能是机器学习中含义最丰富的术语，通常指数据提取、模型开发和训练、调优、生产发布，然后是监控和审查，或这些活动的某种组合。在 Azure 机器学习的上下文中，它是指这些机器学习任务的可重复执行工作流，可以执行数据操作（如归一化、转换、特征化、验证等）、模型构建和训练（如超参数调优、自动模型选择、模型测试和验证）以及模型部署（如部署和批量评分）。模型管道提供了出色的功能，例如：

    - 无人值守执行

    - 多样化的计算资源可访问性

    - 可重现性和可重用性

    - 跟踪和版本控制

5. **模型** – 模型是机器学习的核心：它们通过使用指定的算法，封装了从数据中学习到的知识。在 Azure 机器学习工作区中，你可以注册、创建、管理和跟踪模型。你在工作区其他部分构建的任何模型也会在此存储库中注册，并可在整个平台中使用。

6. **计算** – 机器学习模型、自动化机器学习过程和 Jupyter 笔记本，都需要一个运行（执行计算）的地方。这正是 Azure 计算所提供的——你可以在此创建不同类型的计算目标，如图 10-5 所示。下表详细说明了适用于工作负载的不同训练目标。对于机器学习工作负载，你可以使用本地计算机、计算集群、远程虚拟机以及各种其他训练目标。

7. **数据存储** – 数据存储是数据的存储库；它们是数据集的容器，例如 Azure 存储、Azure Data Lake Storage、Azure SQL 数据库和 Azure Databricks 文件系统 (DBFS)。对于系统存储，Azure 机器学习使用 Azure Blob 容器和文件容器。

8. **标记** – 标记和注释是数据科学生命周期的重要组成部分。使用 Azure 机器学习的标记功能，你可以通过创建标记项目、管理工作流并将其分发给多个标记人员来标记数据集，还可以利用机器学习来帮助你完成标记。

你很快就会在实际操作中进一步了解这些组件。在我们踏上构建机器学习模型的旅程时，了解用于构建模型的正确算法非常重要。
